兼容性脚本

GameMaker版本的用户， GameMaker Studio 2可能会有些混乱，因为对项目资产，编辑和编码的方式进行了许多更改。这些更改意味着IDE和GameMaker Language （ GML ）均不支持下面列出的项目。但是，这并不意味着您不能将使用GameMaker Studio 2中。 GameMaker Studio 2将自动将所有旧版功能转换为兼容性脚本，这些脚本包含将旧版方法转换为更新方法的功能的脚本。

发生的情况是， GameMaker Studio 2将为您使用的所有过时功能和变量解析项目，然后使用这些功能名称创建一系列脚本。这些脚本将包含新的等效GameMaker _Language / GML _Overview / Runtime _Functions.htm“> runtime函数或执行与过时功能相同的操作的简短脚本功能。

首次导入GameMaker Studio 2项目时，将显示如下兼容性报告：

该报告的主体显示了受影响的对象和脚本以及受影响的位置，最后，您可以找到为项目创建的脚本的列表。您可以单击鼠标中键或按在报告中列出的任何脚本上打开它，然后查看已完成的操作。

请注意，如果您看到任何脚本名称前都有一个双下划线（例如： __global_object_depths ），那么这些是由IDE创建以帮助实现兼容性的脚本，而不是从过时的功能转换而来的脚本，因此不应篡改这些脚本，因为它们是其他兼容性脚本正常工作所必需的。

完成兼容性报告的修改后，您可以关闭工作空间，但是如果您需要修改报告中的所有内容，则可以转到Asset Browser的Notes部分再次找到它。

请注意，您可以继续使用这些兼容性脚本，就像使用旧版GameMaker的功能一样，但是，由于它们的工作方式，因此会对性能产生影响。我们建议您更新项目以直接使用新方法并删除兼容性脚本（如果可以的话），因为这将提高性能，并让您深入了解GameMaker Studio 2和GameMaker Language与旧产品相比的工作方式。

另请注意， GameMaker Language引入了一些功能以确保与较早项目的兼容性，这些功能将出现在这些兼容性脚本中。但是，这些功能绝对不能在您自己的项目中使用。您可以从手册的以下部分找到这些功能的列表：
