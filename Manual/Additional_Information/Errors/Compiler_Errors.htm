编译错误

可以为您的游戏报告的第二种错误类型是编译器错误。当您的游戏遇到语法检查程序可能无法检测到的某种类型的错误时，就会发生编译器错误-例如，很容易忘记声明一个变量或错误地将字符串键入为实数-或与如何从“平台偏好设置”中为目标平台设置编译工具有关。 GameMaker Studio 2将捕获这些错误，并且此信息还将显示在Compiler Error Output中，如下所示：

注意：这些错误通常不会弹出一个窗口，而只会在编译器输出中显示“ Build Failed”。如果弹出一个窗口，它通常是一个亚军的错误，这说明这里。

编译器错误消息将全部遵循相同的格式：

[object] - [event] - [Line Number]: [error string]

如果在脚本中发现错误，则将仅是：

[script] - [Line Number]: [error string]

然后您可以双击在任何编译器错误条目上打开标记为给出错误的位置的给定资产，然后可以使用消息中包含的信息来精确指出错误在对象或脚本中的何处发生。但是，有时这些错误似乎有些含糊，因此在下面可以找到所有错误的完整列表以及它们的含义的简要说明：
