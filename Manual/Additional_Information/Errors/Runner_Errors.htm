跑步者的错误

即使在代码编辑器中进行了语法检查，然后让编译器检查你的代码是否有编译错误，仍然有一些情况下会出错。在大多数情况下，这将抛出一个虚拟机（VM）运行器错误（也称为运行时异常），看起来像这样。

Runner错误通常比编译错误更严重，因为它意味着你的代码存在严重的问题，而这些问题是代码编辑器和编译器都无法检测到的。 代码编辑器和编译器都无法检测到，因此你应该注意所有这些错误。当错误发生时，你可以使用弹出窗口中的复制按钮，将错误复制到剪贴板上 然后你可以将其粘贴到文本文件（或其他地方），以供将来参考。

这个错误的结构如下。


  • 这显示错误发生的事件以及对象（如果适用）。

  • 这显示了实际的错误。 因此，它显示了错误的范围 和它的错误类型。括号中的第一个值将是运行代码的实例ID--也可能是下表中列出的特殊值之一--第二个值是YoYo Games用于识别错误的内部值，可以忽略。 用于识别bug的内部值，可以忽略不计。在上面的图片中，它是说 _image 这个变量没有被设置（即：在被访问之前不存在），在对象 obj_EA_DEMO_Control 。

如上所述，某些错误信息不是通过实例ID值来识别范围，而是通过一个负值 。这些值可以用来确定错误的确切性质以及它指的是什么，具体数值如下 可能的。

来自虚拟机运行器的可能错误如下。

误差 留言 运作 描述
DoSet 无效的比较类型 数据类型 这表示运行者试图比较两种不兼容的数据类型，如实数与字符串。
DoConv 执行错误 数据类型 这表示在将一种数据类型转换为另一种数据类型时出现了错误。
DoAdd 执行错误 数学 在使用加法(+)表达式时出了问题。
DoMul 执行错误 数学 在使用乘法(*)表达式时出了问题。
DoSub 执行错误 数学 在使用减法(-)表达式时出了问题。
DoSub 执行引擎 - 无法对字符串类型进行操作 数学 你正在尝试减去错误的变量类型（例如从实数中减去字符串）。
DoDiv 执行错误 数学 在使用除法(/ 或 div)表达式时出了问题。
DoDiv 执行引擎 - 无法对字符串类型进行操作 数学 你正试图对错误的变量类型进行除法（例如用实数除以字符串）。
DoDiv 除以零 数学 你正试图除以0（注意这只在使用整数除法时发生，用（非零）实数除以0会得到无穷大的答案，用0除以0会得到NaN的答案）。你可以用以下方法检查这些值 (is_infinity) 和 (is_nan)
DoMod 执行错误 数学 在使用modulo (mod)表达式时出了问题。
DoMod 执行引擎 - 无法对字符串类型进行操作 数学 你试图在错误的变量类型上使用modulo (mod)(例如 mod 一个字符串乘以一个实数)。
DoAnd 执行错误 按位数计算 在使用位法 "和"(&)表达式时出了问题。
DoAnd 执行引擎 - 无法对字符串类型进行操作 按位数计算 你试图在错误的变量类型上使用位法 "和"(&)（例如试图将一个字符串与一个实数 "和"）。
DoOr 执行错误 按位数计算 在使用比特 "或"(|)表达式时出了问题。
DoOr 执行引擎 - 无法对字符串类型进行操作 按位数计算 你试图在错误的变量类型上使用 "或"(|)（例如试图将一个字符串与一个实数 "或"）。
DoXor 执行错误 按位数计算 在使用位法 "xor"(^)表达式时出了问题。
DoXor 执行引擎 - 无法对字符串类型进行操作 按位数计算 你试图在错误的变量类型上使用 "xor"(^)（例如试图将一个字符串与一个实数 "xor"）。
DoShl 执行错误 按位数计算 当向左移位(<<)一个值时出了问题。
DoShl 执行引擎 - 无法对字符串类型进行操作 按位数计算 你试图对错误的变量类型进行左移(<<)(例如试图对一个字符串进行位移)。
DoShr 执行错误 按位数计算 当向右移位(>>)一个值时出了问题。
DoShr 执行引擎 - 无法对字符串类型进行操作 按位数计算 你试图对错误的变量类型进行右移(>>)(例如试图对一个字符串进行位移)。
DoNeg 执行错误 否定 你试图把一个变量类型变成一个负数，而这个类型不允许这样的操作。
DoNot 执行错误 否定 你正试图 "不 "一个变量类型，而这个类型不允许这样的操作。
Push 执行错误 - 变量索引超出范围 (var) 堆栈 被访问的变量超出了运行者的既定范围。
Push 变量获取 (var) 堆栈 所给的变量未被定义或未知。
Pop 变量指数超出范围 (var) 堆栈 被访问的变量超出了运行者的既定范围。
Pop 变量获取 (var) 堆栈 所给的变量未被定义或未知。
With 不能在with语句中使用全局 与 你试图在 "with" 语句中使用 "global" 作为变量，即。
with (global)
   {
   //do something;
   }
With 不能在with语句中使用local 与 你试图在 "with" 语句中使用 "local" 作为变量，即。
with (local)
   {
   //do something;
   }
DoCall 执行引擎类型错误 发动机 这是虚拟机中的一个未定义的错误。如果发生这种情况，你应该提交一份错误报告（参见：帮助菜单，了解如何做的细节。
Stack Overflow - 发动机 当调用栈上使用了太多的内存时就会发生堆栈溢出，当你的游戏试图使用超过调用栈上的可用空间时（也就是说，当它试图访问超出调用栈界限的内存时，这基本上就是 缓冲区溢出），堆栈就会溢出，导致程序崩溃。重新启动你的电脑和GameMaker。Studio，如果错误仍然存在，请与支持部门联系和/或提交一个错误（如上所述）。
