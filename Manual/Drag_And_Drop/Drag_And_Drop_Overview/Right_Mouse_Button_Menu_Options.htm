鼠标右键菜单选项

在处理DnD™代码时，您可以单击鼠标右键在操作上，在工作区上或在左侧列表中的文本上，以打开“鼠标右键”菜单：

该菜单具有以下选项：

  • 剪切/复制/粘贴：这将允许您剪切，复制或粘贴在工作区中选择的任何操作。请注意，您可以使用+在动作之外拖动或使用/ +在单个动作上，选择多个动作并将命令应用于所有动作。
  • 注释/删除注释：此选项允许您从DnD™代码块中添加和删除注释。选择“注释”时，将在所选DnD™块的右侧打开文本编辑器，您可以在其中添加所需的任何文本，无论是提醒您进行某些更改，还是对教程有用的注释，甚至是说明，为您的队友：
    添加注释后，可以单击并拖动它以垂直和水平放置它-尽管仅在父节点的右侧-并且折叠DnD™节点也会折叠该注释并显示其内容的摘要（在这种情况下，无法重新定位）。如果不再需要注释，则可以删除注释中的所有文本，也可以使用菜单中的“删除注释”选项。请注意，当您复制节点时，注释将与其一起复制，并且如果将节点从DnD™ GML ，则注释将被保留并添加到GML中，如下图所示。 ：
    如上所述，注释只能应用于单个节点，但是您可以通过简单地添加换行符来扩展注释来扩展它以覆盖多个节点-尽管这可能需要您重新放置注释以防止与后续注释重叠节点：
    添加评论后，您可以从人民币菜单中再次选择“评论”选项，也可以双击在其上启用编辑。
  • 禁用/启用：可以使用这些命令禁用或启用所选的动作。禁用某个动作后，该动作仍在工作区中可见，但在编译游戏时不会运行。这样，您可以添加/删除操作以进行测试和调试。下图显示了禁用操作的DnD
    您可以看到“ If Score”和“ Go To Next Room”这些操作已被禁用-它们在视觉上呈灰色显示，因此在测试或编译游戏时将跳过它们。
  • Toggle Breakpoint（切换断点） ：此命令将打开或关闭所选动作的断点。断点只是DnD™代码中的一点，当游戏以“调试模式”运行时，调试器应停止。分配了断点的操作将突出显示，如下图所示（“ If Health”操作已设置为断点）：
  • 删除所有断点：使用此命令，您可以从DnD™代码中删除所有断点。
  • 全部折叠/全部折叠： DnD™动作可以折叠或展开，其中折叠的动作将仅显示动作的名称，而不显示任何内部值，从而最大化工作空间的可用空间。例如：
    如您所见，这些动作占用的空间要少得多，并且您仍然可以从右侧的简短说明中看到它们的作用。您可以选择“全部扩展”选项来为工作空间中的每个节点撤消此操作。
  • 重置布局：这将重置DnD™工作区，将缩放级别设置为100％，聚焦于当前选定的动作，并展开所有（以及所有）折叠动作。
  • 实时预览：选择此选项时，将启用或禁用DnD™到GML （ GameMaker Language ）代码预览窗口。在“将DnD™更改为代码”页面上对此进行了详细说明。
  • 转换为GML ：这会将当前事件中的所有动作转换为GML （ GameMaker Language ）代码。请注意这一点，因为您不能将GML转换回其创建时所用DnD™操作。您可以将DnD™事件，但是任何代码都将简单地包装在“执行代码”操作中。在“将DnD™更改为代码”页面上对此进行了详细说明。
  • 窗口：此部分打开以下子菜单：
    这里可用的菜单项是：
    • 转到对象：这会将工作空间平移到正在调用事件的对象
    • Open Inherited Event ：仅当正在编辑的对象是父对象的子对象时，此选项才可用，并且将打开父对象的正在编辑的事件并将工作空间平移到该对象。
    • 单个窗格/ 2列/ 3列：这些选项可用于将当前工作空间拆分为1个，2个或3个不同的“窗格”。每个窗格是一个单独的工作区，可以将脚本或事件拖到它们上，因此您可以一次编辑和比较2个或更多DnD™代码块。下图在一个窗口中显示了两个脚本，这些窗口显示在它们自己的工作区窗格中：

DnD™节点上打开RMB菜单时，以下两个选项才可用：

  • 平移：在任何节点上选择此选项时，工作空间将平移以将该节点环到中心。
  • 显示帮助... ：选择此选项将在与所选节点相关的页面上的新工作区中打开手册。您还可以通过使用在节点上。
