创建缓冲区

这个动作可以用来在你的游戏中分配一部分内存作为缓冲区，该函数返回唯一的缓冲区ID值，该值应该存储在一个变量中，并用于所有进一步调用缓冲区的函数。你可以提供 一个全局变量，一个实例变量，或者让动作创建一个临时的本地变量来存储缓冲区的ID。注意，你可以通过点击缓冲区旁边的加号图标来创建额外的缓冲区 旁边的加号图标 并选择另一个变量来保存缓冲区的ID。

注意：当你不再需要这样的动态创建的资源时，必须将其从内存中删除，以防止内存泄漏，所以当你完成了你所创建的缓冲区后，你应该再次释放它 使用Delete Buffer动作（例如在Other - Clean Up事件中）。在使用临时局部变量时，这一点尤其要记住。 变量时，这一点尤其重要，因为如果你没有在事件或动作脚本结束前再次删除缓冲区，那么你将无法再访问它，以后就会出现问题。
