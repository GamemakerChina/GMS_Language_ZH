创建网格

该操作将创建一个新的网格数据结构并返回索引值，以便您以后可以通过其他“数据结构”操作访问网格。网格索引将返回到您提供的目标变量，可以使用“分配变量”或“声明温度”在更早的时候创建它，也可以标记“温度”复选框以命名并创建一个临时局部变量来存储该值，直到脚本或事件的结尾。请注意，与其他数据结构不同，不会将网格创建为“空”，并且新创建的网格的每个单元格的值均为0。取决于您如何更改单元格数据或使用“清除”操作将网格自身清除为另一个值网格。请注意，您可以通过单击加号图标来创建其他DS网格在操作旁边，然后选择另一个变量来保存网格ID，并再次给出宽度和高度值。

重要的！创建任何数据结构都会消耗目标平台上的内存，因此，当不再需要使用动作“免费数据结构”时，所有数据结构都应该是空闲的，否则会出现内存泄漏，从而损害游戏性能甚至导致崩溃。当使用临时局部变量存储数据结构索引时，这尤其重要，因为这些变量在代码或事件的末尾被删除，但这并不意味着数据结构也将被删除！数据结构将仍然存在，只有您无法引用它，因此请使用实例变量并在以后释放结构，或者在事件或函数结束之前释放结构（如果其索引存储在其中）一个临时变量。
