数据结构动作库

在游戏中，您通常需要以精确且有序的方式存储信息。例如，您可能需要存储一个人携带的物品的清单，或者您可能希望存储仍然需要访问的地方的网格。现在，您可以为此使用多个变量，但是如果您想执行更复杂的操作（例如，对数据进行排序或搜索特定项目），则需要编写大块的动作块代码，这些代码执行起来很慢，难以执行。调试和真正的痛苦写！

为了解决这个问题， GameMaker Studio 2具有许多内置数据结构，可以通过专业功能进行访问。 Drag and Drop可以使用四种不同类型的数据结构（如果使用GML则有六种），每种类型都有其优缺点，这取决于要存储的信息类型以及希望如何使用它以后-四种类型是：

清单列表是一种数据结构，在添加信息时按顺序存储信息。列表索引从0开始，并且对于添加到列表末尾的每个项目都会上升，尽管您也可以将数据插入到最后一个位置以外的位置。与堆栈数据结构（如下所述）不同，您可以随时从列表的任何索引访问数据。
地图地图数据结构非常有用，因为它可以存储键和值对。例如，您游戏中的一个角色可以具有许多不同的物品（键），并且对于每个单独的物品，他可以具有多个物品（值），例如在RPG中，您可以拥有10个健康药水，5个法力药水和100金币。地图在一处维护所有这样的对，您可以将对添加到地图，并使用一些简单功能搜索与某些键对应的值。映射不会以任何（可识别的）方式进行排序，这意味着要找到某个键，您可能必须遍历整个过程（这很慢），而且也无法保存两个相同的键，您可以为一个键分配两个值吗？
格网网格是值的二维数组，您可以在其中定义其宽度和高度。这将设置网格内网格“单元”的总数（宽度和高度相乘），然后将这些单元用于保存不同的数据值，该数据值可以是任何允许的数据类型。通过这种类型的结构，您可以通过指定网格的x（列）和y（行）位置来设置和检索网格中的像元值（行和列都从0开始，一直到网格宽度-1和网格高度-1）。访问网格数据结构时，应始终尝试对单元位置使用整数值，否则，所有非整数索引都会被GameMaker Studio 2限制。如果这不是您所需要的，那么您将需要先进行四舍五入，然后再传递要检查的索引。
堆栈堆栈数据结构是所谓的后进先出（LIFO）结构。您可以将值推送到堆栈上，然后通过将其从堆栈中弹出来再次将其删除，最近一次在堆栈上压入的值是第一个再次从堆栈中弹出的值（想想一堆硬币，每个硬币您添加的硬币必须先被移除，然后才能进入下方的其余硬币。当有要处理的中断，具有递归功能，甚至为游戏构建基本的AI时，通常会使用堆栈。

本质上，所有数据结构都以相同的方式工作-创建数据结构并将其索引值存储在变量中。然后，您可以使用此索引在可以对其执行操作的所有其他函数调用中引用数据结构。最后，一旦完成，您将再次破坏数据结构以将其从内存中删除。您可以根据需要同时使用任意多个结构，并且所有结构都可以存储字符串和实数值。

注意：与所有动态资源一样，数据结构会占用内存，因此在不再需要时应销毁数据结构，以防止内存泄漏而减慢速度并最终导致游戏崩溃。
