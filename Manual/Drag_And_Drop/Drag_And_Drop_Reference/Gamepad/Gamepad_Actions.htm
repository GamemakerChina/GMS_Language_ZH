游戏手柄动作库

GameMaker Studio 2具有许多专用动作，可用于检测来自多个连接的游戏手柄的模拟和数字控件。这些操作要求您指定游戏手柄索引值，该值是代表所选游戏手柄的数字（从0开始计数）。请注意，当将游戏手柄插入设备（或将其移除）时，将触发异步系统事件（但是，处理该事件可能需要使用代码）。

游戏手柄的“插槽”从0开始索引，插槽0-3（含0和3）仅用于Xinput游戏手柄，即：Xbox360控制器和兼容版本。但是，您还可以检查DirectInput游戏手柄的4-11插槽（含），这意味着通过这些插槽连接时，您可以检测到许多其他型号的控制器。

使用游戏手柄动作时，输入可以来自axis ， button和/或Triggers ， GameMaker Studio 2会将它们分配给以下某些或所有内置常量（请注意，“ hats”通常仅在非标准的情况下才能检测到）控制器）：

持续的描述
gp_face1顶部按钮1（这对应于Xbox 360控制器上的“ A”和PS控制器上的叉号）
gp_face2顶部按钮2（这对应于Xbox 360控制器上的“ B”和PS控制器上的圆圈）
gp_face3顶部按钮3（这对应于Xbox 360控制器上的“ X”和PS控制器上的正方形）
gp_face4顶部按钮4（这对应于Xbox 360控制器上的“ Y”和PS控制器上的三角形）
gp_shoulderl左肩按钮
gp_shoulderlb左肩扳机
gp_shoulderr右肩扣
gp_shoulderrb右肩扳机
gp_select选择按钮（这是DS4控制器上的PS按钮）
gp_start开始按钮（这是PS4控制器上的“选项”按钮）
gp_stickl按下左摇杆（作为按钮）

以下列表显示了跨平台的当前兼容性（请注意，随着将来的更新，兼容性将发生变化）：

  • Windows ，一次最多允许11个连接的设备。请注意，在Windows ，前四个游戏手柄插槽（0-3）是使用XInput dll处理的，这意味着只有XBox控制器可能是100％兼容的，对于其他控制器类型，您应该检查其余的游戏手柄插槽（4- 11）。
  • macOS一次最多支持4个连接的设备，并且这些设备只能是Playstation 3或Xbox 360类型。请注意，Mac Game Options中的“ Build for Mac AppStore ”选项必须为OFF为垫支持工作。
  • Ubuntu也支持GamePad输入，但是您可能需要从Ubuntu存储库安装其他库。您可以通过打开命令行终端并键入以下命令轻松完成此操作，这将为操纵杆以及操纵杆本身安装GUI支持：

sudo apt-get install jstest-gtk
sudo apt-get install joystick.

  • HTML5游戏在Safari之外的大多数主要浏览器上都支持游戏手柄。
  • 通过iCade机箱，游戏手柄支持也扩展到iOS左轴映射到操纵杆控制器（尽管输入是数字的，而不是模拟的），四个“面部”按钮映射到机柜的前部按钮，四个肩部按钮映射到机柜后部的按钮。
  • Android导出支持NYKO控制器和通用蓝牙控制器，但仅在启用它们时才表示支持，这意味着您必须Android游戏选项的常规部分中勾选iCade /蓝牙选项。他们需要API级别12才能正常工作，并且应该注意， GameMaker Studio 2将与您的设备配对的任何蓝牙设备注册为已连接，无论它是否实际连接。因此，在分配和检查“插槽”时应考虑到这一点。
  • 在PS4上，如果要使用触摸板跟踪，则需要使用 device_mouse_* 按钮（对此没有DnD™操作）。
