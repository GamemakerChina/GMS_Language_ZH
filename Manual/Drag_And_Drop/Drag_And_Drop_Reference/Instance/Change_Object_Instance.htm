更改对象实例

通过这个动作，你可以将一个实例变成另一个实例。你需要指定这个实例变成另一个实例的对象，这个动作将改变这个实例，执行初始实例的Destroy Event和Clean Up Event以及新实例的Create Event。 初始实例的Destroy事件和Clean Up事件以及新实例的Create事件。注意，如果你没有指定方向或速度等，那么新的实例将 "继承 "这些值。例如，如果初始实例的image_angle（旋转）为180，以2px的速度移动，那么新实例将 "继承 "这些值。 为180，每个游戏刻度线的移动速度为2px，image_speed（动画速度）为0.5，那么除非在新实例的创建事件中明确设置，否则它所变成的实例将保持这些值。

值得注意的是，改变实例意味着在下一个游戏时间之前，你不能对该实例进行进一步的操作，特别是试图访问变量等......因为这将导致一个错误。基本上，你 改变了实例，但是在当前步骤结束之前，它实际上是不可用的，所以直接访问它所包含的任何变量（例如，调用inst_changed.x）将无法工作。
