设置实例变量

通过此操作，您可以设置一个内置变量。所有对象的所有实例内部都已创建了许多变量，以保存基本信息，例如当前x / y位置，速度，方向，可见性和许多其他内容。您可以使用的实例变量是：

  • X坐标-沿房间中x轴的位置。
  • Y坐标-在房间中沿y轴的位置。
  • 方向-实例的移动方向（默认为0，其中右侧为0°，向上为90°，左侧为180°，向下为270°）。
  • 速度-实例移动的速度（每游戏帧以像素为单位）。
  • 水平速度-实例移动的水平速度（以每个游戏帧为单位的像素）。负速度表示它向左移动，正速度为右，默认值为0。
  • 垂直速度-实例移动的垂直速度（以每个游戏帧为单位的像素）。负速度表示它在向上移动，正速度在下降，默认为0。
  • 可见-可见性切换变量（默认为 true除非在“对象编辑器”中另外设置，否则设置为 false使看不见）。请注意，不可见实例仍将运行其除Draw事件外的所有事件。
  • 持久-持久切换变量（默认为 false除非在“对象编辑器”中另外设置，否则设置为 true使之持久）。这将使实例在每个房间之间“持久”，即：该实例将不会在房间末端被清理，而是将被带到下一个房间。请注意，当实例保留后，它不会调用其Create或Destroy事件，而是会调用Clean Up，Room End和Room Start事件。
  • 层-实例已分配到的层。
  • 深度-分配给图层的深度值。
  • Sprite-分配给实例的Sprite资源。
  • 图像索引-所指定精灵的图像索引。此值设置分配给实例的精灵的帧，如果该值大于动画中的帧数，它将“循环”，这样如果将具有5帧的精灵设置为帧7，则实际上将显示帧2。
  • 图像角度-实例应设置的角度（默认值为0，其中右侧为0°，向上为90°，左侧为180°，向下为270°）。
  • 图像Alpha-实例的Alpha值（默认为1），从0到1，其中0是完全透明的，而1是完全不透明的。
  • 图像混合-要与实例精灵混合的颜色（默认为白色）。
  • 图像速度-实例的动画速度。
  • 水平比例尺-应该绘制实例精灵的水平比例尺（这也会影响实例边界框和碰撞）。比例尺为1表示无比例缩放（1：1），较小的值将按比例缩小（例如0.5，将是精灵的宽度的一半），较大的值将按比例缩放，负值将镜像该精灵并对其进行缩放，除非所使用的值正好为-1（在这种情况下，子画面仅沿y轴镜像，没有缩放比例）。
  • 垂直比例尺-绘制实例精灵的垂直比例尺（这也会影响实例边界框和碰撞）。比例尺为1表示无比例缩放（1：1），较小的值将按比例缩小（例如0.5，将是精灵的高度的一半），较大的值将按比例缩放，而负值将翻转精灵并对其进行缩放，除非所使用的值正好为-1（在这种情况下，仅沿y轴翻转了精灵，没有缩放比例）。
  • Mask-实例的掩码索引。默认值为-1（这意味着该实例将为当前分配的Sprite索引使用碰撞蒙版），但是如果设置为任何其他Sprite资源，则将使用该实例的碰撞蒙版代替分配给该Sprite的Sprite指数。
  • 摩擦-使用游戏的每个游戏框架来减慢实例速度的摩擦。该值将简单地从每个游戏帧的当前速度值中扣除（或在速度值为负数时添加）。
  • 重力-重力将在实例的重力方向（请参见下文）上施加恒定的力，从而影响其速度（默认值为0）。请注意，重力是累积力，如果您选择不限制最终速度，则重力将加速对象。
  • 重力方向-当实例重力大于0（默认值为270）时，可用于设置移动方向。请注意， GameMaker Studio 2中的方向计算为：0度为右，90度为上，180度为左，270度为下。
  • 得分-设置 score多变的。
  • 生活-设置 lives多变的。
  • 健康-设置 health多变的。

选择要使用的内置变量后，即可设置该变量的值。您使用的值将取决于所选变量（有关默认值和范围，请参见上面的列表），并且可以将值标记为相对于变量的当前值。您还可以通过单击加号图标来添加其他变量以在同一操作中进行设置在操作旁边，然后选择另一个实例变量并为其提供值。
