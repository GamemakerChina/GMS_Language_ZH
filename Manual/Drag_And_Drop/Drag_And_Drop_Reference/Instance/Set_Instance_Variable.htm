设置实例变量

通过这个动作，你可以设置其中一个内置的变量。所有对象的所有实例都已经在其中创建了一些变量，以保存基本信息，如当前的X/Y位置、速度、方向、可见度 和许多其他东西。你可以使用的实例变量是。

  • X坐标--沿X轴在房间里的位置。
  • Y坐标- 沿着Y轴在房间里的位置。
  • 方向- 实例的运动方向（默认为0，其中右为0°，上为90°，左为180°，下为270°）。
  • 速度- 实例移动的速度（以每游戏帧的像素为单位）。
  • Horizontal Speed- 实例移动的水平速度（以每游戏帧的像素为单位）。负的速度意味着它在 左边，正的速度是右边，默认值是0。
  • Vertical Speed- 实例移动的垂直速度（以每一游戏帧的像素为单位）。负的速度意味着它在向上移动。 正的速度是向下的，默认值是0。
  • Visible- 可见性切换变量（默认为真，除非在对象编辑器中另外设置。 设置为false可以使其不可见）。)注意，一个不可见的实例仍然运行它的所有事件，除了绘图事件。
  • Persistent- 持久性切换变量（默认为false，除非在对象编辑器中另外设置。 设置为"true"可以使持久化）。)这将使一个实例从一个房间 "持续 "到另一个房间，即：该实例不会在房间结束时被清理掉，而是会被带到下一个房间。请注意，当一个实例被持久化时 它将不会调用它的Create或Destroy事件，但会调用Clean Up、Room End和Room Start事件。
  • Layer- 该实例被分配到的层。
  • Depth- 分配给该层的深度值。
  • Sprite- 分配给实例的精灵资源。
  • 图像索引--分配给精灵的图像索引。这个值设置分配给实例的精灵的帧数，如果 值大于动画中的帧数，它就会 "循环"，比如一个有5个帧的精灵，如果设置为第7帧，实际上会显示第2帧。
  • 图像角度- 实例应设置的角度（默认为0，其中右为0°，上为90°，左为180°，下为270°。 是270°）。)
  • 图像阿尔法--实例的阿尔法值（默认为1），从0到1，其中0是完全透明，1是完全不透明。
  • Image Blend- 要与实例精灵混合的颜色（默认为白色）。
  • Image Speed- 实例的动画速度。
  • Horizontal Scale- 绘制实例精灵的水平比例（这也会影响到实例的边界框和碰撞。 和碰撞）。)1表示不缩放（1:1），较小的值将会缩小（例如0.5，将是精灵宽度的一半），较大的值将会放大，负值将会镜像精灵并缩小它，除非 使用的值正好是-1（在这种情况下，精灵只是沿Y轴镜像，没有缩放）。
  • Vertical Scale- 绘制实例精灵的垂直比例（这也会影响实例边界框和碰撞）。 和碰撞）。)1表示不缩放（1:1），较小的值将缩小比例（例如，0.5将是精灵高度的一半），较大的值将提高比例，负值将翻转精灵并缩放，除非 使用的值正好是-1（在这种情况下，精灵只是沿Y轴翻转，没有缩放）。
  • Mask-实例的掩码索引。默认值是-1（这意味着该实例将使用当前分配的精灵索引的碰撞掩码）。 当前分配的精灵索引），但如果设置为任何其他精灵资源，该实例的碰撞掩码将被使用，而不是分配给该精灵索引的精灵。
  • 摩擦力- 在游戏的每一帧中使用的摩擦力，以减慢实例的速度。这个值将简单地从当前的 扣除（如果速度值是负的，则增加）。
  • 重力- 重力将在实例的重力方向（见下文）施加一个恒定的力，影响其速度（默认为 0).注意，重力是一个累积的力，如果你不选择限制最终速度，它将加速物体。
  • 重力方向- 这可以用来设置当实例的重力大于0时的运动方向（默认为270）。 是270）。)请注意，GameMaker Studio 2中的方向计算为0°为右，90°为上，180°为左，270°为下。
  • 分数- 设置分数变量。
  • 生命- 设置生命变量。
  • 健康- 设置健康变量。

一旦你选择了要使用的内置变量，你就可以为该变量设置值。你所使用的值将取决于所选择的变量（见上面的默认值和范围列表），你可以将该值标记为相对于该变量当前的 变量的相对值。你也可以在同一个动作中添加额外的变量来进行设置，方法是点击旁边的加号图标 旁边的加号图标 旁边的加号图标，然后选择另一个实例变量并给它一个值。
