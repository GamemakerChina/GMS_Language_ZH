创建粒子系统

该操作将初始化粒子系统，以供您创建和使用粒子类型和发射器。您以字符串的形式提供图层名称-例如“ Particle Layer”（必须在房间编辑器中存在）-然后将系统标记为“持久” 。如果将系统标记为“持久”，则系统在创建后将在所有房间中持久保存，并且如果后续房间中不存在初始层，则系统将被分配一个管理层。如果未标记持久性，则系统将在创建房间的末尾自动销毁，与该系统关联的所有发射器也将自动销毁。如果您希望从内存中手动删除系统，则需要使用“销毁粒子系统”操作。

您还需要提供一个将包含唯一粒子系统ID值的目标变量。此ID值将在其他功能中用于设置系统属性或告诉GameMaker Studio 2创建粒子时使用哪个系统。您可以选择将此变量标记为临时局部变量，但这意味着在动作脚本或事件结束时将其丢弃，这意味着该系统仍然存在，但您无权访问它，因此使用此方法时必须小心（通常，系统永远不需要存储在临时局部变量中，但是对于某些事物，例如将粒子绘制到表面上，可能是必需的）。

注意：持久粒子系统一旦创建，便会在游戏过程中一直驻留在系统内存中，因此，如果您需要在任何时候重新创建它（例如，如果您调用“重新启动游戏”操作），则必须首先销毁当前系统，否则会发生内存泄漏。当将资源添加到内存然后“取消引用”时，就会发生内存泄漏，这意味着GameMaker Studio 2无法再与该资源进行交互。这导致内存被不可访问的资源阻塞，进而可能导致游戏问题。要销毁不再需要的持久粒子系统，请使用“销毁粒子系统”操作。
