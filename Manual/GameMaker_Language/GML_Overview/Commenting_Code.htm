注释码

与他人合作，处理大型项目时，甚至只是供您自己参考和调试时，在事件代码和脚本中保留注释非常重要。在GameMaker Studio 2您可以使用各种机制来帮助您留下有关代码部分甚至整个代码块的注释和注释，这些注释和注释可以在当前正在编辑的对象的事件列表中看到。这样，您可以给自己和同事留下笔记，或者解释一段特别棘手的代码，甚至只是提醒您某些事情在做什么，或者您将来想要做什么。

要留下简单的单行注释，您可以做的第一件事是使用 //文字之前。例如：

//initialize variables
sr = 10;
hp = 100;

您也可以留下多行注释，以表示赞赏，省略完整的代码部分进行调试，甚至留下有关将来如何更改代码的注释。为此，您可以使用 /* ... */像这样：

/*
Setup GIF Variables to initial values (these can be edited so you don't need to use the setter functions all the time if you are always using the same values)

IMPORTANT! Please see the individual "set" scripts for more extensive information about each of the different variables.
*/

frame_num = 30;
frame_steps = 5;
frame_width = 256;
frame_height = 256;
frame_quality = gifcam_quality_high;

除了评论，您还可以使用特殊 #region标记以标记代码折叠的一段代码，并且每个区域可以包含一个注释。运作方式是您添加一个 #region标记-以及所有注释文本-置于代码段的顶部，然后添加相应的 #endregion标记到该部分的底部，然后您可以“折叠”代码，即：将其隐藏起来，以便可以随时减少在编辑器中可见的代码行数：

实际的代码如下所示：

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

最后，值得一提的是，在对象编辑器中处理事件时，可以使用特殊的“description在事件开始时添加注释，以说明事件的一般内容，这些内容将显示在“对象编辑器”事件窗口中。该代码应位于代码编辑器的第一行，其格式如下：

/// @description <This Is An Event Description>

最后，如果你正在写的脚本功能，并要加强对他们是如何通过IDE处理一些控制（即：具有自动完成或显示脚本参数），那么你应该使用JSDoc符号作为解释的写你的意见在这里。
