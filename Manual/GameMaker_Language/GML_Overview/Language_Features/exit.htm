退出

exit语句的语法如下。

退出。

exit只是结束当前脚本函数、方法或事件的执行。注意 在这里的使用有细微的差别，这取决于范围的不同。

  • 如果你在一个脚本函数或方法中使用exit，它将简单地退出该函数并返回到调用该函数的代码。
  • 如果你在一个对象的事件中的代码块中使用exit，它将退出整个事件，即使在调用 exit 之后还有各种独立的代码块。
  • 如果你在一个父事件中使用exit，并在一个子对象中使用 event_inherited()来调用该事件，那么只有父事件会退出，而 子对象的事件（调用event_inherited()）将继续。

当在一个事件中使用时，exit通常被用来避免一个实例在满足特定条件（或不满足）时运行任何进一步的代码。下面的代码概述了一个如何使用它的例子，在这种情况下是在一个碰撞事件中。 尽管它可以在任何事件中使用。

if (!visible)
{
退出。
}

other.hp -= attack;
other.coins -= 4;
coins += 4;

上面的代码检查当前实例是否不可见，在这种情况下，它退出代码块，否则它继续运行其余的代码。

注意：它不会结束游戏的执行。为此，你需要使用函数 game_end().
