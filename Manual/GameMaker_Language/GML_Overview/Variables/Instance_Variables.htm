实例变量

一个实例变量是在一个对象的实例内创建的，并且被认为对该实例是唯一的-即：同一对象的许多实例可以具有相同的变量，但是每个变量可以具有不同的值，因为它们对于每个实例都是唯一的。但是如何创建实例变量？好了，您可以通过简单地为它们分配一个值来创建新变量，如下面的小示例所示（这称为声明变量）：

potions = 12;
life = 100;
name = "Jock MacSweeney";
strength = 5.5;
armour = -2;

如您所见，您只需要给出名称，然后给出一个值（该值可以是任何数据类型）来设置该变量，并准备好在您要编码的对象实例中使用它（请注意，该值可以来自函数的返回值或其他变量之间的运算结果等）。然后，可以从实例内部以多种方式使用和修改这些变量，例如，该代码可能在碰撞事件中，并用于减少变量“life “：

life -= 5 + armour;

如果 ”life “现在是100，它将具有97（100-（5 + -2）= 97）的值。现在，这是一个简单的示例，您可以替换“armour ”的实际值-2，但是如果该值在多个位置被硬编码然后决定更改它会发生什么呢？您将必须遍历所有代码并将每个-2更改为新值，这很耗时，而且很容易出错！但是，如果您使用变量，那么您所要做的就是为它重新分配一个新值，然后代码将自动使用该新值，从而使事情变得更加灵活和容易得多。修复应该有的问题。还应该注意的是，即使一个值不改变，记住一个变量“life ”的意思，不只是看一个数字。

GameMaker Studio 2也具有“内置”实例变量的集合，因此您应该注意它们，因为您可能将自己的实例变量命名为相同的名称，或者希望拥有相同名称的全局变量，并想知道为什么遇到错误。但是，它们很容易发现，因为它们在代码编辑器中以不同的颜色显示，并且还自动显示，并在代码编辑器底部的栏中显示。
