asset_get_tags

使用此功能，您可以从资产浏览器检索分配给资产的所有标签。您提供资产名称（作为字符串）或其资产索引，该函数将返回该资产的标签数组。如果未找到标签或存在错误（即：给定的名称字符串不存在），则返回的数组将为空。如果提供资产索引值，那么您将需要提供可选的资产类型参数（常量），因为不同类型的资产可以具有相同的索引，即使它们不能具有相同的名称。下表列出了可用的资产类型：

持续的描述
asset_object给定名称是指对象。
asset_sprite给定名称是指sprite 。
asset_sound给定的名称是指声音。
asset_room给定名称是指room。
asset_tiles给定名称是指图块集。
asset_path给定名称是指路径。
asset_script给定名称是指script。
asset_font给定的名称是指字体。
asset_timeline给定名称是指时间线。
asset_ shader给定名称是shader 。
asset_animationcurve给定名称是指“动画曲线” 。
asset_sequence给定名称是指Sequence 。

句法：

asset_get_tags(name_or_index, [asset_type]);

争论描述
name_or_index资产名称（字符串）或其索引值（整数）。
[asset_type]选修的！获取标签的资产类型，仅在为第一个参数提供索引时使用。

返回值：

Array

例子：

my_tags = asset_get_tags(object_get_name(object_index));

上面的代码将检索分配给创建了运行该代码的实例的对象的所有标记。

返回：资产和标签
