audio_create_buffer_sound

使用此功能，您可以根据缓冲区的内容创建新的声音。缓冲区将先前已创建（有关如何执行此操作的详细信息，请参见缓冲区功能），并且已经添加或加载了数据。然后，使用数据格式将其传递给此函数（仅 buffer_u8或者 buffer_s16目前受支持），采样率（可以在1000hz和48000hz之间）和缓冲区中的偏移量以从中获取数据。您还需要提供缓冲区中的样本数量以及声音所需的通道。这些通道由以下常量之一定义：

持续的描述
audio_mono单声道（单声道）音频。
audio_stereo立体声（双声道）音频。
audio_3D 3D（5.1）音频。

请注意，创建声音后，当不再需要使用该函数时，应释放与其关联的指针索引 audio_free_buffer_sound()。如果您无法执行此操作，然后重新分配变量或更改房间等...，声音ID将会丢失，并且您将发生内存泄漏。另请注意，如果从缓冲区创建了任何声音并且未先释放声音，则无法删除该缓冲区。因此，您将首先释放声音，然后删除缓冲区。还值得注意的是，在从缓冲区中创建任何声音后，向缓冲区中添加任何内容或更改缓冲区的大小都会产生意外的结果，因此不建议-一旦开始从任何缓冲区创建声音，就不应操作之后以任何其他方式进行。

句法：

audio_create_buffer_sound(bufferId, bufferFormat, bufferRate, bufferOffset, bufferLength, bufferChannels);

争论描述
bufferId要使用的缓冲区的ID。
bufferFormat缓冲区中数据的格式（buffer_u8或者 buffer_s16）。
bufferRate缓冲区中数据的采样率。
bufferOffset缓冲区中的偏移量，用于从中读取样本数据（以字节为单位）。
bufferLength缓冲区的长度（音频数据的字节数，不包括标题）。
bufferChannels从下面列出的常量之一使用的通道。

返回值：

Integer (Sound ID)

例子：

var rate = 44100;
var hertz = irandom_range(220, 880);
var samples = 44100;
var bufferId = buffer_create(rate, buffer_fast, 1);
var buffer_seek(bufferId, buffer_seek_start, 0);
var num_to_write = rate / hertz;
var length = buffer_get_size(bufferId) ;
var val_to_write = 1;
for (var i = 0; i < (samples / num_to_write) + 1; i++;)
   {
   for (var j = 0; j < num_to_write; j++;)
      {
      buffer_write(bufferId, buffer_u8, val_to_write * 255);
      }
   val_to_write = (1 - val_to_write);
   }
soundId = audio_create_buffer_sound(bufferId, buffer_u8, rate, 0, length, audio_stereo);

上面创建了一个缓冲区，然后按照程序将其填充数据。然后，该数据用于创建新的声音，并将其存储在变量“ soundId”中。
