audio_play_sound_on

使用此功能，您可以通过发射器播放任何声音资源，并且发射器增益，位置，音高或速度的任何变化都会影响用户听到最终播放声音的方式。您提供要使用的发射器索引，要播放的声音的声音索引，然后指定声音是否循环播放。最后，您可以分配一个优先级，该优先级用于确定当播放的声音数量超过该功能设置的限制时如何处理声音 audio_channel_num()。优先级较高的声音将停止优先级较低的声音，优先级值可以是任何实数（实际值是任意的，并且可以是0到1或0到100，因为GameMaker Studio 2会将它们优先）。需要注意的是优先处理时，数字越大，优先级越高，使得具有优先级100的声音将优于健全优先级为1。

此功能还将为正在播放的声音返回唯一的索引号，然后将其存储在变量中，以便您可以使用适当的功能暂停或停止它。这意味着，如果您一次具有多个播放同一声音的实例，则可以仅使用音频功能来定位该声音的一个实例。

句法：

audio_play_sound_on(emitter, sound, loop, priority);

争论描述
emitter要使用的发射器的索引。
sound要使用的声音的索引。
loop将声音标记为循环播放或不循环播放。
priority设置声音的通道优先级。

返回值：

Index

例子：

if global.SFX
   {
   audio_play_sound_on(s_emit, snd_Explode, false, 1);
   }

上面的代码检查全局变量“ SFX”，并返回 true那么索引在变量“ snd_Explode”中的声音将通过索引在变量“ s_emit”中的发射器播放，优先级低且没有循环。
