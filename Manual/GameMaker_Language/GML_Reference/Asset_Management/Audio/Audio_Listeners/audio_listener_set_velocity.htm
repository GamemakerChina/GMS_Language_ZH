audio_listener_set_velocity

此函数可用于赋予给定的收听者多普勒效果，并基于从给定的相对x，y和z位置解析的矢量来模拟音频运动。默认的侦听器索引为0，但是您可以使用该函数 audio_get_listener_info()获得适用于目标平台的不同索引。

如果给定的听众永远不会移动，或者该移动不是恒定的运动，则通常无需设置这些值，但是，例如，如果您正在制作滚动游戏，其中玩家的底部恒定不变，顶部运动和敌人恒定的上下运动，您可以设置侦听器和发射器的速度（对于发射器，您可以使用 audio_emitter_velocity()）转换为适当的向量，以在它们经过播放器实例时模拟正确的多普勒效果。

下图显示了如何为上述示例游戏进行设置：

句法：

audio_listener_set_velocity(index, x, y, z);

争论描述
index用于获取数据的侦听器（默认为0）。
x侦听器的x速度（默认为0）。
y侦听器的y速度（默认为0）。
z侦听器的z速度（默认为0）。

返回值：

N/A

例子：

var num = audio_get_listener_count();
for(var i = 0; i < num; ++i;)
   {
   var info = audio_get_listener_info(i);
   var data = audio_listener_get_data(info[? "index"]);
   if data[? "vx"] != 0
      {
      audio_listener_set_velocity(info[? "index"], 0, 0, 0);
      }
   ds_map_destroy(info);
   ds_map_destroy(data);
   }

上面的代码检查可用的侦听器的数量，然后遍历它们，如果它们的x速度不为0，则将其速度值设置为0、0、0。

返回：音频听众
