audio_sync_group_debug

他的函数可用于显示有关任何给定同步组的调试信息（组索引是使用该函数创建组时返回的值 audio_create_sync_group()）。您可以使用值为-1的函数来关闭覆盖。

调试音频同步组时，覆盖图将显示：

  • 哪个同步组正在调试
  • 里面有多少个频道
  • 同步组失败了多少次（这意味着它尝试执行过多的工作）
  • 频道清单
    • 红色表示此频道目前没有任何工作
    • 黄色表示频道处于初始状态（尚未解码或播放）
    • 绿色表示曲目正在搜索/解码
    • 紫色表示通道已用完数据
  • 一个配置文件栏，显示解码线程在做什么
    • 蓝色表示首次读取文件
    • 红色表示线程正在读取文件以查找要从何处读取
    • 绿色表示线程正在解码数据
    • 紫色表示线程处于休眠状态，直到需要工作为止

请注意，对于音频的常规调试，您还具有 audio_debug()。

注意：此功能不适用于HTML5目标平台。

句法：

audio_sync_group_debug(group_index);

争论描述
group_index要调试的组索引，或-1以关闭调试。

返回值：

N/A

例子：

if (debug_mode)
   {
   audio_sync_group_debug(sg);
   }
else
   {
   audio_sync_group_debug(-1);
   }

上面的代码将打开或关闭音频同步组调试覆盖图，其中显示了根据变量是否在调试模式下运行而在变量“ sg”中索引的同步组。
