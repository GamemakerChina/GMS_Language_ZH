音频_调试

这个函数可以用来显示关于音频系统的调试信息，true可以打开，false可以关闭。当你以这种方式调试音频时，覆盖层将显示以下信息。

  • Handle - OpenAL源处理（每一个播放的声音都是唯一的）。
  • 数据手柄- OpenAL缓冲区的手柄（对具有相同源手柄的多个声音显示相同的值
  • 已处理的缓冲区- 播放了多少个缓冲区
  • 排队的缓冲区- 有多少缓冲区已经排队（并且仍在等待播放）。
  • Gain - 音频的增益值（0 - 65536）。
  • 播放偏移- 采样位置（当前播放位置）进入当前的缓冲区
  • 排队的缓冲区条形图- 已处理/排队的缓冲区的小方框图，播放偏移量（当前位置）以蓝线绘制。

对于同步的音频组，你还可以使用以下函数 audio_sync_group_debug().

注意： 该功能在HTML5目标平台上不起作用。

语法。

audio_debug(enable)。

争论 描述
启用 启用（设置为真）或禁用（假）音频调试。

返回。

不适用

例子。

如果（debug_mode）
{
audio_debug(true)。
}
else
{
audio_debug(false)。
}

上面的代码将根据游戏是否在调试模式下运行，打开或关闭音频调试覆盖层。
