audio_debug

此功能可用于显示有关音频系统的调试信息，包括 true打开它并 false将其关闭。当您以这种方式调试音频时，覆盖图将显示以下信息：

  • 句柄-OpenAL源句柄（每种播放的声音都是唯一的）
  • 数据句柄-OpenAL缓冲区句柄（对于具有相同源句柄的多种声音显示相同的值）
  • 处理的缓冲区-播放了多少个缓冲区
  • 排队的缓冲区-已排队（并且仍在等待播放）的缓冲区数
  • 增益-音频的增益值（0-65536）
  • 播放偏移量-样本位置（当前播放位置）进入当前缓冲区
  • 排队的缓冲区条-处理/排队的缓冲区的小方框图，播放偏移量（当前位置）绘制为蓝线

对于同步音频组，您还具有 audio_sync_group_debug()。

注意：此功能在HTML5目标平台上不起作用。

句法：

audio_debug(enable);

争论描述
enable启用（设置为 true）或禁用（false ）音频调试。

返回值：

N/A

例子：

if (debug_mode)
   {
   audio_debug(true);
   }
else
   {
   audio_debug(false);
   }

上面的代码将根据游戏是否在调试模式下运行来打开或关闭音频调试覆盖。
