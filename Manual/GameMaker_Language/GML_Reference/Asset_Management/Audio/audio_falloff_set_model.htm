audio_falloff_set_model

为了使音频引擎更具通用性， GameMaker Studio 2允许您选择适合您游戏的衰减模型。此模型将用于游戏或应用程序中的所有音频功能，因此您应确保选择的模型是正确的模型，因为每种模型都会影响听众对通过发射器或通过发射器播放的声音的感知方式。功能 audio_play_sound_at。

当使用适当的功能在游戏中设置衰减时，需要设置三个参数，每个参数都适合于特定的模型，并且会影响玩家根据距离听到最终声音的方式侦听器是从源头发出的。这三个参数是：

  • 参考距离-这是距听众的距离，在该距离下，播放声音的音量通常会下降一半，然后才受到滚降系数或指定的最大距离的影响。
  • 最大距离-设置将不再衰减源声音的距离。这可以是不再听到声音的点或音量不再降低到所选模型定义的最小阈值以下的点。
  • 衰减系数-衰减系数基于反距离模型在距离衰减中使用，并设置具有衰减的声音的最终最小阈值。

受这些参数影响的衰减模型在GameMaker Studio 2由以下常量表示（该表也显示了使用的确切计算）：

持续的功能
audio_falloff_exponent_distance
增益=（侦听器距离/参考距离）^（-falloff_factor）
audio_falloff_exponent_distance_clamped
距离=钳位（listener_distance，reference_distance，maximum_distance）
增益=（距离/ reference_distance）^（-falloff_factor）
audio_falloff_inverse_distance
增益=参考距离/（参考距离+衰减因子*（监听器距离-参考距离））
audio_falloff_inverse_distance_clamped
距离=钳位（listener_distance，reference_distance，maximum_distance）
增益=参考距离/（参考距离+衰减因子*（距离-参考距离））
audio_falloff_linear_distance
距离=最小值（距离，最大距离）
增益=（1-falloff_factor *（距离-参考距离）/（最大距离-参考距离））
audio_falloff_linear_distance_clamped
距离=钳位（listener_distance，reference_distance，maximum_distance）
增益=（1-falloff_factor *（距离-参考距离）/（最大距离-参考距离））
audio_falloff_none
增益= 1


下图是上述常量如何工作并影响播放声音的直观表示：

句法：

audio_falloff_set_model(model);

争论描述
model用于设置衰减模型的常数。

返回值：

N/A

例子：

audio_falloff_set_model(audio_falloff_exponent_distance_clamped);
audio_play_sound_at(snd_Waterfall, x, y, 0, 100, 300, 1, true, 1);

上面的代码设置了衰减模型，然后播放在变量“ snd_Waterfall”中索引的声音，该变量将在其房间位置循环播放，衰减参考值为100，衰减距离为300，衰减系数为1，并且低优先级。
