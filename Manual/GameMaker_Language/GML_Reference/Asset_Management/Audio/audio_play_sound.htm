audio_play_sound

使用此功能，您可以播放游戏中的任何声音资源。您提供声音索引并为其分配优先级，然后该优先级用于确定当播放的声音数量超过该功能设置的限制时如何处理声音 audio_channel_num()。优先级较高的声音将停止优先级较低的声音，优先级值可以是任何实数（实际值是任意的，并且可以是0到1或0到100，因为GameMaker Studio 2会将它们优先）。需要注意的是优先处理时，数字越大，优先级越高，使得具有优先级100的声音将优于健全优先1.最后一个参数是使声音循环并将其设置为 true将使声音循环直到停止并将其设置为 false只会播放一次声音。

此功能还将为正在播放的声音返回唯一的索引号，然后将其存储在变量中，以便您可以使用适当的功能暂停或停止它。这意味着，如果您一次具有多个播放同一声音的实例，则可以仅使用音频功能来定位该声音的一个实例。

句法：

audio_play_sound(index, priority, loop);

争论描述
index播放声音的索引。
priority设置声音的通道优先级。
loop设置声音是否循环播放。

返回值：

Index (sound ID)

例子：

if health <= 0
    {
    lives -= 1;
    audio_play_sound(snd_PlayerDead, 10, false);
    }

上面的代码检查“运行状况”全局变量，如果它小于或等于0，它将从“活动”全局变量中删除1并播放声音。
