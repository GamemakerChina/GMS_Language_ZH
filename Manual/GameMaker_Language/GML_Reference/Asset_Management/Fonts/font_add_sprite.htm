font_add_sprite

使用此功能，您可以使用“ sprite strip”创建新的字体资源，其中每个子图像将是一个单独的符号或字母。索引是要使用的第一个字符的UTF8映射号，并且使用的范围与普通字体相同，以“！”开头。但请记住，您应该只包括需要的内容，并且应该使用 ord()函数以获取起始字母的正确UTF8值。然后，创建的sprite字体会将子图像设置为该字体中的单个字形，其中“第一个”字形的image_index为0。您可以将“ space”定义为所需的任何字符，例如单个字符设置所需空间的大小，然后文本中的所有空间都将以该宽度呈现（但所选图像将永远不会呈现），或者如果您不为空间提供图片，则最宽的宽度将使用精灵字体中的字符代替。

您还可以将字体的间距设置为成比例（true ） 或不 （false ），其中比例字体根据字母的实际宽度（例如，字母“ i”比字母“ w”占用更少的空间）隔开，而非比例字体将根据子字母-图像宽度，如等宽字体。最后，您可以定义书写时每个字母之间的间距，该间距可以是任何整数值，0表示没有空格（如果成比例，字母将“接触”）。该函数返回索引值，该索引值应存储在变量中，因为在引用该字体的所有其他代码中都将需要此索引值；如果函数由于任何原因失败，则它将返回-1。

还值得注意的是，字符图中不能有任何空格，因此，例如，如果使用“。”。和“ 0”而不是“ /”，则无论如何都需要包括“ /”，或者在两个使用的字符之间保留空白的子图像。

精灵本身可以是资源浏览器中的精灵资产，也可以是使用以下命令从文件加载的精灵资产 sprite_add首先获取加载的精灵的索引（然后在函数中使用该索引）。请务必注意，对于字体对齐功能（例如 draw_set_halign()）以使用sprite字体按预期工作时，sprite的原点应设置在左上角。如果使用其他值，则在使用字体绘制文本时需要考虑原点偏移。

注意：将字体加载到GameMaker Studio 2 ，必须记住当不再需要该字体时再次使用（ font_delete（） ）将其删除，否则存在内存泄漏的风险，这会减慢速度并最终导致游戏崩溃。

句法：

font_add_sprite(spr, first, prop, sep);

争论描述
spr用于添加字体的精灵。
first第一个可用字符的索引。
prop是否设置为比例字体。
sep每个字母之间要留出的空间。

返回值：

Real

例子：

global.Font = font_add_sprite(spr_HUD_Font, ord("!"), true, 2);

上面的代码将从一个sprite资产创建一个新的字体资产，并将新字体的索引存储在变量“ global.Font”中，以备后用。
