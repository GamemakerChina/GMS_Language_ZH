font_add_sprite_ext

使用此功能，您可以使用“小精灵条”（小精灵本身必须是资源浏览器中的小精灵资源，或使用以下方式添加到游戏资源中的小精灵：sprite_add ）来创建新的字体资源，其中每个子图像都是一个单独的符号或字母。不像一般 font_add_sprite()对于子画面的子图像有特定的顺序，此函数将基于函数的参数“ string_map”映射子画面子图像。此参数是一个字符串，您可以用来告诉GameMaker Studio 2 Sprite字体的子图像是什么顺序，并且在编写文本时会自动映射它们。因此，例如，如果您的字符串映射为“ AaBbCcDdEeFfGgHh”，则您的精灵字体必须具有以这种方式排序的子图像。您可以将“空格”定义为您想要的任何字符，例如，一行您希望该空格的大小，并且文本中的所有空格都将以该宽度呈现（但是所选的图像将永远不会呈现），或者，如果您不为空间提供精灵，则将使用精灵字体中最宽字符的宽度来代替。

您还可以将字体的间距设置为成比例（true ） 或不 （false ），其中比例字体根据字母的实际宽度（例如，字母“ i”比字母“ w”占用更少的空间）隔开，而非比例字体将根据子字母-图像宽度，如等宽字体。最后，您可以定义书写时每个字母之间的间距，该间距可以是任何整数值，0表示没有空格（如果成比例，字母将“接触”）。该函数返回索引值，该索引值应存储在变量中，因为在引用该字体的所有其他代码中都将需要此索引值；如果函数由于任何原因失败，则它将返回-1。

值得注意的是，字体对齐功能可以正常工作（例如 draw_set_halign()），然后将字体精灵的原点设置在左上角。如果使用其他值，则在使用字体绘制文本时需要考虑原点偏移。

注意：将字体加载到GameMaker Studio 2 ，必须记住当不再需要该字体时再次使用（ font_delete（） ）将其删除，否则存在内存泄漏的风险，这会减慢速度并最终导致游戏崩溃。

句法：

font_add_sprite_ext(spr, string_map, prop, sep);

争论描述
spr用于添加字体的精灵。
string_map从其获取子画面子图像顺序的字符串。
prop是否设置为比例字体。
sep每个字母之间要留出的空间。

返回值：

Real

例子：

global.Font = font_add_sprite_ext(spr_CalcFont, "0123456789+-*/=", true, 2);

上面的代码将根据在变量“ spr_CalcFont”中索引的精灵创建一个新的字体资源，并将新字体的索引存储在变量“ global.Font”中。 Sprite字体的子图像将映射到指定的字符串。
