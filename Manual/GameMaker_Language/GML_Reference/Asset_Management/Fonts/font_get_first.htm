font_get_first（不建议使用）

GameMaker Studio 2定义字体时，可以定义要包括的字符范围。这是因为字体（实际上出于法律原因）实际上并未包含在您的游戏中，而是在纹理页面中包含了字体的图像，这就是您的游戏将使用的图像（就像其他任何图形资源一样）。这意味着您将需要使使用的字符数保持最少，并仅指定游戏所需的字符范围，以使纹理内存尽可能最佳。此功能可用于查找将字体资源添加到游戏中时使用的起始字符（作为ASCII值）。

注意：由于字体处理方式的更改，此功能将始终返回32，因为GameMaker Studio 2将空格字符（32）强制添加到字体中，并且也不允许代码低于32的字符。在以后的更新中将添加执行类似任务的新功能。

句法：

font_get_first(ind);

争论描述
ind要检查的字体索引。

返回值：

Real

例子：

num = font_get_first(fnt_Main);

上面的代码将存储“ fnt_Main”中索引的字体的字体范围首字母的ASCII值。
