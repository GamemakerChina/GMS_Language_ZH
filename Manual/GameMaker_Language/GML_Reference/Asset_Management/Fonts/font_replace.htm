font_replace

此功能可用于从运行游戏的系统上安装的那些字体替换您的游戏中的字体。请注意，要替换的字体不能是资源浏览器中的字体，而必须是以前使用函数添加的字体。font_add() 。

以这种方式替换字体时，可以定义字体的大小（以磅为单位），以及字体应为粗体还是斜体，还可以定义要包括的字符范围。如果函数失败，则它将静默失败，并且不会替换给定的字体。

对于HTML5目标模块，此功能只能用于替换Web字体，并且仅在未启用WebGL的情况下才有效-如果启用了WebGL，则必须使用常规字体资源（即：draw_set_font()以及资源浏览器中的字体）或位图字体（请参见 font_add_sprite()）。

重要的！定位HTML5时，仅当字体资产或Web字体本身包含这些字符时，才支持斜体和粗体文本。它们不会从基本字体中合成。

在所有其他（非Web）目标上，您可以使用此功能替换文件中的字体。该文件GameMaker Studio 2的“包含文件”功能包含在游戏包中，并且必须是 *.ttf格式的字体文件，对于添加非标准字体（如亚洲或阿拉伯）很有用。

警告！：如果您在游戏中包含*.ttf格式的字体，则该字体必须获得许可才能随游戏一起分发。

以这种方式从文件加载字体时，字体大小以像素为单位，并且忽略第一个和最后一个值，这意味着将添加字体中的所有字形。以这种方式添加的字体被分配给自己的纹理页面，因此使用此功能时应格外小心，因为它将在绘制时增加纹理交换的数量。还应注意的是，字体在绘制时可能会显得稍大，因为字形可能具有绘制在为字体定义的边界框之外的部分。您还应该知道，对于特定字体，可以在单个帧中渲染的字形大约有200个限制。

注意：在将字体加载到GameMaker Studio 2 ：Studio中时，必须记住当不再需要它时再次使用font_delete（）删除它，否则存在内存泄漏的风险，该速度会减慢并最终导致游戏崩溃。

句法：

font_replace(ind, name, size, bold, italic, first, last);

争论描述
ind要替换的（先前添加的）字体的索引。
name要使用的字体名称（例如“ Arial”），或者如果包含该字体，则为文件路径 *.ttf文件。
size字体大小-网络字体的点数，像素 *.ttf字体。
bold字体是粗体（true）还是不是（false）。
italics字体是斜体（true）还是不是（false）。
first要包含的第一个字符（如果不确定，请输入32）。
last要包含的最后一个字符（如果不确定，请输入128）。

返回值：

Real

例子：

font_replace(myfont, "Arial", 24, true, true, 32, 128);

这将用大小为24pt，使用“ Arial”且粗体和斜体的新字体替换在变量“ myfont”中索引的字体。字体范围包括大写和小写字母，数字和所有常见标点符号。
