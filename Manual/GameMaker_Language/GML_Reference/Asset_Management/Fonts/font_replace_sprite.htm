font_replace_sprite

使用此功能，您可以使用“ sprite strip”替换以前创建的sprite字体资产，其中每个子图像都是单独的符号或字母。索引是要使用的第一个字符的UTF8映射号，并且使用的范围与普通字体相同，以“！”开头。但请记住，您应该只包括需要的内容，并且应该使用 ord()函数以获取起始字母的正确UTF8值。

您还可以将字体的间距设置为成比例（true ） 或不 （false ），其中比例字体根据字母的实际宽度（例如，字母“ i”比字母“ w”占用更少的空间）隔开，而非比例字体将根据子字母-图像宽度，如等宽字体。最后，您可以定义书写时每个字母之间的间距，该间距可以是任何整数值，0表示没有空格（如果成比例，字母将“接触”）。

如果该函数失败，则它将静默失败，而不替换给定的sprite字体。

t还值得注意的是，字符图中不能有任何空格，因此，例如，如果您使用'。和“ 0”而不是“ /”，则无论如何都需要包括“ /”，或者在两个使用的字符之间保留空白的子图像。下图显示了典型的精灵字体布局：

句法：

font_replace_sprite(ind, spr, first, prop, sep);

争论描述
ind应替换的字体的索引（即，此功能之后对该资源的所有引用将引用新字体，而不是原始字体）。
spr用于添加字体的精灵。
first第一个可用字符的索引。
prop是否应删除每个字母周围的空格（如果为false，则创建的字体将是等宽的，否则它将像大多数常规字体一样）。
sep每个字母之间的空格，0表示一个字母的右边正好碰到下一个字母的左边。

返回值：

N/A

例子：

font_replace_sprite(global.Font, working_directory + "Sprites\spr_Font.png", ord("!"), true, 2);

上面的代码将用来自外部图像文件的新字体资产替换在全局变量“ Font”中索引的字体资产。
