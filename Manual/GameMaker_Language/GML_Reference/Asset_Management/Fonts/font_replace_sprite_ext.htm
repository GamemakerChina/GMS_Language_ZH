font_replace_sprite_ext

该函数将替换您先前使用该函数创建的Sprite字体 font_add_sprite_ext()。它使用一个“ sprite strip”，其中每个子图像都是一个单独的符号或字母，并且根据函数的参数“ string_map”将图像映射为字母。此参数是一个字符串，您可以用来告诉GameMaker Studio 2 Sprite字体的子图像是什么顺序，并且在编写文本时会自动映射它们。因此，例如，如果您的字符串映射为“ AaBbCcDdEeFfGgHh”，则您的精灵字体必须具有以这种方式排序的子图像。

您还可以将字体的间距设置为成比例（true ） 或不 （false ），其中比例字体根据字母的实际宽度（例如，字母“ i”比字母“ w”占用更少的空间）隔开，而非比例字体将根据子字母-图像宽度，如等宽字体。最后，您可以定义书写时每个字母之间的间距，该间距可以是任何整数值，0表示没有空格（如果成比例，字母将“接触”）。该函数返回一个索引值，该索引值应存储在变量中，因为在引用该字体的所有其他代码中都将需要此索引值。

如果该函数失败，则它将静默失败，而不替换给定的sprite字体。

注意：此功能要求您以前使用font_add_sprite（）添加了精灵字体，因为它将替换该字体，并且不能在游戏资源中包含的字体上使用。

句法：

font_replace_sprite_ext(font, spr, string_map, prop, sep);

争论描述
font要替换的字体。
spr用于添加字体的精灵。
string_map从其获取子画面子图像顺序的字符串。
prop是否设置为比例字体。
sep每个字母之间要留出的空间。

返回值：

N/A

例子：

font_replace_sprite_ext(global.Font, spr_CalcFont, "0123456789+-*/=", true, 2);

上面的代码将用在变量“ spr_CalcFont”中索引的精灵图像替换在变量“ global.Font”中索引的精灵字体资产。
