停用实例

GameMaker Studio 2您可以“关闭”实例，从而不再以任何方式对其进行处理。从技术上讲，它们实际上不再存在，除了作为停用过程本身中的指针外，这意味着停用的实例在再次被重新激活之前根本无法进行任何操作或更改。因此，应谨慎使用这些功能，因为当未正确使用这些功能时，尤其是对于持久性对象，它们可能会引起问题，因为已停用的持久性对象将不会移至下一个房间，除非先将其重新激活（有效地）。从游戏中删除它）。还要注意，激活和停用不是瞬时的，以这种方式更改了状态的实例在调用函数的事件结束之前不会被认为是活动/非活动的。

通常没有必要在游戏的每个步骤中停用实例，这实际上可能导致游戏出现性能问题。相反，建议您仅在警报（例如）中每隔几步运行一次这些功能，或者如果摄像机视图已更改位置，则尤其重要的是不要在绘制事件中使用这些功能，因为这可能会导致游戏中的严重错误。

被取消激活的实例实际上在游戏中不再存在，但是单个实例仍然可以访问其变量。你不能使用 with(object)或者 with(instance)但是，您必须使用以下格式的唯一ID直接访问该实例：

val = inst.variable;

在上面的示例中，“ inst”将是一个保存已停用实例ID的变量，这是可以引用已停用实例而无需事先激活它的唯一方法。

注意：如果您在房间启动时停用了一个实例（即：从房间创建代码，或从房间内实例的实例创建事件），则从房间编辑器放置在房间内的所有实例仍将运行其创建事件在停用之前。

警告：停用启用了物理的实例将不会阻止其夹具在物理模拟中进行交互。为此，您应该将其phy_active变量设置为true或 false当您激活/停用实例时。

存在以下用于停用实例的功能：

  • instance_activate_all
  • instance_activate_object
  • instance_activate_region
  • instance_activate_layer
  • instance_deactivate_all
  • instance_deactivate_object
  • instance_deactivate_region
  • instance_deactivate_layer
