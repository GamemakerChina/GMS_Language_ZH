停用实例

GameMaker Studio 2为你提供了 "关闭 "实例的可能性，使它们不再以任何方式被处理。从技术上讲，它们实际上已经不存在了，只是作为停用过程中的一个指针而已 本身，这意味着停用的实例不能以任何方式被操作或改变，直到它被再次激活。所以，这些函数的使用应该非常小心，因为如果使用不当就会引起问题，特别是对于 持久物体，因为一个被停用的持久物体不会被移到下一个房间，除非它被重新激活（有效地从游戏中删除它）。还要注意的是，激活和停用并不是即时的。 而一个以这种方式改变了状态的实例在调用该函数的事件结束之前不会被认为是激活/不激活的。

通常没有必要在游戏的每一步都停用实例，这实际上会导致你的游戏出现性能问题。相反，我们建议你只在警报中每隔几步运行这些函数（例如），或者 如果摄像机视图改变了位置，特别重要的是，你不要在绘制事件中使用这些函数，因为这可能导致你的游戏出现严重的错误。

一个停用的实例实际上在游戏中已经不存在了，但是单个实例仍然可以访问它们的变量。但是你不能使用with(object)或者with(instance)。 而必须使用它的唯一ID，以下列形式直接访问该实例。

val = inst.variable;

在上面的例子中，"inst "将是一个保存停用实例的ID的变量，这是停用的实例可以被引用的唯一方式，而它之前没有被激活。

注意：如果你在房间启动时停用一个实例（即：从房间创建代码中，或从房间内的一个实例的创建事件中），所有从房间编辑器中放置在房间内的实例在停用前仍会运行 他们的创建事件，然后才被停用。

警告： 停用启用了物理的实例不会停止它们的灯具在物理模拟中的互动。为此，你应该把它们的phy_active变量设置为 真实或 false ，因为你激活/停用了这些实例。

以下是停用实例的功能。

  • 实例_激活_所有
  • 实例_激活_对象
  • 实例_激活_区域
  • 实例_激活_层
  • 实例_停用_所有
  • 实例_停用_对象
  • 实例_停用_区域
  • 实例_停用_层
