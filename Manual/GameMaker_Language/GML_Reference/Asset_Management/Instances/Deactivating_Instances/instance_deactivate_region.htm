实例_停用_区域

通过这个功能，你可以在房间里定义一个区域，以停用以前被激活的实例。这个区域可以被标记为 "内部 "或 "外部"，如下图所示。

你可以在上面的图片中看到，"苹果 "实例总是 处于非活动状态，因为即使精灵本身没有与区域重叠，但其边界框却与之重叠。所以，当实例的边界框与区域重叠时，它们被认为是在指定的区域内，而碰撞掩码的状态 的状态（精确与否）不被考虑。请注意，停用不是即时的，以这种方式停用的实例在调用该函数的事件结束前不会被认为是不活动的。 调用。

注意：如果你在房间启动时停用一个实例（即：从房间创建代码，或从房间内一个实例的创建事件），所有从房间编辑器放置在房间内的实例在被停用之前仍然会运行它们的创建事件。

警告：停用启用了物理的实例不会停止它们的灯具在物理模拟中的互动。为此，你应该设置它们的 phy_active变量 为真或假，因为你激活/停用了这些实例。

语法。

instance_deactivate_region(left, top, width, height, inside, notme)。

争论 描述
左边 要停用的矩形区域左边的X坐标。
顶部 要停用的矩形区域顶部的y坐标。
宽度 要停用的区域的宽度。
高度 要停用的区域的高度。
内幕 是否停用区域内部（true）或外部（false）的实例。
不是我 是否将调用实例排除在停用之外（true）或不排除（false）。

返回。

不适用

例子。

instance_activate_all();
var _vx = camera_get_view_x(view_camera[0]);
var _vy = camera_get_view_y(view_camera[0]);
var _vw = camera_get_view_width(view_camera[0]);
var _vh = camera_get_view_height(view_camera[0]);
instance_deactivate_region(_vx - 64, _vy - 64, _vw + 128, _vh + 128, false, false）。)

上面的代码激活了所有的实例，然后停用了房间内的一个区域。
