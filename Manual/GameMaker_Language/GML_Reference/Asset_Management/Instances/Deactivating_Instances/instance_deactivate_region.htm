instance_deactivate_region

使用此功能，您可以在房间内定义一个区域以停用先前已激活的实例。可以将这个区域标记为“内部”或“外部”，如下图所示：

您可以在上图中看到“ apple”实例始终处于不活动状态，因为即使子画面本身不与区域重叠，边界框也会处于活动状态。因此，当实例的边界框与实例重叠时，它们被视为在指定的区域内，并且不考虑碰撞蒙版的状态（精确与否）。请注意，停用不是瞬时的，以这种方式停用的实例在调用函数的事件结束之前不会被认为是非活动的。

注意：如果您在房间启动时停用了一个实例（即：从房间创建代码，或从房间中实例的实例创建事件），则从房间编辑器放置在房间中的所有实例仍将运行其创建事件在停用之前。

警告：停用启用了物理的实例将不会阻止其夹具在物理模拟中进行交互。为此，您应该设置 phy_active可变为 true或者 false当您激活/停用实例时。

句法：

instance_deactivate_region(left, top, width, height, inside, notme);

争论描述
left要停用的矩形区域左侧的x坐标。
top要停用的矩形区域顶部的y坐标。
width要停用的区域的宽度。
height要停用的区域的高度。
inside是停用区域内部（true）还是外部（false）上的实例。
notme是否从取消激活中排除调用实例（是）（否）（否）。

返回值：

N/A

例子：

instance_activate_all();
var _vx = camera_get_view_x(view_camera[0]);
var _vy = camera_get_view_y(view_camera[0]);
var _vw = camera_get_view_width(view_camera[0]);
var _vh = camera_get_view_height(view_camera[0]);
instance_deactivate_region(_vx - 64, _vy - 64, _vw + 128, _vh + 128, false, false);

上面的代码激活所有实例，然后禁用房间内的区域。

返回：停用实例
