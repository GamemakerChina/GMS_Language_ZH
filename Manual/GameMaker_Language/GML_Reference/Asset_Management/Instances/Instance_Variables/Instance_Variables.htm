实例变量

创建新对象时，它将带有某些已经使用默认值初始化的变量。这些变量我们称为实例变量，因为它们将为您将对象放置在游戏室中的每个实例创建，并且一旦创建了实例，这些变量的值对于该实例是唯一的，并且仅是该实例。其中一些变量您会经常使用，例如 x和 y，虽然根据您想要做什么，对其他项的需求较少，但总的来说，它们非常有用，您应该尽可能地使用它们，而不是创建自己的自定义变量。

大多数实例变量都可以设置和读取，从而允许您通过调整特定变量的值来更改实例的属性和行为-例如，可以通过简单地设置实例变量来防止绘制实例 visible内置变量 false。在下面，您可以找到为游戏中所有对象的所有实例初始化的不同变量。

重要的！如果实例位于sequence然后是其中一些变量-例如image_xscale/image_yscale / image_angle /x /y -当序列在开始播放后更新每个步骤时，将被覆盖。

这些变量处理常规实例属性：

  • id
  • visible
  • solid
  • persistent
  • depth
  • layer
  • alarm

这些变量处理实例的位置和移动：

  • direction
  • friction
  • gravity
  • gravity_direction
  • hspeed
  • vspeed
  • speed
  • xstart
  • ystart
  • x
  • y
  • xprevious
  • yprevious

对象属性

以下变量保存创建实例所依据的对象的唯一索引ID：

  • object_index

这些变量都与分配给实例的精灵有关，可用于更改绘制内容和方式：

  • sprite_index
  • sprite_width
  • sprite_height
  • sprite_xoffset
  • sprite_yoffset
  • image_alpha
  • image_angle
  • image_blend
  • image_index
  • image_number
  • image_speed
  • image_xscale
  • image_yscale

这些变量处理碰撞蒙版：

  • mask_index
  • bbox_bottom
  • bbox_left
  • bbox_right
  • bbox_top

这些变量与路径以及实例（如果已分配）与实例的交互方式有关：

  • path_index
  • path_position
  • path_positionprevious
  • path_speed
  • path_scale
  • path_orientation
  • path_endaction

这些变量用于将实例设置为使用时间轴：

  • timeline_index
  • timeline_running
  • timeline_speed
  • timeline_position
  • timeline_loop

顺序

以下变量适用于在序列中使用实例的情况：

  • in_sequence

有许多内置变量可与GameMaker Studio 2的物理功能一起使用，因此为了使内容更清楚，可以在手册的涵盖物理仿真所有内容的部分中找到它们：

  • 物理
