depth

创建对象时，可以为其分配初始深度，该初始深度定义了在玩游戏时如何在房间中绘制该对象的实例，并且在游戏进行时可以使用此变量获取和更改该深度值。跑步。通常，您不需要使用此变量，因为您应该设置要添加到离散图层的实例，然后将其设置为特定的深度，但是可能是您想使用此值更改实例的深度，在这种情况下，将为给定深度的实例专门创建一个“临时层”。请注意，如果没有相同深度的实例，则将从内存中删除此临时层（与常规层不同，即使它们上没有任何东西，它们也将保留）。

重要的！当您修改 depth变量和GameMaker Studio 2管理图层，内置的图层变量将返回-1而不是图层ID，因为不能通过代码来操纵被管理的图层。

在GameMaker Studio 2 ，实例的深度值越低，将绘制该实例的“距摄像机更近”，而深度值越高，表示该实例将“离摄像机更远”绘制，即：-1000在-100的顶部绘制，在-0的顶部绘制，在100的顶部绘制，在1000顶部绘制。

请注意，具有相同的深度可以在上面或彼此甚至低于被绘制时出现它们的对象的实例，以在房间编辑器被放置为一个在另一个的顶部上。如果要保证在其他所有物体上方或下方绘制了某些内容，则应始终明确设置深度（或图层）。还要注意，存在一个最小（-16000）和最大（+16000）深度值，在这些深度值之外将不会绘制实例，尽管它们仍将存在并处理事件。

注意：您不能在其绘制事件中设置实例的深度（其他所有事件都可以）。

句法：

depth;

返回值：

Real (single precision floating point value)

例子：

if y != yprevious
    {
    depth = -y;
    }

上面的代码将检查y位置是否已更改，如果已更改，则深度也将设置为与其对应。

返回：实例变量
