layer

将为房间中的每个实例创建此内置变量，该变量包含实例分配到的图层的图层ID值。然后可以将此值用于其他功能，例如 layer_get_depth()或者可以更改为将实例移动到另一个图层，但是请注意，如果当前房间中不存在分配的图层，则会出现错误，迫使游戏关闭。分配层时，必须提供函数返回的唯一层ID layer_get_id() （当使用命名的房间层时），或者由函数返回 layer_create()（当您在运行时创建自己的图层时）。

重要的！如果您使用instance_create_depth（）函数创建了实例，或者手动更改了depth变量，则分配给实例的图层将变为“托管”图层，这是GameMaker Studio 2自动控制和管理的图层。在这些情况下 layer变量将返回-1。

句法：

layer;

返回值：

Real (integer - resource index value)

例子：

layer = layer_create(-1000);

上面的代码将创建深度为-1000的新层，然后设置实例 layer变量到返回的图层ID，将实例从当前所在的图层移动到正在创建的新图层。

返回：实例变量
