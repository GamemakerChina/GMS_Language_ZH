persistent

可以读取此变量以了解实例是否标记为 persistent是否可以，或者它可以用来设置持久性 true（持续）或 false（不是永久的）实例。持久实例是一个将在每个房间之间“传递”的实例，例如，这意味着它仅在游戏开始时就必须创建一次，并且将出现在所有其他房间中。应谨慎对待持久性，因为它很容易丢失持久性实例的轨迹，而持久性实例可能会在以后的游戏开发中导致问题。

创建在房间中启用了持久性的实例时，此变量将设置为 true并将为其分配一个层或一个深度（取决于您用来创建对象实例的功能）。当房间被改变，如果下面的房间没有使用相同的名称或深度作为一个分配，那么一个新的层将针对跨房间坚持实例被创建一个层。如果您在创建实例时给了命名层，则新层也将与原始层相同，而如果您为实例分配了深度，则新层将被命名为“_layer_XXX “，其中” XXX“是用于为图层赋予唯一名称的十六进制值。这些图层将从Room End的房间中删除。关于图层和持久实例的另一点注意事项是，如果您为实例分配了层创建后，接下来的房间中还有另一个同名层，则无论层的深度如何，持久化的实例都将分配给具有相同名称的层。如果层的深度与创建实例时的深度相同，则不会将其分配给该层，而是将以相同的深度创建一个新层（遵循前面解释的命名约定）。

永久对象仍然会触发其Game Start ， Game End ， Room Start和Room End事件，但是如果您重新启动游戏（例如，使用 game_restart()功能），所有持久对象将被删除，并且只有在游戏再次创建时才存在。还要注意，如果您取消激活持久对象，除非在触发“房间结束”事件之前将其重新激活，否则它将不再从一个房间传递到另一个房间。

句法：

persistent;

返回值：

Boolean

例子：

if !persistent
    {
    persistent = true;
    }

上面的代码将检查实例以查看持久性是否标记为 false如果实例不是持久性的，则持久性标志设置为 true。
