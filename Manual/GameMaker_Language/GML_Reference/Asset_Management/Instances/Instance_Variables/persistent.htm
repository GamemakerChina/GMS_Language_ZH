这个变量可以被读取以了解实例是否被标记为持久化，也可以用来将实例的持久化设置为真（持久化）或假（非持久化）。一个持久化的实例是一个将 从一个房间到另一个房间的 "延续"，这意味着（例如）它只需要在游戏开始时创建一次，它就会出现在所有其他房间里。对持久性要小心，因为很容易失去对持久性实例的跟踪。 实例，这可能会导致在游戏开发的后期出现问题。

当你在一个房间里创建一个启用了持久性的实例时，这个变量将被设置为"真"，并且它将被分配一个层或一个深度（取决于你用哪个函数来创建对象的实例）。当房间被改变时。 如果下面的房间没有一个与所分配的图层名称或深度相同的图层，那么将为该实例创建一个新的图层，该实例将在各个房间中持久存在。如果你在创建实例的时候给了一个命名的图层。 如果你在创建实例时给了一个命名的图层，那么新的图层也将被命名为与原图层相同，而如果你给实例分配了一个深度，那么新的图层将被命名为"_layer_XXX"，其中 "XXX "是一个十六进制值，用于给图层一个唯一的 名称。这些图层将在房间结束时从房间中移除。关于图层和持久化实例需要注意的另一件事是，如果你在创建时给实例分配了一个命名的图层，而在接下来的房间里有另一个同名的图层，那么持久化的实例就会被删除。 相同的名字，那么持久化的实例将被分配到具有相同名字的层，而不考虑该层的深度。最后，如果一个持久化的实例移动到一个房间，其层的深度与该实例创建时的深度相同，那么它将不会被分配到这个层。 将不会被分配到这个层，而是在同一深度创建一个新的层（按照前面解释的命名惯例）。

一个持久化对象仍然会被触发其游戏开始、游戏结束、房间开始和房间结束事件，但是如果你重新启动游戏（例如用game_restart()函数），所有持久化对象都会被删除，只有在游戏重新创建时才会存在。 被删除，只有在游戏再次创建时才会存在。还要注意的是，如果你停用了一个持久性对象，它将不再从一个房间传到另一个房间，除非在房间结束事件被触发之前重新激活。
