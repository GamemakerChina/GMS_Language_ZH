instance_change

您可以使用此功能将一个对象的一个实例更改为另一个对象的另一个实例，并在执行此操作时决定是否执行初始实例Destroy和Clean Up事件以及新实例Create Event。这样，您可以（例如）将炸弹变成爆炸-在这种情况下，“ perf”参数可能是正确的，因为您希望炸弹执行其Destroy Event和Clean Up Event以及爆炸以执行其创建事件-或者您可以将玩家角色更改为其他角色-在这种情况下，“ perf”参数可能为false，因为您不希望实例执行其创建和销毁/清理事件。

值得注意的是，更改实例意味着您不应在下一步之前对该实例执行任何其他操作，尤其是尝试访问变量等时，否则会导致错误。基本上，您可以更改实例，但实际上直到当前步骤结束它才可用，因此可以直接访问它包含的任何变量（例如，调用 obj_Changed.x） 不管用。

警告！更改启用了物理的实例时，物理属性将不会保留到要更改为的新实例中。因此，您应该有适当的代码将当前实例的物理状态“转移”到新实例上，或者在其“创建事件”或对象编辑器中定义了新实例的物理属性。因此，建议您不要将此功能与启用了物理的实例一起使用，而应结合使用instance_destroy（）和instance_create_layer（）的组合。

句法：

instance_change(obj, perf);

争论描述
obj调用对象将变为的新对象。
perf是否执行该新对象的创建和销毁事件（真）或不执行（假）。

返回值：

N/A

例子：

if keyboard_check(vk_enter)
    {
    instance_change(obj_Player_Swimming, false);
    exit;
    }

当用户按下“ Enter”键时，上面的代码会将调用实例更改为“ obj_Player_Swimming”实例，而不执行原始实例destroy事件或新实例create事件，并在完成后退出代码块。
