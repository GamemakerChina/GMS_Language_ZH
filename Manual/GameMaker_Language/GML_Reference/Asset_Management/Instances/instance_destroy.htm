instance_destroy

只要您希望“销毁”实例，通常会触发销毁事件和清理事件，就可以调用此函数。这会将其从房间中删除，直到重新启动房间为止（除非房间是永久性的）。调用不带参数的函数只会破坏当前在作用域中的实例并运行代码，但是您可以提供一个可选的“ id”参数并使用该实例来定位特定实例 id值，也可以使用 object_index。例如：

instance_destroy(other);      // destroy the "other" instance in a Collision Event
instance_destroy(obj_Bullet); // destroy ALL instances of the object "obj_Bullet"

第二个可选标志允许您“关闭”销毁实例的销毁事件。默认情况下，将始终执行“销毁事件”，但是如果将此标志设置为 false那么您可以使实例自行销毁并跳过执行该事件。

重要的！这将跳过销毁事件，但仍将调用清理事件。

值得注意的是，销毁实例时，将在调用destroy函数的代码或操作之后立即调用其destroy事件。同样，尽管执行了destroy事件，该实例仍未立即从游戏中删除，它将继续执行当前事件中包含的代码。仅当当前事件结束时，才会将其从游戏中删除。

因此，例如，如果您具有以下代码：

f hp <= 0 instance_destroy();
score += 10;

即使 instance_destroy函数已被调用，该实例最终将在事件结束时从您的游戏中删除。请注意这一点，就好像您已经为实例创建了一个动态资源（如数据结构），然后在destroy事件中销毁了该资源（如果在destroy函数或动作之后有任何引用）执行完之后，您会收到“未知资源”错误，因为destroy事件将其从游戏中删除了。

句法：

instance_destroy([id, execute_event_flag]);

争论描述
id要销毁的实例ID或object_index（可选，默认为调用实例）
execute_event_flag设置 true或者 false是否执行销毁事件（可选，默认为 true）

返回值：

N/A

例子：

if bbox_right < 0 || bbox_left > room_width || bbox_bottom < 0 || bbox_top > room_height
    {
    instance_destroy(id, false);
    }

如果实例超出房间范围，则以上代码将销毁运行该代码的实例，而无需调用Destroy Event。

返回：实例
