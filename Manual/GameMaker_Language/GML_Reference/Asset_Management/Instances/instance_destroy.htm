实例_销毁

每当你想 "销毁 "一个实例时，你就会调用这个函数，通常会触发一个销毁事件和一个清理事件。 这将把它从房间中移除，直到房间被重新启动（除非房间是持久的）。调用没有参数的函数将简单地销毁当前在范围内并运行代码的实例，但你可以提供一个可选的 "id "参数，并通过使用实例 id 值来锁定一个特定的实例，或者你可以通过使用一个 object_index. 比如说

instance_destroy(other); // 在碰撞事件中销毁 "other "实例
instance_destroy(obj_Bullet); // 销毁 "obj_Bullet "对象的所有实例。

第二个可选标志允许你 "关闭 "被销毁实例的Destroy事件。默认情况下，Destroy事件将一直被执行，但是如果你把这个标志设置为false，那么你可以让 销毁自己，并跳过执行该事件。

重要的是!这将跳过销毁事件，但清理事件仍将被调用。

值得注意的是，当你销毁一个实例时，其销毁事件会在调用销毁函数的代码或动作之后立即被调用。另外，虽然执行了destroy事件，但该实例并没有立即从游戏中删除。 它将继续执行当前事件中包含的代码。只有当当前事件结束时，它才会被从游戏中删除。

所以，如果你有，比如说，这个代码。

f hp <= 0 instance_destroy();
score += 10;

即使调用了instance_destroy函数，变量 "score "也会被递增，在事件结束时，这个实例将最终从你的游戏中删除。请注意这一点，因为如果你已经（例如 为实例创建了一个动态资源，比如一个数据结构，然后在destroy事件中销毁了该资源，如果在destroy函数或动作执行后还有对该资源的引用，那么你会得到一个 "未知的 资源 "的错误，因为destroy事件把它从游戏中删除了。

语法。

instance_destroy([id, execute_event_flag])。

争论 描述
身份证 要销毁的实例ID或object_index（可选，默认为调用实例）。
执行_事件_flag 设置为真或假，以执行或不执行销毁事件（可选，默认为真）。

返回。

不适用

例子。

if bbox_right < 0 || bbox_left > room_width || bbox_bottom < 0 || bbox_top > room_height
{
instance_destroy(id, false);
}

如果实例在房间范围之外，上述代码将销毁运行代码的实例，而不调用Destroy事件。
