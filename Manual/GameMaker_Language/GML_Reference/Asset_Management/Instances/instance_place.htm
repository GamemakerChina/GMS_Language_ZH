instance_place

使用此功能，您可以使用运行检查代码的实例的碰撞屏蔽来检查与另一个实例或对象的所有实例的碰撞位置。使用此功能时，您实际上是在要求GameMaker Studio 2将实例移动到新位置，检查是否有碰撞，然后移回并告诉您是否发现碰撞。这将适用于精确的碰撞，但是仅当实例和要检查的对象都选择了精确的碰撞蒙版时，否则仅应用边界碰撞。此函数将返回唯一实例 id发生碰撞的物体的角度，但是如果不需要，则使用该功能的速度会稍快 place_meeting()。此功能还接受特殊关键字 all和 other并返回关键字 noone如果没有冲突发生，或者如果确实发生冲突，则找到实例的唯一实例ID值。

请注意，在执行检查之前，给定的x / y坐标将四舍五入为最接近的整数，因此，如果这不是您所需要的（或者您一直在使用旧版GameMaker产品），则应 floor检查中的坐标：

instance_place_list(floor(x), floor(y), obj_Enemy, list, true).

句法：

instance_place(x, y, obj);

争论描述
x检查实例的x位置。
y检查实例的y位置。
obj要检查其实例的对象。

返回值：

Real (either an instance ID value or the keyword noone)

例子：

var _inst = instance_place(x, y, obj_Enemy);
if _inst != noone
    {
    hp -= _inst.dmg;
    instance_destroy(_inst);
    }

上面的代码将检查是否与“ obj_Enemy”实例发生冲突，如果存在冲突，它将减少“ hp”变量，减少冲突实例的“ dmg”变量中存储的数量，然后销毁冲突实例。
