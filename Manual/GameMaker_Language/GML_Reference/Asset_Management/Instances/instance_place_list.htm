instance_place_list

使用此功能，您可以使用运行检查代码的实例的碰撞屏蔽来检查与对象的所有实例的碰撞位置。使用此功能时，您实际上是在要求GameMaker Studio 2将实例移动到新位置，检查是否有碰撞，然后向后移动并告诉您是否发现碰撞。这将适用于精确的碰撞，但是仅当实例和要检查对象的实例都选择了精确的碰撞蒙版时，否则，仅应用边界碰撞。该函数将使用唯一的字段填充（先前创建的） DS列表 id对象的所有实例的值都被视为与调用实例发生冲突，并且您可以选择根据从进行检查的实例的原点到发现的实例的原点之间的距离对列表进行排序在碰撞中。请注意，该函数还接受特殊关键字 all，在这种情况下，将列出所有发现冲突的实例。该函数返回找到的实例数，如果找不到则返回0。

请注意，在执行检查之前，给定的x / y坐标将四舍五入为最接近的整数，因此，如果这不是您所需要的（或者您一直在使用旧版GameMaker产品），则应 floor检查中的坐标：

instance_place_list(floor(x), floor(y), obj_Enemy, list, true).

句法：

instance_place_list(x, y, obj, list, ordered);

争论描述
x检查实例的x位置。
y检查实例的y位置。
obj要检查其实例的对象。
list用于存储冲突实例ID的DS列表。
ordered该列表是否应按距离排序（true ） 或不 （false ）。

返回值：

Int (The number of instances found to be in collision)

例子：

var _list = ds_list_create();
var _num = instance_place_list(x, y, obj_Enemy, _list, false);
if _num > 0
    {
    for (var i = 0; i < _num; ++i;)
        {
        instance_destroy(_list[| i]);
        }
    }
ds_list_destroy(_list);

上面的代码将检查与在调用实例位置找到的所有实例是否冲突。这些实例将添加到DS列表中，然后循环遍历以破坏发生冲突的每个实例。
