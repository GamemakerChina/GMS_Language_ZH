生成对象事件

GameMaker Studio 2完全是事件驱动的，并且所有操作都是由于某些事件的发生而发生的，这些事件以特定的顺序或游戏中的特定事件触发。有许多不同的事件，每个事件都有特定的任务或时间安排，其中一些甚至被划分为单独的“子”事件。通常，这些事件由GameMaker Studio 2在检测到游戏内触发或每个游戏滴答时运行（如step和draw事件的情况），但是您也可以将事件应用于当前房间中的实例在一段代码内。

存在以下用于处理事件的功能：

  • event_inherited
  • event_perform
  • event_perform_object
  • event_user

上面列出的功能还可以使用许多只读变量，这些变量对于调试特别有用：

  • event_action
  • event_number
  • event_object
  • event_type

有关事件的更多信息，请参见“对象事件”部分。
