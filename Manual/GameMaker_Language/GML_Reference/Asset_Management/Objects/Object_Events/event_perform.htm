事件_执行

这个函数将对运行代码的实例执行指定的事件，并带有指定的参数。这里有许多选项，可以完全模拟所有可能的事件，但请注意，这只是执行事件中的所有代码 事件中的所有代码，游戏将不会修改任何东西来使其手动触发，例如，如果你选择执行一个键盘按压事件，事件将被触发，但相关的键将不会被识别为已被按下。或者如果你 执行一个警报事件，警报计数器将不会被设置为-1，而是继续倒计时。

下表概述了GameMaker Studio 2中包含的所有用于引用事件类型的常量，以及它们的数字值。事件本身用粗体字标记，与变量的返回值相同。 事件类型 变量的返回值，而它们的数字值则以斜体字标出，这些值与 事件_编号变量的返回值。

不适用

扩展实例


为了帮助理解如何使用这个功能，请看下面几个例子。

event_perform(ev_keypress, ord("W"))。

这将执行与键盘检查按下的 "W "键相关的事件（没有实际产生键盘按下）。

event_perform(ev_step, ev_step_begin)。

这将执行开始步骤事件（如果从任何一个步骤事件中调用，将导致开始步骤事件代码被运行两次）。

event_perform(ev_create, 0);

这将执行 实例的创建 事件。
