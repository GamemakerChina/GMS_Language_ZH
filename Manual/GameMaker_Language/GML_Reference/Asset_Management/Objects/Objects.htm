对象

在GameMaker Studio 2您有对象并且有实例。对象本质上是实例的基本模板，因此永远不会直接存在于房间中……仅将对象的实例放置在房间中。这意味着，如果要为要创建的所有实例更改某些内容，则可以通过更改object来完成。

重要的！更改有关对象的任何内容都不会更改房间中当前存在的任何实例，只会更改那些在更改之后创建的实例。

您还可以访问有关基础对象的信息，并使用它来指示游戏中的行为或更改。例如，您可以检查父对象ID，然后在以下代码中，如果找到具有该父对象的实例，则可以检查其父对象ID。object_index决定应该做什么。

GameMaker Studio 2提供了许多功能，可让您获得有关对象的各种详细信息。请注意，对象不是实例（游戏内实体），它纯粹是用于创建所有实例的资源。这意味着这些函数的返回值可能与房间中实例显示的实际值不同，因为一旦创建实例，就可以通过代码和DnD™动作对其进行更改。

  • object_index
  • object_exists
  • object_get_name
  • object_get_mask
  • object_get_parent
  • object_get_persistent
  • object_get_solid
  • object_get_sprite
  • object_get_visible
  • object_get_physics
  • object_is_ancestor

还有许多功能可让您设置对象的属性。应该注意的是，房间中已经存在的该对象的任何实例都可能不受这些功能的影响，但是将在房间中创建该对象的所有新实例，因此建议您不要在以下情况下更改对象的属性该对象的实例存在于当前房间中。

  • object_set_mask
  • object_set_persistent
  • object_set_solid
  • object_set_sprite
  • object_set_visible
