object_set_persistent

使用此功能，您可以设置特定对象的持久性。持久对象在各个房间之间“保留”，并始终保持其属性（有关持久性的更多信息，请参见“对象编辑器”部分）。设置对象的持久性意味着在更改持久性之后创建的该对象的所有实例将被持久创建，而房间中已经存在的实例可能不会受到影响。

当您创建在房间中启用了持久性的实例时，将为它分配一个图层或深度（取决于您用来创建对象实例的功能）。当房间被改变，如果下面的房间没有使用相同的名称或深度作为一个分配，那么一个新的层将针对跨房间坚持实例被创建一个层。如果您在创建实例时给了命名层，则新层也将与原始层相同，而如果您为实例分配了深度，则新层将被命名为“_layer_XXX “，其中” XXX“是用于为图层赋予唯一名称的十六进制值。这些图层将从Room End的房间中删除。关于图层和持久实例的另一点注意事项是，如果您为实例分配了层创建后，接下来的房间中还有另一个同名层，则无论层的深度如何，持久化的实例都将分配给具有相同名称的层。如果层的深度与创建实例时的深度相同，则不会将其分配给该层，而是将以相同的深度创建一个新层（遵循前面解释的命名约定）。

请注意，这不是实例函数！您可以使用设置单个实例的持久性标志 persistent变量，因此即使有问题的对象在IDE中的标志设置为false，也有十个实例都被标记为持久性实例，即使您使用此功能将对象的持久性标志更改为true，当前房间中的所有实例将保持原样，并且仅在调用函数后创建的实例将以persistent标记为true开始。

句法：

object_set_persistent(index, pers);

争论描述
index要更改的对象的索引。
pers是否应将对象标记为持久性（true ） 或不 （false ）。

返回值：

N/A

例子：

if (room == rm_final)
    {
    object_set_persistent(obj_Player, false);
    }
instance_create_layer(32, 32, "Instances", obj_Player);

上面的代码检查实例当前在哪个房间中，以及它是否位于指定的“ rm_final”中，然后将对象“ obj_Player”的持久性设置为false，然后创建实例。
