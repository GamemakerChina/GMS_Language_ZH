path_start

从一系列链接在一起的定义点创建路径，然后将其用于计划实例的移动。可以使用代码或在路径编辑器中创建它们，并将它们分配给要在游戏中使用的实例。然后，您将使用此功能告诉实例要遵循的路径，遵循的速度（以每步像素为单位），到达路径末端时的行为方式以及遵循绝对路径还是相对路径位置。这最后一部分意味着它要么完全按照您在路径编辑器中设计并放置的路径开始并遵循路径（绝对），要么它从创建实例的位置开始（并相对于路径）。

如您在上图中所看到的，两个实例都遵循以 path_start()，但其中一个实例遵循绝对路径，而另一个实例遵循相对路径。应该注意的是，当relative设置为false（绝对）时，无论将实例放置在房间中的哪个位置，该实例都会“跳转”到路径的起点。

用于定义实例到达路径末端时的行为的常量如下：

持续的描述
path_action_stop结束道路
path_action_restart从起点继续路径，如果路径未闭合，则再次跳至起点位置
path_action_continue从当前位置继续
path_action_reverse再次沿路径后退（通过反转路径移动速度来实现）

句法：

path_start(path, speed, endaction, absolute);

争论描述
path要开始的路径索引。
speed沿着路径的速度（以像素/步为单位），负数表示向后移动。
endaction到达路径末端时该怎么办（请参见下面列出的常数）。
absolute调用实例是应遵循编辑器中的绝对路径（true），还是应遵循其当前位置的相对路径（false）。

返回值：

N/A

例子：

path_start(path, 4, path_action_reverse, 0);

这将使调用实例以4像素/步的速度遵循在变量“ path”中索引的路径。它会按照相对于其自身位置的路径行进，到达终点时，它将反转方向并返回其原路。
