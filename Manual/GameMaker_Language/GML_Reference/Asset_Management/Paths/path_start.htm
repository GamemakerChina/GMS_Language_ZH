路径_开始

路径是由一系列定义点创建的，这些定义点被连接在一起，然后用来规划一个实例的运动。它们可以用代码创建，也可以在路径编辑器中创建，它们被分配给一个实例 以便在游戏中使用。你可以用这个函数来告诉你的实例要遵循哪条路径，遵循路径的速度（以每步的像素为单位），当它到达路径的终点时如何表现，以及是遵循绝对路径还是相对路径。 位置。最后一部分意味着，它要么完全按照你在路径编辑器中设计和放置的路径开始并跟随路径（绝对），要么从实例创建的位置开始并跟随路径 创建实例的位置（相对）。

正如你在上图中所看到的，两个实例遵循的是由path_start()开始的相同路径。 但其中一个实例遵循的是绝对路径，而另一个实例遵循的是相对路径。应该注意的是，当相对路径被设置为false（绝对）时，无论实例放在房间的哪个位置，都会 "跳 "到路径的起点。 房间的哪个位置。

用来定义实例到达路径终点时的行为的常量如下。

恒定 描述
路径_行动_停止 结束路径
路径_行动_重启 从起点继续走，如果路径没有关闭，则再次跳到起点位置
路径_行动_继续 从当前位置继续前进
逆转的路径_行动 沿着路径再次倒退（通过逆转路径移动速度实现）。

语法。

path_start(path, speed, endaction, absolute)。

争论 描述
路 要开始的路径索引。
速度 每一步跟随路径的速度，以像素为单位，负数意味着向后退。
赋予权力 当到达路径的终点时，要做什么（见下面列出的常数）。
绝对的 调用实例是否应遵循它在编辑器中的绝对路径（true）或其当前位置的相对路径（false）。

返回。

不适用

例子。

path_start(path, 4, path_action_reverse, 0)。

这将使调用的实例以每步4像素的速度沿着变量 "path "中索引的路径前进。它将沿着相对于它自己的位置的路径前进，当它到达终点时，它将改变方向并返回到 它的来路。
