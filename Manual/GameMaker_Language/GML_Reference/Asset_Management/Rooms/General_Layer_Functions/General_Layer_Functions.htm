通用层功能

GameMaker Studio 2房间编辑器允许您向房间中添加不同资产的图层。这些可以是图块地图（使用图块集），背景，子画面或序列以及实例，但是请注意，与房间编辑器不同，您不仅限于每层一个资产类型，而且可以将多个资产类型添加到同一层-例如，您可以在同一图层上同时具有背景和图块地图。请注意，添加到层的每个资产都被视为该层的元素。

本节概述了处理房间层时使用的常规功能，以及一些专门用于处理实例层元素的功能：

  • layer_exists
  • layer_get_id
  • layer_get_depth
  • layer_get_id_at_depth
  • layer_get_name
  • layer_get_all
  • layer_get_all_elements
  • layer_get_element_layer
  • layer_get_element_type
  • layer_get_x
  • layer_get_y
  • layer_get_hspeed
  • layer_get_vspeed
  • layer_create
  • layer_destroy
  • layer_x
  • layer_y
  • layer_hspeed
  • layer_vspeed
  • layer_add_instance
  • layer_has_instance
  • layer_instance_get_instance
  • layer_destroy_instances
  • layer_element_move
  • layer_set_visible
  • layer_get_visible
  • layer_depth
  • layer_force_draw_depth
  • layer_is_draw_depth_forced
  • layer_get_forced_depth
  • layer_script_begin
  • layer_script_end
  • layer_ shader
  • layer_get_script_begin
  • layer_get_script_end
  • layer_get_ shader

我们还有一些额外的功能，与在不同房间中定位图层有关。可能是您想更改不是当前房间的房间中的某些东西，或者可能要生成程序性的房间，或者是任何数量的东西。为此，我们使用以下功能：

  • layer_set_target_room
  • layer_get_target_room
  • layer_reset_target_room

您在此处执行的操作是使用适当的功能设置目标房间，然后调用图层功能是正常的。这些图层功能现在仅适用于目标房间，不适用于当前房间。完成后，您将调用reset函数以将代码范围再次返回到当前房间。

警告！在定位另一个房间时，可以使用所有常规图层功能，但不能使用instance_create_layer（）或instance_create_depth（）创建实例，图层功能layer_add_instance（）也将不可用。

最后，这是专门用于图层上的实例的（尽管图层本身可以具有其他元素，但它们不会受到以下功能的影响），您可以使用几个函数来停用或重新激活所有实例：

  • instance_deactivate_layer
  • instance_activate_layer
