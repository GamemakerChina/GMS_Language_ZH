layer_create

此功能用于在当前房间内创建一个新层。您提供新图层要占用的深度，该函数将返回唯一的图层ID值，然后该值可用于在其他功能中访问该图层。您还可以提供一个可选的“名称”参数，这将使您能够为图层指定一个特定的名称（作为字符串），并且可以使用此名称字符串来代替使用图层ID来访问图层，尽管它可以应该注意的是，使用名称字符串比仅使用ID值对性能的影响更大。如果您未指定图层名称，则会为创建的图层分配一个名称，格式为“_layer_XXX ”，其中“ XXX”是用于为图层赋予唯一名称的十六进制值。

提供深度值时，离图层所在的摄像头“越远”越远，而距图层所在的摄像头“越近”越深，因此，如果您有三个深度为-100、0、100的图层，图层将按100、0，-100的顺序绘制，以便“顶部”图层（即最接近摄影机视图的图层，因此在其他所有图层上绘制）将是深度为-100的图层。

下图显示了深度如何用于图层的示意图：

值得注意的是，虽然您可以为一个深度分配多个图层，但无法保证以任何给定的顺序绘制相同深度的图层，因此最好仅分配一个图层到任何给定的深度。另请注意，最小和最大图层深度为-16000到16000，并且不会渲染这些深度之外的任何内容。如果您需要的深度不在该范围内，则需要使用该功能 layer_force_draw_depth()。

句法：

layer_create(depth, [name])

争论描述
depth必须在其上创建层的深度。
name赋予图层的名称（字符串-可选）

返回值：

Layer ID (Real)

例子：

global.Bullet_Layer = layer_create(-100);

上面的代码将创建深度值为-100的新层，并将其ID值分配给全局变量。
