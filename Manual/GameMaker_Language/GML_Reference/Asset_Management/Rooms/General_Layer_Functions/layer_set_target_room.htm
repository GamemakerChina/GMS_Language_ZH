层_set_target_room

当你调用这个函数时，你是在告诉GameMaker Studio 2，所有进一步的图层函数都应该应用于给定的房间。通过这种方式，你可以程序性地改变或生成一个房间的层和层内容，而这个房间不是 当前房间。当你完成了向一个房间添加图层或图层元素，调用函数 layer_reset_target_room()来重置房间的目标（或者再次调用这个函数，其房间参数为 为-1）。)注意，这个函数只能用于当前房间以外的房间，其目的是为了让你可以在当前运行的房间以外的房间添加/删除层和层元素。

警告!当以另一个房间为目标时，你可以使用所有常规的图层功能，但你不能用以下方法创建实例 instance_create_layer() 或 instance_create_depth()创建实例，也不能使用层函数 layer_add_instance() 也不能使用。

语法。

layer_set_target_room(room)

争论 描述
房间 所有进一步层功能的目标空间

返回。

不适用

例子。

layer_set_target_room(rm_Game);
var l = layer_get_id("SpriteAssets");
repeat(50)
{
var _x = random(room_get_width(rm_Game));
var _y = random(room_get_height(rm_Game));
layer_sprite_create(l, _x, _y, spr_Trees);
}
layer_reset_target_room()。

上面的代码将目标房间设置为 "rm_Game "房间，然后获得该房间中名为 "SpriteAssets "的层的ID。这个图层ID然后被用来向该图层添加50个随机的精灵资产，然后图层目标 被重置为当前房间。

返回。一般层的功能
