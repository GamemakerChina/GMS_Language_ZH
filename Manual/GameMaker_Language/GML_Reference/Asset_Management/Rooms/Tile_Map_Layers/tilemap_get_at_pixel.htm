瓷砖地图_获取像素点（tilemap_get_at_pixel

使用这个函数，你可以从瓦片地图元素的某个位置（在房间内）检索瓦片数据。你给瓦片贴图元素的ID（你在创建瓦片贴图元素时使用 layer_tilemap_create() 或当你使用函数 layer_tilemap_get_id())以及房间里的x和y位置来获取瓦片数据，函数将返回瓦片数据 "blob"。这个数据 本质上是一个位掩码，包含瓦片索引、翻转/旋转/镜像布尔运算和任何已经应用的掩码数据（详见 tilemap_set_mask()详细信息），然后产生的数据值可以在 产生的数据值可以在 tile_* 函数中使用，以改变瓷砖的属性。如果你需要从一个特定的瓦片单元中获得瓦片数据，你应该使用函数 tilemap_get()来代替。

重要的是!如果瓷砖地图中的瓷砖没有变化（即：它们没有旋转或翻转等......），那么瓷砖集数据的返回值将完全等于瓷砖集上瓷砖的索引。因此，你可以用一块瓷砖创建 "碰撞 瓷砖的 "碰撞图"，使用瓷砖集中索引为1的瓷砖--例如--然后使用这个函数来检查1或0（空瓷砖）来计算碰撞。

语法。

tilemap_get_at_pixel(tilemap_element_id, x, y)。

争论 描述
tilemap_element_id 获取瓦片数据的瓦片地图元素的唯一ID值。
x 沿着X轴的位置，以获取瓦片数据（房间像素）。
y 沿着Y轴获取瓷砖数据的位置（房间像素）。

返回。

实数（如果有错误则为-1

例子。

var lay_id = layer_get_id("Tiles_sky");
var map_id = layer_tilemap_get_id(lay_id);
var data = tilemap_get_at_pixel(map_id, 64, 128);
data = tile_set_flip(data, true);
tilemap_set_at_pixel(map_id, data, 64, 128) 。

上面的代码得到了 "Tiles_Sky "图层上的瓷砖地图 "Clouds "的ID，然后用它来获取位置（64, 128）的瓷砖数据。这个瓦片数据在被用来设置瓦片地图上的瓦片之前被翻转了一下 再一次。
