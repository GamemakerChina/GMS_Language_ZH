tilemap_set_mask

此功能可用于设置单个图块图的图块位掩码。您提供了平铺地图元素ID（使用 layer_tilemap_create()或使用该功能时 layer_tilemap_get_id()），然后是掩码值。这个想法是，例如，如果您有一个非常小的图块集（16 x 16个图块），那么您的图块集中共有256个不同的图块，它们仅使用图块数据“ blob”中索引的8位”（来自可用的19位数据）。这些位可以被“屏蔽”，然后其余的瓦片数据可以用于存储其他值。

一种简单的获取初始索引掩码值的简单方法是，获取图块集中的图块数并减去1，因此16x16的图块集具有256个图块和掩码值255（或$ff （十六进制）），并且如果瓦数集的瓦数不是2的幂，然后四舍五入到最接近的2的幂以得到蒙版，例如20x20的瓦数集有400个瓦数，您将舍入到512。因此获得的掩码索引值为511（十六进制$1ff除了索引掩码外，通常还希望保留分配给图块的翻转，旋转和镜像值，因此可以使用一些常量来帮助您。可以将它们与索引掩码值一起“或”编辑以保留那些数据位。可以使用以下常量（有关用法，请参见下面的示例代码）：

持续的描述
tile_rotate用于设置/获取tile数据blob的旋转位。
tile_mirror用于设置/获取瓦片数据块的镜像位。
tile_flip用于设置/获取瓦片数据块的翻转位。
tile_index_mask一个特殊常量，用于与图块数据blob进行“与”运算以提取图块索引。

磁贴图位屏蔽是一项高级功能，可让您使用磁贴数据“ blob”的位供自己使用。例如，用户可以在图块数据结构中保留一些意思为“实体”，然后可以检查是否存在快速冲突。基本上，位图是在绘制图块地图时针对图块数据进行“和”编辑的（因此它不会影响其他任何功能），这意味着将忽略不在掩码中的位，但是可以用户仍可出于自己的目的对其进行读写。请注意，有两种不同的蒙版-全局蒙版和每个贴图对应一个-并由GameMaker Studio 2在内部进行“和”编辑，然后再用于绘制。

句法：

tilemap_set_mask(tilemap_element_id, mask)

争论描述
tilemap_element_id瓦片地图元素的唯一ID值，用于设置蒙版
mask掩码值

返回值：

N/A

例子：

var lay_id = layer_get_id("Tiles_sky");
var map_id = layer_tilemap_get_id(lay_id);
var mask = tile_mirror | tile_flip | tile_rotate | 255;
tilemap_set_mask(map_id, mask);

上面的代码从给定的图层获取图块地图ID，然后将图块地图位掩码设置为8bits，并保留图中每个图块的翻转，镜像和旋转值。
