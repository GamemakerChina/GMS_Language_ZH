script_execute

此函数最初是为动态调用脚本资产而设计的，但是随着用户定义函数的引入，现在可以以两种方式使用它：

  • 第一种方法是支持旧脚本，其中每个脚本都是一个函数，而脚本的标题也是该函数的名称。以这种方式使用，它将像以前一样工作，它将运行用户创建的脚本资产并将给定的参数传递给它。请注意，在这种情况下，它实际上不是在调用脚本，而是在脚本内使用相同名称的脚本函数。
  • 第二种方法是从包含多个函数或变量定义的脚本资产中调用函数。在这种情况下，您将提供函数名称和所需的参数，而不是脚本资产名称。请注意，这仅适用于脚本函数而不适用于方法，因为只有脚本函数具有分配给它们的脚本ID。

尽管此功能主要是为旧版支持而设计的，但由于在网络和远程过程调用情况中仍具有它的地位，因此不建议过时。还值得注意的是，此功能很慢，并且大量使用它可能会对性能产生不利影响。

注意：此功能不能与内置功能一起使用。

句法：

script_execute(scr, arg0, arg1, arg2..., etc...);

争论描述
scr您要调用的函数/脚本的名称
arg0, arg1, arg2..., etc...您想要传递给函数/脚本的不同参数

返回值：

Will depend on the return value from the script/function being called

例子：

script_execute(choose(move_up, move_down, move_left, move_right), irandom(5));

上面的示例代码将使用 script_execute从四个用户定义的函数中选择一个（在本例中与移动有关），并将一个随机整数作为第一个参数传递给它。
