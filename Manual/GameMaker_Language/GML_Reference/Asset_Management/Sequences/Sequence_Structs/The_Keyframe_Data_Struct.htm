关键帧数据结构

当您为一个轨道定义关键帧结构时，您还需要定义与之相关的数据。这些数据由不同的通道组成，每个通道都可以根据您设定的轨道类型，赋予特定的 的数据类型，这取决于您为哪种类型的轨道设置关键帧数据。这些属性是：。

  • 通道。这是关键帧数据应该被应用的通道。它是一个从0开始的正整数，值得注意的是，当为 "位置 "或 "比例 "关键帧创建参数轨道时。 那么你需要使用非常具体的通道值。这就是：通道0是X位置或X比例，通道1是Y位置或Y比例。
  • spriteIndex。轨迹中要使用的精灵资产的索引。该属性仅适用于seqtracktype_graphic类型的轨迹，你可以获取或设置它。
  • soundIndex。要用于该轨道的音频资产的索引。这个属性只适用于seqtracktype_audio类型的轨道，你可以获得或设置它。
  • 播放模式。声音的播放模式。这个属性只对seqtracktype_audio类型的音轨有效，你可以获得或设置它。该属性应该是以下常数之一。
恒定 描述 价值
循环播放 播放时，声音会循环播放。 0
语音键_单反相机 声音只会播放一次然后停止。 1

  • 曲线。该属性需要一个动画曲线结构（更多信息请见这里），并且只适用于seqtracktype_real类型的轨道。如果没有使用曲线结构 则该属性的值将为-1。
  • 值。该属性只是一个与关键帧数据通道相关的值，只有在没有提供曲线结构的情况下，才适用于seqtracktype_real类型的轨道。 时，该属性仅适用于seqtracktype_real类型的轨道。
  • color：该属性会返回（或者需要，如果被设置）一个数组，用于格式为[A, R, G, B]的关键帧的颜色值。这仅适用于seqtracktype_colour类型的轨迹。 请注意，每个分量的值应该表示为0到1之间，其中0对应的是HEX值#00，1对应的是HEX值#FF（0-255，如序列编辑器中彩色轨道的颜色选择器所示）。
  • 序列。该属性将返回（或要求，如果被设置）一个序列对象结构，并且仅适用于seqtracktype_sequence类型的轨道。
  • objectIndex。该属性将返回（或者需要，如果被设置）一个对象索引，并且仅适用于seqtracktype_instance类型的轨道。
  • 事件。该属性允许访问与关键帧数据结构相关的事件和广播信息。您可以获取或设置此属性，获取时返回一个 返回一个字符串数组，而设置该属性时应指定一个字符串数组。有关事件的更多信息，请参见 "序列事件和时刻"部分。这个 属性仅适用于seqtracktype_message类型的轨道。
  • 事件。此属性将返回（或可设置为）与关键帧数据结构相关的方法。如果没有指定任何方法，或者您希望删除该方法，那么该属性应该为-1。 该属性仅适用于seqtracktype_moment类型的轨道。
