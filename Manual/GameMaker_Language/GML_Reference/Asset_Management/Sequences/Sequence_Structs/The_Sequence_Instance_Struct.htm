序列实例结构

序列放置在房间中时，将作为元素放置在图层上，并且此元素包含序列实例。此序列实例的struct中将具有以下属性：

  • sequence ：这是创建序列实例所依据的序列对象结构的ID（更多信息，请参见此处）。您可以获取或设置此结构，然后更改实例引用的基本序列。
  • headPosition ：这是序列实例的当前播放头位置（以帧为单位）。您可以获取或设置此值，但是请注意，将其设置为大于或小于当前定义的播放区域的值将产生不同的效果，具体取决于为该序列设置的播放类型。看 layer_sequence_headpos()欲获得更多信息。
  • headDirection ：这是序列实例的当前播放方向。您可以使用以下常量获取或设置此值（请注意，您可以使用图层函数 layer_sequence_get_headdir()和 layer_sequence_headdir()无需直接访问结构即可获取或设置此值）：
持续的描述价值
seqdir_right该序列将从左到右以递增顺序播放帧1
seqdir_left该序列将按从右到左的降序顺序播放帧-1

  • speedScale ：此属性可用于获取或设置播放速度比例。速度标度是一个乘数，其中1是默认播放速度，小于1的值将减慢播放速度，大于1的值将加快播放速度，例如：0.5表示一半的播放速度，而2表示将是两倍的播放速度。请注意，您可以使用图层功能 layer_sequence_get_speedscale()和 layer_sequence_speedscale()获取或设置此值，而不必直接访问该结构。
  • paused ：您可以检查此属性，以查看序列是否已暂停，它将被 true如果有，或者 false否则。这是一个只读属性，但是您可以使用layer函数 layer_sequence_pause()暂停播放，以及 layer_sequence_play()如果需要，可以再次恢复。您还可以使用layer函数检查此属性 layer_sequence_is_paused()而不是直接检查struct中的属性。
  • finished ：您可以检查此属性，以查看序列是否已完成播放，返回 true如果播放完毕，以及 false否则。这是一个只读属性，只会返回 true未设置为循环或乒乓的曲目。您还可以使用layer函数检查此属性 layer_sequence_is_finished()而不是直接检查struct中的属性。
  • elementID ：此属性保存序列元素的ID。该iD是一个简单的识别值，与房间编辑器中的一个图层相关联，然后可以与适当的图层功能一起使用，以查找已分配序列的图层或更改某些序列属性，而无需处理结构。
  • activeTracks ：此属性将保存一个“评估”结构数组，其中包含有关序列（图形，序列，音频等）中每个资产轨道的当前状态的信息。这些只是顶层轨道，返回的数组是只读的。有关每个数组条目中返回的资产跟踪结构的内容，请参见以下部分。

以下列表包含访问轨道结构时可能提供的属性（由 activeTracks父序列结构的属性，如上文所述）。每个资产跟踪结构可能具有以下某些或全部属性，具体取决于跟踪所引用的资产类型（请注意，您可以获取或设置所有这些属性，除非指定为只读），并且返回的所有值都是当前播放头位置的：

  • matrix ：这是用于绘制轨道（精灵，实例和序列轨道）的转换矩阵。
  • posx ：当前播放头位置上曲目的沿X轴的序列中位置的位置（所有资产）。
  • posy ：资产在当前播放头位置沿轨道在Y轴上的顺序位置（所有资产）。
  • scalex ：当前播放头位置上的轨道（精灵，实例和序列轨道）沿X轴的序列中资产的比例。
  • scaley ：当前播放头位置上的轨道（精灵，实例和序列轨道）沿Y轴的序列中的资产比例。
  • xorigin ：轨道资产（所有资产）的X原点。
  • yorigin ：轨道资产（所有资产）的Y原点。
  • width ：轨道（精灵，实例和序列轨道）的资产宽度（以像素为单位）。
  • height ：轨道（精灵，实例和序列轨道）的资产高度（以像素为单位）。
  • imageindex ：当前播放头位置（精灵和实例轨道）上序列中轨道上资产的图像索引。
  • imagespeed ：序列中当前播放头位置（精灵和实例轨道）上的轨道上资产的图像速度。
  • colourMultiply ：在当前播放头位置的序列（精灵，实例和序列轨道）上，轨道上资产的颜色乘数值。该值将是四个ARGB值的数组，格式为[A，R，G，B]。请注意，每个分量的值都表示为0到1，其中0对应于HEX值＃00，1对应于HEX值#FF（如序列中颜色乘法轨道的颜色选择器中所示，0-255编辑）。
  • gain ：当前播放头位置（音轨）上音轨的增益（音量）。
  • pitch ：当前播放头位置（音轨）上的音高。
  • track ：此跟踪基于的跟踪数据结构（所有资产，一个只读值）。
  • parent ：轨道的父序列实例ID（所有资产，一个只读值）。
  • activeTracks ：这是资产轨道包含的每个参数轨道（所有资产，一个只读值）的评估结构数组。参数Track Struct部分中列出了数组中每个结构的内容。
