序列对象结构

序列对象是对从序列资产中检索出的结构的命名，如你游戏的资产浏览器中定义的那样。这个结构将有以下可以改变的属性。

  • name: 这是序列的名称，是一个字符串，你可以根据需要获得或设置这个值。注意，使用函数创建的序列 sequence_create()创建的序列不会有一个名字，而这个 将仅仅是一个空字符串""。
  • loopmode。这是该序列对象的播放模式，可以得到或设置。它将返回（或可以设置为）以下常数。
恒定 描述 价值
独家新闻 序列将播放一次，完成后停止。 0
循环播放（seqplay_loop 序列将循环播放，当播放头到达播放区域的终点时，将回到起点。 1
萍水相逢_萍水相逢 序列将循环播放，当它到达播放区域的末端时，播放头将反转方向。 2

  • playbackSpeed: 这指定了序列的播放速度，根据playbackSpeedType（见下文），它被解释为每秒钟一帧或每游戏帧一帧。你可以获得或设置这个值。
  • playbackSpeedType。这指定了应该如何解释回放速度，你可以获得或设置这个值。你可以使用下列常数之一来设置这个值（或者如果检查这些常数，将返回其中的一个 属性时，这些常数之一将被返回）。)
恒定 描述 价值
速度_帧/秒 指定播放速度应被解释为每秒帧数。 0
每场比赛的速度_帧数 指定playbackSpeed应该被解释为每一帧游戏的帧数。 1

  • 长度。序列的长度，以帧为单位。你可以获得或设置这个值，但请注意，如果引用这个序列的序列实例的播放头被设置为超过新的长度，那么使一个序列变短可能会导致问题。
  • 音量。这是一个从0到1的标度值，用于缩放序列中所有音频轨道的音量。你可以获得或设置这个值，它将修改所有轨道的全局音频输出--例如，如果你有一个音量为0.8的音频轨道 音量为0.8，然后将序列音量属性设置为0.5，那么该音轨的最终音量将为0.4。
  • xorigin。这是序列沿X轴的原点。你可以获得或设置这个值。
  • yorigin。这是序列沿Y轴的原点。你可以获得或设置这个值。
  • messageEventKeyframes。这允许访问该序列的消息事件关键帧。您可以获取或设置这些消息事件，当获取此属性时，将返回一个关键帧结构数组，而设置此属性时，您应提供一个关键帧结构数组 则应提供一个关键帧结构数组。更多信息，请参见 "序列事件和时刻"页面。
  • momentKeyframes。这允许访问序列的时刻事件关键帧。您可以获取或设置这些时刻事件，当获取此属性时，将返回一个关键帧结构数组，而设置此属性时，应提供 一个关键帧结构数组。更多信息，请参见 "序列事件和时刻"页面。
  • 轨道。这允许访问序列顶层的资产轨道列表。您可以获取或设置该属性，获取该属性时，将返回一个轨道结构数组，设置该属性时，您应该 提供一个轨道结构数组。欲了解更多信息，请参见 轨道结构.

请注意，如果你想访问在资产浏览器中已经创建的序列的属性，你必须首先调用函数 sequence_get()来检索序列对象结构。 还要注意的是，对这个序列结构所做的任何改变将意味着这个序列资产的所有进一步的实例也会有这些改变，只要游戏在运行，它们就会被维持，即使你调用 game_restart() 函数。
