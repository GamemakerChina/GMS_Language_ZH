序列对象结构

序列对象是为从序列资产中检索到的结构指定的名称，如游戏的资产浏览器中所定义。此结构将具有以下可以更改的属性：

  • name ：这是序列的名称，为字符串，您可以根据需要获取或设置此值。请注意，使用函数创建的序列 sequence_create()将没有名称，而只是一个空字符串“”。
  • loopmode ：这是序列对象的播放模式，可以获取或设置。它将返回（或可以设置为）以下常量：
持续的描述价值
seqplay_oneshot该序列将播放一次，然后在完成时停止。 0
seqplay_loop该序列将循环播放，当播放头到达播放区域的末尾时，播放头将回到起始位置。 1
seqplay_pingpong当播放头到达播放区域的末尾时，该序列将循环播放，并使播放头反转。 2

  • playbackSpeed ：此选项指定序列的播放速度，视播放速度而定，可以解释为每秒帧数或游戏帧数 playbackSpeedType（见下文）。您可以获取或设置此值。
  • playbackSpeedType ：这指定了 playbackSpeed应该被解释，您可以获取或设置此值。您可以使用以下常量之一来设置值（或者，如果检查该属性，则将返回这些常量之一）：
持续的描述价值
spritespeed_framespersecond指定 playbackSpeed应该解释为每秒帧数0
spritespeed_framespergameframe指定 playbackSpeed应该解释为每游戏帧数。 1

  • length ：序列的长度（以帧为单位）。您可以获取或设置此值，但是请注意，如果引用该序列的序列实例的播放头设置为超过新长度，则使序列更短可能会导致问题。
  • volume ：这是一个从0到1的标量值，用于缩放序列中所有音频轨道的音量。您可以获取或设置此值，它将修改所有轨道的全局音频输出-例如，如果您的音频轨道的音量为0.8，然后设置顺序 volume属性设置为0.5，则音轨的最终音量为0.4。
  • xorigin ：这是沿着X轴的序列的原点。您可以获取或设置此值。
  • yorigin ：这是沿Y轴的序列的原点。您可以获取或设置此值。
  • messageEventKeyframes ：这允许访问序列keyframes您可以获取或设置这些消息事件，并且在获取此属性时，将keyframe结构数组，并且对于设置该属性，您应提供一个keyframe结构数组。有关更多信息，请参见“序列事件和时刻”页面。
  • momentKeyframes ：这允许访问序列keyframes您可以获取或设置这些时刻事件，并且在获取此属性时，将keyframe结构数组，并且对于设置该属性，您应提供一个keyframe结构数组。有关更多信息，请参见“序列事件和时刻”页面。
  • tracks ：这允许访问序列顶层的资产轨道列表。您可以获取或设置此属性，并且在获取此属性时，将返回一个跟踪结构数组，并且对于设置该属性，您应该提供一个跟踪结构数组。有关更多信息，请参见“轨道结构”部分。

请注意，如果要访问在资源浏览器中创建的序列的属性，则必须首先调用该函数 sequence_get()在资产索引上检索序列对象结构。另请注意，对此序列结构进行的任何更改都将意味着此序列资产的所有其他实例也将具有这些更改，并且只要游戏正在运行，它们就将保持不变，即使您调用 game_restart()功能。
