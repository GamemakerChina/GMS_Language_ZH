轨道结构

轨道结构既可用于顶层资产轨道，也可用于子轨道参数轨道。 轨迹的行为由两点定义，即它的名称和它的类型。轨迹结构是通过函数 sequence_track_new() 创建，并且可以使用Sequence Instance Struct的activeTracks属性或Sequence Object Struct的tracks属性从序列资产中检索。

轨迹结构可用的属性是。

  • 类型。以下常量都可以用来定义在序列中创建的轨迹类型（这些常量在生成关键帧和关键帧数据时也会用到，在下面的章节中会解释）。

    • "rotation（旋转）" - 当在一个类型为seqtracktype_real的轨道上设置时，表示该轨道的内容应被用作旋转数据。

    • "image_speed（图像速度）--当设置在类型为seqtracktype_real的轨道上时，表示该轨道的内容应作为图像速度值使用。
    • " image_index "--当设置在seqtracktype_real类型的轨道上时，表示此轨道的内容应被用作图像索引值。
    • "image_angle" - 当设置在类型为seqtracktype_real的轨道上时，表示此轨道的内容应被用作图像角度值。

  • visible。表示该轨迹是否可见（值为true）或不可见（值为false）。你可以获取或设置这个值，如果一个轨道不可见，那么它的子轨道也不会被绘制出来。
