in_sequence

这是一个内置变量，是为游戏中每个对象实例创建的实例变量的一部分。如果这个实例是由一个序列控制的。 这个变量将返回 true ，否则它将返回 false 。这是一个只读的变量，不能被改变。

注意，在控制的序列结束后，这个变量将变成假的，即使它的序列元素仍然存在，如果该序列元素再次被播放，它将变成真的。

你可以在你的玩家（或CPU）控制的对象中使用这个变量，以确保它们只在不在序列中的时候被代码移动。这可以证明对插曲很有用，当你需要在一个序列中使用相同的实例时（使用 函数），而你需要确保在序列中使用相同的实例。 sequence_instance_override_object()函数），并且你需要确保当该场景处于活动状态时，它们不能被它们的常规代码所移动。

语法。

in_sequence

返回。

布尔型

例子。

if (!in_sequence)
{
x += move_x;
y += move_y;
}

上述代码检查 in_sequence 变量，如果它是 false （意味着该实例没有被一个序列控制），那么它将move_x和move_y添加到该实例的 的位置，确保它只在不在序列中时才移动。

返回。序列
