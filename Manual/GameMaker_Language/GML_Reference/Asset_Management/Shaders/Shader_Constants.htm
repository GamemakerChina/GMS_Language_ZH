Shader常数

除了在OpenGL ES着色语言（GLSL ES） 参考页中shader功能和常量之外，您还可以使用GameMaker Studio 2独有的许多shader常量。

shader数组常量时，以下显示矩阵常量可用作数组索引 gm_Matrices：

持续的描述
MATRIX_VIEW此数组索引常量将索引保留到当前视图矩阵。调用时，返回的索引将用作数组值 gm_Matricesshader代码中的常量。
MATRIX_PROJECTION该数组索引常数保持当前投影矩阵的索引。调用时，返回的索引将用作数组值 gm_Matricesshader代码中的常量。
MATRIX_WORLD该数组索引常量保持当前世界矩阵的索引。如果您在世界空间中拥有灯光信息，则可以将其用于照明之类的事情。调用时，返回的索引将用作数组值 gm_Matricesshader代码中的常量。
MATRIX_WORLD_VIEW此数组索引常量将对世界的结果索引和视图矩阵相乘在一起。这通常用于雾之类的东西。调用时，返回的索引将用作数组值 gm_Matricesshader代码中的常量。
MATRIX_WORLD_VIEW_PROJECTION此数组索引常量将对世界，视图和投影矩阵相乘的结果的索引保持不变。这是用于顶点位置的法线变换矩阵。调用时，返回的索引将用作数组值 gm_Matricesshader代码中的常量。
MATRIX_MAX这不是数组索引，而是返回顶点shader矩阵数组的大小。
MAX_VS_LIGHTS这不是数组索引，而是返回顶点shader中的灯光数量。

shader使用以下预定义的矩阵均匀性和常量来访问GameMaker Studio 2特定值：

持续的描述
gm_Matrices[matrix]此数组常量返回一个转换矩阵，并且是GameMaker Studio 2创建的可用预定义制服之一，供您在shader代码编辑器中使用。数组索引是从上面列出的常量之一中选择的。
gm_BaseTexture这是一个2D采样器常量，该常量返回GameMaker Studio 2设置的当前对象的纹理。因此，在不调用shader情况下进行绘制时通常将使用当前的精灵，表面或纹理。
gm_LightingEnabled使用3D时，可用于获取或设置GameMaker Studio 2
gm_FogStart这可用于获取多边形开始与雾色混合的距离。
gm_RcpFogRange这可以用来获取雾最大的距离，并且什么也看不到。
gm_PS_FogEnabled这将返回 true或者 falseGPU是否启用了像素模糊。
gm_FogColour这可以用来获取GameMaker Studio 2使用的雾色。
gm_VS_FogEnabled这将返回 true或者 falseGPU是否启用了顶点雾。
gm_AlphaTestEnabled这可用于在shader alpha测试。看 gpu_set_alphatestenable()有关alpha测试的更多信息。
gm_AlphaRefValue这可用于获取当前的Alpha测试参考值。看 gpu_set_alphatestref()有关alpha测试参考的更多信息。
