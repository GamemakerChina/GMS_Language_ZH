着色器常量

除了OpenGL ES着色语言（GLSL ES）参考页中定义的着色器函数和常量之外，还有一些 的着色器常量，这些常量是GameMaker Studio 2所特有的。

在使用着色器阵列常量gm_Matrices时，以下显示矩阵常量可以作为阵列索引。

恒定 描述
MATRIX_VIEW 这个数组索引常数持有当前视图矩阵的索引。当在着色器代码中调用gm_Matrices常量时，返回的索引将作为数组值使用。
矩阵投影 这个数组索引常数持有当前投影矩阵的索引。当在着色器代码中调用gm_Matrices常量时，返回的索引将作为数组值使用。
MATRIX_WORLD 这个数组索引常数保存了当前世界矩阵的索引。如果你有世界空间的光照信息，这可以用于照明等事情。当在着色器代码中调用gm_Matrices常量时，返回的索引将作为一个数组值使用。
矩阵_世界观 这个数组索引常数持有世界和视图矩阵相乘的结果的索引。这通常用于像雾一样的东西。当在着色器代码中调用gm_Matrices常量时，返回的索引将作为一个数组值使用。 的时候，返回的索引将作为数组值使用。
矩阵_世界观_投影 这个数组索引常数持有世界、视图和投影矩阵相乘的结果的索引。这是用于顶点位置的普通变换矩阵。返回的索引将作为一个数组值用于 在着色器代码中调用gm_Matrices常量时，返回的索引将被用作数组值。
MATRIX_MAX 这不是一个数组索引，而是返回顶点着色器中矩阵阵列的大小。
MAX_VS_LIGHTS 这不是一个数组索引，而是返回顶点着色器中灯光的数量。

以下常数也可用于照明。

恒定 描述
MAX_VS_LIGHTS 着色器中可用的最大的点光源和定向光源的数量。

以下预定义的矩阵制服和常数可以在你的着色器中使用，以访问GameMaker Studio 2的特定值。

恒定 描述
gm_Matrices[矩阵] 这个数组常数返回一个变换矩阵，是GameMaker Studio 2为你创建的可用的预定义制服之一，在着色器代码编辑器中使用。数组索引是从上面列出的常数中选择的。
gm_BaseTexture 这是一个2D采样常数，它返回当前对象的纹理，正如GameMaker Studio 2所设定的那样。因此，它将是当前的精灵、表面或纹理，在没有调用着色器的情况下，通常会在绘图时使用。
gm_LightingEnabled 在使用3D时，这可以用来获取或设置GameMaker Studio 2的照明。
gm_FogStart 这可以用来获得多边形开始与雾色混合的距离。
gm_RcpFogRange 这可以用来得到雾气最大的距离，在这个距离上什么都看不到了。
gm_PS_FogEnabled 如果GPU是否启用了像素雾，这将返回真或假。
gm_FogColour 这可以用来获得GameMaker Studio 2使用的雾的颜色。
gm_VS_FogEnabled 如果GPU启用了顶点雾化，这将返回真或假。
gm_AlphaTestEnabled 这可以用来在着色器中获得alpha测试。参见 gpu_set_alphatestenable()获取更多关于alpha测试的信息。
gm_AlphaRefValue 这可以用来获得当前的阿尔法测试参考值。参见 gpu_set_alphatestref()获取更多关于alpha测试参考的信息。
