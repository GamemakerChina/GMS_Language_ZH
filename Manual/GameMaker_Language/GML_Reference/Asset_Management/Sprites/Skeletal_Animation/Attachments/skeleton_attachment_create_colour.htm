骨架_附件_创建_颜色

当你创建骨架动画精灵时，你可以指定附件槽和附件，以放入这些附件。这些简单的图像（精灵）与动画是分开的，但当它们被连接时，会随着它们所连接的骨骼一起移动。 连接到的骨骼上。通常你会在你的动画程序（Spine）中分配这些附件，但有了这个函数，你可以在运行时使用游戏中的精灵资产来创建你自己的附件。 一个字符串），然后设置 魉_指数和 图像_指数以及X和Y原点（可以是 可以与精灵属性中定义的原点不同），你还可以设置你希望在附加时应用于图像的任何变换，以及与图像混合的颜色和它的alpha（透明度）。 值。如果成功创建了附件，该函数将返回1；如果没有成功创建（你提供了一个无效的精灵索引，或者基础精灵不是Spine精灵），那么它将返回1。

值得注意的是，实例精灵可以有一个混合颜色和alpha设置(图像混合和 图像_alpha), 和附件槽一样（见函数 skeleton_slot_colour_set())，因此，分配给槽的附件精灵的最终颜色和阿尔法值将是所有这些值的组合。 所有这些值的组合。

语法。

skeleton_attachment_create_colour（name, sprite, ind, xorigin, yorigin, xscale, yscale, rot, color, alpha）。

争论 描述
名称 要创建的附件的名称（字符串）。
精灵 用于获取附件图像的sprite_index。
ind 用于获取附件图片的image_index。
xorigin 正在使用的图像的X原点。
原产地 正在使用的图像的Y原点。
xscale 图像的水平缩放比例，作为一个乘数。1=正常缩放，0.5是一半等等...
yscale 图像的垂直缩放，作为一个乘数。1=正常缩放，0.5是一半等等...
腐烂 图像的旋转。0=正常，90=逆时针旋转90°等。
颜色 要使用的颜色值（一个常数、整数或十六进制值）。
阿尔法 要使用的alpha值（从0到1）。

返回。

如果成功则为1，不成功则为-1

例子。

skeleton_attachment_create_colour("sword", spr_Weapons, 0, 0, 80, 1, 1, 90, c_red, 1);
skeleton_attachment_create_colour("killer", spr_Weapons, 1, 0, 45, 1, 90, c_yellow, 1);
skeleton_attachment_create_colour("crossbow", spr_Weapons, 0, 10, 30, 1, 1, 0, c_white, 0.5);
skeleton_attachment_set("slot_leftHand", choose("sword", "kill", "crossbow"));

上述代码将检查当前分配给名为 "slot_leftHand "的插槽的附件名称，如果返回一个空字符串，就会附加一个新的精灵。

返回。附件
