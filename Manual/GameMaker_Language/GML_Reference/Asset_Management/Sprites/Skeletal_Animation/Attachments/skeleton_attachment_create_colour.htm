skeleton_attachment_create_colour

创建骨骼动画精灵时，可以分配附件插槽和附件以放入其中。这些只是与动画分开的图像（子画面），但是在附加时会随它们所附加的骨骼一起移动。通常，您可以在动画程序（Spine）中分配这些附件，但是使用此功能，您可以在运行时使用游戏中的Sprite资产创建自己的附件。该功能要求您为附件命名（以字符串形式），然后设置 sprite_index和 image_index以及x和y的原点（可能与sprite属性中的sprite定义的原点不同），并且还可以设置希望附加在图像上的任何变换，以及与图像混合的颜色及其alpha（透明度）值。如果附件创建成功，函数将返回 1如果不是（您提供的Sprite索引无效，或者基本Sprite不是Spine Sprite），它将返回 -1。

值得注意的是，实例精灵可以具有混合颜色和alpha设置（image_blend和 image_alpha），以及附件插槽（请参见功能 skeleton_slot_colour_set()），因此为该插槽分配的附件精灵将具有的最终颜色和Alpha将是所有这些值的组合。

句法：

skeleton_attachment_create_colour(name, sprite, ind, xorigin, yorigin, xscale, yscale, rot, colour, alpha);

争论描述
name要创建的附件的名称（作为字符串）。
sprite从中获取附件图像的sprite_index。
ind用于获取附件图像的image_index。
xorigin使用的图像的x原点。
yorigin所使用图像的y原点。
xscale图像的水平缩放比例，作为乘数：1 =正常缩放比例，0.5等于一半，依此类推...
yscale图像的垂直缩放比例，作为乘数：1 =正常缩放比例，0.5等于一半，依此类推...
rot图像的旋转：0 =正常，90 =逆时针旋转90°等。
Colour要使用的颜色值（常数，整数或十六进制值）。
Alpha要使用的Alpha值（从0到1）。

返回值：

1 if successful, -1 if not

例子：

skeleton_attachment_create_colour("sword", spr_Weapons, 0, 0, 80, 1, 1, 90, c_red, 1);
skeleton_attachment_create_colour("knife", spr_Weapons, 1, 0, 45, 1, 1, 90, c_yellow, 1);
skeleton_attachment_create_colour("crossbow", spr_Weapons, 0, 10, 30, 1, 1, 0, c_white, 0.5);
skeleton_attachment_set("slot_leftHand", choose("sword", "knife", "crossbow"));

上面的代码将检查名为“ slot_leftHand”的插槽当前分配的附件名称，如果返回空字符串，则将附加一个新的精灵。
