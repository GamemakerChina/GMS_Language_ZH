骨架_骨骼_状态_获取

你的骨骼动画是由一些 "骨头 "组成的，你将在你的动画程序中定义并给它们命名，这个函数可以用来在任何时候获得命名的骨头的某些数据。注意，这些数据是指 注意，这些数据指的是骨架的当前姿势，这取决于所使用的动画集，而且该函数需要一个先前创建的DS图，然后它将有以下键和它们的 调用该函数后，它将有以下键和它们的等值。

  • "x":骨骼相对于父骨骼的局部X位置。
  • "y":骨骼相对于父骨骼的局部Y位置。
  • "角度"。骨骼相对于父骨骼的局部旋转。
  • "xscale"。骨骼的局部水平比例，参照父骨骼。
  • "yscale"。骨骼的局部垂直比例，参照父骨骼。
  • "worldScaleX"。世界尺度X的大小（总是正的）（这是一个只读值）。
  • "worldScaleY"。世界尺度Y的大小（总是正的）（这是一个只读值）。
  • "worldAngleX"。X轴的世界旋转（这是一个只读值）。
  • "worldAngleY"。Y轴的世界旋转（这是一个只读值）。
  • "worldX"。世界的X位置（这是一个只读值）。
  • "worldY"。世界的Y位置（这是一个只读值）。
  • "appliedAngle"。这是用于摆放骨架骨骼的局部旋转。
  • "父"。父骨的名称（一个字符串）。

返回的地图数据类似于你在使用默认姿势时返回的数据。 skeleton_bone_data_get()时返回的地图数据，只是现在你有额外的 "世界 "键。这些指的是骨头的位置 相对于骨骼动画精灵的根（原点）的位置，并且返回的值不考虑任何通过设置内置精灵变量（如image_xscale或image_angle）而进行的缩放或旋转。 世界值是只读的，不能被设置。

提供这个函数是为了让你可以 "拦截 "动画数据，并在它被绘制到屏幕上之前对其进行修改，因此你会希望在其他--动画更新事件中使用它，因为这个事件正好在绘制事件之前被触发。

语法。

skeleton_bone_state_get(bone, map);

争论 描述
骨骼 骨头的名称（字符串）。
地图 存储骨骼数据的（先前创建的）DS地图。

返回。

不适用

例子。

var map = ds_map_create();
skeleton_bone_state_get("head", map);
var xx = ds_map_find_value(map, "worldX");
var yy = ds_map_find_value(map, "worldY");
var deltax = mouse_x-（x + xx）;
var deltay = mouse_y - (y + yy);
var angle = -radtodeg(arctan2(deltax, deltax));
ds_map_replace(map, "angle", angle);
skeleton_bone_state_set("head", map);
ds_map_destroy(map);

上面的代码创建了一个DS地图，然后用名为 "头 "的骨头的数据填充它。然后，它提取该骨骼的世界位置，并使用该数据来设置该骨骼的 "角度"，以跟踪游戏中的鼠标位置。 游戏。
