skeleton_bone_state_get

您的骨骼动画由许多“骨骼”组成，您将在动画程序中定义这些骨骼并为其指定名称，并且该函数可随时用于获取命名骨骼的某些数据。请注意，此数据指的是骨骼的当前姿势，具体取决于所使用的动画集，并且该函数需要先前创建的DS贴图，然后在调用该函数后将具有以下键及其等效值：

  • “ x”：骨骼相对于父骨骼的局部x位置。
  • “ y”：骨骼相对于父骨骼的局部y位置。
  • “角度”：骨骼相对于父骨骼的局部旋转。
  • “ xscale”：骨骼的局部水平比例，以父骨骼为参考。
  • “ yscale”：骨骼的局部垂直比例，以父骨骼为参考。
  • “ worldScaleX”：世界比例X的大小（始终为正）（这是一个只读值）。
  • “ worldScaleY”：世界范围Y的大小（始终为正）（这是一个只读值）。
  • “ worldAngleX”： X轴的世界旋转（这是一个只读值）。
  • “ worldAngleY”： Y轴的世界旋转（这是一个只读值）。
  • “ worldX”：世界X位置（这是一个只读值）。
  • “ worldY”：世界Y位置（这是一个只读值）。
  • “ appliedAngle”：这是用于摆姿势骨骼的局部旋转。
  • “ parent”：父骨骼的名称（字符串）。

使用时返回的地图数据类似于默认姿势返回的地图数据 skeleton_bone_data_get()，直到现在您才拥有额外的“世界”键。它们是指骨骼相对于骨骼动画精灵的根（原点）的位置，并且返回的值不考虑通过设置内置的精灵变量（例如，image_xscale或者 image_angle。世界值是只读的，无法设置。

提供此功能是为了使您可以“拦截”动画数据并在将其绘制到屏幕上之前对其进行修改，因此，您希望在“其他-动画更新”事件中使用它，因为此事件在“绘制”之前触发大事记

句法：

skeleton_bone_state_get(bone, map);

争论描述
bone骨骼的名称（以字符串形式）。
map （先前创建的）DS映射，用于存储骨骼数据。

返回值：

N/A

例子：

var map = ds_map_create();
skeleton_bone_state_get("head", map);
var xx = ds_map_find_value(map, "worldX");
var yy = ds_map_find_value(map, "worldY");
var deltax = mouse_x - (x + xx);
var deltay = mouse_y - (y + yy);
var angle = -radtodeg(arctan2(deltay, deltax));
ds_map_replace(map, "angle", angle);
skeleton_bone_state_set("head", map);
ds_map_destroy(map);

上面的代码创建了一个DS映射，然后使用名为“ head”的骨骼中的数据填充它。然后，提取骨骼的世界位置，并使用该数据设置骨骼的“角度”以跟踪游戏中的鼠标位置。
