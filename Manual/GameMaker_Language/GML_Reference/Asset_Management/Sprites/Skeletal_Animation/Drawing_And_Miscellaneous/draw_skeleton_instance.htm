draw_skeleton_instance

该功能仅适用于使用骨骼动画程序Spine创建的具有指定精灵的实例。虽然您可以使用常规绘制功能绘制这些子画面（例如 draw_self()），有时您可能想绘制一个框架或显示皮肤纹理的变化而实际上不更改实例的sprite属性。在这些情况下，您可以使用此功能绘制已分配给实例的子画面，在其中您提供唯一的实例ID，然后给出从中获取帧的动画名称（一个字符串，该字符串在程序中用于精灵）。接下来，给出要使用的皮肤名称（同样，是制作精灵时设置的名称）和帧值。帧值用于告知GameMaker Studio 2要显示动画的哪一部分，并且是介于0和 image_number。其余参数与用于普通精灵绘制的参数相同，它们将更改要绘制的最终动画帧的比例，角度，混合颜色和Alpha。

警告！由于骨骼动画是在各种“关键”帧之间进行插值的方式，因此此调用会产生额外的开销，因此，除非绝对必要，否则建议您避免使用此方法绘制基于骨骼的精灵。

句法：

draw_skeleton_instance(instance, animname, skinname, frame, x, y, xscale, yscale, rot, colour, alpha);

争论描述
instance从其绘制精灵的实例的索引。
animname要从中获取帧的动画的名称（一个字符串）。
skinname要使用的皮肤名称（字符串）。
frame要绘制的动画帧（从0到image_number-1）。
x绘制精灵的位置的x坐标。
y绘制子画面的y坐标。
xscale Sprite的水平缩放比例，作为乘数：1 =正常缩放比例，0.5等于一半，依此类推...
yscale Sprite的垂直缩放比例，作为乘数：1 =正常缩放比例，0.5等于一半，依此类推...
rot精灵的旋转。 0 =正常，90 =逆时针旋转90度，依此类推。
colour混合精灵的颜色。
alpha子画面的Alpha（从0到1，其中0是透明的，而1是不透明的）。

返回值：

N/A

例子：

draw_skeleton_instance(obj_Player, "jump", "skin1", 0, x, y, 1, 1, 0, c_white, 1);

上面的代码将使用“跳转”动画集和定义为“ skin1”的皮肤绘制分配给给定实例的骨骼动画精灵的第一帧。
