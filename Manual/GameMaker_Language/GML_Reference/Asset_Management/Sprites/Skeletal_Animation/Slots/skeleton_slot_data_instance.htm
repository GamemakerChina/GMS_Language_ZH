skeleton_slot_data_instance

使用此功能，您可以为（预先创建的） DS列表填充与当前实例相关联的Spine Sprite的每个可用附件插槽的数据（包括任何附件修改）。此数据在DS映射的论坛中返回，该映射包含以下键/值对：

  • “名称”：插槽的名称。
  • “骨头”：骨头的名称。
  • “附件”：如果使用附件，则为附件的名称；如果未分配附件，则为“（无）”。

每个键的值将是字符串，然后可以在这些类型的Sprite的其他骨架附加函数中使用。请注意 DS Map创建的地图不会被销毁，因此您需要遍历DS列表并自己销毁每个创建的地图。

句法：

skeleton_slot_data_instance(list);

争论描述
list用DS映射填充的DS列表的ID。

返回值：

N/A

例子：

var list = ds_list_create();
var open = true;
slot_name = "";
skeleton_slot_data_instance(list);
for (var i = 0; i < ds_list_size(list); i++;)
    {
    var map = list[| i];
    if open
        {
        if map[? "attachment"] == "(none)"
            {
            open = false;
            slot_name = map[? "name"];
            }
        }
    ds_map_destroy(map);
    }
ds_list_destroy(list);

上面的代码创建一个DS列表，然后使用实例Sprite的广告位数据填充它。然后解析该数据以提取带有时隙数据的单个DS映射。然后检查是否有空插槽，如果有，则在销毁DS映射之前为变量“ slot_name”分配空插槽名称。最后，我们销毁了不再需要的DS列表。
