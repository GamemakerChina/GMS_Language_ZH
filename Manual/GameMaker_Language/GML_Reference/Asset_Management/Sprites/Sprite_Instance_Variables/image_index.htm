image_index

子画面由一个或多个子图像组成，这些子图像可以使子画面在从一个子画面切换到另一个子画面时显得生动，或者可以在代码之间进行切换以提供不同的“状态”，就像按钮中的按钮一样。视窗。如果精灵是动画的，则可以通过检查image_index变量来获取动画的当前帧，或者如果要更改静态精灵的状态，则可以通过将该变量设置为所需的子图像来选择新的子图像。子画面的子图像。请注意，要使此变量的更改可见，实例应没有绘制事件（因此GameMaker Studio 2将默认绘制精灵），或使用诸如draw_self()或draw_sprite_ext() （通过将image_index提供到适当的参数中）。

请注意，直接将此变量设置为一个帧将不会触发该帧上可能存在的任何广播消息。

还要注意，在使用骨骼动画精灵时，您仍然可以获取并设置image_index值-有关如何执行此操作的示例，请参见函数skeleton_animation_get_duration()

句法：

image_index;

返回值：

Real (single precision floating point value)

例子：

if (image_speed > 0)
    {
    if (image_index >= image_number - 1) instance_destroy();
    }

上面的代码检查sprite是否正在设置动画，如果是，则检查当前image_index是否在最后一帧，并在这种情况下破坏实例。
