sprite_add_from_surface

此功能的工作方式与 sprite_create_from_surface()它只是代替从选择的索引曲面的区域创建新的精灵，而是将曲面的定义区域作为新的子图像添加到先前创建的精灵（这意味着您不能将其直接添加到来自资源树，但仅适用于先前从表面创建的树，或已使用的资源精灵复制的树 sprite_duplicate()）。

应该注意的是，以这种方式添加子画面时使用的内存将比您预期的大。这是因为GameMaker Studio 2会将精灵存储为纹理页面，并且还将存储在GPU内存中，因此总内存将大于相同精灵图像文件的预期内存。

句法：

sprite_add_from_surface(index, surface, x, y, w, h, removeback, smooth);

争论描述
index要向其添加新图像的子画面的索引。
surface从中获取图像的表面的索引。
x要复制的x位置。
y要复制的y位置。
w要复制区域的宽度（从x位置开始）。
h要复制区域的高度（从y位置开始）。
removeback指示是否使所有具有背景色（左下像素）的像素透明。
smooth指示是否平滑边缘。

返回值：

N/A

例子：

spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
var i;
for (i = 1; i < 8; i +=1)
    {
    sprite_add_from_surface(spr_Custom, surf, i, 0, 32, 32, true, true, 16, 16);
    }

上面的代码从变量“ surf”中索引的曲面创建一个精灵，将其索引分配给变量“ spr_Custom”，然后使用 for循环以在整个表面上移动并捕获添加到子画面中的各个部分作为子图像。
