精灵合并

这个函数将合并参数1（"ind2"）中索引的精灵和参数0（"ind1"）中索引的精灵。图像本身并没有被合并在一起，而是图像的索引被合并。 合并，精灵 "ind2 "的子图像被附加到精灵 "ind1 "的图像上，即：它们被加在最后。请注意，如果精灵的尺寸不同，那么附加的精灵会被拉伸以适应 "ind1 "的图像尺寸 缩放到 "ind1 "上。

这种改变是永久性的，从你使用这个函数的那一刻起，直到游戏关闭或精灵被删除，被合并的精灵将被改变，然而被合并的精灵将保持不变。

注意：无论是被合并的图像资产，还是用于合并的图像资产，都不能是原始游戏资源之一。你必须使用 sprite_duplicate()首先 来创建两个新的资源（如果你正在合并包含的游戏资产），然后再合并这些资源。

注意：这个函数只对位图精灵有用，对SWF或JSON（Spine）精灵不起作用。

语法。

sprite_merge(ind1, ind2)。

争论 描述
nd1 要合并的精灵的索引。
nd2 ind1要被合并的精灵的索引。

返回

不适用

例子。

var tspr;
tspr[0] = spr_add(working_directory + "Sprite Assets\explode1.png", 16, true, true, 0, 0);
tspr[1] = sprite_duplicate(spr_Explosion);
sprite_merge(tspr[0], tspr[1]);
sprite_index = t_spr[0];
sprite_delete(tspr[1]) 。

上面的代码将一个精灵加载到一个局部变量中，然后将其与另一个被复制的精灵合并。最后，合并后的精灵被分配给实例sprite_index，而加载的精灵被删除 以释放其使用的内存。请注意，在这个操作之后的某个时刻，比如在 "销毁 "或 "房间结束 "事件中，你还需要用 sprite_delete()将合并后的精灵 从内存中删除。
