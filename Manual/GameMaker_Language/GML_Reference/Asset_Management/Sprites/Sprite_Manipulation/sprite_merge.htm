sprite_merge

此函数会将参数1（“ ind2”）中索引的精灵与参数0（“ ind1”）中索引的精灵合并。图像本身不会合并在一起，而是图像索引会合并，将来自子画面“ ind2”的子图像附加到子画面“ ind1”的子图像上，即：将它们添加在末尾。请注意，如果精灵大小不同，则将附加的精灵拉伸以适合“ ind1”的图像大小。

此更改是永久性的，从使用此功能直到游戏关闭或删除小精灵，要合并到的小精灵都会更改，但是与之合并的小精灵将保持不变。

注意：合并到的图像资产或要合并的图像资产都不能成为原始游戏资源之一。您必须先使用sprite_duplicate（）创建两个新资源（如果您要合并包含的游戏资产），然后再合并这些资源。

注意：此功能仅对位图精灵有用，不适用于SWF或JSON（Spine）精灵。

句法：

sprite_merge(ind1, ind2);

争论描述
ind1要合并的精灵的索引。
ind2与ind1合并的sprite的索引。

例子：

var tspr;
tspr[0] = spr_add(working_directory + "Sprite Assets\explode1.png", 16, true, true, 0, 0);
tspr[1] = sprite_duplicate(spr_Explosion);
sprite_merge(tspr[0], tspr[1]);
sprite_index = t_spr[0];
sprite_delete(tspr[1]);

上面的代码将一个精灵加载到局部变量中，然后将其与另一个已经重复的精灵合并。最后，将合并的精灵分配给实例 sprite_index同时删除已加载的精灵以释放其使用的内存。请注意，执行此操作后的某个时刻，例如在销毁或房间结束事件中，您还需要使用 sprite_delete()也从内存中删除合并的精灵。
