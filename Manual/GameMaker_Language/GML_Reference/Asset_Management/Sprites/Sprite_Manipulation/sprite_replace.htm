sprite_replace

此功能的工作方式几乎与 sprite_add()，它只会覆盖先前创建的Sprite索引，而不是返回要导入的Sprite的索引。使用此功能时，您应使用已创建的精灵索引，并使用其他函数将其存储在变量中 sprite_add()或者 sprite_create_from_surface()， 甚至 sprite_duplicate()，而不是资源树精灵资产。您可以使用此功能从游戏资产中替换精灵，但这意味着您将丢失要替换的精灵的参考ID，无论您是否调用 game_restart()是否，因此不建议使用。无论替换了哪个精灵，此功能都会为该精灵创建一个新的纹理页面，因此在使用它时应格外小心，因为它可能会通过增加渲染所需的纹理交换次数而对性能产生不利影响。

要加载的图像文件应始终位于 *.png格式，所有要转换为动画精灵的图像都应具有“条带”格式（请参见下图）。按照规则sprite width = strip width / sub images，它们将被划分为指定的子图像数。

如上图所示，该图片已放置在深紫色背景上，可以通过将“ removeback”参数设置为 true。通过检查子画面左下角像素的颜色，然后将其用作要删除的颜色，可以进行此操作。例如，在上图中，如果我们左下方的像素颜色为绿色，则该精灵的所有绿色部分都将被删除，而紫色背景的其余部分将被忽略。

如果选择“删除”选项，则可能还希望GameMaker Studio 2通过将“ smooth”参数设置为来使子画面的边缘平滑。true 。所有这些操作是在除去背景后，在精灵的边缘周围创建一个半透明的边框。

最后，您还可以指定子画面的x和y原点。这是将子画面“固定”到使用它的实例的位置，并且始终相对于一个子画面子图像的0,0左上角进行计算。因此，例如，将这些值设置为（16,16）的32 x 32像素的精灵将以其原点为中心。

默认情况下，所有新的精灵都会自动计算其边界框（确切的bbox取决于该精灵的大小和透明度），但是您可能希望对其进行自定义，在这种情况下，您还应该使用函数 sprite_collision_mask()。

注意：根据所选择的目标平台，您可以从何处保存和加载文件受到限制。有关更多信息，请参见文件处理。

注意：请注意，如果您正在HTML5目标游戏中使用此功能从外部服务器加载资源，则由于浏览器中的XSS保护，跨域加载资源的尝试可能会被阻止，并且可能会返回空白结果。

句法：

sprite_replace(ind, fname, imgnumb, removeback, smooth, xorig, yorig);

争论描述
ind要永久替换的精灵的索引。
fname制作新图片的图片文件名。
imgnumb子画面将被水平切割成的帧数。 1 =一幅单张图片/*.gif 。
removeback指示是否使所有具有背景色（左下像素）的像素透明。
smooth指示是否平滑边缘。
xorig原点的x坐标，相对于sprite的左上角。
yorig相对于精灵左上角的原点的y坐标。

例子：

sprite_replace(spr_banner, "gravemaker.png", 1, false, false, 0, 0);

上面的代码将用从外部源加载的另一资源替换“ spr_banner”中索引的图像资产。
