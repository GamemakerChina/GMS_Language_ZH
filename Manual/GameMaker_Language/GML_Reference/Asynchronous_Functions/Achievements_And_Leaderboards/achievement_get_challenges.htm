achievement_get_challenges

此功能将向服务器发送请求以获取有关所有当前挑战的信息，并将触发一个回调社交异步事件，该事件包含 async_load映射填充了相关的键/值对。这个的ID键 DS Map用于标识正确的回调（对于任何给定的异步事件，可以有多个触发器函数），并将其与常量配对 achievement_challenge_list_received以及每个挑战的许多其他键/值对。该地图的确切内容如下所示：

注意：此功能仅适用于iOS 。

  • “ id ”-对于此功能，应为 achievement_challenge_list_received
  • “ numchallenges ”-当前可用的挑战数（本地和远程）。
  • “ ChallengeNplayerid ”-挑战的玩家ID，其中“ N”是整数，例如EG：“ Challenge5playerid”是列表中第五个挑战的玩家ID。
  • “ ChallengeNissuerid ”-发出挑战的人的ID，其中“ N”是整数，例如EG：“ Challenge2issuerid”是列表中第二个挑战的发行者ID。
  • “ ChallengeNstate ”-挑战状态“ N”，对于无效，未决，完成或拒绝，其值为0-3（作为字符串）。
  • “ ChallengeNmessage ”-挑战“ N”的短信。
  • “ ChallengeNissueddate ”-挑战“ N”的发布日期。
  • “ ChallengeNcompleteddate ”-挑战“ N”的完成日期。
  • “ ChallengeNtype ”-给定的挑战类型。可以是两个常量之一：
持续的描述
achievement_filter_friends_only仅获取玩家朋友的分数。
achievement_filter_all_players获取所有玩家的所有分数。

  • “ ChallengeNidentifier ”-挑战的标识字符串。
  • “ ChallengeNscore ”-与挑战并列的分数。

扩展示例：

在玩家使用以下方式登录其游戏帐户后，可能会调用以下代码 achievement_login()：

achievement_get_challenges();

这将发出对当前质询信息的请求，并生成一个特殊的异步回调 async_loadds_map包含以下数据：

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_challenge_list_received
   {
   var numentries = ds_map_find_value(async_load,"numchallenges");
   for(var i = 0; i < numentries; i++;)
      {
      player_id[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"playerid");
      issuer_id[i] = ds_map_find_value(async_load, "Challenge" + string(i) +"issuerid");
      state[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"state");
      message[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"message");
      date_completed[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"completeddate");
      date_issued[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"issueddate");
      ach_type[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"type");
      ach_ident[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"identifier");
      ach_score[i] = ds_map_find_value(async_load, "Challenge" + string(i)+"score");
   }

上面的代码检查返回的 DS Map在社交异步事件中，如果其“ id”与要检查的常量匹配，则它将循环遍历映射，将所有不同的值存储在多个数组中。然后，可以使用此信息来例如在游戏中创建自己的个性化挑战页面。
