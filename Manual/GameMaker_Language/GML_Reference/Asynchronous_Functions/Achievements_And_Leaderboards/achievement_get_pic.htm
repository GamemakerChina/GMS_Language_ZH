achievement_get_pic

此功能将向服务器发送有关玩家或朋友的图像的请求，并触发包含以下内容的回调社交异步事件：async_load 映射填充了相关的键/值对。这个的ID键 DS Map用于标识正确的回调（对于任何给定的异步事件，可以有多个触发器函数），并将其与常量配对 achievement_pic_loaded以及许多其他键/值对。该地图的确切内容如下所示：

注意：此功能仅适用于iOS 。

  • “ id ”-对于此功能，应为 achievement_pic_loaded
  • “ playerid ”-我们正在接收其图像的玩家或朋友的ID。
  • “ spriteid ”-图像的精灵ID，然后可以使用任何draw sprite函数将其用于在屏幕上绘制

警告！这个函数创建精灵，因此，它们的ID应该存储在变量中，然后使用sprite_delete（）函数从内存中删除，否则重复使用会导致内存泄漏，最终会减慢游戏速度或使游戏崩溃。

句法：

achievement_get_pic(char)

争论描述
char要获取其图像的人的ID（从朋友列表或页首横幅中）

返回值：

N/A

扩展示例：

使用以下方法获得好友列表或页首横幅列表后，将调用以下代码 achievement_load_friends()或者 achievement_load_leaderboard()。这些函数将为每个人返回唯一的“ id”值，然后在函数中使用它们来检索他们的个人资料图片：

achievement_get_pic(global.player_id[0]);

这将发出对与存储在全局数组中的玩家ID关联的图像的请求，并生成带有特殊内容的异步回调 async_loadDS地图包含以下数据：

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_pic_loaded
    {
    var picid = ds_map_find_value(async_load, "playerid");
    var spriteid = ds_map_find_value(async_load, "spriteid");
    if global.playerid[0] == picid)
        {
        global.playerpic[0] = spriteid;
        }
    }

上面的代码检查返回的 DS Map在社交异步事件中，如果其“ id”与要检查的常数匹配，则它检查与图像关联的id是否与所请求的ID相同，如果是，则设置另一个全局数组来保存该图像的唯一ID图像（可用于将图像绘制为精灵）。
