achievement_load_leaderboard

此功能将向服务器发送请求，以获取给定排行榜上的信息。排行榜必须事先在您的游戏的开发人员仪表板上创建，并且给出的ID必须作为功能的第一个参数提供。它将触发一个回调社交异步事件，其中包含 async_load映射填充了相关的键/值对。这个的ID键 DS Map用于标识正确的回调（对于任何给定的异步事件，可以有多个触发器函数），并将其与常量配对 achievement_leaderboard_info以及每个玩家的许多其他键/值对。该地图的确切内容如下所示：

注意：此功能仅适用于iOS 。

  • “ id ”-对于此功能，应为 achievement_leaderboard_info
  • “ Leaderboardid ”-提供商面板上定义的排行榜的唯一ID。
  • “ numentries ”-排行榜中已收到的条目数。
  • “ PlayerN ”-玩家的名称，其中“ N”是一个整数值，对应于他们在排行榜条目列表中的位置。
  • “ PlayeridN ”-玩家的唯一用户ID，“ N”。
  • “ RankN ”-排行榜中玩家“ N”的排名。
  • “ ScoreN ”-玩家“ N”的分数。

您可以通过提供最小和最大索引值来指定排行榜的确切部分，也可以使用以下过滤器常量，要求对结果进行过滤以仅检索朋友或所有玩家发布的得分：

持续的描述
achievement_filter_friends_only仅获取玩家朋友的分数。
achievement_filter_all_players获取所有玩家的所有分数。

句法：

achievement_load_leaderboard(ident, minindex, maxindex, filter)

争论描述
ident排行榜的唯一ID，如开发人员仪表板上所示。
minindex从中获取排行榜数据的起始索引值。
maxindex从中获取排行榜数据的最大索引值。
filter是否将结果过滤为好友列表中的结果或所有玩家的结果（请参见下面的说明）

返回值：

N/A

扩展示例：

在玩家使用以下方式登录其游戏帐户后，可能会调用以下代码 achievement_login()Android或iOS上获取游戏的所有好友排行榜位置的列表：

if os_type == os_ios
    {
    achievement_load_leaderboard("CgkIs9_51u0PEAIQBw", 1, 100, achievement_filter_friends_only);
    }

这将在给定的排行榜上发出对信息的请求，并生成带有特殊内容的异步回调 async_loadDS地图包含以下数据：

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_leaderboard_info
    {
    var lbid = ds_map_find_value(async_load,"leaderboardid");
    if lbid == "CgkIs9_51u0PEAIQBw" || lbid == "leaderboard1id"
        {
        global.numentries = ds_map_find_value(async_load,"numentries");
        for(var i = 0; i < numentries; i++;)
            {
            global.playername[i] = ds_map_find_value(async_load, "Player" + string(i));
            global.playerid[i] = ds_map_find_value(async_load, "Playerid" + string(i));
            global.playerrank[i] = ds_map_find_value(async_load, "Rank" + string(i));
            global.playerscore[i] = ds_map_find_value(async_load, "Score" + string(i));
            achievement_get_pic(global.playerid[i]);
            }
        }
    }

上面的代码检查返回的 DS Map在社交异步事件中，如果其“ id”与要检查的常量匹配，则它将在循环遍历地图并将所有不同值存储在多个数组中之前，确保发送的排行榜与原始函数调用匹配。它还会为地图中的每个条目请求一张图片（请参见 achievement_get_pic()）。
