achievement_login

此功能将用户登录到适当的排行榜和成就服务。如果该服务不可用，则将用户登录到“假装”游戏中心，并将所有成就和分数存储在设备上，以便在实际服务可用时可以上载这些详细信息。该函数将触发一个回调社交异步事件，其中包含 async_load映射填充了相关的键/值对。这个的ID键 DS Map用于标识正确的回调（对于任何给定的异步事件，可以有多个触发器函数），并将其与常量配对 achievement_our_info以及每个挑战的许多其他键/值对。该地图的确切内容如下所示：

  • “ id ”-对于此功能，应为 achievement_our_info
  • “ name ”-当前登录的用户名。
  • “ playerid ”-当前登录用户的唯一玩家ID。

句法：

achievement_login()

返回值：

N/A

扩展示例：

以下代码可能会在游戏开始时立即调用，或者从您放置在房间中的特殊按钮对象中调用，并要求玩家登录其Google Game Services或Apple Game Center帐户：

achievement_login();

这将发出对当前玩家信息的请求，并生成带有特殊内容的异步回调 async_loadDS地图包含以下数据：

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_our_info
    {
    var name = ds_map_find_value(async_load, "name" );
    var playerid = ds_map_find_value(async_load, "playerid" );
    global.OurName = name;
    global.OurId = playerid;
    if os_type == os_android
        {
        achievement_load_leaderboard("CgkIs9_51u0PEAIQBw", 1, 100, achievement_filter_friends_only);
        }
    else
        {
        achievement_load_leaderboard("leaderboard1id", 1, 100, achievement_filter_friends_only);
        }
    achievement_get_challenges();
    achievement_load_friends();
    achievement_load_progress();
    }

上面的代码检查返回的 DS Map在社交异步事件中，如果其“ id”与要检查的常量匹配，则它将在调用其余函数以准备排行榜，挑战和好友列表之前，循环遍历映射，将所有不同值存储在变量中。
