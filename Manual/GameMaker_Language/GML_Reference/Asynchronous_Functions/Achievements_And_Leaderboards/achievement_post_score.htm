achievement_post_score

您可以使用此功能将分数发送到所选的排行榜和成就服务。在Android ，您将分数表的ID作为字符串发送（这是设置排行榜时分配的唯一排行榜ID），而在所有其他平台上，您将定义的排行榜名称作为字符串提供，然后给出实际得分值。该函数将触发社交异步事件，其中返回的 async_loadDS映射将具有以下键/值对：

  • “ type ”-这是已触发的事件的类型，将是字符串“achievement_post_score_result ”。
  • “ id ”-这是事件的ID，对于此函数，它将返回GML常量 GooglePlayServices_EVENT_ID_POST_SCORE。
  • “ Leaderboard_id ”-这是已更新的页首横幅的唯一ID。
  • “ status ”-如果分数更新成功，则为1；如果请求失败，则为0
  • “ daily_new_best ”-如果提交的分数是用户在过去一天中的最高分数，则为1，否则为0。
  • “ daily_best ”-这将保留用户从比赛的最后一天开始的最高分。
  • “ weekly_new_best ”-如果提交的分数是上周用户的最佳成绩，则为1，否则为0。
  • “ weekly_best ”-保留用户上周的最高分。
  • “ all_time_new_best ”-如果提交的分数是用户有史以来最好的分数，则为1，否则为0。
  • “ all_time_best ”-保留用户有史以来的最高分。

句法：

achievement_post_score(leaderboard, score)

争论描述
leaderboard页首横幅的名称/ ID。
score要发送的分数。

返回值：

N/A

扩展示例：

if achievement_available()
    {
    achievement_post_score(global.LeaderboardID, 1000);
    }

上面的代码检查以查看所选排行榜和成就服务是否可用，如果可用，它将分数发布到称为“排行榜”的分数表中。这将触发一个社交异步事件，可以将其解析如下：

if async_load[? "id"] == GooglePlayServices_EVENT_ID_POST_SCORE
    {
    if async_load[? "type"] == "achievement_post_score_result"
        {
        if async_load[? "status"] == 1
            {
            global.Best_Day = async_load[? "daily_best"];
            global.Best_Week = async_load[? "weekly_best"];
            global.Best = async_load[? "all_time_best"];
            }
        }
    }

上面的代码检查 async_loadDS映射与该功能相对应的“ id”和“ type”键，然后检查事件的状态。如果状态为1，则DS映射中的某些值存储在全局变量中。
