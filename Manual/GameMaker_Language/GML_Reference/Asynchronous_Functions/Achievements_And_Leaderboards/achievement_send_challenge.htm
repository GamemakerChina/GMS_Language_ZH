achievement_send_challenge

此功能将通过网络向选定的玩家发送挑战。您可以使用获得playerid achievement_load_friends()或者 achievement_load_leaderboard()功能，并且还必须提供Challengeid ，它是在iTunes Connect或Google Play仪表板上创建挑战时给定挑战的唯一值。您还必须提供一个分数，一条短消息以及设置挑战类型。这可以是以下常量之一：

持续的描述
achievement_type_score_challenge基于得分值的挑战。
achievement_type_achievement_challenge基于成就的挑战。

此函数将为要接收挑战的玩家触发回调社交异步事件，在这种情况下，您将在变量中引用DS映射 async_load。这个的ID键 DS Map用于标识正确的回调（对于任何给定的异步事件，可以有多个触发器函数），并将其与常量配对 achievement_challenge_received以及每个玩家的许多其他键/值对。该地图的确切内容如下所示：

注意：用户还可以收到关于收到的挑战的吐司通知，但这取决于您使用功能Achievement_show_challenge_notifications（）选择的值。

  • “ id ”-对于此功能，应为 achievement_leaderboard_info
  • “ playerid ”-挑战的玩家ID。
  • “ issuerid ”-挑战的发行者ID。
  • “状态”-挑战的状态，对于无效，待处理，已完成或已拒绝，其值为0-3（作为字符串）。
  • “ message ”-质询短信。
  • “ completeddate ”-挑战的完成日期。
  • “issueddate” -于债券发行日的挑战。
  • “类型”-发出挑战的类型。可以是上面列出的两个常数之一
  • “标识符”-挑战的标识字符串，在提供商仪表板上设置
  • “得分”-与挑战相关的得分（如果适用）。

句法：

achievement_send_challenge(playerid, challengeid, score, type, message)

争论描述
playerid要挑战的玩家的唯一ID。
challengeid唯一质询ID（在提供商信息中心上找到）。
score比分击败。
type发出挑战的类型-以下给出的两个常数之一。
message附加到挑战的信息。

返回值：

N/A

扩展示例：

要通过网络发送挑战，您需要以下代码：

achievement_send_challenge(global.playerid[0], global.challengeid[0], score, achievement_type_score_challenge, "Beat that sucker!");

然后，此请求将在您的游戏中为挑战所针对的玩家触发社交事件，可以通过以下方式处理：

var ident = ds_map_find_value(async_load, "id");
if ident == achievement_challenge_received
    {
    player_id = ds_map_find_value(async_load, "playerid");
    issuer_id = ds_map_find_value(async_load, "issuerid");
    state = ds_map_find_value(async_load, "state");
    message = ds_map_find_value(async_load, "message");
    date_completed = ds_map_find_value(async_load, "completeddate");
    date_issued = ds_map_find_value(async_load, "issueddate");
    ach_type = ds_map_find_value(async_load, "type");
    ach_ident = ds_map_find_value(async_load, "identifier");
    ach_score = ds_map_find_value(async_load, "score");
    }

上面的代码检查返回的 DS Map在社交异步事件中，如果其“ id”与要检查的常量匹配，则它将提取映射中每个键的相关值，并将它们存储在变量中以备将来使用。
