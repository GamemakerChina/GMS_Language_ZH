异步功能

GameMaker Studio 2的上下文中，异步功能是一种发送到Web服务器或请求用户输入甚至流数据的功能，同时允许GameMaker Studio 2继续运行而不阻止游戏运行。收到信息回调后，将触发一个特殊的异步事件，该事件进而允许分配了异步事件的任何实例执行进一步的功能（尽管不一定如此）。因此，基本上，这是一种与某些外部资源（服务器，用户或设备）进行通信的方法，而游戏或项目在等待回复时不会停止。

为什么这很重要？好吧，因为它允许GameMaker Studio 2在发送或接收数据的同时继续运行，这意味着您可以做很多事情而不会阻塞游戏循环，例如：

  • 将数据从设备流传输到游戏中，而无需玩家等待加载的内容
  • 只有在收到正确的信息后，才有回调事件来执行操作
  • 与Web服务器通信和交换数据

这使所有功能以更加流畅和不干扰的方式运行。应当注意的是，异步事件将为所有拥有它们的实例触发，因此您可以在一个实例中使用异步功能，并让异步事件在另一个实例中甚至在其他各种实例中处理答复。您可以在此处获取有关异步事件如何工作的更多信息-异步事件。
