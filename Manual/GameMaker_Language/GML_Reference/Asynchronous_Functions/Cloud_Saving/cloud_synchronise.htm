cloud_synchronise

通常在新游戏开始时会调用此函数，该函数用于在游戏启动时检索云服务的当前状态。该函数返回一个唯一的id值，然后该值将在“异步云事件”中用于从创建的DS映射中检索相关信息。

此功能将发送到云以获取数据，然后将触发适当的异步事件。在这种情况下，您可以检查返回的 async_loadDS映射以下值：

  • “状态”：保存状态代码，其中负值表示错误，其描述将包含在“ errorString”中。值为0（或正值）表示成功（请参见下面的确切值），并且“ resultString”将包含返回的数据或成功消息。
  • “ id”：从被调用函数返回的id。如果您开除一系列 cloud_请求，那么您需要知道您要获得答复的对象，因此您将使用此值与最初发送请求以查找正确请求时存储的值进行比较。
  • “描述”：最后上传文件的描述。
  • “ resultString”：它包含一个字符串，它是从云返回的数据blob。
  • “ errorString”：返回任何错误的错误字符串。

下表解释了返回的“状态”映射条目的确切含义：

状态值errorString / resultString描述
-1 errorString =“未登录<SERVICE>” You have not successfully logged in to the given Cloud Service
0 resultString =恢复的数据New game data downloaded from the cloud (following a cloud_synchronise call)
1 resultString =“已经同步” No new data since you last called cloud_synchronise
2 resultString =“ ConflictDeferral” A conflict was encountered, but the gamer chose to ignore it
3 resultString =“ GameUploadSuccess” data from cloud_string_save() or cloud_file_save() was successfully uploaded to the cloud
-n errorString =错误说明Any other negative number means a synchronisation failure

句法：

cloud_synchronise();

返回值：

Real

扩展示例：

在诸如游戏开始事件之类的事件中或在游戏第一个房间中放置的对象中将调用此函数，其想法是您检查云服务器中的当前数据blob以查看是否达到是否约会。

cloud_check = cloud_synchronise();

然后，您要检查返回的 DS Map在异步Cloud Event中获取状态和返回的字符串（如果有的话），其内容类似于以下代码：

if ds_map_find_value(async_load, "id") == cloud_check
   {
   if ds_map_find_value(async_load, "status") < 0
      {
      show_message_async("Cloud Services not available. Please check connectivity.");
      }
   else
      {
      if ds_map_find_value(async_load, "status") == 0
         {
         var file = file_text_open_write("Save.txt");
         file_text_write_string(file, ds_map_find_value(async_load, "resultString"));
         file_text_close(file);
         }
      }
   }

上面的代码检查以确保正在修改正确的异步函数调用，然后继续获取返回的云DS映射的状态。如果状态为负数，则出了点问题，并通知用户，否则代码将继续并检索同步的数据并将其写入文本文件以供以后使用。
