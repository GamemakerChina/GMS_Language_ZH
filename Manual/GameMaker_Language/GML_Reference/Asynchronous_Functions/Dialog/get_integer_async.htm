get_integer_async

这个函数打开一个窗口，显示信息以及一个供用户输入数值的空间（它将包含提供的默认值开始）。这是一个异步函数，因此GameMaker Studio 2在等待答案时不会阻止正在运行的设备。 它正在运行的设备在等待答案，而是继续正常地运行事件。一旦用户输入了他们的字符串并按下了 "Okay "按钮，一个异步的 Dialog 事件被触发，仅在该事件的持续时间内，将有一个DS地图存储在变量 Async_load.

这个映射将包含三个键，"id"、"status "和"value"。"id "是函数调用时返回的值，"status "将是 "Okay "按钮被按下时的true，或者是消息被取消时的false（在适用情况下，不是所有目标平台都允许消息被取消）。最后，"value"将返回用户输入的整数。 作为一个字符串（如果没有输入，将返回一个空字符串""）。

注意。这个功能 只用于调试和测试，不应该在已发布的游戏中使用。为此，你应该创建一个自定义的用户界面来接收玩家的输入，这样你就可以完全控制对话框的外观和行为。

语法。

get_integer_async(string, default);

显示该信息的对象的鼠标左键事件（例如）将有以下代码。

msg = get_integer_async("How old are you?", 0);

上面将显示一条信息，要求用户输入一个字符串并按下 "好"。该函数ID存储在变量 "msg "中，并将在异步对话事件中使用，如下所示。

var i_d = ds_map_find_value(async_load, "id");
if i_d == msg

if ds_map_find_value(async_load, "status")
{
global.Age = ds_map_find_value(async_load, "value");
}
}

上述代码将返回的DS Map的 "id"键 与存储在变量 "msg "中的值进行检查。如果它们相同，它就会检查 "OK "是否被按下（而不是窗口被关闭/取消）。 如果返回为真，它就从返回的整数中设置一个全局变量。
