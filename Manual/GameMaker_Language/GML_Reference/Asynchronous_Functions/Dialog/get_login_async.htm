get_login_async

此功能将打开一个窗口，要求用户输入用户名和密码。这些参数可以设置为空字符串，或者可以存储以前输入的值以供使用。这是一个异步函数，因此GameMaker Studio 2不阻止它正在对在等待答案的运行，而是在设备上运行的事件作为正常保持。用户输入详细信息并按下“确定”按钮后，将触发异步用户交互事件，该事件仅在该事件的持续时间内将DS映射存储在变量中 async_load。

该映射将包含两个键，“用户名”和“密码”，并在每个中存储了相应的用户输入。由于它是已创建的DS地图，因此可以例如由 json_encode()功能已准备好发送到服务器或写入所选设备上的文件。

值得注意的是，此函数将返回创建的DS映射的索引号，然后可以在相应事件中检查该索引号，以便在您期望触发多个异步事件时可以“定位”特定的DS映射（也许来自其他功能）。

句法：

get_login_async(name, password);

争论描述
username默认用户名
password默认密码

返回值：

Real

扩展示例：

控制我们用户登录的对象的create事件（例如）将具有以下代码：

ini_open("Profile.ini");
u = ini_read_string("User","0","");
p = ini_read_string("User","1","");
ini_close();
login = get_login_async(u,p);

上面的代码打开一个ini文件（如果不存在则创建一个），并获取存储在该文件中的名称和密码。如果它们不存在，则返回默认的空字符串（“”）。然后将这些值用于 get_login_async()向用户询问其用户名和密码详细信息，并将请求索引存储在变量“ login”中。请注意，在等待用户提供必要的信息时，游戏及其事件将继续正常运行。

现在，异步代码已被触发，我们需要通过以下方式检查Dialogs的异步事件中的返回值：

if ds_map_find_value(async_load, "id") == login
    {
    u = ds_map_find_value(async_load, "username");
    p = ds_map_find_value(async_load, "password");
    }

上面的代码检查了 async_loadDS映射，如果它具有与“ login”变量中存储的值相同的值，则将映射详细信息读取到相应的变量中，然后可以将其存储或用于检查数据库值等。
