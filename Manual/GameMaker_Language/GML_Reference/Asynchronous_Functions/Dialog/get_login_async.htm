get_login_async

这个函数打开一个窗口，要求用户输入用户名和密码。这些参数可以设置为空字符串，或者你可以存储以前输入的值来使用，如果你愿意。这是一个异步函数，因此GameMaker Studio 2在等待回答时不会阻止它所运行的设备，而是继续正常地运行事件。一旦用户输入细节并按下 "好 "的按钮，一个异步的用户交互事件 被触发，在该事件的持续时间内，将有一个DS地图存储在变量 async_load.

这个地图将包含两个键，"用户名 "和"密码"，每个键中都存储有相应的用户输入。由于它是一个已经创建的DS地图，它可以被使用，例如，由 json_encode()函数准备发送到服务器或写入所选设备上的文件。

这个函数将返回被触发的异步事件的索引号，然后可以在相应的事件中进行检查，这样你就可以 "瞄准 "特定的数据，以防你期望有一个以上的异步事件被触发 (也许来自其他函数)。

注意。这个功能 只用于调试和测试，不应该在已发布的游戏中使用。为此，你应该创建一个自定义的用户界面来接收玩家的输入，这样你就可以完全控制对话框的外观和行为。

语法。

get_login_async(name, password)。

争论 描述
帐号 默认的用户名称
密码 默认密码

返回。

真实的

扩展实例。

控制用户登录的对象的创建事件（例如）将有以下代码。

ini_open("Profile.ini");
u = ini_read_string("User", "0","");
p = ini_read_string("User", "1",");
ini_close();
login = get_login_async(u,p);

上面的代码打开了一个ini文件（如果不存在，则创建一个），并获得存储在该文件中的名称和密码。如果它们不存在，则返回默认的空字符串（""）。这些值随后被用于get_login_async()，以询问用户的用户名和密码细节，请求索引被存储在变量 "login "中。注意，在等待用户提供必要的信息时，游戏和它的事件将继续正常运行。 正常运行。

现在，异步代码已经启动，我们需要在对话框的异步事件中以如下方式检查返回值。

if ds_map_find_value(async_load, "id") == login
{
u = ds_map_find_value(async_load, "username");
p = ds_map_find_value(async_load, "password") ;
}

上述代码检查 async_load DS地图的 "id "键，如果它与存储在 "login "变量中的值相同，地图的细节就会被读到相应的变量中，然后你就可以存储或用来检查数据库的值等。 然后你可以存储或用于检查数据库的值等等。
