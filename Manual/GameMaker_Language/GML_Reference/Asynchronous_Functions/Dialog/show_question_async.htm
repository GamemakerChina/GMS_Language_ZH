show_question_async

这个函数打开一个窗口，向用户显示你在函数中定义的问题。这是一个异步函数，因此GameMaker Studio 2在等待答案时不会阻止它所运行的设备，而是 继续正常地运行事件。该函数有两个按钮，分别显示 "是 "和 "否"，一旦用户按下一个，就会触发一个异步对话事件，在该事件的持续时间内，将 变量中存储有一个DS地图 async_load.

这个映射将包含两个键，"id "和 "status"。"id"是函数调用时返回的值，而 "status "将分别为 "是 "和 "否 "的真或假。

注意。这个功能 只用于调试和测试，不应该在已发布的游戏中使用。为此，你应该创建一个自定义的用户界面来接收玩家的输入，这样你就可以完全控制对话框的外观和行为。

语法。

show_question_async(string);

争论 描述
字符串 要向用户提出的问题。

返回。

以上将显示一个带有给定字符串的问题，要求用户按 "是 "或 "否"。该函数ID存储在变量 "msg "中，并将在异步对话事件中使用，如下所示。

var i_d, stat;

coins -= armour[0,5];

上面的代码将返回的DS Map的 "id "键与存储在变量 "msg "中的值进行检查。如果它们相同，它就会检查两个按钮中是否有一个被按下，如果返回为真，它就会从一个变量中扣除一个值，并向一个全局变量添加一个值。
