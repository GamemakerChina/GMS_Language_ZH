http_get

使用此功能，您可以连接到指定的URL以便检索信息。由于这是一个异步功能，因此GameMaker Studio 2在等待回复时不会阻塞，但会一直运行，除非它获取回调信息。此信息将采用字符串形式，并且将在其对象属性中定义了一个的任何实例中触发异步事件。您还应该注意，使用此功能时，HTTP请求参数（当您在网页上提交表单时有时会“附加到” URL末尾的位）是完全可以接受的，例如：

http_get("http://www.YoYoGames.com/logon?username="+name);

会将变量“名称”中包含的字符串传递给服务器，并从该URL检索数据。因此，从本质上讲，每当需要将简单的短数据从客户端传递到服务器时，这都是要使用的功能，这是合理的选择。

注意：您应该意识到，由于浏览器中的XSS保护，跨域对资源的请求和加载尝试都将被阻止，并且可能会返回空白结果。有关更多详细信息，请参阅“跨域问题”部分。

此函数将生成多个“回调”，这些回调将由任何HTTP Events拾取。这些将产生一个 DS Map（通常称为“字典”），此事件专用，并存储在特殊变量async_load中。根据是否有数据返回，此DS映射将包含不同的值。例如，如果您已请求文件，则事件将在接收到每个数据包时触发多次，因此您可以显示进度条。 DS地图的一般结构如下：

  • id：从命令返回的ID。如果您开除一系列 http_请求，那么您需要知道您要获得答复的对象，因此您将使用此值与最初发送请求以查找正确请求时存储的值进行比较。
  • status：对于错误返回小于0的值，对于完整返回0，对于接收数据包返回小于1的值（有关更多详细信息，请参见下文）。
  • 结果：接收到的数据（仅字符串）。
  • url：您请求的完整URL。
  • http_status：原始的http状态代码（如果有）。这将为大多数浏览器返回标准的网络状态代码，例如：“未修改”为304，“无内容”为204，等等。

如果有多个数据包返回给您的游戏，则回调“状态”键将返回1，在这种情况下，DS映射将具有以下附加键：

  • “ contentLength”：这是Web服务器说您应该期望接收的文件大小（如果服务器不返回此数据，则为-1）。
  • “ sizeDownloaded”：已经下载的数据的大小。

句法：

http_get(url);

这 http_get()可以从任何事件调用该函数，并且由于它是异步的，所以回调几乎是瞬时的，也可能需要几秒钟。调用函数很简单，看起来像这样：

get = http_get("http://www.MacSweeneyGames.com/logon?username=" + name);

上面的代码会将保存在变量“名称”中的字符串传递给给定服务器，并从该URL检索数据，从而触发一个异步事件，该事件将包含async_load DS映射（async_load映射索引将存储在该变量中“ get”，以便您可以检查正确的回调）。触发的异步事件将是HTTP子事件，在该事件中，您将具有以下内容：

if ds_map_find_value(async_load, "id") == get
    {
    if ds_map_find_value(async_load, "status") == 0
        {
        r_str = ds_map_find_value(async_load, "result");
        }
    else
        {
        r_str = "null";
        }
    }

上面的代码将首先检查已创建的DS映射的ID，然后检查回调的“状态”。如果该值等于0（信号发送成功），则来自回调的结果将存储在变量中以备将来使用，否则该变量将设置为默认值（在这种情况下为“ null”）。
