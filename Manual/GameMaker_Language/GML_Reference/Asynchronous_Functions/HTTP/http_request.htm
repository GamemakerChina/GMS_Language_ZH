http_request

使用此功能，您可以创建HTTP标头请求以定义HTTP事务的操作参数，如果您使用RESTful API，则可以将其用于许多事情，例如通过HTTP标头进行身份验证。该功能需要服务器的完整IP地址以及发出请求的类型（作为字符串，请参见下面的注释）：“ GET”，“ HEAD”，“ POST”，“ PUT”，“删除”，“跟踪”，“选项”或“连接”。您还需要提供一个 DS map键/值对（作为字符串，其中键是标头字段，值是标头的必需数据），最后一个参数是可选数据字符串，您可以将其添加到请求中，如果不需要那么它可以是0或空字符串“”。请注意，您也可以发送缓冲器（见部分缓冲区的更多详细信息），在这种情况下，最后一个参数将是缓冲区送的“手柄”。

注意：HTTP标头通常遵循“ key：value”格式，但是由于GMS从ds_map为您创建了这些对，因此不需要在映射键或值字符串中包含冒号“：”。

注意：当对body参数使用缓冲区时，如果缓冲区查找位置位于开始位置（0），则不会发送任何正文，并且缓冲区中将充满来自http调用的响应，但是如果缓冲区查找位置不是-零，然后将缓冲区字符串内容作为正文发送。

注意：您应该意识到，由于浏览器中的XSS保护，跨域对资源的请求和加载尝试都将被阻止，并且可能会返回空白结果。有关更多详细信息，请参阅“跨域问题”部分。

此事件将生成一个“回调”，该回调将被任何HTTP事件拾取，在这种情况下，它将生成一个 DS Map（通常称为“字典”），此事件专用，并存储在特殊变量async_load中。该DS映射具有以下两个与请求功能相关的键：

  • id：从命令返回的ID。如果您开除一系列 http_请求，那么您需要知道您要获得答复的对象，因此您将使用此值与最初发送请求以查找正确请求时存储的值进行比较。
  • response_headers：如果该值的值大于或等于0，则它将保存一个DS映射，该映射包含对HTTP请求的响应返回的HTTP标头。

句法：

http_request(url, method, header_map, body);

争论描述
url您希望从中获取信息的服务器的网址
method请求方法（通常为“ POST”或“ GET”，但支持所有方法）
header_map具有必填标头字段的ds_map
body标头之后要传输的数据（可以是二进制缓冲区句柄）
