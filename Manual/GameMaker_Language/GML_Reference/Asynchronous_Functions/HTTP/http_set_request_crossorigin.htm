http_set_request_crossorigin

使用此功能，可以设置从文件加载图像时使用的跨原点类型。该功能专用于HTML5平台，仅在使用 sprite_add()功能。请注意，当设置为“ use-credentials”时，您不再需要为要加载的精灵提供绝对路径，而是需要提供相对路径（如下例所示）。默认情况下，交叉原点类型设置为“匿名”。

句法：

http_set_request_crossorigin(origin_type);

争论描述
origin_type要使用的交叉原点类型（字符串）

返回值：

N/A

例子：

if string_lower(http_get_request_crossorigin()) != "use-credentials"
    {
    http_set_request_crossorigin("use-credentials");
    }
sprite_add("sprites/player_outfit_1.png", 0, false, false, 0, 0);

上面的代码将首先检查当前设置的跨原点类型，如果不是“ use-credentials”，则将其设置为“ use-credentials”，然后从文件中添加一个sprite。
