http_set_request_crossorigin

通过这个函数，你可以设置从文件中加载图像时使用的跨源类型。该函数是专为HTML5平台设计的，并且只在使用 sprite_add()函数从文件中加载精灵时才有用。 请注意，当设置为 "use-credentials "时，你不再需要给出被加载的精灵的绝对路径，而是给出一个相对路径（如下例所示）。默认情况下，跨源类型被设置为 "anonymous"。

语法。

http_set_request_crossorigin(origin_type)。

争论 描述
原产地_类型 要使用的交叉原产地类型（一个字符串）。

返回。

不适用

例子。

if string_lower(http_get_request_crossorigin()) != "use-credentials"
{
http_set_request_crossorigin("use-credentials")。
}
sprite_add("sprites/player_outfit_1.png", 0, false, false, 0, 0) 。

上述代码将首先检查当前设置的cross origin类型，如果它不是 "use-credentials"，那么它将被设置为 "use-credentials"，然后从一个文件中添加一个sprite。
