推送通知

在移动设备上，通常任何时候任何时候前台都只能激活一个应用程序，但是许多游戏和应用程序都在基于时间或互连的环境中运行，当该应用程序不在前台时，可能会发生用户感兴趣的事件。在这些情况下，本地和远程推送通知可以使这些游戏在事件发生时通知其用户。

当操作系统传递本地或远程推送通知且目标应用程序不在前台运行时，它将以横幅形式向用户显示通知。如果有通知警报并且用户点击了该警报，则将启动游戏并调用一个异步推送事件，您可以在其中检查指定的回调字符串并根据其值解析推送。如果在传递通知时应用程序在前台运行，则该应用程序仍将收到一个异步推送事件，但是除非您自己创建它，否则用户将不会看到任何消息。

支持的平台之间有一些差异需要注意：

  • 在Android ，将存储应用未运行/在后台运行时收到的所有通知，以及下次应用运行时由async事件传递的数据，您需要在Android游戏选项中启用推送通知，以便它们工作。
  • 在iOS ，对于特定的通知数据将只选择了通知时交付（或如果应用程序在前台运行接到通知的时候），你将需要启用内推送通知iOS游戏选项为他们的工作。

本地通知

本地推送通知对于安装游戏的设备是本地的，并且不需要后端服务器。您只需设置通知的日期和时间，如果游戏当前未在运行，它将显示给用户。此类通知对于设置“提醒”，以便用户再次玩游戏或为玩游戏提供每日奖励等很有用……当用户点击通知时，它将启动游戏并传递数据然后可以由游戏解析的字符串，以提供奖励或其他奖励。

注意：此功能仅限于iOS和Android目标模块。对于Android您需要安装Google Push Notification扩展。

以下功能可用于本地通知：

  • push_local_notification
  • push_get_first_local_notification
  • push_get_next_local_notification
  • push_cancel_local_notification

服务器将远程通知消息发送到设备平台应用商店提供的服务，然后将这些消息转发到安装了应用程序的所有设备上。 iOS和Android目标模块上的GameMaker Studio 2支持此功能。

GameMaker Studio 2中没有处理远程通知的功能，因为它们必须全部由您的服务器生成并由相应的应用商店处理。但是，一旦正确设置， GameMaker Studio 2游戏将收到这些通知，然后可以像本地通知那样在“异步推送事件”中进行处理。

请注意，不能保证将发送远程推送通知，并且不能保证允许的数据有效载荷很小。不同平台之间的情况有所不同，但是iOS受到特别限制-苹果服务仅提供最新的通知，收件人必须选择最新的通知才能将有效负载数据传递到您的异步事件，并且这些通知的最大有效负载大小为256byte 。通常，例如，远程推送通知仅指示您的服务器上有可用的新数据。

注意：服务器端的实现完全取决于最终用户， YoYo Games除了从其知识库中可获得的基本设置信息外，不对该端提供任何支持。
