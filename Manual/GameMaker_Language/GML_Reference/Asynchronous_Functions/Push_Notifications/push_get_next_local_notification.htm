push_get_next_local_notification

这个函数将用一系列的键/值对来填充一个预制的DS地图，以便在队列中显示下一个本地推送通知。如果没有更多的通知，它将返回-1。 则返回-1，否则返回一个真实的值，代表该通知的ID。然后这个ID可以用来取消通知，使用函数 push_cancel_local_notification().你应该 调用该函数 push_get_first_local_notification()来获得队列中的第一个通知，然后用这个函数继续完成。

注意：该功能仅限于在 iOS和 安卓系统目标模块。

他的DS地图将持有以下键/值对。

  • "标题" - 通知的标题（这一点只在 安卓平台上可用，因为iOS不存储标题信息)
  • "信息" - 信息主体文本
  • " data"- 数据包字符串

语法。

push_get_next_local_notification(map)。

争论 描述
地图 要使用的DS地图ID。

返回。

真实的

例子 :

var map = ds_map_create();
var ntf = push_get_first_local_notification(map);
while(ntf >= 0)
{
var data = ds_map_find_value(map, "data");
if data == "Daily_Reward"
{
push_cancel_local_notification(ntf)。
}
ntf = push_get_next_local_notification(map);
}
ds_map_destroy(map)。

上面的代码创建了一个DS地图，然后请求队列中第一个可用的通知的数据。如果有通知存在，地图就会被填充，然后代码会检查 "data "键是否有一个特定的字符串。如果这个字符串 被发现，该通知就会被取消，并继续检查下一个通知，直到队列中的所有通知都被检查过。
