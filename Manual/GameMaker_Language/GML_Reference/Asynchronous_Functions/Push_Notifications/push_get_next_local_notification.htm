push_get_next_local_notification

此功能将使用一系列键/值对填充预制的DS映射，以显示队列中的下一个本地推送通知。如果没有其他通知排队，则返回-1，否则返回一个实际值，否则表示该通知的ID。然后，可以使用此ID使用该功能取消通知 push_cancel_local_notification()。您应该调用该函数 push_get_first_local_notification()获取队列中的第一个通知，然后使用此功能继续进行操作。

注意：此功能仅限于iOS和Android目标模块。

DS映射将包含以下键/值对：

  • “ title ”-通知的标题（仅在Android平台iOS不存储标题信息）
  • “ message ”-邮件正文
  • “ data ”-数据包字符串

句法：

push_get_next_local_notification(map);

争论描述
map这 DS Map要使用的ID。

返回值：

Real

例子 ：

var map = ds_map_create();
var ntf = push_get_first_local_notification(map);
while(ntf >= 0)
    {
    var data = ds_map_find_value(map, "data");
    if data == "Daily_Reward"
        {
        push_cancel_local_notification(ntf);
        }
    ntf = push_get_next_local_notification(map);
    }
ds_map_destroy(map);

上面的代码创建了一个 DS Map然后为队列中的第一个可用通知请求数据。如果存在通知，则将填充地图，然后代码检查“数据”键中的特定字符串。如果找到了字符串，则取消通知，并继续检查下一个通知，直到检查了队列中的所有对象为止。

返回：推送通知
