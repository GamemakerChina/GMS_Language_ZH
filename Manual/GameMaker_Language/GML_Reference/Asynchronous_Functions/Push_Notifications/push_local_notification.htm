push_local_notification

这个函数可以用来设置一个本地通知，在给定的日期和时间显示。fire_time "是通知应该被推送到用户设备上的日期/时间（你可以使用GameMakerStudio 2 Date and Time Functions来获得这个信息），你可以给通知一个标题和信息文本，以及一个有效载荷字符串，当用户点击通知时，它将被传递给你的游戏。

注意："标题 "参数在iOS上被忽略，而是显示游戏名称。

点击通知将在设备上启动游戏，并触发一个异步推送通知事件，在这个事件中 异步推送通知（async_load DS地图可以被解析，以检查有效载荷字符串，并执行任何需要的操作。

注意：该功能仅限于在 iOS和 安卓系统目标模块。

语法。

push_local_notification(fire_time, title, message, data)。

争论 描述
火灾_时间 发出通知的日期/时间
职称 通知的标题（一个字符串）。
信息 通知信息文本（一个字符串）。
数据 发送给你的游戏的数据包（一个字符串）。

返回。

不适用

扩展实例

在这个例子中，我们将使用以下代码发送一个本地推送通知。

var fireTime = date_inc_day(date_current_datetime(), 1);
var data = "daily_reward";
push_local_notification(fireTime, "Ahoy!", "Catch The Haggis Has A Present", data);

这将设置一个计时器，在一天过去后向设备 "推送 "一个通知。当一天结束时，如果你的游戏在后台或没有运行，一个通知将显示给用户，并给出标题和信息（在 iOS中，游戏名称会被显示，而标题则被忽略），然后一个异步推送通知事件将被调用。请注意，如果在通知时间到来时游戏处于前台，它将不会被显示，但异步的 事件仍然会被触发。在事件本身中，你会像这样处理回调：异步事件的方式如下。

var type = ds_map_find_value(async_load, "type");
var status = ds_map_find_value(async_load, "status");
if status == 0
{
//出现某种错误
var error = ds_map_find_value(async_load, "error");
show_debug_message("error=" + string(error));
}
else
{
if type == "register"
{
global.reg_id = ds_map_find_value(async_load, "reg_id");
}
else
{
var data = ds_map_find_value(async_load, "data");
if data == "day_reward"
{
global.Gold += 1000;
}
}
}
