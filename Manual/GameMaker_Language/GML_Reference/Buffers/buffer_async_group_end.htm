buffer_async_group_end

此功能完成了缓冲区保存组的定义。您必须事先调用过该函数 buffer_async_group_begin()发起小组，然后调用函数 buffer_save_async()对于您要保存的每个文件。最后，您调用此函数，它将开始文件的保存。该函数将为保存返回唯一的ID值，然后可以在“异步保存/加载”事件中使用该ID值来解析保存的结果。async_load DS地图。

句法：

buffer_async_group_end();

返回值：

Real

扩展示例：

这 buffer_async_group_end()可以从任何事件调用该函数，并且由于它是异步的，所以回调几乎是瞬时的，也可能需要几秒钟。调用函数很简单，看起来像这样：

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
save_id = buffer_async_group_end();

上面的代码启动一个缓冲区组，然后将其设置为异步保存4个文件。然后，结束组定义（此时将开始保存），将函数调用的ID存储在变量“ save_id ”中。保存完成后，将触发异步Save / Load事件，您可以解析 async_load映射功能的正确ID，如下所示：

if ds_map_find_value(async_load, "id") == saveid
    {
    if ds_map_find_value(async_load, "status") == false
        {
        show_debug_message("Save failed!");
        }
    }

上面的代码将首先检查已创建的DS映射的ID，然后检查回调的状态，并在出现任何问题时发布调试消息。
