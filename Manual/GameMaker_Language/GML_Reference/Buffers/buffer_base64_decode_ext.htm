buffer_base64_decode_ext

使用此功能，您可以解码base64编码的字符串（使用 buffer_base64_encode()函数）放入缓冲区。与功能不同 buffer_base64_decode()，这不会为您创建缓冲区，而是您应该已经创建了缓冲区（请参见 buffer_create()），然后您将在此函数中使用其ID。 “偏移量”是缓冲区中解码给定字符串（以字节为单位）的位置。

句法：

buffer_base64_decode_ext(buffer, string, offset);

争论描述
buffer将字符串解码为的缓冲区的索引。
string要解码的base64编码字符串。
offset数据偏移值。

返回值：

N/A

例子：

buff = buffer_create(16384, buffer_grow, 2);
ini_open("Save.ini");
var str = ini_read_string("Save", "Slot1", "");
buffer_base64_decode_ext(buff, str, 0);
ini_close();

上面的代码将创建一个缓冲区，并将其唯一标识存储在变量“ buff”中，然后打开一个ini文件，并将其中的字符串读取到局部变量“ str”中。然后，此字符串将解码到新创建的缓冲区中，然后再次关闭ini文件。

返回：缓冲区
