buffer_create_from_vertex_buffer_ext

您可以使用此函数为游戏中的缓冲区分配一部分内存，该内存中填充了先前创建的顶点缓冲区中的数据。该函数返回一个唯一的缓冲区ID ，该ID应该存储在变量中，并用于对该缓冲区的所有其他函数调用。该函数获取要使用的顶点缓冲区的ID（由函数返回）vertex_create_buffer() （例如），并使用以下常量来定义缓冲区类型：

持续的描述
buffer_fixed固定大小的缓冲区。
buffer_grow缓冲区将随着数据的添加而动态“增长”
buffer_wrap数据将“包装”的缓冲区。当所添加的数据达到缓冲区大小的限制时，覆盖将被放回缓冲区的开头，并且从该点开始将继续写入。
buffer_fast特殊的“剥离”缓冲区，读取/写入速度非常快。只能与 buffer_u8数据类型，并且必须对齐1个字节。
buffer_vbuffer这种类型的缓冲区仅用作顶点缓冲区。

除了缓冲区类型之外，您还必须设置缓冲区的字节对齐方式。该值将根据您希望存储在缓冲区中的数据而有所不同，并且在大多数情况下，值为1完全可以。但是，请注意，对于某些操作，特定的对齐方式是必不可少的，错误的对齐方式可能会导致错误（有关对齐的更多详细信息，请参见Buffer Alignment ）。以下是一般指南，以显示哪些值最适合每种数据类型（如有疑问，请使用1的对齐方式）：

  • 字符串应对齐1个字节。
  • 有符号或无符号8位整数可以与任何值对齐，但是请注意，对于快速缓冲区（请参见 buffer_write()）它必须与1对齐。
  • 有符号或无符号16位整数应对齐2个字节。
  • 有符号或无符号32位整数应对齐4个字节
  • 最多16位的浮点数应对齐2个字节。 （当前不支持！）
  • 最多32位的浮点数应对齐4个字节。
  • 最多64位的浮点数应对齐8个字节。

注意：顶点缓冲区是1字节对齐的，但是您可以根据要如何对待数据来创建任何对齐的缓冲区，因为顶点数据只是到缓冲区中的原始内存副本。

最后两个参数用于指定要复制到新创建的缓冲区中的顶点数据的范围。起始顶点可以在0到顶点数量-1之间的任何位置，然后您可以指定从该点开始复制的顶点数量。您可以使用该功能 vertex_get_number()在顶点缓冲区中获取存储的顶点总数。

注意：重要的是，当不再需要这样的动态创建的资源时，请从内存中删除这些资源，以防止内存泄漏，因此，当您完成创建的缓冲区后，应该使用buffer_delete（）再次释放它。还要注意，用于创建新缓冲区的顶点缓冲区也不会从内存中删除，并且当不再需要该函数时，应该使用功能vertex_delete_buffer（）。

句法：

buffer_create_from_vertex_buffer_ext(vertex_buffer, type, alignment, start_vertex, num_vertices)

争论描述
vertex_buffer要使用的顶点缓冲区索引。
type要创建的缓冲区的类型（请参见下面的常量列表）。
alignment缓冲区的字节对齐
start_vertex起始顶点。
num_vertices要复制的顶点总数。

返回值：

Int (Vertex buffer ID)

例子：

var v_num = vertex_get_number(model_buff); player_buffer = buffer_create_from_vertex_buffer(model_buffer, buffer_grow, 1, 0, v_num - 1);

上面的代码将内存分配给缓冲区，然后将所有顶点的数据从给定的顶点缓冲区复制到其中，返回存储在变量“ player_buffer”中的新缓冲区的索引，以备将来使用。

返回：缓冲区
