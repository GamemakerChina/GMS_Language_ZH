缓冲区_加载_async

通过这个函数，你可以将你之前用 buffer_save()函数（或其他任何用于保存缓冲区的函数）创建的文件载入缓冲区。偏移量 "定义了加载缓冲区的起始 偏移量 "定义了加载缓冲区的起始位置（以字节为单位），而 "大小 "是要从该偏移量开始加载的缓冲区的大小（也以字节为单位）。你可以为size参数提供一个-1的值，整个缓冲区都将被加载。 注意，该函数将从一个 "默认 "文件夹加载，它不需要作为你提供的文件路径的一部分。如果这个文件夹不存在，或者当你用"... "保存文件时，这个文件夹将被创建。 buffer_save_async().

该函数返回一个唯一的ID值，然后可以在保存/加载异步事件中使用该值来检查 异步_加载 ID值，如下面的扩展例子所示。事件中的async_load映射将有以下两个键/值对。

  • "id"： 保存函数返回的异步函数的ID 。
  • "status": 如果数据被正确保存/加载，将 返回true，否则返回false。

注意：在HTML5上，如果你是从服务器而不是本地存储中加载文件，这是首选的方法，因为同步加载在大多数浏览器上已被废弃，最终将被淘汰。

语法。

buffer_load_async(buffer, filename, offset, size)。

争论 描述
缓冲区 要加载的缓冲区的索引。
文件名 要加载的文件的名称。
补偿 缓冲区内要加载的偏移量（以字节为单位）。
尺寸 要加载的缓冲区的大小（以字节为单位）。

返回。

真实的

扩展实例。

buffer_load_async()函数可以从任何事件中调用，由于它是异步的，所以回调可以是瞬间的，也可以是几秒钟的。调用该函数很简单，看起来像这样。

loadid = buffer_load_async(buff, "Player_Save.sav", 0, 16384);

上述代码将文件"Player_Save.sav "的内容加载到给定的缓冲区，将函数调用的ID存储在变量" loadid "中。当加载完成后，异步保存/加载事件将被触发。 触发，你可以解析async_load地图，以获得正确的函数ID，像这样。

if ds_map_find_value(async_load, "id") == loadid
{
if ds_map_find_value(async_load, "status") == false
{
show_debug_message("加载失败！");
}
}

上面的代码将首先检查已经创建的DS地图的ID，然后检查回调的状态，如果有任何问题，将发布一个调试信息。
