buffer_read

此功能可用于从先前创建的缓冲区读取数据。返回值将取决于您正在读取的数据类型，其本身由以下常量定义：

持续的描述
buffer_u8一个无符号的8位整数。这是从0到255的正值。
buffer_s8有符号的8位整数。可以是-128到127之间的正值或负值（0被分类为正值）。
buffer_u16一个无符号的16位整数。这是从0-65,535的正值。
buffer_s16有符号的16位整数。可以是-32,768到32,767之间的正值或负值（0分类为正值）。
buffer_u32一个无符号的32位整数。这是从0到4,294,967,295的正值。
buffer_s32有符号的32位整数。这可以是正值或负值，范围是-2,147,483,648至2,147,483,647（0分类为正值）。
buffer_u64一个无符号的64位整数。
buffer_f16 16位浮点数。这可以是正值或负值，范围为+/-65504。 （当前不支持！）
buffer_f32一个32位浮点数。这可以是正值或负值，范围为+/- 16777216。
buffer_f64 64位浮点数。
buffer_bool布尔值。只能是1或0（true或者 false）
buffer_string任何大小的字符串。
buffer_text任何大小的字符串，不带最终的空终止符。

如果函数成功，它将返回给定类型的值，但是，如果函数失败，则将导致运行程序错误。

注意：对于读取的数据使用错误的数据类型将导致返回错误的值。

句法：

buffer_read(buffer, type)

争论描述
buffer要读取的缓冲区的索引。
type从缓冲区读取的数据类型（请参见上面的常量列表）。

返回值：

Real/String

例子：

var cmd = buffer_read(buff, buffer_s16);

上面的代码从ID存入变量“ buff”的缓冲区中读取一个带符号的16位值到局部变量“ cmd”中。

返回：缓冲区
