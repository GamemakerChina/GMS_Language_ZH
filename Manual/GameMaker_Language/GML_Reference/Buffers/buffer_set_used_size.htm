buffer_set_used_size

这个函数主要是在扩展中使用的，它允许你设置给定缓冲区的 "已用 "大小，也就是已经写入的字节数。

当你从扩展写数据到缓冲区时，GameMaker不知道缓冲区有多少被扩展代码填满，也就无法读取这些数据。这个函数可以被扩展调用，以告诉引擎有多少字节的数据被写入缓冲区 被写入缓冲区，所以引擎可以读取这些数据。

语法。

buffer_set_used_size(buffer, size)。

争论 描述
缓冲区 要使用的缓冲区的索引。
尺寸 要设置为 "已用 "大小的字节数。

返回。

不适用

例子。

buffer_write(_bufferAddress, buffer_u8, 1);
buffer_write(_bufferAddress, buffer_u8, 2);
buffer_write(_bufferAddress, buffer_u16, 400);

buffer_set_used_size(_bufferAddress, 4);

上述函数将在扩展中被调用，以将一些数据写入GameMaker缓冲区（通过其内存地址），然后设置被写入的字节数，以便引擎能够读取这些数据。
