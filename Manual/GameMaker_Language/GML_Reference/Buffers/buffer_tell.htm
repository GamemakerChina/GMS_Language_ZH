缓冲_告诉

当你在缓冲区中读取或写入数据时，使用 buffer_read()或 buffer_write()读取或写入数据时，当前的 "寻找 "位置会被写入或读取的字节提前，通过 这个函数可以得到当前的 "寻求 "位置，以便在其他缓冲区函数中使用。例如，如果你的缓冲区对齐方式被设置为4字节，你写了一个1字节大小的数据，然后做一个buffer_tell()。 但是，如果你写另一个数据，也是1字节大小，然后做buffer_tell()，你会得到一个5的返回值，因为对齐方式已经把数据 "填充 "到这个位置。

语法。

buffer_tell(buffer)。

争论 描述
缓冲区 要使用的缓冲区的索引。

返回。

真实的

例子。

var pos = buffer_tell(buff); buffer_seek(buff, buffer_seek_start, 0);
val[0] = buffer_read(buff, buffer_S16);
val[1] = buffer_read(buff, buffer_S16);
val[2] = buffer_read(buff, buffer_S16);
buffer_seek(buff, buffer_seek_start, pos) 。

上面的代码将把变量 "buff "中索引的缓冲区内的当前查找位置存储到本地变量 "pos "中。然后，缓冲区的搜索位置将被设置为缓冲区的起始位置，并将三条数据读入一个数组。 读取到一个数组中，最后再将缓冲区的寻道位置重新设置到之前的位置。
