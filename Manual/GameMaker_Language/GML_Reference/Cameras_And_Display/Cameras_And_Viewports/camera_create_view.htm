摄像机_创建_视图

此函数将创建一个新的摄像机，然后设置其进入房间的视角，再返回唯一的ID值，以便在以后的函数调用中识别该摄像机。摄像机的创建将使用你在此函数中设置的位置、尺寸 和其他细节，与您在房间编辑器中设置的值相关联（见这里）。请注意，这个函数可以接受不同数量的参数，只有X/Y位置和宽/高 参数是必须的。任何没有提供的参数将使用其默认参数。

请注意，摄像机和所有的动态资源一样，在创建时将占用内存。这意味着你需要注意不要不必要地创建相机，并通过调用函数销毁那些你不需要的相机 camera_destroy(). 这也意味着，如果分配给全局变量，摄像机将在各个房间之间持续存在（分配给实例或局部变量时也会持续存在，但你在更换房间后无法访问它们，因此会有内存泄漏），所以 例如，你可以创建一个新的摄像机，将其分配给一个全局变量，然后在每个房间开始时使用camera_set_*函数来设置属性，再将其分配给一个视图端口。

语法。

camera_create_view(room_x, room_y, width, height, [angle, object, x_speed, y_speed, x_border, y_border]。

争论 描述
房间_x 摄像机视图所捕捉到的左边区域的X位置（在房间内）。
房间_y 摄像机视图捕捉到的区域顶部的y位置（在房间内）。
宽度 视图捕获的区域的宽度。
高度 视图捕捉到的区域的高度。
角度 设置摄像机视角的角度（从0°-360°，默认为0）。
对象 要跟踪的摄像机视图的对象ID（或实例ID）（默认为-1，没有对象）。
x_speed 视图可以移动的最大水平速度（设置为-1表示 "即时 "跟随，默认为-1）。
y_speed 视图可以移动的最大垂直速度（设置为-1表示 "即时 "跟随，默认为-1）。
x_border 视图区域的边缘和它被设置为跟随的实例之间的最小水平空间（在视图开始移动之前，如果它被设置为跟随一个实例，默认为0）。
y_border 视图区域的边缘和它被设置为跟随的实例之间的最小垂直空间（在视图开始移动之前，如果它被设置为跟随一个实例，默认为0）。

返回。

真实（唯一的相机ID值）。

例子。

camera_destroy(view_camera[0]);
view_camera[0] = camera_create_view(0, 0, 640, 480, 0, obj_Player, 5, 5, -1, -1) 。

上面的代码删除了当前分配给视口[0]的摄像机，然后创建一个新的摄像机，定义其视图属性，然后将其分配给视口[0]。
