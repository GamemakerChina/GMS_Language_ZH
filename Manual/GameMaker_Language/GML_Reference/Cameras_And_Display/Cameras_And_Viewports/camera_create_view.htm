camera_create_view

此函数将创建一个新的摄像头，然后将其视图设置到房间中，然后返回唯一的ID值以在以后的函数调用中标识该摄像头。将使用您在此功能中为位置，大小和其他详细信息设置的值来创建摄像机，这些值与您在“房间编辑器”中设置的值相关（请参见此处）。请注意，此函数可以接受可变数量的参数，仅x / y位置和width / height参数是必需的。任何未提供的参数将使用其默认参数。

请注意，相机就像所有动态资源一样，在创建时会占用内存。这意味着您需要注意不要不必要地创建相机，并通过调用函数销毁不需要的相机 camera_destroy()。这也意味着，如果将摄像机分配给 global变量（它们在分配给实例变量或局部变量时也将保留，但是在更改空间后将无法访问它们，因此会导致内存泄漏），以便您可以（例如）创建一个新的摄像机，将其分配给全局变量，然后在每个房间的开头使用 camera_set_*函数，用于在将属性分配给视口之前对其进行设置。

句法：

camera_create_view(room_x, room_y, width, height, [angle, object, x_speed, y_speed, x_border, y_border])

争论描述
room_x相机视图捕获的区域左侧的x位置（房间内）。
room_y相机视图捕获的区域顶部的y位置（在房间内）。
width视图捕获的区域的宽度。
height视图捕获的区域的高度。
angle为相机视图设置的角度（从0°-360°，默认为0）。
object摄像机视图要遵循的对象ID（或实例ID）（默认为-1，无对象）。
x_speed视图可以移动的最大水平速度（以下“即时”设置为-1，默认值为-1）。
y_speed视图可以移动的最大垂直速度（以下“即时”设置为-1，默认值为-1）。
x_border视图区域的边缘与设置为跟随该实例的实例之间的最小水平空间（在视图开始移动之前，如果设置为跟随一个实例，则默认值为0）。
y_border视图区域的边缘与设置为跟随该实例的实例之间的最小垂直空间（在视图开始移动之前，如果设置为跟随一个实例，则默认值为0）。

返回值：

Real (unique camera ID value)

例子：

camera_destroy(view_camera[0]);
view_camera[0] = camera_create_view(0, 0, 640, 480, 0, obj_Player, 5, 5, -1, -1);

上面的代码将删除分配给viewport [0]的当前摄像机，然后创建一个新摄像机，定义其视图属性，然后将其分配给viewport [0]。
