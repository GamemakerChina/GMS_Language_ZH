window_get_visible_rects

使用此功能，您可以在每个连接的显示器上找到由（x1，y1）到（x2，y2）定义的矩形的重叠区域。该函数将返回一个每个显示屏具有8个值的数组（即：如果有两个显示屏，则数组的长度为16个索引），其中值[0... 3]对应于overlayx1，overlapy1，overlapx2， overlayy2-定义此显示器上的重叠区域，如果没有重叠，则将其设置为0,0,0,0-值[4... 7]对应于monitorx1，monitory1，monitorx2，monitory2-的坐标虚拟显示空间中的显示。例如，这可用于测试保存的窗口位置是否可见（用户可能已断开外部监视器的连接或将窗口移离屏幕，从而使保存的窗口位置无效）。

句法：

window_get_visible_rects(x1, y1, x2, y2);

争论描述
x1要检查的矩形的左边缘
y1要检查的矩形的上边缘。
x2要检查的矩形的右边缘
y2要检查的矩形的底边。

返回值：

Array

例子：

var wx = window_get_x();
var wy = window_get_y();
var ww = window_get_width();
var wh = window_get_height();
display_data = window_get_visible_rects(wx, wy, wx + ww, wy + wh);
display_num = array_length(display_data) / 8;

上面的代码将生成一个保存在变量中的一维数组 display_data包含有关显示的信息以及创建变量 display_num保留找到的活动显示的数量。

返回：游戏窗口
