display_set_timing_method

此功能可用于设置用于渲染游戏的计时方法。该方法可以是下列常量之一：

持续的描述
tm_sleep使用睡眠余量值作为主要计时方法
tm_countvsyncs使用vsync计时作为主要计时方法（所有受支持平台的默认设置）

vsync计时方法使用目标平台支持的垂直同步为游戏渲染计时计算提供锚点，而将模式设置为睡眠余量将仅尝试确保每个帧持续正确的时间量（例如1/30或1/60秒）。通常，默认的vsync时序将提供最平滑的结果，但是请注意，使用vsync方法时，睡眠余量值仍然有意义，尽管会减少影响，我们建议在使用此方法时将其保留为默认值。

在PS4， Ubuntu和HTML5以外的所有平台上GameMaker Studio 2将使用vsync计时方法，而在不受支持的平台上，只能使用睡眠裕度计时。

如果您想设置睡眠余量，可以使用功能 display_set_sleep_margin()您可以找到该函数当前使用的计时方法 display_get_timing_method()。

句法：

display_set_timing_method(method);

争论描述
method使用的计时方法（请参见上面的常量列表）

返回值：

N/A

例子：

if display_get_timing_method() != tm_sleep
{
display_set_timing_method(tm_sleep);
if display_get_sleep_margin() != 20
    {
    display_set_sleep_margin(20);
    }
}

上面的代码将检查计时方法，然后将其设置为 tm_sleep它将被设置为该值，睡眠裕度将被设置为20。
