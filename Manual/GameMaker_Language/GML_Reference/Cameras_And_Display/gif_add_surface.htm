gif_add_surface

使用此功能，您可以从表面将框架保存到GIF文件。您提供GIF文件ID（由函数返回）gif_open() ）和表面ID（由函数返回）surface_create() ），以及帧之间的延迟时间。延迟时间的计算单位为1/100秒，但请注意，一旦创建了GIF，例如，如果将其用于促销目的，则不同的浏览器对帧延迟的解释可能会略有不同-有关详细信息， 请参见此处。您可以（可选）提供用于添加到GIF的曲面的X和Y偏移值，在这种情况下，绘制到GIF的零件将从偏移开始，而不是默认（0,0）位置，从而允许您仅选择要添加的曲面的一部分。您还可以提供一个量化值，该值是0到3之间的整数，其中0是完全量化，3是零量化（较低的值会导致质量下降，但会生成较小的GIF）。请注意，默认质量2和最高质量3之间没有太大差异，使用3会大大减慢GIF的创建速度，因此在使用最大值时应格外小心。

请注意，GIF中的帧数或这些帧的大小没有内置限制，但是如果使用过多内存，则该函数将失败并返回-1，否则将返回0。

句法：

gif_add_surface(gif_index, surface, delay_time, [xoffset], [yoffset]);

争论描述
gif_index将表面数据添加到的gif的ID
surface用作添加框架的表面的ID
delay_time帧的延迟时间（以百分之一秒为单位）
[xoffset] [可选]沿曲面的X轴添加的偏移量
[yoffset] [可选]沿曲面的Y轴添加的偏移量
[quantization] [可选]量化数量，范围为0到3（最低质量为0，最高质量为3，默认值为2）

返回值：

Integer

例子：

if save_gif == true
{
if count == 0
    {
    gif_image = gif_open(room_width, room_height);
    }
else if count < 30
    {
    gif_add_surface(gif_image, application_surface, 6/100);
    }
else
    {
    gif_save(gif_image, "GameCapture.gif");
    count = 0;
    save_gif = false;
    }
count++;
}

上面的代码将创建一个从应用程序表面获取30帧的GIF图像文件，然后将其保存。
