屏幕_保存部分

通过这个功能，你可以保存游戏中某一部分的截图，就像它目前在游戏显示屏中出现的那样。选择的坐标是基于显示器（如果不是全屏，则是窗口）的绝对尺寸，所以（0,0）总是屏幕（或窗口）的左上角。 所以（0,0）总是屏幕（或窗口）的左上角。下面是一张图片，显示了这个功能的工作原理。

该文件将被保存到游戏的工作目录中。 游戏的工作目录。值得注意的是，该函数被设计为在Draw GUI End Event中调用，这应该在所有平台上得到一致的结果。在任何其他事件中使用它可能不会像预期的那样工作，并且可能在不同的目标甚至设备上得到不同的 可能会在不同的目标甚至是设备上产生不同的结果。

注意： 这个功能在HTML5目标上不起作用。

语法。

screen_save_part(fname,x,y,w,h)。

争论 描述
fname 要保存的屏幕截图的文件名
x 要采集的屏幕区域的左上角的X坐标。
y 要采集的屏幕区域的左上角的y坐标。
w 要捕获的屏幕区域的宽度
h 要捕获的屏幕区域的高度

返回。

不适用

例子。

if keyboard_check_pressed(ord("S"))
{
screen_save_part("Screens\Screen_" + string(num++) + ".png", 0, 0, 100, 100)
}

上面的代码在用户按下 "S "键时，对游戏屏幕上100像素的部分进行截图。这个屏幕截图被保存到一个特定的文件夹，并使用一个变量进行编号，然后递增。
