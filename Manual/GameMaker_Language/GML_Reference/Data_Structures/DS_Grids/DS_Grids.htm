DS网格

DS网格基本上是具有2维的数组类型。您可以通过设置网格的整数宽度和高度来定义网格，然后再设置网格内“单元格”的总数（宽度和高度相乘），然后将这些单元格用于保存不同的数据值（实数或字符串）。

这种类型的结构允许您通过给出网格的x和y索引来设置和检索网格中的像元值（x和y轴的索引都从0开始，直到宽度-1和高度-1）。但是，您也可以在特定的网格区域中设置值，添加单个值，并检索给定区域中的总和，最大值，最小值和平均值，从而使该数据结构在您可用的所有数据中成为最通用的。

访问DS网格数据结构时，应始终尝试对单元位置使用整数值，否则，所有非整数索引都将由GameMaker Studio 2 。如果这不是您所需要的，那么您将需要先进行四舍五入，然后再传递要检查的索引。

注：与所有的动态资源，数据结构占用内存，当不再需要以防止内存泄漏，这将减缓，并最终崩溃的游戏，所以应该总是被销毁。

注意：如果需要检查数据结构是否存在，可以使用ds_exists（）函数。

除了下面列出的特定功能，您都可以使用表达式（称为accessor ）来添加或修改DS网格的内容。该访问器看起来类似于具有以下语法的2D数组：

grid_index[# xpos, ypos]

您可以从Accessors GML概述页面上找到更多信息以及示例。
