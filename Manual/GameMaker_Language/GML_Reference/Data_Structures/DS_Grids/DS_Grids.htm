DS网格

DS网格基本上是一种 有两个维度的 数组。你通过设置其整数的宽度和高度来定义网格，然后设置网格中的 "单元格 "的总数 (宽度和高度相乘)，然后这些单元格被用来保存不同的数据值（实数或字符串）。

这种类型的结构允许你设置和检索网格中单元格的值，通过给出它的x和 y索引（x轴和y轴都从0开始，一直到宽度-1和高度-1）。但你也可以在特定的网格区域内设置数值，添加单个数值，并检索特定区域内的总和、最大、最小和平均值。 平均值，使这个数据结构成为所有可用的数据结构中最通用的。

当访问DS网格数据结构时，你应该总是尽量使用整数值来表示单元格的位置，如果你不这样做，所有非整数的指数都会被GameMaker Studio 2浮动。如果这不是你所需要的，那么你将需要 在传递你希望检查的索引之前，你需要自己做四舍五入的工作。

注意： 与所有的动态资源一样，数据结构会占用内存，所以当不再需要时，应该总是销毁，以防止内存泄漏，这将减缓并最终使你的游戏崩溃。

注意：如果你需要检查一个数据结构是否存在，你可以使用 ds_exists()函数。

除了下面列出的具体功能外，你都可以使用一个表达式（称为访问器）来添加或修改你的DS网格的内容。这个访问器看起来类似于一个2D 数组，其语法如下。

grid_index[# xpos, ypos]

你可以从GML概述的Accessors页面中找到更多的信息和例子。
