DS清单

DS列表是一种数据结构，它在添加信息时按顺序存储信息（非常类似于array ）。 DS列表是非常灵活的数据结构，允许您在列表的末尾添加值或将它们插入列表的中间位置，还可以让您随机排序（随机化值）或对它们进行升序排序或降序排列。

访问DS列表数据结构时，应始终尝试对列表位置使用整数值GameMaker Studio 2将对所有非整数索引进行下限。如果这不是您所需要的，那么您将需要先进行四舍五入，然后再传递要检查的索引。

注：与所有的动态资源，数据结构占用内存，当不再需要以防止内存泄漏，这将减缓，并最终崩溃的游戏，所以应该总是被销毁。

注意：如果需要检查数据结构是否存在，可以使用ds_exists（）函数。

除了下面列出的特定功能之外，您都可以使用表达式（称为accessor ）添加或修改DS列表的内容。该访问器看起来类似于具有以下语法的一维数组：

list_index[| index]

您可以从Accessors GML概述页面上找到更多信息以及示例。

存在处理DS列表的以下功能。

  • ds_list_create
  • ds_list_destroy
  • ds_list_clear
  • ds_list_empty
  • ds_list_size
  • ds_list_add
  • ds_list_set
  • ds_list_delete
  • ds_list_find_index
  • ds_list_find_value
  • ds_list_insert
  • ds_list_replace
  • ds_list_shuffle
  • ds_list_sort
  • ds_list_copy
  • ds_list_read
  • ds_list_write

Ds列表也可以用于保存DS映射和其他DS列表，因此我们具有一些特殊功能，用于在列表中“标记”或“标记”位置以保存这些数据结构。通常，这不是必需的，但是它的优点是，在销毁父列表时，可以自动对标记的数据结构进行垃圾回收（销毁），并且与 json_encode()函数，以便对列表及其数据进行正确编码。其功能是：

  • ds_list_mark_as_list
  • ds_list_mark_as_map
  • ds_list_is_list
  • ds_list_is_map
