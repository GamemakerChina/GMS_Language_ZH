ds_list_destroy

此函数将从内存中删除给定的列表数据结构，释放其正在使用的资源并删除其包含的所有值。当您使用完ds_list后，应始终使用此功能，以防止内存泄漏会减慢游戏并使其崩溃。请注意，销毁列表将取消引用存储在其中的任何数据结构，从而导致内存泄漏，因此您需要遍历该列表并手动销毁所有数据结构项，然后再销毁它以防止这种情况。唯一不需要的时间是将列表中的任何项目标记为另一个DS列表或DS映射时，在这种情况下，这些项目将被销毁并且其内存也将自动清除。

重要的！创建数据结构时，用于标识它的索引值是一个从0开始的整数值。这意味着不同类型的数据结构可以具有相同的索引值，因此如果有疑问，应在使用ds_exists（）函数之前访问它们。还要注意，索引已被重新使用，因此新创建的索引可能会被破坏后的数据结构索引值使用，因此我们建议始终将破坏DS索引的变量设置为-1。

句法：

ds_list_destroy(id);

争论描述
id要删除的数据结构的ID。

返回值：

N/A

例子：

if (lives == 0)
    {
    ds_list_destroy(AI_list);
    AI_list = -1;
    room_goto(rm_Menu);
    }

上面的代码将检查内置全局变量“ lives”的值，如果该值为0，它将销毁在变量“ AI_list”中索引的DS列表，然后更改房间。

返回： DS列表
