ds_list_mark_as_list

此功能将“标记”（或“标记”）先前创建的DS列表中的给定位置以保存另一个DS列表。编码JSON字符串时可以使用此功能（可以使用 json_encode()），但在嵌套数据结构时也很有用，因为当父DS列表被销毁时，以此方式标记的项目将自动进行垃圾回收（销毁）。这意味着您在销毁“父”清单之前不必手动浏览清单内容并单独销毁标记的数据结构。但是，如果您分别删除列表位置，则其中包含的DS列表将不会被垃圾回收，您应在从父列表位置删除项目之前调用适当的DS列表销毁函数。另请注意，如果您调用该函数 ds_list_clear()在列表上，清除列表时，标记为列表的所有项目也会被销毁。

还值得注意的是，如果已标记的位置在列表中更改，则“标记”将随之移动，因此，如果您在列表中标记了位置3（例如），然后在其下插入2个其他项，它会移动到位置5，但仍会标记为列表。

句法：

ds_list_mark_as_list(id, pos);

争论描述
id要标记的列表的ID。
pos列表中要标记的位置。

返回值：

Real (integer - ID of the DS list if successful or -1 if it fails)

例子：

var j_list = ds_list_create();
var sub_list = ds_list_create();
ds_list_add(sub_list, health);
ds_list_add(sub_list, lives);
ds_list_add(sub_list, score);
ds_list_add(j_list, sub_list);
ds_list_mark_as_list(j_list, 0);

上面的代码创建了两个DS列表，然后使用全局变量中的各个值填充其中一个。然后将此列表添加到第二个列表中，并在位置上进行“标记”，以便以后可以对其进行正确编码。

返回： DS列表
