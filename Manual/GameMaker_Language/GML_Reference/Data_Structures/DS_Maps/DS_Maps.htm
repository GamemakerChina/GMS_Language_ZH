DS地图

DS地图数据结构是一种非常有用的结构，因为它可以存储键和值对。例如，您游戏中的角色可以具有许多不同的物品（键），并且对于每个单独的物品，他可以具有多个物品（值），例如在RPG中，您可以拥有10个健康药水，5个法力药水和100金币。地图在一处维护所有这样的对，您可以将对添加到地图，并使用一些简单功能搜索与某些键对应的值。但是，在使用地图之前，您需要了解几件事！映射不会以任何（可识别的）方式排序，这意味着要找到某个键，您可能必须遍历整个过程（这很慢）。也没有办法保持两个相同的键，也不能为一个键分配两个值。

注：与所有的动态资源，数据结构占用内存，当不再需要以防止内存泄漏，这将减缓，并最终崩溃的游戏，所以应该总是被销毁。

注意：如果需要检查数据结构是否存在，可以使用ds_exists（）函数。

除了下面列出的特定功能，您都可以使用表达式（称为accessor ）来添加或修改DS映射的内容。该访问器看起来类似于具有以下语法的一维数组：

map_index[? key]

您可以从Accessors GML概述页面上找到更多信息以及示例。

存在以下处理DS映射的功能。

  • ds_map_exists
  • ds_map_create
  • ds_map_add
  • ds_map_clear
  • ds_map_copy
  • ds_map_replace
  • ds_map_delete
  • ds_map_empty
  • ds_map_size
  • ds_map_find_first
  • ds_map_find_last
  • ds_map_find_next
  • ds_map_find_previous
  • ds_map_find_value
  • ds_map_keys_to_array
  • ds_map_values_to_array
  • ds_map_set
  • ds_map_read
  • ds_map_write
  • ds_map_destroy

还有另外四个功能可用于保存和加载DS地图。这些功能将混淆地图，并使用一种文件格式将其存储在目标平台上的安全位置，这意味着最终文件无法在设备之间传输：

  • ds_map_secure_save
  • ds_map_secure_save_buffer
  • ds_map_secure_load
  • ds_map_secure_load_buffer

DS映射的用途之一是在使用JSON时，因此有一些特定于此的特殊功能：

  • ds_map_add_list
  • ds_map_add_map
  • ds_map_replace_list
  • ds_map_replace_map
  • ds_map_is_list
  • ds_map_is_map

注意：虽然这些功能允许您在映射中添加列表和映射，但是它们对于JSON以外的其他功能都没有用，并且如果将嵌套的映射和列表写入磁盘或以任何其他方式访问，它们将无法正确读取。
