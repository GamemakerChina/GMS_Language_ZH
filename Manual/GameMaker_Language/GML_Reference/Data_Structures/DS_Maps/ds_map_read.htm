ds_map_read

这个函数将接收一个之前由函数创建的字符串 ds_map_write()创建的字符串，然后将其读入先前创建的DS地图 。如果该字符串被读入的地图 被读入的地图包含任何键/值对，在重新构建保存的地图之前，这些键/值对将首先被清除。注意，如果数据结构是用GameMaker Studio 2以前的版本创建的，你应该添加一个可选参数 "legacy"，将其设置为true，因为这个版本的字符串格式已经改变了。

语法。

ds_map_read(id, str [, legacy])。

争论 描述
身份证 要读入字符串的数据结构的ID
弦外之音 要读取的字符串
遗产（可选 可以是真或假，也可以完全省略。

返回。

不适用

例子。

inventory = ds_map_create();
ini_open("map.ini");
var t_string = ini_read_string("Saved", "0", "");
if (t_string != " ")
{
ds_map_read( inventory, t_string);
}
ini_close()。

上面的代码创建了一个新的DS地图，并将其id索引存储在变量 "invent "中。然后，它打开一个ini文件，从该文件中读取一个字符串到临时变量 "t_string"。最后，它检查以确保该 字符串是有效的（而不是默认的ini值""），如果是的话，它就把字符串读到新创建的DS地图中，然后再次关闭ini文件。

返回。DS地图
