DS队列

DS队列在某种程度上类似于DS堆栈，但它以先进先出（FIFO）的方式工作。首先被放入队列的值也从它除去第一。您可以将其视为商店中的待付款队列，其中先排队的人先付款然后离开商店。队列通常用于存储仍然需要由实例完成的动作或行为（例如，以脚本的形式），但是还有许多其他用途。

注：与所有的动态资源，数据结构占用内存，当不再需要以防止内存泄漏，这将减缓，并最终崩溃的游戏，所以应该总是被销毁。

注意：如果需要检查数据结构是否存在，可以使用ds_exists（）函数。

存在处理DS队列的以下功能：

  • ds_queue_create
  • ds_queue_destroy
  • ds_queue_clear
  • ds_queue_empty
  • ds_queue_size
  • ds_queue_dequeue
  • ds_queue_enqueue
  • ds_queue_head
  • ds_queue_tail
  • ds_queue_copy
  • ds_queue_read
  • ds_queue_write
