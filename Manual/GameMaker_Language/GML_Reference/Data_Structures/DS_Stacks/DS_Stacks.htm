DS堆栈

DS堆栈是所谓的后进先出（LIFO）数据结构，类似于DS队列。您可以将值压入堆栈，然后通过将其从堆栈中弹出来再次将其删除，最近一次在堆栈上压入的值是再次从堆栈中弹出的值（想想一堆硬币，每个硬币您添加的硬币必须先被移除，然后才能进入下方的其余硬币。当有要处理的中断，具有递归功能，甚至为游戏构建基本的AI时，通常会使用堆栈。

注：与所有的动态资源，数据结构占用内存，当不再需要以防止内存泄漏，这将减缓，并最终崩溃的游戏，所以应该总是被销毁。

注意：如果需要检查数据结构是否存在，可以使用ds_exists（）函数。

存在以下处理DS堆栈的功能。

  • ds_stack_create
  • ds_stack_destroy
  • ds_stack_clear
  • ds_stack_empty
  • ds_stack_size
  • ds_stack_copy
  • ds_stack_top
  • ds_stack_pop
  • ds_stack_push
  • ds_stack_read
  • ds_stack_write
