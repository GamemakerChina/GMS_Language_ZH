数据结构

在游戏中，您通常需要以精确且有序的方式存储信息。例如，您可能需要存储一个人携带的物品的清单，或者您可能希望存储仍然需要访问的地方的网格。现在，您可以为此使用数组，但是如果您想执行更复杂的操作（例如，对数据进行排序或搜索特定项目），则需要编写大块的GML代码，这些代码执行起来很慢，难以调试和一个真正的痛苦写！

为了解决这个问题， GameMaker Studio 2具有许多内置数据结构，可以通过专业功能进行访问。有六种不同类型的数据结构，每种类型都有其自己的优势，这取决于您要存储的信息类型以及以后希望如何使用它：堆栈，队列，列表，地图，优先级队列和网格。

本质上，所有数据结构都以相同的方式工作-创建数据结构并将其索引值存储在变量中。然后，您可以使用此索引在可以对其执行操作的所有其他函数调用中引用数据结构。最后，一旦完成，您将再次破坏数据结构以将其从内存中删除。您可以根据需要同时使用任意多个结构，并且所有结构都可以存储任何可用的数据类型。

注：与所有的动态资源，数据结构占用内存，当不再需要以防止内存泄漏，这将减缓，并最终崩溃的游戏，所以应该总是被销毁。

重要的！创建数据结构时，用于标识它的索引值是一个从0开始的整数值。这意味着不同的数据结构可以具有相同的索引值，因此如果有疑问，应在访问它们之前使用ds_exists函数。另请注意，索引已被重新使用，因此，新创建的索引可能会被破坏后的数据结构索引值使用，并且我们建议您在破坏数据结构后将任何将DS索引保持为-1的变量设置为1。

以下各节中提供了有关不同数据结构的信息：

  • 格网
  • 清单
  • 地图
  • 优先队列
  • s列
  • 堆栈

在游戏中使用数据结构之前，您还应该意识到，在某些时刻，由于舍入错误，您可能会得到与预期不同的结果。这可以通过使用以下功能更改DS精度来解决：

  • ds_set_precision

还有一个特殊功能，用于检查是否存在任何给定类型的数据结构：

  • ds_exists
