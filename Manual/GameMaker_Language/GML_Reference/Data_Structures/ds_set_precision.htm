ds_set_precision

比较值时（例如，在地图中搜索或对列表进行排序时）， GameMaker Studio 2必须确定两个值何时相等。对于字符串和整数值，这是显而易见的，但对于实数，由于浮点舍入错误，看似相等的数字很容易变得不相等。例如， （5/3）* 3可能不等于5！为了避免这种情况，在所有实数函数上使用一个精度值，并且当两个数字之间的差小于此精度时，它们被视为相等。除非为此功能更改，否则所有数据结构函数的默认精度均为0.0000001。

注意：此精度可用于所有数据结构，但不能用于GML其他比较中！

句法：

ds_set_precision(prec);

争论描述
prec精度值（默认为0.0000001）

返回值：

Returns:N/A

例子：

ds_set_precision(0.0001);

上面的代码将更改所有数据结构函数的默认精度设置。

返回：数据结构
