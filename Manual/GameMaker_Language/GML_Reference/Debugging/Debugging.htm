调试

在制作游戏时，通常非常重要的一点是，您必须在特定的时间点了解游戏机制的某些细节，或者能够为某些功能输入某些键值，以测试游戏的不同方法和想法。为此， GameMaker Studio 2为您提供了许多调试功能，这些功能将在调试模式下（或在Windows目标上）显示消息并要求输入，但在为其他目标模块编译时将被忽略。函数将游戏置于闭环状态，直到输入解决。

注意：尽管它们可能在某些平台上运行，但除调试外，不得用于其他任何用途。

  • debug_mode
  • debug_event
  • debug_get_callstack
  • exception_unhandled_handler
  • get_integer
  • get_string
  • show_error
  • show_message
  • show_question
  • show_debug_message
  • show_debug_overlay
  • code_is_compiled
  • fps
  • fps_real

除了这些专业功能以外， GameMaker Studio 2还提供一长串错误消息，这些错误消息可以在编写游戏代码，编译游戏或运行游戏时显示，以帮助您创建游戏。这些消息指向有问题的特定错误，可用于查明特定对象或脚本函数中的问题，从而极大地帮助调试游戏。但是，错误消息的含义可能并不总是很明显，因此以下各节列出了所有可能的消息并解释了它们的含义，以使您的生活更轻松：

  • 语法错误
  • 编译器错误
  • 转轮错误

您还可以在“调试器”部分中找到有关如何调试游戏的更多信息。
