debug_event

此函数会生成一个自定义调试事件，当在“调试模式”下运行游戏时，该事件将显示在调试器的“图形视图”中。如果您要求在非调试模式下显示消息，则应该使用 show_debug_message()。值得注意的是，该函数还将采用两个保留的字符串，以帮助使用Visual Studio这样的外部应用程序执行调试。这些字符串是：

  • ”OutputDebugOn “-这样可以拨打 OutputDebugString对于Windows目标，这意味着所有调试输出-您在“输出”窗口中看到的所有内容-都可以由Visual Studio或第三方应用程序查看。
  • ”BreakOnError “-此选项仅适用于Windows YYC构建，这意味着runtime检测到错误，项目将不会显示通常的代码错误屏幕，而是继续进行并崩溃。这使您可以在调试时Visual Studio 。

句法：

debug_event(string)

争论描述
string要使用的定制调试事件字符串。

返回值：

N/A

例子：

if !surface_exists(global.EffectsSurface)
   {
   debug_event("Recreating Effects Surface");
   global.EffectsSurface = surface_create(room_width, room_height);
   }

上面的代码检查表面是否存在，如果不存在，则在调试器的图形视图中触发调试事件（游戏必须已经在“调试模式”下运行才能看到），然后重新创建表面。

返回：调试
