debug_get_callstack

这个函数生成一个字符串数组作为 "callstack"，其中当前脚本被列在第一位，然后是所有为执行当前脚本而运行的其他脚本。确切的字符串格式将取决于 确切的字符串格式将取决于所选择的目标平台，但大多数情况下，它将包含脚本/事件名称，然后是冒号：和行号，类似于这样。

"gml_Script_script2:1"
"gml_Script_script1:32"
"gml_Script_script0:64"
"gml_Object_object0_Create_0:1"

该函数允许传入一个可选的参数，即返回的callstack的最大深度。这个值是返回的数组中从当前脚本开始所包含的脚本数量。如果没有指定这个参数 指定，那么将返回完整的callstack。

注意，在列出调用栈脚本后，返回的数组在最后一个位置总是包含0。

语法。

debug_get_callstack([maxdepth])。

争论 描述
[最大深度] 呼叫堆栈的最大深度

返回。

阵列

例子。

如果debug_mode
{
if keyboard_check(vk_escape)
{
var _a = debug_get_callstack(4);
for (var i = 0; i < array_length_id(_a); ++i;)
{
show_debug_message(_a[i])。
}
}
}

上面的代码检查是否启用了调试模式，如果是，则检查是否有按键被按住。在这种情况下，它会将当前的调用栈输出到控制台，最大深度为4。
