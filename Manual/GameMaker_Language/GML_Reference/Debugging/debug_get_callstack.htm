debug_get_callstack

此函数生成一个字符串数组作为“调用堆栈”，其中首先列出当前脚本，然后列出所有其他为了执行当前脚本而运行的脚本。确切的字符串格式将根据所选的目标平台而有所不同，但大多数情况下将具有脚本/事件名称，然后是冒号：和行号，类似于此：

"gml_Script_script2:1"
"gml_Script_script1:32"
"gml_Script_script0:64"
"gml_Object_object0_Create_0:1"

该函数允许传入一个可选参数，这是返回的调用栈的最大深度。此值是从当前脚本开始，包含在返回数组中的脚本数。如果未指定此参数，则将返回完整的调用堆栈。

请注意，列出了调用堆栈脚本之后，返回的数组的最后位置将始终包含0。

句法：

debug_get_callstack([maxdepth])

争论描述
[maxdepth]调用堆栈的最大深度

返回值：

Array

例子：

if debug_mode
    {
    if keyboard_check(vk_escape)
        {
        var _a = debug_get_callstack(4);
        for (var i = 0; i < array_length_id(_a); ++i;)
            {
            show_debug_message(_a[i]);
            }
        }
    }

上面的代码检查是否启用了调试模式，如果已启用，则检查是否按住了某个键。在这种情况下，它将当前调用堆栈输出到控制台，最大深度为4。

返回：调试
