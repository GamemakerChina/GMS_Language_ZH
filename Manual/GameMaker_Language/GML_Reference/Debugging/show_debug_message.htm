show_debug_message

这个函数将创建一个自定义的调试信息，在运行时显示在编译器窗口。注意，信息可以是一个字符串或一个实数，但如果你需要这两种信息，那么数字必须先被转换为字符串，使用 函数将数字转换为字符串（见下面的例子）。 string()函数转换为字符串（见下面的例子），如果数字有两个以上的小数位，那么你应该用 string_format()来显示它们 因为默认情况下，小数会被四舍五入到最接近的两位小数（所以 "1.2468 "在输出窗口会显示为 "1.25"）。

用这个函数显示的调试信息将显示在IDE底部的编译器输出窗口，以及在调试模式下运行游戏时的图形视图中。 在调试模式下运行游戏时，用该函数显示的调试信息将显示在IDE底部的编译器输出窗口以及调试器的图形视图中。如果你只想在调试模式下看到信息，那么你可能应该使用 debug_event()来代替。

语法。

show_debug_message(string)。

争论 描述
绳子 要显示的自定义调试信息。

返回。

不适用

例子。

if !instance_exists(obj_Exit)
{
show_debug_message("Exit not created!");
show_debug_message("Error Num: " + string(global.error));
game_end();
}

上述代码检查实例是否存在，如果不存在，编译窗口会显示一条调试信息并结束游戏。
