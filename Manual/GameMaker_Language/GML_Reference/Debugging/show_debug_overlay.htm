show_debug_overlay

测试游戏时，可以使用此功能打开和关闭标准调试覆盖图，并且默认情况下处于禁用状态。调试覆盖图会在实际游戏窗口本身中显示一个图形CPU / GPU使用情况栏，以及当前的实际fps值，纹理交换的数量和顶点批处理的数量（请注意，纹理交换和顶点批处理永远不会为零，通常会显示值2或3，因为即使房间是空的， GameMaker Studio 2仍然没有对象可以绘制和批处理东西。

此条分为多个部分，每个部分为1/60秒。从上图可以看到，该栏由多种颜色组成：

  • 绿色-输入/输出处理（即：键盘，鼠标，游戏板，网络等）
  • 红色-步骤事件的更新速度
  • 黄色-抽奖活动所需的时间
  • 橙色-调试更新时间，仅在使用调试模块时通常可见
  • 白色-GPU剩余时间，这是等待GPU完成下一帧开始之前的帧渲染所花费的时间
  • 青色-文字呈现时间
  • 灰色-清除每个绘图步骤所需的时间
  • 蓝色-垃圾收集器运行所需的时间
  • 暗红色-GPU刷新，这是GPU从内存中清除图像所需的时间

使用此功能，无论是否在调试模式下，您都可以添加叠加层，这样，您就可以查看游戏的运行效率，并直观地了解游戏如何使用可用资源，而无需付出过多的开销调试器并排运行。

注意：由于此功能与所有其他目标相比工作方式不同，因此无法在HTML5目标上使用。

句法：

show_debug_overlay(enable);

争论描述
enable打开 （true ）或关闭（false ) the debug overlay.

返回值：

N/A

例子：

if global.debug
   {
   show_debug_overlay(true);
   }
else
   {
   show_debug_overlay(false);
   }

上面的代码将根据全局变量的值打开或关闭调试。

返回：调试
