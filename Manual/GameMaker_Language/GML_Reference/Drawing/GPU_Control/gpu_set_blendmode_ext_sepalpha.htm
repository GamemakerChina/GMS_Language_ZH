gpu_set_blendmode_ext_sepalpha

当GameMaker Studio 2要绘制一个像素时，有一个源颜色（我们要绘制的像素的颜色）和一个目标颜色（我们要绘制的像素中已经有的颜色），所以当确定像素的新颜色时 时，源色和目的色是根据所选择的混合模式来计算的。颜色的每一个分量都被存储为0到1之间的浮点值，新的颜色是通过将源颜色的每一个分量乘以某个系数来计算的。 源颜色的每个分量乘以某个系数，目标颜色的每个分量乘以另一个系数，然后将结果逐个分量相加来计算。

这个函数允许你设置不同的分量部分，这些分量部分应该被因子化以创建一个自定义的混合模式，为RGB分量和Alpha分量选择不同的混合模式因子。因此，"src" 和 "dest "参数将使用前3个混合模式值（RGB），"alphasrc "和 "alphadest "参数将采用混合模式因子的最后一部分。例如，如果你的RGB源为bm_zero。 那么将使用（0，0，0）。如果你将Alpha源设置为bm_one，那么将使用（1），最终的源混合模式为（0, 0, 0, 1）。

下图显示了每种混合模式的RGBA值，其中（Rs, Gs, Bs, As）被认为是源值，（Rd, Gd, Bd, Ad）是目的值。可供使用的11个常数既可以是源值也可以是目的值 (或两者兼而有之)，在这个函数中使用。

恒定 混合系数（红、绿、蓝、Alpha）。
bm_zero (0, 0, 0, 0)
bm_one (1, 1, 1, 1)
bm_src_colour (Rs, Gs, Bs, As)
bm_inv_src_colour (1-Rs, 1-Gs, 1-Bs, 1-As)
bm_src_alpha (如、如、如、如)
bm_inv_src_alpha (1-As, 1-As, 1-As, 1-As)
bm_dest_alpha (广告、广告、广告、广告)
bm_inv_dest_alpha (1-Ad, 1-Ad, 1-Ad, 1-Ad)
bm_dest_colour (Rd, Gd, Bd, Ad)
bm_inv_dest_colour (1-Rd, 1-Gd, 1-Bd, 1-Ad)
bm_src_alpha_sat (f, f, f, 1) 其中f = min(As, 1-Ad)

  • bm_src_colour
  • bm_inv_src_colour
  • bm_dest_colour
  • bm_inv_dest_colour
  • bm_src_alpha_sat

请注意，你可以为这个函数提供四个单独的参数，也可以提供一个参数数组（如由函数 gpu_get_blendmode_ext_sepalpha()返回的参数）。) 如果提供的是一个数组，它应该有以下四个元素。

  • [0] = 源混合模式减去alpha成分（默认为bm_src_alpha（As, As, As））。
  • [1] = 目的地混合模式减去α成分（默认为bm_inv_src_alpha（1-As, 1-As, 1-As））。
  • [2] = 源阿尔法分量（默认为bm_src_alpha(As)）。
  • [3] = 目的地alpha分量（默认为bm_inv_src_alpha（1-As））。

为了帮助你从混合模式中获得最大的收益，并帮助理解它们是如何工作的，以及它们是如何影响绘制在屏幕上的最终图像的，我们建议你阅读以下指南。

语法。

gpu_set_blendmode(src, dest, alphasrc, alphadest)。

争论 描述
画面 RGB的源混合模式系数（见上述常数）。
目的地 RGB的目标混合模式系数（见上面的常数）。
缩写为alphasrc 源混合模式系数Alpha（见上述常数）。
杏仁饼（Alphadest 目的地混合模式系数Alpha（见上面的常数）。

返回。

不适用

例子。

var bm = gpu_get_blendmode_ext_sepalpha();
bm[0] = bm_src_alpha;
bm[1] = bm_inv_src_alpha;
gpu_set_blendmode_ext_sepalpha(bm) 。

上面的代码创建了一个本地数组，并获得了当前的扩展混合模式与单独的alpha值。然后对这个数组进行操作并返回，以设置混合模式。
