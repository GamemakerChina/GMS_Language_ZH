gpu_set_zwriteenable

这个函数可以用来切换使用z-buffer的深度排序（真）和关闭（假）。当一个实例在GameMaker Studio 2中被渲染时，生成的像素的深度（它的z坐标）被存储在z-buffer中。 然后，如果该场景的另一个实例必须在同一像素处渲染，那么这两个深度将被比较（z-testing - 见 gpu_set_ztestenable())，然后将 "更接近 "观察者的那一个保存到 到Z缓冲区中，取代原来的那个。然而，这并不总是你希望发生的，有些时候，你可能希望某个东西被画在其他东西上面，不管它的Z值是多少。在这些情况下，你可以使用这个函数 来关闭对Z缓冲区的写入，从而迫使正在绘制的东西被绘制在其他东西之上。注意，这里给出的描述是针对默认的z-testing比较模式的，但是可以更改

默认值是启用z-buffering（true）。

语法。

gpu_set_zwriteenable(enable)。

争论 描述
启用 使用的比较模式（见上面的列表）。

返回。

不适用

例子。

gpu_set_zwriteenable(false);
with (obj_Menu_Containers)
{
draw_self();
}
gpu_set_zwriteenable(true)。

上面的代码关闭了z-buffer的写入，绘制了一些对象，然后再次打开，继续绘制。
