part_system_create_layer

此功能将在给定的层上创建一个新的粒子系统。您提供函数返回的唯一层ID layer_create()或用作字符串的层的名称（例如“ instance_layer”），然后将系统标记为持久性或非持久性。如果系统未标记为持久系统，那么它将在创建该系统的房间的末尾自动销毁（这与调用该函数的作用相同）part_system_destroy()并且还会破坏与系统关联的所有发射器）。然而，当为持久标记时，系统将进行到下一个房间当房间被改变，并且如果满足下列室不具有相同的名称或深度指派的一种的层，然后一个新层将被创建对于持续存在于整个房间中的系统，它将被命名为与原始层相同。更改房间时，如果随后的房间中还有另一个具有相同名称的图层，则持久层实例将被分配给具有该名称的图层，而与图层的深度无关。最后，如果持久性系统移动到一个房间，该房间的某个层的深度与创建该系统的层的深度相同，则不会将其分配给该层，而是将以相同的深度创建一个新层（具有与原始图层同名）。

该函数将为粒子系统返回唯一的ID值，该值应在需要提供系统ID的所有其他函数调用中使用。

重要的！如果将粒子系统标记为持久性系统，则在不使用粒子系统时（以及分配给它的所有发射器）都将在不使用时使用适当的销毁功能进行手动清理，否则可能会导致内存泄漏，从而对最终游戏产生负面影响。

句法：

part_system_create_layer(layer_id, persistent);

争论描述
layer要将粒子系统分配给的图层ID值或字符串（可以是任何图层类型）
persistent将粒子系统标记为持久性（设置为 true）或不（设置为 false）

返回值：

Real (unique particle system ID value)

例子：

global.p_sys = part_system_create_layer("effects_layer", true);

上面的代码将在给定的层上创建一个新的粒子系统，并将其标记为在后续房间中持久存在。粒子系统的ID存储在全局范围变量中，以备将来参考。
