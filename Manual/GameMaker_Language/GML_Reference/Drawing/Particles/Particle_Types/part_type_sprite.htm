part_type_sprite

此功能可用于设置粒子类型，以使用游戏资源中的自定义精灵。您可以选择子画面，然后告诉GameMaker Studio 2使用子画面并对其进行动画处理，如果选择动画，则还可以选择以一定的间隔更改子图像，以使动画开始和结束在创建的每个粒子的生命周期的开始和结束时（因此，如果您将生命周期设置为最小15步长和最大30步长，则无论使动画的寿命达到什么值，动画的运行速度都将更快寿命越短，速度越慢）。您也可以选择选择随机子图像，以便如果未对精灵进行动画处理，则将随机选择其他图像来创建精灵，或者如果对粒子进行了动画处理，则它将在随机点处开始动画处理。

句法：

part_type_sprite(ind, sprite, animate, stretch, random);

争论描述
ind要更改的粒子类型的索引。
sprite要将粒子类型设置为的精灵的索引。
animate是否正常（true）或否（false）跟随精灵的动画。
stretch是否拉伸精灵的动画以匹配粒子寿命（真）或不匹配（假）。
random是否选择随机子图像（真）或不选择（假）。

返回值：

N/A

例子：

global.p1 = part_type_create();
part_type_sprite(global.p1 , spr_Coins, 1, 0, 0);
part_type_size(global.p1, 1, 3, 0, 0);
part_type_scale(global.p1, 1, 1);
part_type_colour1(global.p1, c_white);
part_type_alpha2(global.p1, 1, 0);
part_type_speed(global.p1, 2, 4, 0, 0);
part_type_direction(global.p1, 0, 180, 0, 0);
part_type_gravity(global.p1, 0.20, 270);
part_type_orientation(global.p1, 0, 0, 0, 0, 1);
part_type_blend(global.p1, 1);
part_type_life(global.p1, 15, 60);

上面的代码将在全局变量“ p1”中索引的粒子类型设置为在“ spr_Coins”中索引的精灵的形状，然后设置各种其他粒子属性。
