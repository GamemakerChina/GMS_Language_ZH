粒子

GameMaker Studio 2复杂事物，通常需要一个对象并在房间周围创建该对象的实例。但是，对于图形效果而言，这可能会很昂贵，因为每个实例的处理过程都带有“成本”，这是因为实例中包含的变量和在不同事件中具有的代码。您可以通过使用图块图图层和资产图层来满足您的图形需求来降低此成本，但是这些图层通常是静态的，无法进行更改或移动。但是，还有另一种方法可以在游戏中绘制快速而通用的图形效果，即使用粒子。

粒子是具有在粒子系统内定义的某些属性的图形资源。这些属性不能直接用于单个粒子，而可以通过用于定义单个粒子及其所属系统的代码进行更改。它们对于在没有使用实例所拥有的CPU开销的情况下，在游戏中创建精美，浮华的效果（或微妙而谨慎的效果）（例如爆炸，贴花，雨，雪，星空和碎片）非常有用。

粒子系统的基本设置分三个步骤，第三步是可选步骤，具体取决于您希望如何在房间内创建粒子效果：

  • 创建粒子系统：粒子系统就像一个容器，我们将使用它来保存我们准备使用的不同粒子类型。我们使用代码为粒子定义一系列视觉方面和行为，然后将它们放置在“容器”中，以便我们可以取出它们并在以后需要的任何时间和地点使用它们。
  • 创建粒子类型：粒子类型是图形效果本身。您可以有许多不同的类型，每种类型都有各自的颜色，alpha，大小和运动范围，但是请务必注意，您无法控制单个粒子。您定义了一系列参数，粒子将被创建为具有从中选择的行为的随机分布。
  • 创建发射器：发射器是一个选项，可用于从非常明确定义的范围内爆裂或流化粒子。它们是可选的，因为您可以使用来从任何实例创建粒子 part_particles_create()和 part_particles_create_colour()功能，但它们并不总是适合每种情况。

尽管粒子是创建效果的出色工具，但它们确实带有某些限制和良好做法，除非您希望游戏性能不佳甚至可能崩溃，否则它们必须遵循：

  • 粒子系统，粒子和发射器会占用内存，因此，您应该非常小心地使用它们，因为这很容易导致内存泄漏，这会减慢速度并最终导致游戏崩溃。解决此问题的一种方法是拥有一个全局系统，该系统在游戏开始时就定义了所有内容，并在游戏结束时将其删除了，但是如果您要使用动态系统，则每个粒子和发射器（以及系统本身）都应该在瞬间被破坏不需要。
  • 粒子在CPU上可能很快且很轻，但是它们仍需要进行一些处理，因此您不应一次在屏幕上爆炸40,000。将它们限制为实现特定效果所必需的对象，仅此而已。
  • 如果您定义自己的粒子精灵而不是使用其中包含的14个精灵之一，则应尝试使其尽可能小以达到所需的效果。
  • 粒子不与任何东西相互作用。如果您需要它们与用户或游戏中的任何其他实例进行任何类型的交互，则应考虑使用实例，因为粒子是纯图形的。
  • 即使您可以在一个游戏中创建的系统，发射器和粒子的数量没有技术限制，您也应尝试将所有内容限制在最小数量，以保持尽可能低的内存使用量。
  • 在移动设备上，请格外小心，因为如果它们覆盖屏幕的较大区域，则绘制它们的速度可能会很慢（移动设备上的过度绘制是导致速度下降的主要原因之一）。
  • 使用HTML5时，不会进行添加剂混合，除非启用了WebGL，否则也不能进行颜色混合。

GameMaker Studio 2在游戏中创建粒子的最简单方法是使用内置的效果机制。这些效果是使用内部粒子系统创建的，该系统基本上是仅用于绘制图形效果的非常快速的方法，因此，使用这些功能时您不必担心所有细节（例如内存管理）。您只需指定效果的类型，必须创建的位置，所需的大小以及最后的颜色，然后让GameMaker Studio 2完成所有工作。

存在以下用于创建内置粒子效果的功能：

  • effect_create_below
  • effect_create_above
  • effect_clear

即使这些效果在范围和自定义方面受到限制，它们仍然可以轻松地用于创建一些简单而出色的效果。例如，通过在每步移动的导弹下方创建一小撮灰色烟雾，就会创建出令人信服的烟雾轨迹，因此，即使您是粒子专家，也要记住这些效果的存在，因为它们仍然可以为您节省一些时间。
