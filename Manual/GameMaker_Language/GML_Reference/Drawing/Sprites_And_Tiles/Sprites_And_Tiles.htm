精灵和瓷砖

即使您可以在房间编辑器中设置背景，也可以将精灵指定给实例，但通常这对于您制作的游戏来说不够灵活。例如，如果您有一个必须绘制菜单的实例，则可能需要一次显示各种按钮精灵，或者如果您的游戏带有视差背景滚动，则需要能够通过代码在菜单中设置它们。某种类型的控制器实例。在这些情况下，您应该使用以下功能，这些功能可以在房间的任何地方绘制精灵和背景素材，其中一些功能还允许您操纵精灵的形状，大小，颜色和Alpha以获得特殊效果。

您可以为实例分配一个精灵，默认情况下， GameMaker Studio 2将使用各种精灵变量给定的属性来绘制它，但是，如果在draw事件中添加任何内容，则您是“控制人”，必须准确GameMaker Studio 2画什么。这意味着，如果必须使用以下函数来确切定义实例将要绘制的内容以及绘制对象的位置：

  • draw_self
  • draw_sprite
  • draw_sprite_ext
  • draw_sprite_general
  • draw_sprite_part
  • draw_sprite_part_ext
  • draw_sprite_stretched
  • draw_sprite_stretched_ext
  • draw_sprite_pos
  • draw_sprite_tiled
  • draw_sprite_tiled_ext

如果您使用的是SWF格式精灵，则还有其他选项可用于控制绘图时使用的抗锯齿（有关这些类型的精灵的更多信息，请参见导入非位图精灵）。请注意，这些函数在全局级别上起作用，并且将在调用它们后绘制的所有矢量SWF精灵上设置AA。

  • draw_enable_swf_aa
  • draw_set_swf_aa_level
  • draw_get_swf_aa_level

除了上面列出的那些功能，也有专为骨骼动画使用两种特殊功能（见导入非位图精灵对于这些类型的精灵的更多信息，以及部分骨骼动画，了解与这些类型的精灵的特定功能） ：

  • draw_skeleton
  • draw_skeleton_instance
  • draw_skeleton_collision
  • draw_skeleton_time

将根据包含的数据，使用的图块集和图块地图属性自动绘制图块。但是，您可以使用以下功能绘制单个图块或图块地图：

  • draw_tile
  • draw_tilemap
