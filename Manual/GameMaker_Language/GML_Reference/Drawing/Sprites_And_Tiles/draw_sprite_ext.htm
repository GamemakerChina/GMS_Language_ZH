这个函数将绘制给定的精灵，如函数中的 draw_sprite()中的做法，但会有额外的选项来改变被绘制的精灵的比例、混合、旋转和alpha。改变这些值并不以任何方式修改资源（只修改它的绘制方式），而且你可以使用任何可用的精灵变量来代替函数中所有的 参数的直接值。下面的图片说明了不同的值如何影响精灵的绘制。

注意：只有在启用WebGL时，才建议对HTML5目标进行颜色混合，不过，如果没有启用WebGL，你仍然可以设置混合颜色，它将像正常一样混合精灵。然而，所有以这种方式进行的混合都会创建一个重复的 然后存储在缓存中并在需要时使用。这远非最佳状态，如果你使用多种颜色变化，除非你激活WebGL，否则会降低你的游戏性能。 字体缓存的大小，以便在必要时使用该函数来限制它。 sprite_set_cache_size().

注意：当使用骨架动画精灵时，这个函数可能不会像预期的那样工作，你可能会发现这个函数只画了默认姿势的第一帧。你应该使用draw_skeleton_*函数来代替。

语法。

draw_sprite_ext ( sprite, subimg, x, y, xscale, yscale, rot, color, alpha ) 。

争论 描述
精灵 要绘制的精灵的索引。
亚img 要绘制的精灵的subimg（帧）（image_index或-1与对象中的当前动画帧相关）。
x 绘制精灵的X坐标。
y 绘制精灵位置的y坐标。
xscale 精灵的水平比例，作为一个乘数。1=正常缩放，0.5是一半等等...
yscale 精灵的垂直缩放比例是一个乘数。1=正常缩放，0.5是一半等等...
腐烂 该精灵的旋转。0=向右上方，90=逆时针旋转90度等等。
颜色 混合精灵的颜色。c_white是正常显示。
阿尔法 精灵的alpha（从0到1，其中0是透明的，1是不透明的）。

返回。

不适用

例子。

draw_sprite_ext(sprite_index, image_index, x, y, image_xscale, image_yscale, image_angle, image_blend, image_alpha) 。

这将以所有默认值绘制实例指定的精灵（本质上与使用draw_self 相同）。
