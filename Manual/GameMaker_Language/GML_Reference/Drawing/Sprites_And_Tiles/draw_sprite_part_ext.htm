draw_sprite_part_ext

该函数将按照与以下规则相同的规则在给定位置绘制所选精灵的一部分 draw_sprite_part()，直到现在您才可以缩放零件，为其混合颜色或在将其绘制到屏幕上时更改其Alpha（与使用 draw_sprite_ext()）。您应该注意，如果纹理页面允许自动裁剪，则此功能可能无法按预期方式工作，因为将删除小精灵周围多余的“空”空间以创建纹理页面。要解决此问题，您将需要设置纹理页面设置（在“纹理组编辑器”中）以禁用“自动裁剪” 。

注意：使用此功能进行绘制时，将忽略子画面x偏移和y偏移，并且将在房间中指定的x / y位置的左上角绘制子画面零件。

注意：此功能仅对位图精灵有用，不适用于SWF或JSON（Spine）精灵。

注意：仅当启用WebGL时，才建议将混合颜色用于HTML5目标，但是如果未启用混合颜色，则仍可以设置混合颜色，并且它将正常混合精灵。但是，以这种方式进行的所有混合都会创建一个重复的精灵，然后将其存储在缓存中，并在需要时使用。这远非最佳选择，如果您使用多种颜色更改，除非您激活WebGL，否则它将降低游戏性能。如果您不想使用WebGL，则可以使用函数设置字体缓存大小以尝试限制此大小 sprite_set_cache_size() 。

句法：

draw_sprite_part_ext(sprite, subimg, left, top, width, height, x, y, xscale, yscale, colour, alpha);

争论描述
sprite要绘制的精灵的索引。
subimg要绘制的精灵的子图像（帧）（image_index或-1与对象中的当前动画帧相关）。
left要绘制的区域左上角的子画面上的x位置。
top要绘制的区域左上角的sprite上的y位置。
width要绘制的区域的宽度。
height要绘制的区域的高度。
x绘制精灵的位置的x坐标。
y绘制子画面的y坐标。
xscale Sprite的水平缩放比例，作为乘数：1 =正常缩放比例，0.5等于一半，依此类推...
yscale Sprite的垂直缩放比例，作为乘数：1 =正常缩放比例，0.5等于一半，依此类推...
colour混合精灵的颜色。 c_white是正常显示。
alpha子画面的Alpha（从0到1，其中0是透明的，而1是不透明的）。

返回值：

N/A

例子：

draw_sprite_part_ext(sprite_index, image_index, 8, 8, sprite_width-16, sprite_height-16, x, y, 2, 0.5, c_black, 1);

这将绘制实例指定的精灵（sprite_index）及其当前的动画帧（image_index），但是它将从精灵的所有四个侧面削掉8px的边距。然后将其拉伸为通常宽度的两倍，但为其常规高度的一半，并且尽管alpha仍为1，但它将与黑色混合（将其变为轮廓）。

背面：精灵和瓷砖
