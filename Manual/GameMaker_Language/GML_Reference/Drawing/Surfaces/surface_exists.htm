surface_exists

由于其易挥发特性，在处理表面时，此功能必不可少。表面始终保留在纹理内存中，这意味着它们可以在一瞬间到下一瞬间被销毁（例如，当屏幕保护程序在Windows上启动时或在Android设备上最小化时），因此在操作之前，应始终检查表面是否存在使用它进行任何操作（包括将其绘制到屏幕上）。下面的示例代码显示了该命令在实例的draw事件中的典型用法，以检查表面并在表面被移除后重新创建（请注意，表面最初是在对象的create事件中创建的） ）。

句法：

surface_exists(surface_id);

争论描述
surface_id要检查的表面的ID。

返回值：

Boolean

例子：

if !surface_exists(surf)
    {
    surf = surface_create(1024, 1024);
    }
else
    {
    if view_current = 1
        {
        draw_surface(surf,0,0);
        }
    }

上面的代码将检查是否存在在变量“ surf”中索引的曲面，如果不存在，它将重新创建该曲面。如果存在，则检查当前正在绘制哪个视图，如果是view [1]，则绘制表面。
