编码和散列

在处理外部文件时，最终用户总是有可能打开并更改其中包含的信息，从而改变您的游戏。这可能会导致游戏玩法元素损坏或在线得分GameMaker Studio 2为您提供了一些基本的编码功能以及对字符串和文件执行哈希检查以确保它们具有在使用前保持其完整性。还提供了一些用于编码和解码JSON格式字符串的功能。

注意：编码不是加密！ base64编码使文件无法肉眼读取，并且需要代表用户做出努力才能解码，但这种安全对黑客而言是不安全的。建议您将这些功能与自己的加密混合使用（ GameMaker脚本功能可用）。

  • json_encode
  • json_decode
  • json_stringify
  • json_parse
  • base64_encode
  • base64_decode
  • md5_string_utf8
  • md5_string_unicode
  • md5_file
  • sha1_string_utf8
  • sha1_string_unicode
  • sha1_file
  • zip_unzip
  • load_csv

值得注意的是，还有许多与编码和散列缓冲区有关的函数，您可以从以下链接中找到它们：

  • buffer_md5
  • buffer_sha1
  • buffer_crc32
  • buffer_base64_encode
  • buffer_base64_decode
  • buffer_base64_decode_ext
