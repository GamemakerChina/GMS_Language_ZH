编码和哈希

当处理外部文件时，总是有这样的可能性，即最终用户可以打开并改变它们所包含的信息，从而改变你的游戏。这可能会导致游戏元素的损坏或在线分数的欺诈（例如）。 因此，GameMaker Studio 2为你提供了一些基本的编码功能，以及对字符串和文件进行散列检查的功能 和文件，以确保它们在使用前保持其完整性。还有一些函数提供给你用于编码和解码JSON格式的字符串。

注意： 编码并不是加密!base64编码使文件无法被肉眼看清，需要用户努力解码，但它并不安全，无法防止黑客攻击。建议你将这些功能与 你自己的加密（有许多形式的加密和GameMaker的脚本功能可在互联网上找到）。

  • json_encode
  • json_decode
  • json_stringify
  • json_parse
  • base64_encode
  • base64_decode
  • md5_string_utf8
  • md5_string_unicode
  • md5_file
  • sha1_string_utf8
  • sha1_string_unicode
  • sha1_file
  • zip_unzip
  • 加载_csv

值得注意的是，还有一些与编码和散列Buffers有关的功能，你可以从以下链接中找到。
