工作目录

Working_directory实际上可以返回两个不同的值，这取决于你对它的使用情况。如果你正在将文件写入磁盘，working_directory指向目标设备上为你的游戏保留的本地存储区域。 在windows下这是 "%LOCALAPPDATA%/gamedir/"，其中 "gamedir "是你游戏名称的目录）。然而，当从磁盘读取时，working_directory 既可以是本地存储，也可以是包含文件的存储区域。

注意：working_directory变量将返回包括最后一个反斜杠的路径。

例如，如果你有一个默认的*.txt文件包含在你的游戏中，并使用working_directory将其读入内存，那么该文件将从包含的文件存储区域中取出。如果你选择将该信息写入 信息写入文本文件，这将把文件写入目标平台的本地存储，下次将从那里读取，除非你指示GameMaker Studio 2删除该文件，在这种情况下，working_directory将再次指向包含文件的存储区域。

值得注意的是，一般来说，你不需要使用这个，只需调用文件名（加上任何额外的路径信息）就足够了，例如这样。

file_text_open_read(working_directory + "my_file.txt")。

最好是这样做。

file_text_open_read("my_file.txt")。

警告!由于GameMaker Studio 2是沙盒式的，这个功能可能不会像你预期的那样工作!请参阅文件系统部分以了解更多信息。

例子。

ini_open(working_directory + "temp_ini.ini")。

这将从游戏的工作目录中打开一个ini文件（如果它不存在，则创建它）。这可能是本地存储区，也可能是包含文件的区域，取决于要寻找的文件是否存在于这两个地方。
