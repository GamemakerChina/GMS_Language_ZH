working_directory

Working_directory实际可以返回两个不同的值，具体取决于您使用它的目的。如果您要将文件写入磁盘，working_directory指向目标设备上为游戏保留的本地存储区域（在Windows中，该区域为“%LOCALAPPDATA%/gamedir/ “，其中” gamedir“是带有游戏名称的目录。）但是，从磁盘读取时，working_directory可以是本地存储或其中包含的文件的存储区域。

注意：working_directory变量将返回包含最终反斜杠的路径。

例如，如果您有默认 *.txt游戏随附的文件，并使用将其读入内存 working_directory，那么该文件将从存储包含文件的区域中获取。如果您随后选择使用以下命令将该信息写入文本文件 working_directory，这会将文件写入目标平台本地存储，除非您指示GameMaker Studio 2删除文件，否则下次将在该文件中读取该文件。working_directory将再次指向包含文件的存储区域。

值得注意的是，通常您不需要使用它，只需调用文件名（加上任何其他路径信息）就足够了，例如：

file_text_open_read(working_directory + "my_file.txt");

这样做会更好：

file_text_open_read("my_file.txt");

警告！ GameMaker Studio 2沙盒化，因此此功能可能无法按预期工作！请参阅文件系统部分以获取更多信息。

句法：

working_directory

返回值：

String

例子：

ini_open(working_directory + "temp_ini.ini");

这将从游戏的工作目录中打开一个ini文件（如果尚不存在，则创建该文件）。这可以是本地存储，也可以是包含文件的区域，这取决于要查找的文件是否存在于任何一个位置。
