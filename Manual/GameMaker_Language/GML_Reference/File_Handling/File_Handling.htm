文件处理

在创建游戏的某个时刻，有必要在外部存储信息并能够检索此信息。例如，您可能希望存储登录详细信息或高分，或者玩家在各个关卡中的进度。 GameMaker Studio 2时，您都需要了解如何使用可用的不同文件功能。

GameMaker Studio 2默认为“沙盒”是非常重要的，这意味着使用可用的文件系统功能时会有很多限制和小麻烦，并且值得注意的是文件系统的工作方式会稍有不同取决于您要定位的平台。有关更多信息，请参见以下部分：

  • 文件系统

您可以从所选平台的“游戏选项”中关闭桌面目标（ Windows ， macOS和Ubuntu （ Linux ））上的沙箱，这意味着您可以在操作系统权限限制内的任何位置进行保存和加载。

Steam授予您使用其专用API进行游戏的权限，您会发现还有一组新的文件功能可供您使用。这些内容在本手册Steam API部分中进行了说明。

最后，您还应该注意，在使用文件功能时，某些目标平台之间会有一些明显的区别。这些是：

  • iOS ：在iOS ，如果您要从文件夹中加载包含的文件，即：路径为“Backgrounds/background1.png ”，则无需在路径中包含该文件夹，只需提供文件名就足够了。在所有其他平台上，您必须提供该文件夹作为文件路径的一部分。
  • macOS和Ubuntu （ Linux ） ：在macOS和Ubuntu ，文件名和路径在这些平台上区分大小写，因此，为了防止出现任何潜在问题， GameMaker Studio 2默认会自动使所有名称/路径使用小写字母。
  • HTML5 ：在HTML5上，可以按照不同功能页中的说明进行工作，并可以从本地存储进行保存和加载，但是，如果要从服务器加载文件，则只能使用该功能 buffer_load_async()由于大多数浏览器均已弃用同步加载，因此最终将被淘汰。另请注意，您无法在浏览器本地存储中创建或破坏目录。

有关不同文件功能的信息可以在以下页面中找到：

  • 文件系统
  • Ini文件
  • 文字档案
  • 二进制文件
  • 文件目录

GameMaker Studio 2还包括许多用于对文件和数据进行编码和哈希处理的功能，您可以在以下位置找到它们：

  • 编码和散列

另请注意，以下页面提供了一些用于保存游戏的基本文件功能（请注意，这些功能是专为初学者设计的，以使其尽快启动并运行基本的保存系统，但对于更复杂的项目，则应创建您自己的保存系统，使用上面各节中列出的功能）：

  • game_load
  • game_load_buffer
  • game_save
  • game_save_buffer

最后，您可以在以下页面上找到更高级的缓冲区功能，这些功能可用于加载和保存数据：

  • buffer_save
  • buffer_save_ext
  • buffer_save_async
  • buffer_load
  • buffer_load_ext
  • buffer_load_async
  • buffer_load_partial
  • buffer_async_group_begin
  • buffer_async_group_option
  • buffer_async_group_end
