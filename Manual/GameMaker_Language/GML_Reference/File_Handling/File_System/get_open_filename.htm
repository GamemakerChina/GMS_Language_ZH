get_open_filename

此功能将打开一个对话框，并要求播放器提供要使用给定过滤器打开的文件名。过滤器的形式为“ name1 | mask1 | name2 | mask2 |... ”，其中，掩码可能包含不同的选项，并且选项之间以分号分隔，您也可以使用“ *”来表示任何字符串。例如：“ bitmaps | *.bmp; *。wmf ”，将要求用户从系统中选择位图文件的名称，但仅选择具有两个指定扩展名的文件（bmp和 png）。此功能不打开文件本身！它仅返回带有文件全名和路径的字符串。如果用户按下“取消”，则返回一个空字符串“”。

使用此功能时要注意的重要一点是，它会在游戏期间为您授予该文件某些权限。因此，一旦从用户那里获得了文件路径，就可以一次又一次地访问它而无需询问。但是，由于此功能用于读取文件，因此授予的权限仅用于读取和保存，操作系统将不允许这些权限。如果需要读写文件，可以使用函数获得许可 get_save_filename()。

注意：此功能仅在Windows和macOS目标上有效。

句法：

get_open_filename(filter, fname);

争论描述
filter文件类型过滤器和/或名称。
fname建议使用的文件名。

返回值：

String

例子：

var file;
file = get_open_filename("text file|*.txt", "");
if file != ""
    {
    file_text_open_read(file);
    }

上面的代码将要求用户选择一个要打开的文本文件，然后检查返回的文件名和路径，如果存在则将打开该文件进行读取。
