get_open_filename

这个函数打开一个对话，要求玩家提供一个文件名，用给定的过滤器打开。过滤器的形式是 "name1|mask1|name2|mask2|..."，其中mask可以包含不同的选项，中间用分号隔开。 你也可以用 "*"来表示任何字符串。例如："bitmaps|*.bmp;*.wmf"，将要求用户从系统中选择一个位图文件的名称，但只有那些具有两个指定扩展名（bmp和png）的文件。 png）。这个函数并不打开文件本身。它只返回一个包含文件全名和路径的字符串。如果用户按"取消"，将返回一个空字符串""。

在使用这个功能时，需要注意的一件事是，它授予你该文件的某些权限，在你的游戏过程中。因此，一旦你从用户那里得到了文件路径，你就可以一次又一次地访问它，而不需要询问。然而。 因为这个功能是用来读取文件的，所以授予的权限只用于读取和保存，不会被操作系统所允许。如果你需要对文件进行读写，你可以通过以下函数获得权限 get_save_filename().

注意：该功能仅在Windows和macOS目标上有效。

语法。

get_open_filename(filter, fname)。

争论 描述
过滤器 文件类型过滤器和/或名称。
fname 建议使用的文件名。

返回。

字符串

例子。

var file;
file = get_open_filename("text file|*.txt", "");
if file !"="
{
file_text_open_read(file)。
}

上述代码将要求用户选择一个文本文件进行打开，然后检查返回的文件名和路径，如果它存在，将打开该文件进行阅读。
