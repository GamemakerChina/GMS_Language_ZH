get_save_filename_ext

这个函数打开一个对话，要求玩家提供一个文件名，用给定的过滤器来保存。过滤器的形式是 "name1|mask1|name2|mask2|..."，其中mask可以包含不同的选项，中间有分号 而你也可以用 "*"来表示任何字符串。例如："bitmaps|*.bmp;*.wmf"，将要求用户从系统中选择一个位图文件的名称，但只有那些具有两个指定扩展名（bmp和png）的文件。这个函数不会保存文件本身它只返回一个包含文件全名和路径的字符串。如果用户按"取消"，将返回一个空字符串""。你也可以添加一个 目录来开始搜索，并在窗口的顶部显示一个标题，而不是默认的标题。

在使用这个功能时，需要注意的一个重要问题是，它授予你该文件的某些权限，在你的游戏期间。所以，一旦你从用户那里得到了文件路径，你就可以一次又一次地访问它，而不需要询问。 权限同样适用于读和写。

注意：该功能仅在Windows和macOS目标上有效。

语法。

get_save_filename_ext(filter, fname, directory, caption)。

争论 描述
过滤器 文件类型过滤器和/或名称。
fname 建议使用的文件名。
目录 要开始搜索的目录。
标题 窗口的标题。

返回。

字符串

例子。

var file;
file = get_save_filename_ext("screenshot|*.png", "", working_directory, "Save a level here");
if file != ""
{
screen_save(file);
}

上述代码将要求用户选择一个图像文件名文件保存在工作目录中，然后检查返回的文件名和路径，如果不是一个空字符串，它将拍摄一个屏幕截图并使用给定的名称保存。
