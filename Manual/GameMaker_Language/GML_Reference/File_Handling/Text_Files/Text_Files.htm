文本文件

文本文件是一种从游戏外部存储大量数据的有用方法。与ini文件相反，你最多可以同时打开32个文本文件，你也可以向它们读写大块的数据。然而，这取决于你 然而，由于没有像*.ini文件那样的固定的 "部分 "或 "关键 "结构，所以数据应该如何结构是由你决定的。

请注意，对于有本地化的游戏来说，可以从外部文件中读取重音字母是非常重要的。这意味着你应该先创建UTF8格式的txt文件，然后将其添加到GameMaker Studio 2中，作为一个包含的文件，以便在运行游戏时将其导出并使用，而不是在游戏中使用。 然后将其添加到GameMaker Studio 2中，使其在运行游戏时被导出，并取代默认的ANSI格式txt文件，该文件由GameMaker Studio 2的文件功能创建。 功能在没有提供文件的情况下创建的。通过这种方式，你可以正确地阅读和写入它，所有的重音和非罗马字母都被保留。

存在以下处理文件的功能。

  • 文件_text_open_read
  • 文件_text_open_write
  • 文件_text_open_append
  • 从字符串打开的文本
  • 文件_文本_阅读_真实
  • file_text_read_string
  • 文件_文本_阅读
  • 文件_文本_写作_真实
  • 文件_text_write_string
  • file_text_writeln
  • 文件_文本_eoln
  • 文件_text_eof
  • 文件_文本_关闭
