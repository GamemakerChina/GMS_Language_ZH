file_text_open_append

此函数打开具有指定文件名的文本文件进行写入（如果文件不存在，则创建该文件），并返回该文件的唯一ID ，该ID应存储在变量中，因为它将用于所有后续操作处理该文件。文件中要写入的位置设置为文件包含的文本的最后一行。请注意，如果无法创建文件（例如，由于文件名非法），该函数将返回-1。

注意：一次最多只能打开32个文件。完成后，您还应该始终关闭文件，因为这将写入信息并释放与文件关联的内存。

警告！ GameMaker Studio 2沙盒化，因此此功能可能无法按预期工作！请参阅文件系统部分以获取更多信息。

句法：

file_text_open_append(fname);

争论描述
fname要追加的文件名。

返回值：

Real

例子：

file = file_text_open_append(working_directory + "save.txt");

这将从与游戏相同的目录中打开“ save.txt”，并将文件ID存储在变量“ file”中。

返回：文本文件
