file_text_write_real

使用此功能，您可以将数字写入先前打开的文本文件。请注意，由于要写入的值可以是实数，因此所有小数都将以“。”表示。点作为分隔符。如果文件已经包含信息，则除非您使用 file_text_open_append()。您也可以通过使用 file_text_readln()功能以及 file_text_eof()函数循环浏览文件的内容，直到结束，然后开始写入。

重要的是要注意，当使用此功能将大量数字写入文本文件时，它可能会转换为标准的简化格式，例如“ 6.6624e + 003”，无法正确读回GameMaker Studio 2 。为避免出现此类问题，您应该将值转换为字符串并使用该函数 file_text_write_string()反而。

句法：

file_text_write_real(fileid, val);

争论描述
fileid要编辑的文件的ID。
val写入文件的实际值。

返回值：

Real

例子：

var file = file_text_open_write(working_directory + "hiscore.txt");
for (var i = 0; i < 10; ++i;)
    {
    file_text_write_real(file, scr[i]);
    file_text_writeln(file);
    file_text_write_string(file, scr_name[i]);
    file_text_writeln(file);
    }
file_text_close(file);

上面的代码打开一个要写入的文件，然后遍历两个数组，将每个数组的值写入文件的新行。循环结束后，文件将关闭。

返回：文本文件
