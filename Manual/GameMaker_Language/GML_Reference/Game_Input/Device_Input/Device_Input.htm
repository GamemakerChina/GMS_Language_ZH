设备输入

当你的游戏是针对那些允许多点触摸的平台时，你应该使用这些功能。它们的工作方式与PC机连接多个鼠标时的工作方式相同，被检查的常数与那些 被检查的常数与标准鼠标功能所使用的常数相同。

恒定 描述
mb_left 鼠标左键
mb_middle 鼠标中键（这可能不是对所有目标平台都有效）。
mb_right 鼠标右键
mb_side1* 鼠标侧键1
mb_side2* 鼠标侧键2
mb_any 鼠标的任何一个按钮
mb_none 没有鼠标按钮


*注意：mb_side1和mb_side2 按钮仅适用于Windows、macOS、Ubuntu和HTML5。

这些函数都可以接受一个对应于设备显示屏上的触摸的 "触摸 "值，然而可以检测到的最大触摸次数在很大程度上取决于设备和正在运行的操作系统。例如，一些低端的安卓 设备可能只能同时检测到2或3个触摸，而UWP设备可能能够检测到多达10个触摸。还要注意的是，在触摸屏设备上，双击被返回为等同于鼠标右键事件。因此，如果你 需要检测快速连续的多次点击，例如，你将需要同时检查mb_left和mb_right。这个功能可以通过以下方式开启或关闭 device_mouse_dbclick_enable().

下面你可以找到所有设备特定控制功能的列表。

  • 设备_鼠标_检查按钮
  • 设备_鼠标_检查_按钮_按下
  • 设备_鼠标_检查_按钮_释放
  • device_mouse_dbclick_enable
  • 设备_mouse_x
  • 设备_mouse_y
  • 设备_mouse_raw_x
  • 设备_鼠标_raw_y
  • 设备_mouse_x_to_gui
  • 设备_鼠标_y_to_gui
  • 设备_is_keypad_open
  • 装置_获取倾斜度_x
  • 装置_获取倾斜度_y
  • 装置_获取倾斜度_z

请注意，在移动设备（Android、iOS和Windows Phone）上，你可以随时使用后退键，因为它被映射到键盘退格键，也就是GameMaker Studio 2的键盘常量 vk_backspace，所以你可以对该键进行检查，并在这些设备上执行操作，就像使用了后退键一样。还要注意的是，在使用这些函数时，HTML5目标将只正确接受 "设备0 "的输入。
