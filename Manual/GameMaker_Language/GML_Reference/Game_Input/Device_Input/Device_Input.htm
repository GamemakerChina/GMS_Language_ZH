设备输入

当您的游戏针对允许多点触控的平台时，您应该使用这些功能。如果连接了多个鼠标，并且它们要检查的常量与用于标准鼠标功能的常量相同，则它们的工作方式与PC的工作方式相同：

持续的描述
mb_left鼠标左键
mb_middle鼠标中键（可能不适用于所有目标平台）
mb_right鼠标右键
mb_none没有鼠标按键
mb_any任何鼠标按钮

这些功能都可以采用与设备显示屏上的触摸相对应的“触摸”值，但是可以检测到的最大触摸次数在很大程度上取决于设备和运行的OS。例如，某些低端Android设备可能只能同时检测2或3次触摸，而UWP设备则可能最多检测10次。请注意，在触摸屏设备上，双击也会返回等效于鼠标右键事件。因此，例如，如果您需要快速连续检测多个轻拍，则需要检查两个 mb_left和 mb_right。可以使用以下功能打开或关闭此功能 device_mouse_dbclick_enable()。

您可以在下面找到所有设备特定控制功能的列表：

  • device_mouse_check_button
  • device_mouse_check_button_pressed
  • device_mouse_check_button_released
  • device_mouse_dbclick_enable
  • device_mouse_x
  • device_mouse_y
  • device_mouse_raw_x
  • device_mouse_raw_y
  • device_mouse_x_to_gui
  • device_mouse_y_to_gui
  • device_is_keypad_open
  • device_get_tilt_x
  • device_get_tilt_y
  • device_get_tilt_z

请注意，在移动设备（ Android ， iOS和Windows Phone ）上，您可以随时使用“后退”按钮，因为它已映射到键盘退格键，即GameMaker Studio 2键盘常量 vk_backspace，因此您可以检查该键并在这些设备上执行操作，就像使用了后退按钮一样。另请注意，使用这些功能时，HTML5目标将仅正确接受“设备0”的输入。
