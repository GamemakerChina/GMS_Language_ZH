设备_mouse_x_to_gui

此函数返回设备上触摸的X位置，与GUI层的关系。由于GUI层可以被设置成独立于屏幕分辨率和房间大小的不同尺寸，因此有时要获得用户触摸屏幕时在GUI层上的确切位置会很复杂。 用户触摸屏幕时在GUI层上的准确位置。有了这个函数，你可以根据实际的GUI尺寸得到这个位置，使GUI按钮和HUD元素更容易创建和互动。如果你是在一个 HTML5或PC和Mac模块上运行，那么只要设备（通常是鼠标）被插入，这个值就会持续更新，但是对于移动设备，这个值只会在屏幕被触摸时更新，并且注意，可以检测到的最大数量 请注意，能够检测到的最大触摸次数在很大程度上取决于所使用的设备和它所运行的操作系统。还要注意的是，在HTML5上，只有 "设备0 "能正常工作，其他设备可能会给出错误的值。

语法。

device_mouse_x_to_gui(device)。

争论 描述
设备 被检查的设备（从0-n）。

返回。

真实的

例子。

if device_mouse_check_button(0, mb_left)
{
if device_mouse_x_to_gui(0) > 0 && device_mouse_x_to_gui(0) < 32
{
pressed = true;
}
else
{
pressed = false;
}
}

上面的代码检查设备的鼠标是否被按下，如果是，它就会在GUI层上轮询设备的X位置，看它是否在参数范围内。如果是的话，它将变量 "pressed "设置为真，反之则设置为假。 则将其设置为false。
