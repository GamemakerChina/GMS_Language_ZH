keyboard_set_map

有时，在制作游戏时，您可能希望一个键能做到与另一个相同。例如，许多人使用WASD键进行移动，但随后许多人还使用箭头键！那么该怎么办？好的，您可以对运动系统进行两次编码，但这有点复杂，而且很幸运，因为此功能允许您将一个键“映射”到另一个，因此任何一个键的输入都将解释为相同。为此，请选择要映射的键（键2-这将是为其编写代码的键）和要映射到的键（键1）。 GameMaker Studio 2会将对任一键的按键解释为来自key2。您还可以使用此功能设计一个系统，用户可以通过简单地将用户输入键映射到您已编码到游戏中的键来定义自己的游戏键。

该函数采用函数返回的键码值 ord()（仅限AZ的大写字母或0-9的数字），或任何 vk_*键盘输入主页面上列出的常量。

重要的！一旦您使用此功能，映射到（key1）的键将不再可用，因为它是实际的键符号。例如，如果将向上箭头映射到“ W”，那么您将不再能够将“ W”键的按下检测为“ W”，它将始终被视为向上箭头。要撤消此操作，可以将密钥映射到自身-因此key1和key2都将是 ord("W")-或使用功能 keyboard_unset_map()。

句法：

keyboard_set_map(key1, key2);

争论描述
key1这是key1要映射到的密钥
key2这是要映射的密钥

返回值：

Boolean

例子：

keyboard_set_map(ord("A"), vk_left);

上面的示例代码会将“ A”键映射到左箭头键。这意味着玩家可以使用“ A”或向左箭头键，并且为左箭头编写的所有代码也将响应所使用的“ A”键。
