虚拟键和虚拟键盘

在开发用于触摸屏设备的游戏时，从用户那里获取键盘输入通常很重要。使用GameMaker Studio 2可以通过以下两种方式之一进行：

  • 创建自己的屏幕虚拟按键以获取输入
  • 弹出OS虚拟键盘并从中获取输入

使用的内容在很大程度上取决于游戏和所需的内容，但通常，当您想要使用屏幕上的按钮来控制播放器或菜单时，通常会使用虚拟键；而当您需要播放器的输入时，会使用虚拟键盘，例如获取名称或设置值。

为了使您的游戏在带触摸屏的移动设备上正常运行，您可能需要能够分配屏幕区域以响应用户触摸某些事物。这是通过向内部映射的标准键盘键分配“虚拟键”，然后使用它们生成的键盘事件来控制您的应用程序来完成的。值得注意的是，这些“虚拟键”被映射到屏幕位置而不是房间位置，因此x / y值基于显示大小是绝对的。这意味着您不必担心使用视图或相对房间坐标，而只需在对象的“创建事件”中定义键（您只需要为每个房间定义一个虚拟键），然后在Draw GUI Event中绘制关键精灵，将GUI层的大小设置为与屏幕相同。

注意：这些功能在Windows ， Ubuntu （ Linux ）和MacOS目标平台上将不起作用，尽管它们会以“触摸”方式接收鼠标点击。

可用的虚拟键功能包括：

  • virtual_key_add
  • virtual_key_show
  • virtual_key_hide
  • virtual_key_delete

如果需要在触摸屏设备上的游戏中输入文本或数字，则可以使用提供的虚拟键盘功能来使用设备OS键盘。这些功能将调出或隐藏OS键盘并接受其输入，但是不会触发常规的键盘事件，而是更新 keyboard_string用户输入变量（这是由于虚拟键盘可以输入整个单词，而不仅仅是单个字符）。

打开或关闭虚拟键盘将生成系统异步事件，其中 async_loadDS映射将填充以下键/值对：

  • “ event_type ”-接收到的系统事件的类型，对于虚拟键盘，将是“虚拟键盘状态”。
  • “ screen_height ”-虚拟键盘的高度（以像素为单位）。如果键盘不可见，则该值为0。
  • “ keyboard_status ”-键盘的当前状态，以下列字符串之一返回：
    • “隐藏”
    • “隐”
    • “正在显示”
    • “可见的”

可用的虚拟键盘功能包括：

注意：这些功能仅对Android （包括Android TV ）， iOS （包括tvOS ）和UWP目标平台有效。

  • keyboard_virtual_show
  • keyboard_virtual_hide
  • keyboard_virtual_status
  • keyboard_virtual_height
