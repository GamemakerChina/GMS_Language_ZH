keyboard_virtual_show

此功能可用于在运行游戏的设备上显示虚拟键盘。调用此函数时，需要为前三个参数中的每一个提供以下常量之一：

  • 键盘类型：此常数用于设置虚拟键盘上可用的键集。可用的键盘类型为：
键盘类型描述
kbv_type_default当前系统的默认键盘类型。
kbv_type_ascii仅ASCII的键盘。
kbv_type_url针对URL输入进行了优化的普通键盘。通常以“.com”或其他域键以及“ /”和“。”为特色。键。
kbv_type_email优化用于电子邮件输入的普通键盘。通常以“ @”和“。”为特征。人物。
kbv_type_numbers仅数字键盘，通常显示为数字键盘。
kbv_type_phone电话键盘。通常仅使用数字“ *”和“＃”键。
kbv_type_phone_name经过优化的键盘可同时输入电话号码和姓名。通常类似于ASCII键盘，但特殊字符选择有限。

  • 返回类型：此常数用于设置虚拟键盘的Return / Action键上显示的内容。可用的返回类型为：
返回类型描述
kbv_returnkey_default当前系统的默认返回键标题。
kbv_returnkey_go将返回键标题设置为“ Go”。
kbv_returnkey_google将返回键标题设置为“ Google”，或在某些情况下设置为通用搜索图标。
kbv_returnkey_join将返回键标题设置为“ Go”。
kbv_returnkey_next将返回键标题设置为“ Next”。
kbv_returnkey_route将返回键标题设置为“ Route”。
kbv_returnkey_search将返回键标题设置为“搜索”，或在某些情况下设置为通用搜索图标。
kbv_returnkey_send将返回键标题设置为“发送”。
kbv_returnkey_yahoo将返回键标题设置为“ Yahoo”，或在某些情况下设置为常规搜索图标。
kbv_returnkey_done将返回键标题设置为“完成”。
kbv_returnkey_continue将返回键标题设置为“ Continue”。
kbv_returnkey_emergency将返回键标题设置为“紧急呼叫”。

  • 自动大写类型：此常数用于确定如何通过虚拟键盘键入的单词自动大写。可用的自动大写类型为：
自动大写类型描述
kbv_autocapitalize_none自动大写已禁用。
kbv_autocapitalize_words单词将自动大写。
kbv_autocapitalize_sentences句子将自动大写。
kbv_autocapitalize_characters所有字符将大写。

最后一个参数是启用/禁用预测性文本，这将设置为 true允许它，并且 false否则，但是请注意，仅因为允许使用，并不意味着它将被使用，因为这取决于用户在设备上的偏好。在预测文本模式下，虚拟键盘将不会生成正常的GameMaker Studio 2按键事件。相反，它将仅更新最后按下的字符和键盘字符串变量。这是由于无法检测用于检测按键状态的内部文本字段中的更改是来自实际的虚拟键盘键还是来自文本建议。在这种情况下，您需要阅读 keyboard_string输入，而不是读取任何种类的原始键输入。

同样需要注意的是，根据平台操作系统的不同，用户将获得具有不同功能的不同键盘，并针对每个目标使用以下警告：

  • UWP ：UWP实施不支持 return_key_type，autocapitalization_type或者 predictive_text_enabled。将这些参数传递给函数不会导致任何错误，它们只会对键盘没有任何影响。

  • Android ：在Android ，键盘类型 kbv_type_phone_name不支持，而是显示ASCII键盘，并且返回键类型只能是 kbv_returnkey_go，kbv_returnkey_search ，kbv_returnkey_next ，kbv_returnkey_send或者 kbv_returnkey_done。如果该平台上使用了其他返回键类型，则系统将默认使用以下列出的替换返回键：
返回键不可用更换钥匙
kbv_returnkey_google
kbv_returnkey_yahoo		 kbv_returnkey_search
kbv_returnkey_join
kbv_returnkey_route
kbv_returnkey_emergency		 kbv_returnkey_go
kbv_returnkey_continue kbv_returnkey_next

  • Android TV / FireTV ：ASCII键盘不支持自定义返回键类型-默认返回键将始终显示。

  • tvOS ：预测文本和自动大写不支持tvOS ，和键盘的高度，通过系统事件和该函数返回 keyboard_virtual_height()tvOS上禁用了键盘矩形尺寸功能，因此默认为屏幕高度。另请注意，除非已打开虚拟键盘，否则不会广播物理（蓝牙）键盘事件。

调用此函数将生成一个系统异步事件，其中 async_loadDS映射将填充以下键/值对：

  • “ event_type ”-由虚拟键盘功能触发时将为“虚拟键盘状态”。

  • “ screen_height ”-虚拟键盘的高度（以像素为单位）。如果键盘不可见，则该值为0。

  • “ keyboard_status ”-键盘的当前状态，以下列字符串之一返回：
    • “隐藏”
    • “隐”
    • “正在显示”
    • “可见的”

句法

keyboard_virtual_show(keyboard_type, return_key_type, autocapitalization_type, predictive_text_enabled);

争论描述
keyboard_type确定虚拟键盘上可用的键集。
return_key_type确定在虚拟键盘的回车/动作键上显示的内容。
autocapitalization_type确定如何/是否通过虚拟键盘键入的单词自动大写。
predictive_text_enabled设置 true/false启用/禁用预想输入法。

返回值：

N/A

例子：

if input == false
    {
    input = true;
    keyboard_virtual_show(kbv_type_numbers, kbv_returnkey_default, kbv_autocapitalize_none, false);
    }

如果给定变量未设置为，则上面的代码将显示OS虚拟键盘。true 。
