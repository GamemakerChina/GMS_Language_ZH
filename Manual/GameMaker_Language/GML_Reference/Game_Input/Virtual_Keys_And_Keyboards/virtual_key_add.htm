virtual_key_add

此功能使您可以将屏幕区域的“触摸”映射到键盘事件。这意味着，一旦您为虚拟按键分配了一个区域，对该区域的所有触摸都会触发与您映射到该区域的按键相对应的键盘事件。您也可以将定义的每个虚拟键分配给一个变量，然后可以在其他虚拟键功能中使用它来显示，隐藏和删除它们。这些键是按房间分配的，更改房间时GameMaker Studio 2会自动删除它们。

虚拟键的实际位置基于屏幕位置而不是房间位置，因此x / y值在屏幕上是绝对的。这意味着您不必担心使用视图或相对房间坐标，而只需在对象的“创建事件”中定义键（您只需要为每个房间定义一个虚拟键），然后在Draw GUI Event中绘制关键精灵，将GUI层的大小设置为与屏幕相同。

句法

virtual_key_add(x, y, w, h, keycode);

争论描述
x屏幕上虚拟键的x坐标（左侧）
y屏幕上虚拟键的y坐标（顶部）
w虚拟键的宽度
h虚拟钥匙的高度
keycode触摸此区域应触发哪个键盘键事件

返回值：

index of virtual key

例子：

global.Left = virtual_key_add(32, 32, 64, 64, vk_left);

上面的代码创建了一个虚拟按键，其大小为64x64像素正方形，位于屏幕上的（32，32）处，当触摸该虚拟按键时将触发vk_left事件，并将该虚拟按键的索引分配给全局变量。
