虚拟键_添加

这个功能使你能够将屏幕区域的 "触摸 "映射到键盘事件。这意味着，一旦你把一个区域分配给一个虚拟键，所有对该区域的触摸都将触发与你所映射的键相对应的键盘事件 到该区域。你也可以把你定义的每个虚拟键分配给一个变量，然后可以在进一步的虚拟键功能中使用，以显示、隐藏和删除它们。这些键是以每个房间为单位分配的，当改变房间时，GameMaker Studio 2会自动删除这些键。 当更换房间时，GameMaker Studio 2会自动删除。

虚拟键的实际位置是基于屏幕位置而不是房间位置，因此x/y值在屏幕上是绝对的。这意味着你不需要担心使用视图或相对房间坐标的问题。 你可以简单地在对象的创建事件中定义按键（你只需要为每个房间定义一次虚拟按键），然后在绘图GUI事件中绘制你的按键精灵。 将GUI层的大小调整到与屏幕相同。

语法

virtual_key_add(x, y, w, h, keycode)。

争论 描述
x 屏幕上虚拟键的X坐标（左侧）。
y 虚拟键在屏幕上的Y坐标（顶面）。
w 虚拟钥匙的宽度
h 虚拟钥匙的高度
关键代码 触摸这个区域应该触发哪个键盘按键事件

返回。

虚键的索引

例子。

global.Left = virtual_key_add(32, 32, 64, 64, vk_left)。

上面的代码创建了一个64x64像素的正方形虚拟键，在屏幕上的位置是（32，32），当触摸到这个虚拟键时将触发vk_left事件，并将这个虚拟键的索引分配给一个全局变量。
