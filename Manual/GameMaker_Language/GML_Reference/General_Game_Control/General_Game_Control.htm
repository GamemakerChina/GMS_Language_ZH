通用游戏控制

这些功能和变量都与游戏特定功能有关，例如加载/保存或重新启动等。

以下全局作用域内置变量对于调试游戏非常有用-例如，在出现问题时，它们可用作日志文件的输出-以及显示当前的构建ID等。用户：

  • game_id
  • game_save_id
  • game_display_name
  • game_project_name

以下功能用于一般游戏控制：

  • game_end
  • game_restart
  • game_load
  • game_load_buffer
  • game_save
  • game_save_buffer
  • game_get_speed
  • game_set_speed

为了帮助您创建和维护游戏的高分， GameMaker Studio 2创建了一个由10个位置组成的全局高分数组，您可以访问，添加和更改该位置，以创建自己的自定义高分列表而不必大惊小怪。此功能是游戏的本地功能，可在所有平台上使用，因此，您可以使用这些功能以及文件处理功能轻松地存储和维护基本的高分表。

存在以下处理内部高分列表的功能。

  • highscore_add
  • highscore_name
  • highscore_value
  • highscore_clear

GameMaker Studio 2还包括用于将高分列表作为列表绘制在屏幕上的功能，但是它是非常基本的，建议您使用它进行除其他以外的其他调试工作，因为制作自己的高分表很容易并且会您可以更好地控制其显示方式：

  • draw_highscore

最后，我们有一个函数允许您将精灵指定给鼠标光标：

  • cursor_sprite
