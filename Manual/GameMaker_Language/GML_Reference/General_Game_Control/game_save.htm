游戏保存

这是一个传统的功能，可以用来保存游戏的当前状态，不建议再使用。使用文件函数来创建一个自定义的保存系统，你只保存 和加载特定的游戏数据。

这个函数将保存游戏的原状，但它不会保存当时正在使用的任何动态资源，如数据结构、表面、运行时添加的资产等。如果加载这样的保存文件，任何在保存后进行的游戏更新都将不可见。 如果加载这样的保存文件，任何在保存后进行的游戏更新将不可见，因为它将恢复创建保存时使用的游戏版本。

请注意，使用此功能创建的保存文件可能不被更新的版本所支持。 运行时间版本的支持，所以用一个新的保存系统取代这个功能是非常重要的，以确保与未来的GameMaker更新的兼容性。 替换为新的保存系统是必要的，以确保与未来的GameMaker更新兼容，并且由于上述原因，对你自己的游戏进行的更新。

语法。

game_save(filename)。

争论 描述
文件名 保存游戏的文件名。

返回。

不适用

例子。

if keyboard_check_pressed(ord("S"))
{
global.Saved = true;
game_save("Save.dat");
}

上面的代码将设置一个全局变量为真，然后在按下 "S "键时将游戏保存到 "Save.dat "文件。

返回。一般游戏控制
