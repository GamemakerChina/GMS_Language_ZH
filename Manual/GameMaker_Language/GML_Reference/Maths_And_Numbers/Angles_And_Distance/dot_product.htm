dot_product

点积是表示两个向量之间的角度关系的值，是通过将两个向量相乘，相乘然后求和而得到的。名称“点积”是从通常用于指定此操作的居中点“·”派生而来的（替代名称“标量积”强调结果的标量而不是矢量性质）。

实际的数学公式可以这样写：

因此，在2D中向量的点积 a[x1,y1]和 b[x2,2]是 x1x2 + y1y2，表示GameMaker Studio 2中的dot_product计算如下：

a · b = (x1*x2)+(y1*y2);

关于点积的奇怪之处在于它与输入向量形成的角度之间的关系，可以表示为：

a · b = (length of a) * (length of b) * cos(angle)

也就是说，两个向量的点积将等于这些向量之间的角度的余弦值乘以每个向量的长度。这是一张图片说明：

注意：有关向量的更多信息，请参见此处。

现在，我们可以从任意两个向量中得出与它们的点积结果有关的一些信息：

  • 如果a和b垂直（彼此成90度），则点积的结果将为零，因为 cos(ang)将为零
  • 如果a和b之间的角度小于90度，则点积将为正（大于零），如 cos(ang)将是积极的
  • 如果a和b之间的角度大于90度，则点积将为负（小于零），如 cos(ang)将为负

那么这对我们这些做游戏的人意味着什么呢？好的，这种数学关系可以在很多情况下使用，但最好的观察方法是构建实际场景并亲自观察发生的情况。最简单的方法之一是在平台游戏中为敌人生成简单的“视线”检查，这样，如果敌人位于玩家任一侧的90°范围内，敌人就会“看到”玩家矢量法线。

基本上，我们必须从敌人那里获得法线（即，由方向和视距组成），然后我们需要从玩家那里获得敌人的向量。然后，我们将获得这些向量的dot_product，如果结果为正，则表示已经看到该玩家，如果结果为负，则没有。下面的示例提供了实现此功能的实际代码。

句法：

dot_product(x1, y1, x2, y2)

争论描述
x1第一个向量的x坐标。
y1第一个向量的y坐标。
x2第二个向量的x坐标。
y2第二个向量的y坐标。

返回值：

Real

例子：

var x1, y1, x2, y2;
x1 = lengthdir_x(1, image_angle);
y1 = lengthdir_y(1, image_angle);
x2 = o_Player.x - x;
y2 = o_Player.y - y;
if dot_product(x1, y1, x2, y2) > 0 seen=true else seen=false;

上面的代码使用实例图像角度创建矢量，然后将播放器对象“ o_Player”的矢量获取到自身。最后，它计算这两个向量的点积，如果大于0，则将变量“ seen”设置为true，如果等于或小于0，则将其设置为“ false”。
