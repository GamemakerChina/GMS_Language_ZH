数字功能

GameMaker Studio 2中的实数被认为是双精度浮点数--也就是说，它们有一个小数点，在小数点之前或之后都没有固定的数字。 或整数--也就是说，它们是没有小数点值的整数。例如，2是一个整数，但2.01是一个浮点实数。

注意：在HTML5目标上，所有实数都是双数。

在处理跨平台开发时，整数和浮点数之间的区别非常重要，因为在WindowsPC上进行计算的精度与在移动设备上进行计算的精度不一样 在移动设备上的计算精度是不一样的。这意味着你在进行比较时要特别注意，比如说。

if (image_index == 1)
{
do something...
}

在上面的例子中，如果我们一直将image_speed设置为0.25--例如--那么在四步之后，你可能会认为image_index的值是1，但它可能是 但由于浮点数学的工作方式，它可能是一个像1.0000002这样的值，所以评估将不真实。这些类型的错误可能相当难以调试，因此意识到这些错误总是好的，提前计划并使用 其他检查数值的方法，或者使用适当的落地或四舍五入函数（列在下面）将要检查的数字转换成整数（关于浮点数学的更多信息以及为什么这是一个问题，请看这里）。例如，上面的代码可以写成。

if (floor(image_index) == 1)
