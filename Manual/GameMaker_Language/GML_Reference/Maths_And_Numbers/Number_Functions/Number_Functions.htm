数字功能

GameMaker Studio 2中，实数被认为是双精度浮点数-也就是说，它们中有一个小数点，在该点之前或之后没有固定的数字-或整数-也就是说它们是没有小数点值的整数。例如，2是整数，而2.01是浮点实数。

注意：在HTML5目标上，所有实数均为双精度。

整数和浮点数之间的区别是非常重要的，当处理跨平台开发作为上进行计算精度Windows PC是不一样的，当在移动设备上进行这些相同的计算精度。这意味着进行比较时应特别注意，例如：

if (image_index == 1)
    {
    do something...
    }

在上面的示例中，如果我们一直在设置 image_speed到0.25（例如），然后经过四个步骤，您可以假定 image_index值将为1，但由于浮点数学的工作方式，其值可能类似于1.0000002，因此求值结果将不会是 true。这些类型的错误可能很难调试，因此始终要意识到这些错误并预先计划并使用其他方式来检查值，或者使用适当的下限或舍入函数（如下所列）将数字转换为整数，这始终是一件好事。检查一个整数（有关浮点数学以及为什么这是一个问题的更多信息，请参见此处）。例如，上面的代码可以写成：

if (floor(image_index) == 1)
    {
    do something...
    }

还值得注意的是，在使用YoYo Compiler目标时，所有表达式和函数都是从左到右评估的，而在所有其他目标平台上，它们是从右到左评估的，这意味着-例如-根据不同的结果在平台上：

val = max(num, ++num, num++);

注意：有关更多信息，请参阅评估顺序部分。

您还可以使用GameMaker Studio 2提供的特殊功能来设置浮点数学运算的epsilon值。当实数四舍五入到最接近的浮点数时，ε（也称为“机器ε”）形成相对误差的上限，您可以使用以下函数来获取和设置ε值：

  • math_set_epsilon
  • math_get_epsilon
