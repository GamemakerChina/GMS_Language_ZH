碰撞_圆圈_列表

这个函数与 Collision_circle()函数一样，只是它不是一次只检测一个碰撞的实例，而是检测多个实例。你提供要检查的圆形区域中心的x/y位置以及半径。 的x/y位置，以及半径和要检查的对象，你可以设置检查是精确的（在这种情况下，所有被检查的实例都需要有精确的碰撞掩码），以及检查是否应该包括 是否包括调用的实例。

你也提供一个DS列表，所以 id任何与调用实例发生冲突的实例的值将被添加到给定的列表的末尾。 被添加到给定列表的末尾。你可以在调用这个函数之前清除这个列表，这样它就只包含这个函数调用的结果。你还可以选择将 根据它们从圆形区域的原点到它们的原点的距离来排列实例。该函数返回找到的实例的数量，如果没有找到，则返回0。

注意，你可以提供实例关键字 all ，以检查当前房间内的所有实例，这可能包括运行代码的实例，而不是对象索引。 代码的实例（取决于 notme 参数）。

语法。

collision_circle_list(x1, y1, rad, obj, prec, notme, list, ordered)。

争论 描述
x1 要检查的圆心的X坐标。
y1 要检查的圆心的y坐标。
辐射 半径（从其中心到边缘的距离，以像素为单位）。
obj 检查实例碰撞的对象。
沉淀 检查是基于精确的碰撞（真，比较慢）还是基于其一般的边界盒（假，比较快）。
不是我 如果相关的话，调用实例是否应该被排除（true）或不排除（false）。
列表 用来存储碰撞实例的ID的DS列表。
有序 列表是否应按距离排序（true）或不排序（false）。

返回。

Int (发现有碰撞的实例数量)

例子。

var _list = ds_list_create(;
var _num = collision_circle_list(x, y, 100, obj_Enemy, false, true, _list, false);
if (_num > 0)
{
for (var i = 0; i < _num; ++i;)
{
instance_destroy(_list[| i]);
}
}
ds_list_destroy(_list)。

上面的代码将检查调用实例位置周围半径为100像素的圆形区域是否与 "obj_Enemy "的实例发生碰撞。如果有任何碰撞，那么预先创建的列表就会被循环使用，每个发生碰撞的实例 都会被销毁。
