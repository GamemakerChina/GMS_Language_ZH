collision_ellipse

Collision_ellipse使用第一个四个参数（X1，Y1，X2，Y2）来定义当前房间内椭圆的宽度和高度，然后检查是否在由“OBJ”参数定义的任何目的是在碰撞与区。可以精确地检查此冲突，也可以选择检查是否运行代码本身的实例。考虑这张图片：

在这里，中间的实例使用碰撞椭圆检查球对象。现在，蓝色的框没有精确的边界框，如您所见，即使子图形实际上没有接触椭圆，碰撞仍会发生（即使您将函数中的精确选项设置为true）也是如此。该Sprite的边界框重叠了冲撞圆定义的椭圆区域。另一方面，只有当实际的子图形重叠定义的椭圆时，才会将绿色球视为碰撞。请记住，精确的碰撞被认为是两个物体精灵和碰撞功能必须有精确的标记为上。还应注意，函数的返回值可以是被认为发生冲突的任何实例的id。

请注意，您可以提供实例来代替对象索引 id检查特定实例或实例关键字 all， 或者 other（取决于事件和当前代码范围）。

句法：

crash_ellipse（x1，y1，x2，y2，obj，prec，notme）;

争论描述
x1要检查的椭圆左侧的x坐标。
y1要检查的椭圆顶侧的y坐标。
x2要检查的椭圆右侧的x坐标。
y2要检查的椭圆底边的y坐标。
obj检查实例冲突的对象。
prec检查是否基于精确的碰撞（true ，速度较慢）或其边界框通常（false ， 快点）。
notme是否应排除调用实例（如果相关）（true ） 或不 （false ）。

返回值：

实例ID或无人

例子：

如果collision_ellipse（50、50、200、100，obj_Player，false，true）
{
instance_create_layer（obj_Player.x，obj_Player.y，“效果”，obj_Splash）;
}

这将检查对象“ obj_Player”在50x，50y和200x，100y边界内的椭圆区域。如果与该对象发生碰撞，则它将在obj_Player的x / y坐标处创建“ obj_Splash”的实例。
