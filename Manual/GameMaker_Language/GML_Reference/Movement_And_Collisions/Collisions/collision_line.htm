collision_line

Collision_line沿着从点x1，y1到点x2，y2的线检查是否与参数“ obj”指定的对象的任何实例发生碰撞。这种检查可以是准确与否，而是启用精确的碰撞，对象或实例您检查也必须对他们的精灵启用精确的碰撞。如果不是，则默认检查基于边界框。下图说明了它是如何工作的：

请记住，精确的碰撞被认为是两个物体精灵和碰撞功能必须有精确的标记为上。还应注意，该函数的返回值可以是被认为与该行发生碰撞的任何实例的ID，因此，如果三个实例与定义的行重叠，则其ID中的任何一个都可以作为返回值功能的

请注意，您可以提供实例来代替对象索引 id检查特定实例或实例关键字 all， 或者 other（取决于事件和当前代码范围）。

句法：

collision_line(x1, y1, x2, y2, obj, prec, notme);

争论描述
x1行首的x坐标。
y1行首的y坐标。
x2行尾的x坐标。
y2行尾的y坐标。
obj检查实例冲突的对象。
prec检查是否基于精确的碰撞（true ，速度较慢）或其边界框通常（false ， 快点）。
notme是否应排除调用实例（如果相关）（true ） 或不 （false ）。

返回值：

Instance id or noone

例子：

with (obj_Enemy)
   {
   if collision_line(100, 400, 100, 600, id, false, false) instance_destroy();
   }

此代码获取“ obj_Enemy”的所有实例，以沿着从100x，400y到100x，600y的直线检查与它们自身的碰撞，如果存在，则将其销毁。
