碰撞线

Collision_line 沿着从点x1,y1到点x2,y2的一条线检查是否与参数 "obj "指定的任何对象实例发生碰撞。这种检查可以是精确的，也可以是不精确的。 碰撞，你要检查的对象或实例也必须为其精灵启用精确碰撞。如果没有，默认的检查是基于边界盒的。下面的图片说明了这是如何工作的。

记住，要考虑精确碰撞，对象精灵和碰撞函数都必须将精确标记为开。 和碰撞函数都必须将精确标记为开启。还应该注意的是，函数的返回值可以是沿线被认为是碰撞的任何一个实例的id，所以如果有三个实例重叠在 定义的线，他们中的任何一个id都可以成为函数的返回值。

注意，你可以提供一个实例来代替对象索引 id来检查一个特定的实例，或者实例关键字 all ，或者 other （取决于事件和当前的代码范围）。

语法。

collision_line(x1, y1, x2, y2, obj, prec, notme)。

争论 描述
x1 线条起点的X坐标。
y1 线条起点的y坐标。
x2 线条末端的X坐标。
y2 线条末端的y坐标。
obj 检查实例碰撞的对象。
沉淀 检查是基于精确的碰撞（真，比较慢）还是基于其一般的边界盒（假，比较快）。
不是我 如果相关的话，调用实例是否应该被排除（true）或不排除（false）。

返回。

实例ID或没有人

例子。

with (obj_Enemy)
{
if collision_line(100, 400, 100, 600, id, false, false) instance_destroy();
}

这段代码让 "obj_Enemy "的所有实例沿着100x，400y到100x，600y的线检查是否与自己相撞，如果有的话，它们就被销毁。
