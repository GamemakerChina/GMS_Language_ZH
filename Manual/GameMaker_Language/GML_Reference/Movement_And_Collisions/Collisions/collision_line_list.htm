碰撞线列表

这个函数与 Collision_line()函数一样，只是它不是一次只检测一个碰撞的实例，而是检测多个实例。你提供要检查的线的起点和终点的x/y位置以及要检查的对象。 以及要检查的对象的X/Y位置，你可以设置检查是精确的（在这种情况下，所有被检查的实例都需要有精确的碰撞掩码），以及检查是否应该包括调用的实例 或不包括。

你也提供一个DS列表，所以 id任何与调用实例发生冲突的实例的值将被添加到给定的列表的末尾。 被添加到给定列表的末尾。你可以在调用这个函数之前清除这个列表，这样它就只包含这个函数调用的结果。你也可以选择将 实例的顺序是根据它们从线的中心到它们的起点的距离。该函数返回找到的实例的数量，如果没有找到，则返回0。

注意，你可以提供实例关键字 all ，以检查当前房间内的所有实例，这可能包括运行代码的实例，而不是对象索引。 代码的实例（取决于 notme 参数）。

语法。

collision_line_list(x1, y1, x2, y2, obj, prec, notme, list, ordered)。

争论 描述
x1 线条起点的X坐标。
y1 线条起点的y坐标。
x2 线条末端的X坐标。
y2 线条末端的y坐标。
obj 检查实例碰撞的对象。
沉淀 检查是基于精确的碰撞（真，比较慢）还是基于其一般的边界盒（假，比较快）。
不是我 如果相关的话，调用实例是否应该被排除（true）或不排除（false）。
列表 用来存储碰撞实例的ID的DS列表。
有序 列表是否应按距离排序（true）或不排序（false）。

返回。

整数（发现有碰撞的实例数量）。

例子。

var _list = ds_list_create();
var _num = collision_line_list(x - 100, y, x + 100, y, obj_Enemy, false, true, _list, false);
if _num > 0
{
for (var i = 0; i < _num; ++i;)
{
instance_destroy(_list[| i]);
}
}
ds_list_destroy(_list)。

上面的代码将检查调用实例位置两侧100像素的行，以检查与 "obj_Enemy "实例的碰撞。如果有任何碰撞，那么预先创建的列表就会被循环使用，每个处于碰撞中的实例都会被销毁。 碰撞的实例被销毁。

返回。碰撞
