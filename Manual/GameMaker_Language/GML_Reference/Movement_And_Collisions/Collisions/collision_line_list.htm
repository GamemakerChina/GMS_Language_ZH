collision_line_list

此功能与 collision_line()功能，它不仅检测一次冲突中的一个实例，还将检测多个实例。您提供要检查的行的起点和终点的x / y位置以及要检查的对象，并且可以将检查设置为精确（在这种情况下，所有被检查的实例都需要具有精确的碰撞蒙版）以及检查是否应包括调用实例。您还提供了一个DS列表，其中将填充唯一列表 id对象的所有实例的值（被认为与调用实例发生冲突），并且您可以选择基于从线的中心到发现发生冲突的实例的原点的距离来对列表进行排序。该函数返回找到的实例数，如果找不到则返回0。

请注意，您可以提供instance关键字来代替对象索引all，以检查当前房间中的所有实例，其中可能包括运行代码的实例（具体取决于 notme争论）。

句法：

collision_line_list(x1, y1, x2, y2, obj, prec, notme, list, ordered);

争论描述
x1行首的x坐标。
y1行首的y坐标。
x2行尾的x坐标。
y2行尾的y坐标。
obj检查实例冲突的对象。
prec检查是否基于精确的碰撞（true ，速度较慢）或其边界框通常（false ， 快点）。
notme是否应排除调用实例（如果相关）（true ） 或不 （false ）。
list用于存储冲突实例ID的DS列表。
ordered该列表是否应按距离排序（true ） 或不 （false ）。

返回值：

Integer (The number of instances found to be in collision)

例子：

var _list = ds_list_create();
var _num = collision_line_list(x - 100, y, x + 100, y, obj_Enemy, false, true, _list, false);
if _num > 0
    {
    for (var i = 0; i < _num; ++i;)
        {
        instance_destroy(_list[| i]);
        }
    }
ds_list_destroy(_list);

上面的代码将检查调用实例位置两侧100像素的行是否与“ obj_Enemy”实例发生冲突。如果存在任何冲突，则循环遍历预创建的列表，并且销毁冲突中的每个实例。
