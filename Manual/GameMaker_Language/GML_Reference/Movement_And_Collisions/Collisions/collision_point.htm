collision_point

Collision_point检查参数x1，y1指定的点是否与参数“ obj”指定的对象的任何实例发生冲突。这种检查可以是准确与否，而是启用精确的碰撞，对象或实例您检查也必须对他们的精灵启用精确的碰撞。如果不是，则默认检查基于边界框。下图说明了它是如何工作的：

请记住，精确的碰撞被认为是两个物体精灵和碰撞功能必须有精确的标记为上。还应注意，函数的返回值可以是被认为发生冲突的任何实例的ID，因此，如果三个实例在该点重叠，则它们的任何ID都可以是函数的返回值。

请注意，您可以提供实例来代替对象索引 id检查特定实例或实例关键字 all， 或者 other（取决于事件和当前代码范围）。

句法：

collision_point(x, y, obj, prec, notme);

争论描述
x要检查点的x坐标。
y要检查点的y坐标。
obj检查实例冲突的对象。
prec检查是否基于精确的碰撞（true ，速度较慢）或其边界框通常（false ， 快点）。
notme是否应排除调用实例（如果相关）（true ） 或不 （false ）。

返回值：

Instance id or noone

例子：

if collision_point(x, y, obj_Cursor, false, true)
   {
   Score += 10S;
   }

在这里，我们正在检查具有对象“ obj_Cursor”代码的对象位置的点。如果有1，则将10加到score变量上。
