collision_rectangle

Collision_rectangle使用前四个参数（x1，y1，x2，y2）定义当前房间内的区域，然后检查是否由“ obj”参数定义的任何对象与该区域发生冲突。可以精确地检查此冲突，也可以选择检查是否运行代码本身的实例。考虑这张图片：

在这里，中间的实例使用碰撞矩形检查球对象。现在，蓝色的框没有精确的边界框，如您所见，即使精灵实际上没有接触矩形，碰撞仍会发生（即使您将函数中的精确选项设置为true）也是如此。该小精灵的边界框与collision_rectangle重叠。另一方面，只有当实际的精灵重叠在矩形上时，才会将绿色球视为碰撞。请记住，精确的碰撞被认为是两个物体精灵和碰撞功能必须有精确的标记为上。还应注意，函数的返回值可以是被认为发生冲突的任何实例的id。

请注意，您可以提供实例来代替对象索引 id检查特定实例或实例关键字 all， 或者 other（取决于事件和当前代码范围）。

句法：

collision_rectangle(x1, y1, x2, y2, obj, prec, notme);

争论描述
x1要检查的矩形左侧的x坐标。
y1要检查的矩形顶部的y坐标。
x2要检查的矩形右侧的x坐标。
y2要检查的矩形底部的y坐标。
obj检查实例冲突的对象。
prec检查是否基于精确的碰撞（true ，速度较慢）或其边界框通常（false ， 快点）。
notme是否应排除调用实例（如果相关）（true ） 或不 （false ）。

返回值：

Instance id or noone

例子：

var inst;
inst = collision_rectangle(50, 50, 200, 100, obj_Ball, false, true);
if inst != noone
   {
   with (inst) instance_destroy();
   }

此短代码使用collision_rectangle检查房间中从50x，50y（矩形的左上角）到200x，200y（矩形的右下角）的区域，以查找名为“ obj_ball”的对象的实例。它将返回值存储在一个临时变量中，然后将其检查以查看该值是否是实例ID或关键字noone 。如果不是没有人则使用存储的实例ID销毁对象。
