相撞的矩形

Collision_rectangle 使用前四个参数（x1,y1,x2,y2）来定义当前房间内的一个区域，然后检查由 "obj "参数定义的任何物体是否与该区域发生了碰撞。 区域发生碰撞。这种碰撞可以被检查为精确或不精确，你也可以选择检查运行代码的实例本身或不检查。考虑一下这个图片。

这里，中间的实例正在使用一个碰撞矩形来检查 球的物体。现在，蓝色的没有一个精确的边界框，正如你所看到的，即使精灵实际上没有接触到矩形，碰撞仍然会发生（即使你将函数中的精确选项设置为true 因为该精灵的包围盒与碰撞矩形重叠。另一方面，只有当实际的精灵与矩形重叠时，绿球才会被视为发生碰撞。记住，要考虑精确碰撞，对象精灵和碰撞函数都必须将精确标记为开启。还应该注意的是，该函数的返回值可以是任何一个被认为是碰撞的实例的ID。

注意，你可以提供一个实例，而不是一个对象索引 id来检查一个特定的实例，或者实例关键字 all ，或者 other （取决于事件和当前的代码范围）。

语法。

collision_rectangle(x1, y1, x2, y2, obj, prec, notme)。

争论 描述
x1 要检查的矩形左边的X坐标。
y1 要检查的矩形顶面的y坐标。
x2 要检查的矩形右侧的X坐标。
y2 要检查的矩形底边的y坐标。
obj 检查实例碰撞的对象。
沉淀 检查是基于精确的碰撞（真，比较慢）还是基于其一般的边界盒（假，比较快）。
不是我 如果相关的话，调用实例是否应该被排除（true）或不排除（false）。

返回。

实例ID或没有人

例子。

var inst;
inst = collision_rectangle(50, 50, 200, 100, obj_Ball, false, true);
if inst != noone
{
with (inst) instance_destroy();
}

这段简短的代码使用collision_rectangle检查房间里从50x, 50y（矩形的左上角）到200x, 200y（矩形的右下角）的一个叫做 "obj_ball "的物体实例。它将返回值存储在一个临时 变量中，然后检查该值是否是一个实例id，或者是关键字 noone.如果它不是 noone，那么它就使用存储的实例ID来销毁该对象。
