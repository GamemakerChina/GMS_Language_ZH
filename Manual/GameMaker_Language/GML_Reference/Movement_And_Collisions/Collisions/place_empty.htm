place_empty

您可以使用此函数来检查并查看调用实例是否会与对象的任何其他实例或游戏中的所有实例发生冲突。为了使冲突能够正确解决，运行代码的实例必须具有有效的冲突掩码（对于Sprite本身或通过 mask_index），并且只会注册与具有有效掩码的实例的冲突。

该函数正在测试是否应将调用实例放置在特定位置上没有冲突，并且您可以提供一个可选参数来完善检查，以仅检查位置是否与给定类型的实例没有冲突。请注意，如果未提供可选的对象ID，则将针对房间内的所有实例进行检查。碰撞检查将是精确的，还是基于实例的边界框，具体取决于已选择哪种碰撞遮罩，但是为了使精确的碰撞正常工作，检查中的所有实例都应具有精确的碰撞遮罩。

注意，在执行检查之前，给定的x / y坐标将被取整为最接近的整数。

句法：

place_empty(x, y, [object_id]);

争论描述
x要检查的x位置。
y要检查的y位置。
[object_id]选修的！要检查的对象。

返回值：

Boolean

例子：

if place_empty(mouse_x, mouse_y, obj_Enemy)
   {
   x = mouse_x;
   y = mouse_y;
   }

上面的代码将检查与对象“ obj_Enemy”的任何其他实例的冲突，就像调用实例将与鼠标放在相同的位置一样。如果未检测到冲突，则实例的x / y坐标设置为鼠标的x / y坐标。
