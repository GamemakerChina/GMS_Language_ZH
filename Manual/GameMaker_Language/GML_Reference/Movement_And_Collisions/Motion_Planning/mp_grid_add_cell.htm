mp_grid_add_cell

使用此功能，我们可以将给定MP网格中的各个单元标记为“禁止”，这意味着路径查找功能永远不会越过它们。现在，通常您使用的是x / y房间坐标，而不是MP网格单元坐标。因此，让我们考虑下图，该图显示了两者如何相互关联：

如您所见，mp_grid被编号为一系列单元格（从左上角的0,0开始），当您将此网格放置在房间上方时，房间坐标之间也存在直接相关性（也从0开始） ，0在左上角）和网格中的单元格。在图像中，您可以看到位置260,130的实例落在网格单元8,4上。通过将数字除以网格分辨率然后四舍五入，或者如果使用的是2个网格分辨率的乘方（例如8、16、32等），则可以数学计算出某个位置容易落在的确切像元，可以使用按位运算符。应当注意，按位运算速度更快，下面的示例显示了这两种方法。

句法：

mp_grid_add_cell(id, h, v);

争论描述
id要使用的mp_grid的索引
h单元格在网格中的水平位置
v单元格在网格中的垂直位置

返回值：

N/A

例子：

with (obj_Wall)
    {
    mp_grid_add_cell(grid, floor(x / 32), floor(y / 32));
    }
with (obj_Door)
    {
    mp_grid_add_cell(grid, x>>5, y>>5);
    }

上面的代码将使所有“ obj_Wall”和所有obj_Door将它们的等效单元格位置添加到在变量“ grid”中索引的mp_grid中。在示例的第一部分中，这是通过获取对象的x / y坐标并将其除以网格的分辨率（使用 floor（将值保持为整数），在第二部分（由于网格分辨率为2的幂），是通过将x，y坐标向右移动5个位来完成的。

背面：运动计划
