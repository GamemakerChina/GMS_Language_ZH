mp_grid_add_instances

这个函数使用实例碰撞掩码（由精灵属性或调用实例的掩码索引决定），将MP网格中的单元格标记为禁止或不禁止，其中禁止的单元格不能被任何寻路函数穿过。 你可以在函数中指定是否考虑精确碰撞，被标记的禁区单元将根据这一设置而改变。这张图片说明了这种行为。

上面的两个实例已经被添加到MP网格中，使用 mp_grid_add_instances() 函数，精确设置为true。正如你所看到的，绿色实例只将它 "接触 "的网格方格标记为禁止，这是因为它的精灵掩码也被设置为精确。然而，第二个实例已经将其他 也标记了其他的单元格，而这些单元格看起来并没有被触碰。这是因为精灵遮罩没有设置为精确，这意味着即使你在函数中使用精确设置，也只考虑精灵的边界框。这 如果你把函数的精确参数设置为false，也会发生这种情况--即使有精确的精灵遮罩的实例也会根据其边界框被添加到网格中。

语法。

mp_grid_add_instances(id, obj, prec)。

争论 描述
身份证 将要使用的mp_grid的索引。
obj 要添加到mp_grid中的实例的对象索引，或实例id。
沉淀 检查是否基于像素完美碰撞（true = 慢）或其一般的边界盒（false = 快）。

返回。

不适用

例子。

mp_grid_add_instances(grid, obj_Wall, true)。

上述代码将把 "obj_Wall "的所有实例添加到变量 "grid "中索引的mp_grid中，使用精确的碰撞掩码而不是边界框。
