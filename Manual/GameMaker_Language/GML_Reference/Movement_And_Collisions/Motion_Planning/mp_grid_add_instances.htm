mp_grid_add_instances

此函数使用实例碰撞遮罩（由sprite属性或调用实例的遮罩索引决定）在MP网格中将单元标记为禁止或禁止，其中任何寻路功能都无法穿越禁止的单元。您可以在函数中指定是否考虑精确的碰撞，标记的禁止使用的单元格将根据此设置而改变。此图说明了这种行为：

上面的两个实例已使用 mp_grid_add_instances()精确设置为true的功能。如您所见，绿色实例仅将其“接触”的网格正方形标记为禁止的，这是因为其精灵蒙版也已设置为精确。但是第二个实例也标记了其他似乎未触摸的单元格。这是因为子画面蒙版未设置为精确，这意味着即使您在函数中使用精确设置，也将仅考虑子画面的边界框。如果将函数的精确参数设置为false，也会发生这种情况-即使具有精确Sprite蒙版的实例也将基于其边界框添加到网格中。

句法：

mp_grid_add_instances(id, obj, prec);

争论描述
id要使用的mp_grid的索引
obj要添加到mp_grid的实例的对象索引或实例ID
prec该检查是基于像素完美碰撞（true =慢）还是通常基于其边界框（false =快速）。

返回值：

N/A

例子：

mp_grid_add_instances(grid, obj_Wall, true);

上面的代码将使用精确的碰撞掩码而不是边界框将“ obj_Wall”的所有实例添加到在变量“ grid”中索引的mp_grid中。
