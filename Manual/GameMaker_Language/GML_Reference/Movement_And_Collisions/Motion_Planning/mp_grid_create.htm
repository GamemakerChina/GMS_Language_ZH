mp_grid_create

使用此功能，可以为运动计划功能创建一个mp_grid。它返回一个索引，该索引应存储在变量中，以便在所有其他MP网格函数调用中使用。 x和y坐标指示网格左上角的位置，hcell和vcell指示网格内要创建的水平和垂直像元的数量，像元宽度和像元高度指示像元的大小（以像素为单位） 。如果需要将它们用于不同的事情，则可以同时创建和维护多个网格结构，但是请注意，每个网格都需要内存，并且它们具有的单元越多，所需的内存就越多。还要注意，MP功能的使用会占用大量处理器，因此您调用它们的次数越多，单元格分辨率越高，那么性能下降的可能性就越大。

句法：

mp_grid_create(left, top, hcells, vcells, cellwidth, cellheight):

争论描述
left在房间里开始mp_grid的x坐标
top房间中mp_grid的起始y坐标
hcells mp_grid将包含的水平单元格数
vcells mp_grid将包含的垂直单元格数
cellwidth mp_grid每个单元格的宽度（以像素为单位）
cellheight mp_grid每个单元格的高度（以像素为单位）

返回值：

Real

例子：

global.grid = mp_grid_create(0, 0, room_width / 32, room_height /32, 32, 32);

上面的代码创建一个全局变量“ grid”，然后生成一个mp_grid并将其索引（id）分配给该变量，以用于所有其他mp_grid函数调用。网格覆盖了整个房间，其单元分辨率为32x32像素。这意味着，例如，在640x480的房间中，网格将包含300个单元：20个水平单元（640/32）乘以15个垂直单元（480/32）得到300个单元

背面：运动计划
