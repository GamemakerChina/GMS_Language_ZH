联网

GameMaker Studio 2网络功能基于套接字，这是所有平台上的行业标准。使用套接字，您可以使用简单的GML创建客户端和服务器，这允许您使用客户端/服务器模型（甚至可以在以后轻松添加多人游戏的方式）创建单人游戏。

基本上，套接字是一个对象，可以发送，接收，连接和侦听网络上的端口。这些功能仅处理TCP / IP连接，这是Internet所基于的（更精确地说是IPv4）。从本质上讲，这使您可以将使用IP地址（例如192.168.1.10）的一个套接字连接到另一个IP地址（例如192.168.1.11）的另一个套接字。连接后，这两个插座可以使用 network_send_*功能（如 network_send_packet()）以及由它们触发的专用异步事件。此异步事件将生成一个DS映射，其中包含接收到的数据以及其他详细信息，您可以在“网络异步事件”部分中找到完整的详细信息。

IP地址还处理端口（编号从0到65535），而不是程序必须读取和处理进入计算机的网络流量的每一个比特。这使每个程序只能获取其感兴趣的数据包，而不是每个程序发送的所有内容。这也节省了大量CPU时间，因为它大大减少了数据处理，因此，使用GameMaker Studio 2您还可以“监听”端口。因此，服务器将创建一个套接字，然后将其绑定到要侦听的特定端口。然后，它将收到有关该套接字的连接，断开连接和数据传输的通知。

数据“数据包”本身由取自缓冲区的二进制数据组成。 GameMaker Studio 2 Buffer函数创建和操作此缓冲区。

值得注意的是，如果您正在为HTML5目标创建项目，则由于浏览器的限制，您将无法使用服务器创建功能，并且必须使用Electron或NodeJS之类的工具为此平台编写自己的服务器。还要注意，如果您的游戏是跨平台的，并且希望从移动或台式设备上运行的服务器连接到游戏的HTML5实例，则您的服务器只能使用Web Socket协议与HTML5实例进行通信（此操作已完成）使用常数 network_socket_ws- 看 network_create_server()欲获得更多信息）。这意味着您的服务器可能必须在不同的端口上创建两个套接字，一个用于连接非HTML5目标（ Android ， Windows等），另一个用于连接HTML5目标。

存在以下功能供您用于联网：

  • network_create_server
  • network_create_server_raw
  • network_create_socket
  • network_create_socket_ext
  • network_connect
  • network_connect_async
  • network_connect_raw
  • network_connect_async_raw
  • network_resolve
  • network_set_config
  • network_set_timeout
  • network_send_broadcast
  • network_send_packet
  • network_send_raw
  • network_send_udp
  • network_send_udp_raw
  • network_destroy
