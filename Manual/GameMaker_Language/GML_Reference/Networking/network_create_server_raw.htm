network_create_server_raw

此功能用于使用允许的连接协议之一为您的游戏创建一个新的网络服务器（请参阅下面列出的常数）。您提供服务器类型，然后为其提供使用的端口，最后是在任何时候都允许与服务器的最大连接数（请注意，该数量取决于您，但是连接的客户端过多将使服务器饱和）。网络或设备CPU无法处理该数量的播放器，因此请谨慎使用）。该函数返回一个唯一的ID ，该ID应该存储在变量中，并用于在所有其他网络功能中标识服务器；如果连接失败，则返回小于0的值。

持续的描述
network_socket_tcp使用TCP创建套接字。
network_socket_udp使用UDP创建套接字。
network_socket_ws使用TCP创建Web套接字（仅用于连接到HTML5项目）。



由于这将创建“原始”服务器，因此它将不接受也不使用任何GameMaker内置数据头进行通讯，因此您应该使用以下功能 network_send_raw()和 network_send_udp_raw()将未格式化的数据发送到创建的服务器。

句法：

network_create_server_raw(type, port, max_client);

争论描述
type要创建的服务器的类型（请参见下面列出的常量）。
port服务器将使用的端口。
max_client一次可以连接的最大客户端数量。

返回值：

Real

例子：

var port = 6510;
server = network_create_server_raw(network_socket_tcp, port, 32);
while (server < 0 && port < 65535)
    {
    port++
    server = network_create_server(network_socket_tcp, port, 32);
    }

上面的代码将尝试通过端口6510使用TCP创建服务器。如果该端口不可用，它将在端口之间循环查找一个。

返回：联网
