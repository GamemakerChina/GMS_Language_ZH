network_send_udp_raw

使用此功能，您可以使用UDP通过网络将数据发送到服务器。该函数获取要通过其连接的套接字ID，要连接的URL和要使用的端口。然后，您必须提供缓冲区ID ，其中包含要发送的数据（有关缓冲区的更多信息，请参见参考-Buffers ），最后是数据的大小（以字节为单位）。 UDP是“无连接”的，因为您实际上没有进行连接，您只是直接向IP发送数据包，服务器从IP地址获取传入数据，因此必须“按原样”处理。发送的数据没有通过GameMaker Studio 2进行任何格式化，接收设备将以流的形式获取数据，这意味着您必须自己处理它。该函数将返回已发送数据的字节数，如果发送失败，则返回小于0的数字。

注意：此功能在HTML5目标上运行的项目中使用时将不起作用，HTML5项目也将无法接收UDP。

句法：

network_send_udp_raw(socket, url, port, buffer, size);

争论描述
socket要使用的套接字的ID。
url要连接的URL或IP（一个字符串）。
port要连接的端口。
buffer从中获取数据的缓冲区的ID。
size数据大小（以字节为单位）。

返回值：

Real

例子：

network_send_udp_raw(sock, "www.macsweeneygames.com", 6510, buff, buffer_tell(buff));

上面的代码将原始UDP数据包发送到端口6510上的URL所定义的服务器。数据取自变量“ buff”中索引的缓冲区。
