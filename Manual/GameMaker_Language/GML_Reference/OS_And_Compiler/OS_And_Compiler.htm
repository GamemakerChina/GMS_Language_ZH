操作系统和编译器

在创建跨平台游戏时，获得运行游戏的设备所使用的操作系统的详细信息通常至关重要。语言，版本或网络状态之类的东西都可以用于使您的游戏适应运行该游戏的系统，并使最终用户获得最佳体验，因此GameMaker Studio 2具有许多特殊功能，可用于获得必要的信息。

您可以使用以下函数和变量来获取有关运行游戏的操作系统或浏览器的某些详细信息：

  • os_browser
  • os_device
  • os_type
  • os_version
  • os_is_paused
  • os_is_network_connected
  • os_get_config
  • os_get_language
  • os_get_region
  • os_get_info
  • os_powersave_enable
  • os_lock_orientation
  • os_check_permission
  • os_request_permission

某些功能在您运行游戏时会进行预编译（即：它们用于定义最终游戏的编译方式），而某些功能在编译游戏以获取有关runtime环境的详细信息时也将使用。在这里，您可以找到这些功能的完整列表：

  • GM_build_date
  • GM_version
  • GM_ runtime _version
  • gml_release_mode
  • gml_pragma
  • parameter_count
  • parameter_string
  • environment_get_variable

在Windows ，运行游戏时，默认情况下Windows线程调度程序的分辨率设置为1ms。以下功能可用于runtime Windows线程调度程序的分辨率：

  • scheduler_resolution_get
  • scheduler_resolution_set
