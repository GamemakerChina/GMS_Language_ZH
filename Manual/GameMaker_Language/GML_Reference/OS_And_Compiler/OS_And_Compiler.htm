操作系统和编译器

在创建跨平台游戏时，获得运行你的游戏的设备所使用的操作系统的细节往往是至关重要的。诸如语言、版本或网络状态等，都可以用来使你的游戏适应运行的系统，并使最终用户的体验达到最佳。 所以GameMaker Studio 2有一些特殊的功能，可以用来获取必要的信息。

有以下函数和变量供你使用，以获得有关运行你的游戏的操作系统或浏览器的某些细节。

  • os_browser
  • os_device
  • 种类
  • 操作系统版本
  • 暂停使用
  • os_is_network_connected
  • os_get_config
  • os_get_language
  • os_get_region
  • 读取信息（os_get_info
  • 启用OS_POWERSAVE_ENABLE
  • 锁定方向
  • os_check_permission
  • os_request_permission

某些函数是在你运行游戏时预先编译的（即：它们被用来定义最终游戏的编译方式），有些函数也是在游戏被编译后使用的，以获得有关运行环境的具体细节。在这里 你可以找到这些函数的完整列表。

  • GM_build_date
  • GM_version
  • GM_runtime_version
  • gml_release_mode
  • gml_pragma
  • 参数_count
  • 参数_字符串
  • 环境_获取变量

在Windows上，当你的游戏运行时，Windows线程调度器的分辨率被默认设置为1ms。以下函数可以在运行时检索和改变Windows线程调度器的分辨率。

  • 调度器_分辨率_获取
  • 调度器_分辨率_设置
