external_define

此函数可用于定义对特定dll（对于Windows ）或dylib（对于Mac）的外部函数调用。该文件可以是包含的文件，也可以是扩展名的一部分。您提供文件名（和路径），然后提供要定义的函数的名称。接下来，您需要定义要使用的调用约定（请参见下面的常数列表）以及预期的结果类型（也如下所示是常数）。最后，您必须提供函数可以接受的数字参数（从0到15），并且对于每个参数，您还必须指定其类型。请注意，对于具有4个或更多参数的函数，所有参数必须均为类型 ty_real。

注意：仅适用于已作为“包含文件”添加到GameMaker Studio 2 IDE的dll或dylib。它不适用于作为扩展名添加的文件，因为这些文件需要您在扩展包本身中定义功能。

持续的描述
dll_cdecl这是默认的C，C ++调用
dll_stdcall这是标准的WinAPI调用（ Windows dll只要）
ty_real实数参数
ty_string空终止的字符串参数

句法：

external_define(dll, name, calltype, restype, argnumb, argtype[0], argtype[1], ...argtype[10]) ;

争论描述
dll dll文件的名称（字符串）
name函数名称（字符串）
calltype使用的调用约定
restype预期结果的类型
argnumb参数个数（0-10）
argtype[0... 10]使用的不同类型的参数

返回值：

Real (Unique function ID value)

例子：

my_funcion = external_define("MyDLL.dll", "MyMin", dll_cdecl, ty_real, 2, ty_real, ty_real);

上面的示例代码将定义一个带有两个参数的名为“ MyMin”的外部函数。

返回： OS和编译器
