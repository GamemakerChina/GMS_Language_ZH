gml_release_mode

编译游戏时，它还包含某些内部函数，可在运行时进行错误检查。这些内部检查涵盖了游戏代码的许多方面，包括范围，参数和内部GML编译器的一些常规方面，并且在许多情况下是产生最终错误消息的原因。但是，可以使用此功能关闭这些内部检查（即：将它们从已编译的游戏中完全删除），从而加快最终项目的速度。

请注意，这意味着您应该在游戏代码中做一些不正确的操作，编译后的游戏可能会崩溃，或者可能会遇到一些非常意外的行为。例如，如果 DS Grid在定义的大小之外的坐标处访问，在开发人员模式下，您会收到编译器窗口错误消息，但进行适当的检查将允许游戏继续运行。但是，在释放模式下，这些检查将不存在，因此代码可能返回NULL指针或错误的指针，后者又将指向一些随机内存。因此，充其量您可能会得到错误的数据返回，或者最糟糕的是您可能会覆盖内存中的某些内容-数据，图形甚至代码本身。另一个示例是在构建图元时-如果以与创建顶点格式时定义的顺序不同的顺序提供颜色，位置，纹理等数据，则游戏在发布时可能会崩溃或产生意外结果模式已开启。

句法：

gml_release_mode(flag);

争论描述
flag任何一个 true设置发布模式或为一般开发设置为false。

返回值：

N/A

例子：

gml_release_mode(true);

上面的示例代码会将游戏设置为在编译时使用发布模式。

返回： OS和编译器
