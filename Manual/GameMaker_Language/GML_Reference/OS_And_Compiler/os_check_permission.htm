os_check_permission

使用此功能，您可以检查用户是否已授予游戏特定权限。您可以使用以下格式提供以字符串形式检查的权限：android.permission.<permission> ”，因此要检查 RECORD_AUDIO您会调用的权限（例如）

os_check_permission("android.permission.RECORD_AUDIO");

该函数将返回将返回三个常量之一-如下所示-告诉游戏许可状态如何。有关应用权限的更多信息，请参阅Android文档。

重要的！此功能仅适用于Android目标。

句法：

os_check_permission(permission)

争论描述
permission检查权限（字符串）

返回值：

Constant

持续的描述
os_permission_granted这表明该权限已被授予
os_permission_denied这表明尚未授予该权限
os_permission_denied_dont_request这表明该权限已被电话设置所阻止，或者该用户先前已拒绝该请求并选择了“不再询问”。

例子：

if (os_type == os_android)
    {
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
        {
        os_request_permission("android.permission.INTERNET");
        }
    }

上面的代码检查操作系统类型，如果是Android ，则对权限进行检查，如果尚未授予“ INTERNET”权限，则请求该权限。
