os_get_info

该函数返回一个 DS Map以及有关运行游戏的操作系统的详细信息。返回的确切信息将取决于操作系统和设备，因此建议您在希望定位的所有平台上测试此功能，并使用类似于 show_debug_message()。请注意 DS Map不会自动从内存中清除，您应该使用 ds_map_destroy()当您不再需要包含的数据时可以使用该功能。

在下面，您可以找到一些额外信息的详细信息，这些信息将提供给特定于以下平台的DS映射，并且应注意，在除HTML5之外的所有平台上，该映射均具有键“ is64bit“ true或者 false如果平台是不是64位操作系统：

Windows / UWP

在Windows桌面和Windows UWP ，您将通过DX映射中可用的以下键从DX11中获得一些额外信息（对扩展有用）：

  • video_d3d11_device-指向DX11设备的指针
  • video_d3d11_context-指向DX11上下文的指针
  • video_adapter_vendorid-带有适配器供应商ID的字符串
  • video_adapter_deviceid-带有适配器设备ID的字符串
  • video_adapter_subsysid-带有适配器子系统ID的字符串
  • video_adapter_revision-带有适配器修订号的字符串
  • video_adapter_dedicatedvideomemory-具有视频内存大小的字符串
  • video_adapter_dedicatedsystemmemory-具有系统内存大小的字符串（由DX11适配器使用）

Android

在Android平台上，您将通过DS映射中提供的以下键从各种不同的API获取一些额外的信息：

  • android_tv -这将返回 true如果检测到该设备是Android TV设备，或者 false否则
  • GL_VERSION-驱动程序报告的OpenGL版本
  • GL_VENDOR-驱动程序报告的OpenGL供应商
  • GL_RENDERER-驱动程序报告的OpenGL渲染器
  • GL_EXTENSIONS-驱动程序报告的可用OpenGL扩展
  • GL_SHADING_LANGUAGE_VERSION-此OpenGL驱动程序支持的GLSL版本
  • GL_MAX_TEXTURE_SIZE-此OpenGL驱动程序支持的最大纹理大小
  • SDK_INT-android.os.Build.VERSION.SDK_INT的值，请参见此处以获取更多信息。
  • RELEASE -android.os.Build.VERSION.RELEASE的值，请参见此处以获取更多信息。
  • MODEL -android.os.Build.MODEL的值，有关更多信息，请参见此处。
  • DEVICE -android.os.Build.DEVICE的值，请参见此处以获取更多信息。
  • MANUFACTURER -android.os.Build.MANUFACTURER的值，请参见此处以获取更多信息。
  • CPU_ABI -android.os.Build.CPU_ABI的值，请参见此处以获取更多信息。
  • CPU_ABI2 -android.os.Build.CPU_ABI2的值，有关更多信息，请参见此处。
  • BOOTLOADER -android.os.Build.BOOTLOADER的值，请参见此处以获取更多信息。
  • BOARD -android.os.Build.BOARD的值，有关更多信息，请参见此处。
  • VERSION Android的os.version的价值 System.getProperty方法
  • REGIONAndroid中user.region的值 System.getProperty方法
  • VERSION_NAME-此软件包的值versionName-有关更多信息， 请参见此处。
  • PHYSICAL_KEYBOARD -如果我们认为可以使用物理键盘，则字符串“ TRUE”，否则为“ FALSE”

注意：此功能仅适用于Android ， iOS ，Mac， Ubuntu和Windows （台式机+ UWP），在所有其他目标平台上它将返回-1而不是a DS Map 。

句法：

os_get_info()

返回值：

Real (ds_map index)

例子：

os_map = os_get_info();
if (os_map != -1)
   {
   var size = ds_map_size(os_map);
   var key = ds_map_find_first(os_map);
   for (var i = 0; i < size - 1; i++;)
      {
      map_data[i] = ds_map_find_value(os_map, key);
      key = ds_map_find_next(os_map, key);
      }
   ds_map_destroy(os_map);
   }

上面的代码将检查操作系统信息，以查看是否 DS Map返回。如果是，则代码将遍历映射并将所有值分配给数组。
