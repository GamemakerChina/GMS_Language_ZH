读取信息（os_get_info

这个函数返回一个 DS地图的详细信息，其中包括游戏所运行的操作系统。返回的确切信息将取决于操作系统和设备，因此建议 所以建议你在你希望的所有平台上测试这个函数，并使用类似 show_debug_message() 的函数输出返回值。注意，DS地图不会自动从内存中清除 注意，DS地图不会自动从内存中清除，你应该在不再需要这些数据的时候使用这个 ds_map_destroy()函数，当你不再需要它所包含的数据时。

下面你可以找到一些关于DS地图特定于以下平台的额外数据的细节，需要注意的是，除了HTML5之外，在所有的平台上，地图将有一个键"is64bit" 这将是 true 或 false ，取决于该平台的操作系统是否为64位。

Windows/ UWP

在Windows桌面和Windows UWP上，你将从DX11中获得一些额外的数据，在DS地图中可获得以下按键（对扩展有用）。

  • video_d3d11_device - 指向DX11设备的指针
  • video_d3d11_context - 指向DX11上下文的指针。
  • video_adapter_vendorid - 适配器供应商ID的字符串。
  • video_adapter_deviceid - 包含适配器设备ID的字符串。
  • video_adapter_subsysid - 包含适配器子系统ID的字符串。
  • video_adapter_revision - 包含适配器修订号的字符串。
  • video_adapter_dedicatedvideomemory - 包含视频内存大小的字符串。
  • video_adapter_dedicatedsystemmemory - 包含系统内存大小的字符串（由DX11适配器使用）。

安卓

在 安卓平台上，你将从各种不同的API中获得一些额外的数据，在DS地图上有以下键。

  • android_tv- 如果设备被检测到是一个AndroidTV设备，这将返回true，否则返回false。
  • GL_VERSION- 驱动程序所报告的OpenGL的版本
  • GL_VENDOR- 驱动程序所报告的OpenGL的供应商
  • GL_RENDER- 驱动程序所报告的OpenGL的渲染器
  • GL_EXTENSIONS- 驱动程序所报告的OpenGL的可用扩展。
  • GL_SHADING_LANGUAGE_VERSION- 该OpenGL驱动支持的GLSL版本。
  • GL_MAX_TEXTURE_SIZE- 该OpenGL驱动支持的最大纹理尺寸
  • SDK_INT- android.os.Build.VERSION.SDK_INT的值，更多信息见这里。
  • RELEASE- android.os.Build.VERSION.RELEASE的值，更多信息见这里。
  • MODEL- android.os.Build.MODEL的值，更多信息见这里。
  • DEVICE- android.os.Build.DEVICE的值，更多信息见这里。
  • MANUFACTURER- android.os.Build.MANUFACTURER的值，更多信息见这里。
  • CPU_ABI- android.os.Build.CPU_ABI的值，更多信息见这里。
  • CPU_ABI2- android.os.Build.CPU_ABI2的值，更多信息见这里。
  • BOOTLOADER- android.os.Build.BOOTLOADER的值，更多信息见这里。
  • BOARD- android.os.Build.BOARD的值，更多信息见这里。
  • VERSION- 来自Android System.getProperty方法的os.version的值。
  • REGION- 来自Android System.getProperty方法的user.region值。
  • VERSION_NAME- 这个包的版本名称的值 - 更多信息请看这里。
  • PHYSICAL_KEYBOARD- 如果我们认为物理键盘是可用的，那么字符串 "TRUE"，否则 "FALSE"。

例子。

if (os_type == os_android)
{
var _info = os_get_info();

if (_info[? "android_tv"])
{
global.android_tv = true;
}
}

上面的代码检查当前的操作系统是否是安卓，获得操作系统信息并读取返回的地图，以检查游戏是否在安卓电视上运行。在这种情况下，它将一个全局变量设置为真。
