os_is_paused

针对移动设备进行开发时，您需要有一种方法来检查设备是否仍然专注于游戏，尤其是当您的游戏是在iOS ， Android或Windows Phone设备上时，它可能会被电话中断和其他应用。您可以使用此功能检查设备是否暂停。

值得注意的是，此功能将运行设备上代码块中包含的任何代码，但是之后，当设备将游戏作为后台进程缓存游戏时， GameMaker Studio 2因此，如果您具有下面提供的示例代码之类的东西，则直到从缓存状态重新启动游戏后，要创建的实例才会真正出现或运行其任何事件。这意味着您可能更希望拥有 os_is_paused函数调用警报，然后在警报事件中（将在设备暂停时设置，但要在未暂停之前才运行）将创建一个暂停菜单，重新启动音乐等。

Windows ， Ubuntu （ Linux ），MacOS或HTML5平台上使用此功能，则仅将功能切换一个步骤即可 true当游戏失去焦点时，它将返回 false直到游戏回到焦点（并再次失去）。这是因为这些平台不缓存应用程序和游戏，因此它们将继续在后台运行。

注意：在移动设备上，当操作系统暂停时，声音和音乐将停止。除非您使用适当的功能专门启动它们，否则它们不会再次重新启动。

句法：

os_is_paused()

返回值：

Boolean

例子：

if os_is_paused()
    {
    if !instance_exists(obj_PauseMenu)
        {
        instance_create_layer(0, 0, "Controllers", obj_PauseMenu);
        }
    }

上面的代码检查游戏是否已暂停，如果已经暂停，则创建对象“ obj_PauseMenu”的实例（如果尚不存在）。

返回： OS和编译器
