os_check_permission

使用此功能，您可以请求特定的权限。您可以使用以下格式提供以字符串形式请求的权限：android.permission.<permission> ”，因此要求 RECORD_AUDIO您会调用的权限（例如）

os_request_permission("android.permission.RECORD_AUDIO");

该函数将返回触发一个系统异步事件，其中内置 async_loadDS映射将包含以下键/值对：

  • “ type ”：这是触发的事件类型，其值将是字符串“permission_request_result ”。
  • “ <permission> ”：这将以字符串的形式请求函数所请求的许可，其中将包含以下常量之一：
    持续的描述
    os_permission_granted这表明该权限已被授予
    os_permission_denied这表明尚未授予该权限

GameMaker Studio 2本机支持以下许可，但Google认为以下许可“危险”，因此必须明确请求它们（请注意，可以使用Android游戏选项来请求某些许可，而无需此功能）：

  • android.permission.WRITE_EXTERNAL_STORAGE
  • android.permission.READ_PHONE_STATE
  • android.permission.RECORD_AUDIO

有关应用权限的更多信息，请参阅Android文档。

重要的！此功能仅适用于Android目标。

句法：

os_request_permission(permission)

争论描述
permission请求权限（字符串）

返回值：

N/A

例子：

if (os_type == os_android)
    {
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
        {
        os_request_permission("android.permission.INTERNET");
        }
    }

上面的代码检查操作系统类型，如果是Android ，则对权限进行检查，如果尚未授予“ INTERNET”权限，则请求该权限。
