os_check_permission

通过这个函数你可以请求一个特定的权限。你可以使用"android.permission.<permission>"的格式提供要请求的权限的字符串，所以要请求RECORD_AUDIO权限（比如说 你可以调用

os_request_permission("android.permission.RECORD_AUDIO")。

该函数将返回触发一个系统异步事件，其中内置的 异步_加载 DS地图将包含以下键/值对。

  • "类型"。这是被触发的事件类型，值将是字符串 "permission_request_result "。
  • "<permission> ":这将函数所请求的权限作为一个字符串，它将包含以下常数之一。
    恒定 描述
    os_permission_granted 这表示该权限已被授予
    os_permission_denied 这表示该权限没有被授予

值得注意的是，GameMaker Studio 2原生支持以下权限，但被Google认为是 "危险 "的，因此必须明确申请。 也请注意，有些权限可以使用Android游戏选项申请，而不需要这个功能）。)

  • android.permission.WRITE_EXTERNAL_STORAGE
  • android.permission.READ_PHONE_STATE
  • android.permission.RECORD_AUDIO

关于应用程序权限的更多信息，请参见Android文档。

重要的是!此功能只针对 安卓系统目标。

语法。

os_request_permission(permission)

争论 描述
许可 请求的权限（一个字符串）。

返回。

不适用

例子。

if (os_type == os_android)
{
if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
{
os_request_permission("android.permission.INTERNET")。
}
}

上面的代码检查操作系统的类型，如果是安卓，它就对权限进行检查，如果 "INTERNET "的权限还没有被授予，它就会请求它。
