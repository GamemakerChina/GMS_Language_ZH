physics_fixture_bind

一旦定义了夹具，就必须将其绑定到实例。这意味着其属性将转移到选定的实例，而不是实际的夹具本身，因此，如果所有夹具都具有相同的属性，则一个夹具可以绑定到多个实例。您可以为目标指定一个对象索引，此时房间中存在的所有实例都将获得该灯具属性（但以后不会创建该对象的任何新实例），或者可以使用特殊关键字other和all 。您甚至可以指定一个父对象，并且具有该父对象的所有子实例也将获得固定装置。将装置绑定到所需的所有实例之后，可以将其删除（如果不再需要），并且具有该装置属性的实例将不受影响并保持这些属性。

夹具将绑定到实例，质心位于实例的原点，多边形夹具将根据点相对于原点的位置进行绑定。如果您需要将灯具绑定到原点以外的其他点，则应使用 physics_fixture_bind_ext()。

该函数还将为绑定的夹具（不是夹具本身）返回唯一的“ id”值，然后可以使用该函数从实例中删除（“解除绑定”）物理特性。physics_remove_fixture() 。这使您可以在不破坏和重新创建对象的情况下从实例添加和删除物理属性。

注意：不再需要夹具时，应将其删除，否则将导致内存泄漏，从而减慢速度并最终导致游戏崩溃。

句法：

physics_fixture_bind(fixture, target)

争论描述
fixture要绑定的装置
target要接收固定装置的目标实例（可以是实例ID，对象ID， other或all ）

返回值：

Real

例子：

var fix, inst;
fix = physics_fixture_create();
physics_fixture_set_circle_shape(fix, 16);
physics_fixture_set_density(fix, 1.0);
inst = instance_create_layer(x, y, "Instances", genericBodyObject);
my_fix = physics_fixture_bind(fix, inst);
physics_fixture_delete(fix);

上面的代码将创建一个固定装置，并将其索引分配给变量“ fix”。然后，在将其绑定到使用存储在变量“ my_fix”中的绑定灯具的索引创建的实例之前，它定义灯具的形状和密度。最后，删除固定装置以防止内存泄漏，因为不再需要它。

返回：照明灯
