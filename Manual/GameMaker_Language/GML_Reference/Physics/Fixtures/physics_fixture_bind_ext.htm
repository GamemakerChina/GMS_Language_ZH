物理学_夹具_绑定_文本

一旦我们定义了我们的灯具，它就必须被绑定到一个实例上。这意味着它的属性被转移到选定的实例上，而不是实际的灯具本身，所以一个灯具可以被绑定到多个实例上，如果所有的 拥有相同的属性。你可以为目标指定一个对象索引，当时房间里的所有实例都会收到该灯具的属性（但不包括后来创建的对象的任何新实例），或者你可以使用特殊的 关键字other和all。你甚至可以指定一个父对象，所有与该父对象有关的子实例也将接受该灯具。一旦夹具被绑定到你需要的所有实例上，如果不再需要，它可以被删除。 而具有该夹具属性的实例将不会受到影响并保持这些属性。

通常情况下，夹具将被绑定到实例上，质心被定位在实例的原点上，然而这并不总是你所需要的，所以这个函数也允许你将夹具的X和Y位置偏移 被绑定的X和Y的位置进行给定的偏移（如果你不需要这个，那么就用 physics_fixture_bind()来代替）。)值得注意的是，一个灯具只能支持一个偏移量，因为在一个灯具上添加多个偏移量是不支持的。 因为Box2D不支持在一个灯具上添加多个偏移。

该函数也将为绑定的灯具返回一个唯一的 "id "值（不是灯具本身），然后可以使用该函数从实例中移除（"解除绑定"）物理属性。 physics_remove_fixture().这允许你 从一个实例中添加和删除物理属性，而不需要破坏和重新创建对象。

注意：当不再需要时，应该删除固定装置，因为不这样做可能导致内存泄漏，这将减慢并最终使你的游戏崩溃。

语法。

physics_fixture_bind_ext(fixture, target, xoffset, yoffset)

争论 描述
固定设备 要绑定的灯具
目标 接收夹具的目标实例（可以是一个实例ID，一个对象ID，其他，或所有）。
xoffset 沿着X轴的偏移量
yoffset 沿着Y轴的偏移量

返回。

真实的

例子。

var fix, inst;
fix = physics_fixture_create();
physics_fixture_set_circle_shape(fix, 16);
physics_fixture_set_density(fix, 1.0);
inst = instance_create_layer(x, y, "Instances", genericBodyObject);
my_fix = physics_fixture_bind_ext(fix, inst, sprite_width / 2, -(sprite_height / 2));
physics_fixture_delete(fix);

上面的代码将创建一个夹具，并将其索引分配给变量 "fix"。然后定义夹具的形状和密度，再根据精灵的宽度和高度以偏移量将其绑定到实例上。绑定的夹具的索引 绑定的夹具的索引被存储在变量 "my_fix "中。最后，夹具本身被删除，以防止内存泄漏，因为它不再需要了。
