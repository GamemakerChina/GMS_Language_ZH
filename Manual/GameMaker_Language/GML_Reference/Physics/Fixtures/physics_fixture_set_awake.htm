物理学_灯具_设置觉醒

当你用物理学启动一个房间，或者将灯具绑定到实例上时，灯具总是被认为是 "醒着的"......也就是说，它们正在处理事件并与周围的实例互动。然而，这有时会导致 问题，特别是当一个房间开始的时候，如果你有许多带有灯具的实例并排在一起（想想由各种带有灯具的实例组成的塔），因为它们是 "醒着的"，它们会移动并可能改变位置 由于突然开始的重力和碰撞影响它们。在这些情况下，你可以使用这个函数将灯具标记为 "睡眠"，在这种情况下，当它在房间里第一次创建时，它将不会处理任何东西。

语法。

physics_fixture_set_awake(fixture, state)

争论 描述
固定设备 夹具的索引
旗帜 一个灯具是否醒着（真）或不醒（假）。

返回。

不适用

例子。

physics_fixture_set_awake(fix_Cloud, true)。

上面的代码在绑定到一个实例时将灯具标记为 "唤醒"。
