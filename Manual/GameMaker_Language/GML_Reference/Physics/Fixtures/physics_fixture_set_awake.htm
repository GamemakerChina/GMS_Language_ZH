physics_fixture_set_awake

当您以物理开始一个房间，或将灯具绑定到实例时，这些灯具始终被认为是“清醒的”……也就是说，它们正在处理事件并与周围的实例进行交互。但是，这有时可能会导致问题，尤其是当您有多个实例，这些实例在一个房间开始时并排放置（例如，一个由多个装有夹具的实例构成的塔），而当它们处于“清醒状态”时，它们会移动并可能由于重力的突然开始和碰撞影响位置而改变位置。在这些情况下，您可以使用此功能将灯具标记为“处于睡眠状态”，在这种情况下，在房间中首次创建灯具时，它将不会进行任何处理。

句法：

physics_fixture_set_awake(fixture, state)

争论描述
fixture夹具的索引
flag灯具是否苏醒（true ） 或不 （false ）

返回值：

N/A

例子：

physics_fixture_set_awake(fix_Cloud, true);

上面的代码将夹具绑定到实例时将其标记为“唤醒”。
