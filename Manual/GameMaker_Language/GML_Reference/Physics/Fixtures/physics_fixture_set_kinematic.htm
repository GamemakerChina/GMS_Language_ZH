physics_fixture_set_kinematic

在基于物理的游戏中，某些情况下您想四处走动，但又不想受到重力等力的作用，也不想与动态物体碰撞而产生的力（例如，在平台游戏中移动平台） ）。对于此类对象，只需将固定装置的密度设置为0，就意味着物理学将假定该对象是静态的。但是，您应该设置 phy_speed_x，phy_speed_y或者 phy_angular_velocity绑定了固定装置的实例的变量，它将被转换成运动学对象并开始运动。在处理方面（取决于物理世界中实例的状态），将实例从静态更改为运动中间仿真可能会非常昂贵，因此建议在将此功能绑定到之前，使用此功能将夹具设置为运动型实例，这样它就不会受到碰撞和力（例如重力）的影响，但可以以恒定的线性和/或角速度运动。

句法：

physics_fixture_set_kinematic(fixture)

争论描述
fixture夹具的索引

返回值：

N/A

例子：

physics_fixture_set_kinematic(fix_Cloud);

上面的代码将指示在“ fix_Cloud”中索引的灯具应标记为运动学的。

返回：照明灯
