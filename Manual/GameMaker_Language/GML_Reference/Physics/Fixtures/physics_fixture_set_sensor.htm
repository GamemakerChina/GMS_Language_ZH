physics_fixture_set_sensor

有时，您希望游戏检测并识别两个带有灯具的实例发生碰撞，但对碰撞没有任何物理反应。这可以通过将固定装置转换为传感器来完成，这基本上意味着它们将生成碰撞事件，但没有物理响应，因此您可以将这些实例用作“触发”，以在游戏室中发生其他事件。任何固定装置都可以标记为传感器，并且如果绑定的实例是静态的或处于移动中，则没有任何区别。

注意：仅在首次发生碰撞时，传感器固定装置才会触发碰撞事件，这意味着当两个物体继续重叠时（传统情况下会发生这种情况），您不会收到碰撞事件流。如果它们停止重叠并随后重叠，则将触发另一个碰撞事件。

句法：

physics_fixture_set_sensor(fixture, state)

争论描述
fixture夹具的索引
state灯具是否为传感器（true）或不是（false）

返回值：

N/A

例子：

physics_fixture_set_sensor(fix_Cloud, 1);

上面的代码会将“ fix_Cloud”中索引的灯具的传感器状态变为true。
