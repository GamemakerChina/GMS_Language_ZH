physics_apply_local_impulse

GameMaker Studio 2的物理将力和重力应用于对象，还可以施加脉冲。这与力略有不同，因为在施加力时会立即影响物体的速度，并可能会影响物体的扭矩（或“旋转”），特别是如果选择施加脉冲的点的矢量不与质心对齐（请注意：质心不一定与原点相同！）。此功能在本地将脉冲应用于实例。这意味着脉冲的强度和方向是根据实例的原点（如果没有精灵，则是位置）来计算的，而不考虑其在游戏室或物理世界中可能具有的方向或旋转。这是一个例子：

如您所见，图像中的青蛙从下面受到冲动，使其“跳跃”。确切的脉冲是由我们从分量ximpulse / yimpulse相对于xpos / ypos坐标获得的矢量定义的-这简单地表示，脉冲的计算是从xpos / ypos到ximpulse / yimpulse的距离（以牛顿为单位）以及方向是我们从xpos / ypos到ximpulse / yimpulse的角度。

注意：物理世界使用米来进行所有计算，并且在房间编辑器中或使用physics_world_create（）定义了每米的像素比率。

句法：

physics_apply_local_impulse(xpos, ypos, ximpulse, yimpulse)

争论描述
xpos相对于将应用脉冲的原点的x坐标
ypos相对于将施加脉冲的原点的y坐标
ximpulse脉冲矢量的x分量（相对于xpos坐标）
yimpulse脉冲矢量的y分量（相对于ypos坐标）

返回值：

N/A

例子：

if jump
    {
    physics_apply_local_impulse(0, 20, 0, 50);
    }

如果变量“ jump”为true，则上面的代码将施加30牛顿的脉冲，并相对于实例原点从底部到顶部方向移动。
