physics_joint_distance_create

最简单的关节之一是距离关节，它表示两个实例上两点之间的距离必须恒定。指定距离关节时，应该已经创建了两个实例并分配了夹具，然后在房间坐标中定义了两个锚点。第一个锚点连接到实例1，第二个锚点连接到实例2，这些点之间的距离意味着距离约束的长度。下图显示了它是如何工作的：

如您所见，锚点被指定为房间坐标，因此在定义它们时必须格外小心，尤其是在实例与关节同时创建而不是通过房间编辑器放置在房间中的情况下。您还应该认识到，关节的创建与实例的精灵大小或它们所附着的夹具的大小无关。因此，如果在实例的原点以外的地方创建距离关节，则该距离关节仍然有效，并且会将两个实例相对于其创建位置进行约束。如果您将“ col”值设置为 true那么这两个实例可以交互并且彼此碰撞，但前提是它们有碰撞事件，但是如果设置为 false，无论如何它们都不会发生冲突。

句法：

physics_joint_distance_create(inst1, inst2, w_anchor1_x, w_anchor1_y, w_anchor2_x, w_anchor2_y, col)

争论描述
inst1与关节连接的第一个实例
inst2与关节连接的第二个实例
w_anchor1_x游戏世界中关节的第一个x坐标
w_anchor1_y游戏世界中关节的第一个y坐标
w_anchor2_x游戏世界中关节的第二个x坐标
w_anchor2_y游戏世界中关节的第二个y坐标
col两个实例是否可以发生冲突（真）或否（假）

返回值：

Integer (index of the joint)

例子：

var mainFixture, o_id;
mainFixture = physics_fixture_create();
physics_fixture_set_circle_shape(mainFixture, sprite_get_width(sprite_index) / 2);
o_id=instance_create_layer(x+300, y, "Instances", obj_Rudder);
physics_fixture_bind(mainFixture, id);
physics_fixture_bind(mainFixture, o_id);
physics_joint_distance_create(id, o_id, x + 50, y, o_id.x - 50, o_id.y, 0);
physics_fixture_delete(mainFixture);

上面的代码创建并定义了一个新的Fixture，然后创建了“ obj_Rudder”的实例。然后将夹具分配给正在运行代码的实例以及新创建的实例，并在它们之间创建一个关节。最后，由于不再需要该固定装置，因此将其删除。
