physics_joint_prismatic_create

像旋转关节一样，棱柱形关节也只有一个自由度，但这种关节相对于轴是定向的，而不是旋转的，实际上阻止了任何形式的旋转。这是一张图像，可帮助您直观地了解其工作原理：

我们在锚点处连接了两个实例，其中一个运动轴由矢量定义，两个w_axis x / y坐标相对于物理世界的（0,0）坐标（因此x分量为0，y分量为y） 1（0,1）将使该关节成为垂直棱柱形关节）。然后，该关节只能相对于该轴运动，例如弹簧或活塞。如果设置上下限，则基本上是在限制沿该轴的移动量，其中0位置是用w_anchor x / y定义的确切位置，因此负值将位于“左侧”沿轴的那个点的位置，以及“ right”的正值，如下图所示（认识到“ left”和“ right”是相对项！）：

您也可以将关节定义为是否具有电机。这意味着，在不受碰撞影响的情况下，关节将沿轴的方向移动。该方向由电动机速度定义，正数朝向轴坐标（“右”），负数朝向锚点（“左”）。 “ max_motor_force”参数用于限制运动速度，以免获得永不加速的电动机，并限制碰撞可能对运动产生的影响。通过这种方式，可以通过将关节速度设置为零并将最大力设置为较小但有效的值，来使用关节电动机来模拟关节摩擦。电机将尝试防止关节移动，但会屈服于较大的负载。

与所有关节一样，如果将“ col”值设置为 true那么这两个实例可以交互并且彼此碰撞，但前提是它们有碰撞事件，但是如果设置为 false，无论如何它们都不会发生冲突。

句法：

physics_joint_prismatic_create(inst1, inst2, w_anchor_x, w_anchor_y, w_axis_x, w_axis_x, lower_trans_limit, upper_trans_limit, limit, max_motor_force, motor_speed, motor, col)

争论描述
inst1与关节连接的第一个实例
inst2与关节连接的第二个实例
w_anchor_x游戏世界中关节锚定的x坐标
w_anchor_y游戏世界中关节锚定的y坐标
w_axis_x轴向量的x分量
w_axis_y轴向量的y分量
lower_trans_limit关节运动的允许下限
upper_trans_limit关节运动的允许下限
limit是否限制关节的运动（真）或不假（假）
max_motor_force设置电机的最大移动速度
motor_speed这是电动机应该移动的速度
motor电机是否应处于活动状态（真）或不激活（假）
col两个实例是否可以发生冲突（真）或否（假）

返回值：

Integer (index of the joint)

例子：

var mainFixture, o_id;
mainFixture = physics_fixture_create();
physics_fixture_set_circle_shape(mainFixture, sprite_get_width(sprite_index) / 2);
o_id = instance_create_layer(x+25, y, "Instances", obj_Piston);
physics_fixture_bind(mainFixture, id);
physics_fixture_bind(mainFixture, o_id);
physics_joint_prismatic_create(id, o_id, x, y, 0, 10, 0, 0, 0, 5, 0, 1, 1);
physics_fixture_delete(mainFixture);

上面的代码创建并定义了一个新的灯具，然后创建了一个“ obj_Piston”实例，将创建的灯具绑定到两个新对象。然后将它们通过棱柱形接头连接起来，其锚固位置位于初次实例的相同x / y坐标，并且该轴由x / y位置的向量和x / y轴形成（在这种情况下为0,10，这是“向下”）。沿该轴的移动量没有限制，但是我们添加了速度为0且最大力为5的电动机来模拟关节摩擦。
