物理学_关节_绳索_创建

绳索连接是用来连接两个你想保持恒定距离的实例，无论其他力量作用于它。使用距离连接，你可以得到 "连接拉伸"，在这种情况下，两个固定装置会分离，并且在连接上施加太多的压力时表现得很奇怪。 如果在接头上施加了太多的压力，就会表现得很奇怪，但是绳索接头不会这样做，它不会比定义的最大长度拉伸得更远。当您创建一个绳索连接时，两个实例应该已经被创建，并且已经分配了一个夹具。 有分配的夹具，然后你在房间坐标中定义两个锚点。第一个锚点连接到实例1，第二个锚点连接到实例2，距离和maxlength参数设置关节的最大 长度的约束。下面的图片显示了这是如何工作的。

正如你所看到的，锚点被指定为房间坐标，所以在定义锚点的时候必须小心。 它们，特别是如果实例是和关节同时创建的，而不是通过房间编辑器放在房间里的。你也应该意识到，关节的创建与实例的精灵的大小无关 或它们所连接的固定装置的大小无关。所以，如果你在实例的原点以外的地方创建了一个绳索关节，它仍然有效，并且会相对于创建它的位置约束两个实例。如果你将 "col "的值设置为 值为"true"，那么这两个实例就可以相互作用和碰撞，但前提是它们有碰撞事件，然而如果它被设置为"false"，它们无论如何都不会碰撞。

语法。

physics_joint_rope_create(inst1, inst2, w_anchor1_x, w_anchor1_y, w_anchor2_x, w_anchor2_y, maxlength, col)

争论 描述
inst1 第一个例子是与联合的连接
仪表盘2 第二种情况是与联合体连接
w_anchor1_x 在游戏世界中，关节的第一个X坐标。
w_anchor1_y 在游戏世界中，关节的第一个Y坐标。
w_anchor2_x 关节的第二个X坐标，在游戏世界中。
w_anchor2_y 该关节的第二个Y坐标，在游戏世界中。
最大长度 关节可以 "伸展 "的最大长度
殖民地 两个实例是否可以碰撞（真）或不（假）。

返回。

整数（关节的索引）。

例子。

var mainFixture, o_id;
mainFixture = physics_fixture_create();
physics_fixture_set_circle_shape(mainFixture, sprite_get_width(sprite_index) / 2);
o_id=instance_create_layer(x+300, y, "Instances", obj_Rudder);
physics_fixture_bind(mainFixture, id);
physics_fixture_bind(mainFixture, o_id);
physics_joint_rope_create(id, o_id, x + 50, y, o_id.x - 50, o_id.y, 300, 0);
physics_fixture_delete(mainFixture) 。

上面的代码创建并定义了一个新的夹具，然后创建了一个 "obj_Rudder "的实例。然后，该夹具被分配给正在运行代码的实例以及新创建的实例，并在它们之间创建一个绳索连接。 最后，该夹具被删除，因为它不再需要了。
