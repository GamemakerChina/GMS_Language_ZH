物理变量

与“传统”碰撞和运动系统一样，物理系统在创建对象时也具有分配给对象的每个实例的一系列实例变量。这些属性中的一些可以由用户在GML定义，但是许多属性是只读的，还有更多属性仅适用于碰撞事件。

以下变量存在于所有启用了物理学的对象中，并且可以使用实例和脚本函数中的代码读取和写入（即更改）它们：

  • phy_active
  • phy_angular_velocity
  • phy_angular_damping
  • phy_linear_velocity_x
  • phy_linear_velocity_y
  • phy_linear_damping
  • phy_speed_x
  • phy_speed_y
  • phy_position_x
  • phy_position_y
  • phy_position_xprevious
  • phy_position_yprevious
  • phy_rotation
  • phy_fixed_rotation
  • phy_bullet

某些变量是只读变量，这意味着它们包含的信息无法通过实例代码或脚本功能GameMaker Studio 2可以在内部对其进行更改，这就是为什么它们是变量而不是常量）的原因。以下列表显示了所有启用了物理的对象中存在的只读变量：

  • phy_speed
  • phy_com_x
  • phy_com_y
  • phy_dynamic
  • phy_kinematic
  • phy_inertia
  • phy_mass
  • phy_sleeping

最后，在启用了物理学的对象中存在许多临时变量，这些临时变量仅在碰撞事件中创建。这些变量类似于上面列出的变量，并且是只读的：

  • phy_collision_points
  • phy_collision_x
  • phy_collision_y
  • phy_col_normal_x
  • phy_col_normal_y
