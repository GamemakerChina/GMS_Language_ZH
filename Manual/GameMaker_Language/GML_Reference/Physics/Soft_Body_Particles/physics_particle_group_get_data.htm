physics_particle_group_get_data

此函数使用已检查的给定标志返回物理模拟中有关一组粒子的各种信息。组索引（其ID）是该函数返回的索引 physics_particle_group_end()，并且使用的缓冲区必须事先使用函数创建 buffer_create()。它应该是“增长”类型，其大小大约是预期返回数据的大小。标志本身是使用下面给出的常量设置的，您可以使用按位或“ |”创建单个标志值以获得所需的信息。

持续的描述数据类型
phy_particle_data_flag_typeflags粒子的标志值。 buffer_u32
phy_particle_data_flag_position粒子的x和y位置。 2 x buffer_f32
phy_particle_data_flag_velocity水平和垂直速度。 2 x buffer_f32
phy_particle_data_flag_colour颜色和Alpha值（十六进制）。 buffer_f32
phy_particle_data_flag_category粒子类别（在创建它所属的组时定义）。 buffer_u32

句法：

physics_particle_group_get_data(group, buffer, flags)

争论描述
group要从中获取数据的粒子组的组索引（ID）。
buffer用于存储数据的（先前创建的）缓冲区。
flags用于提取有关特定粒子类型的数据的标志。

返回值：

Buffer data

例子：

var count = physics_particle_group_count(gp);
var flags = phy_particle_data_flag_position | phy_particle_data_flag_colour;
if (count > 0)
    {
    var buffer = buffer_create(count * 12, buffer_grow, 4);
    physics_particle_group_get_data(gp, buffer, flags);
    for (var n = 0; n < count; n++;)
        {
        var xx = buffer_read(buffer, buffer_f32);
        var yy = buffer_read(buffer, buffer_f32);
        var argb = buffer_read(buffer, buffer_u32);
        var alpha = (argb >> 24) & 255;
        draw_sprite_ext(sprBlob, 0, xx, yy, 1, 1, 0, c_green, alpha);
        }
    buffer_delete(buffer);
    }

上面的代码获取变量“ gp”索引的组中每个粒子的位置和速度，将缓冲区数据存储在多个变量中，然后使用它们在组中每个粒子的位置绘制一个精灵。
