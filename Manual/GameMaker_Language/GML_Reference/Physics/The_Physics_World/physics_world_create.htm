physics_world_create

即使您已经为对象定义了物理原理，但如果没有放置它们的世界，它们都将默认使用传统的GameMaker Studio 2碰撞系统，因此您需要此功能为对象事件中指定房间的物理原理创建物理系统，或房间本身的创建代码（也可以在不使用房间编辑器中的代码的情况下进行定义）。此功能会将一个物理“世界”与房间关联，并且在该房间中创建的所有实例将使用为它们和世界本身定义的物理行为。

您会注意到，我们有一个参数定义了像素大小和米之间的比率。这是因为物理功能使用真实世界的测量值工作，因此我们必须使用“像素到米的比例”将像素坐标和像素大小转换为这些真实世界的测量值。您需要调整pixeltometrescale设置，直到您使用的对象的平均像素大小大致转换为适当大小的模拟物理对象为止，并且在上面您可以看到我们将其设置为32像素，相当于1米。应该注意的是，物理系统将以更大或更小的比率工作，但是最佳性能是当对象的真实世界测量值不小于0.1米且不大于50米时。

句法：

physics_world_create(pixeltometrescale);

争论描述
pixeltometrescale定义现实世界中屏幕上的像素与仪表的比率。 32：1的比例将指定为1/32（或0.03125）。

返回值：

N/A

例子：

physics_world_create(1/32);

上面的代码将在当前房间中创建一个物理世界，其比例为32像素，相当于1米。
