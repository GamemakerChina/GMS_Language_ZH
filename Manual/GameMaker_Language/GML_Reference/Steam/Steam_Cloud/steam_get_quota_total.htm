steam _get_quota_total

使用Steam Cloud存储和同步文件时，必须设置游戏所需的空间配额。此配额是在每个启用云的游戏上强制执行的，基于每个用户每个游戏，因此，例如，如果Game X的配额为1 MB，则拥有Game X的Steam ，与云端游戏相关的1兆字节数据。用户拥有的任何其他具有云功能的游戏（例如，游戏Y）都不会受到游戏X存储的数据的影响。新Steam作品游戏的默认配额为1 GB，但是您可以从Steam作品控件中更改此设置游戏面板。

注意：一旦配额用完，文件写入将失败。如果您认为游戏用户的配额可能会用尽，则应创建代码来处理该配额，因为如果不执行任何操作，则会使用户处于无法修复的情况，这将导致您的游戏体验不佳。

句法：

steam_get_quota_total();

返回值：

Real

例子：

if steam_is_cloud_enabled_for_app()
    {
    quota = steam_get_quota_total();
    }

上面的代码检查是否steam云，如果启用了蒸汽云，它将获取存储配额的大小并将其存储在变量中。

背面： Steam云
