steam _upload_score_buffer

此功能会将分数与从缓冲区创建的数据包一起发送到给定的排行榜。缓冲区的大小不应超过256个字节-超出的任何内容都会被砍掉-并可以包含您需要的任何数据。要上传的分数应为实数，并且排行榜名称是使用函数创建排行榜时定义的字符串 steam_create_leaderboard()。

如果函数失败，它将返回值-1，而成功的发布将返回上传的唯一ID，并触发异步Steam事件。此事件将创建一个 DS Map在内置变量中 async_load具有以下键/值对：

  • “ event_type ”-当此函数触发时，此键将保留值“ leaderboard_upload”
  • “ lb_name ”-此键保存发布到的排行榜的名称
  • “成功”-如果发布成功，则为1，失败为0
  • “ updated ”（更新）-如果排行榜分数实际上已更新（即：新分数更好），则为1，否则为0。请注意，如果乐谱已更新，则上传的缓冲区数据也不会添加。
  • “ post_id ”-此密钥应与上载调用返回的ID值匹配
  • “ score ”-此键保存已发布的分数

请注意，下载排行榜时，async_load地图现在将具有“数据”键，因此您可以检索上载的数据缓冲区（有关更多详细信息， Steam事件扩展代码示例）。该数据缓冲区将以Base 64编码，因此您需要使用该函数 buffer_base64_decode()从缓冲区读取数据之前先对数据进行处理。

句法：

steam_upload_score_buffer(lb_name, score, buffer);

争论描述
lb_name您要将分数上传到的排行榜的名称（一个字符串）。
score要上传的分数（实数）。
buffer要附加的缓冲区的ID。

返回值：

Real

扩展示例：

在此示例中，我们首先上传分数，然后解析 async_load成功返回地图。下面的代码显示了一个典型的上传示例，其中创建了一个缓冲区来容纳一个字符串，该字符串告诉我们分数从哪个级别上传：

if (hp <= 0)
    {
    var buff = buffer_create(256, buffer_fixed, 1 );
    buffer_write(buff, buffer_string, "Uploaded on level " + string(global.Level));
    upload_ID = steam_upload_score("Game Scores", score, buff);
    if (!upload_ID)
        {
        alarm[0] = room_speed;
        }
    buffer_delete(buff);
    }

请注意，如果呼叫失败，我们将设置警报。这将用于稍后再次尝试上载，您可以在此处添加额外的代码以重试上载，或者如果乐谱继续失败，则将乐谱保存到文本文件中，依此类推。另外请注意，我们会立即删除缓冲区，因为该功能不再需要它。现在，我们将以下内容添加到用于控制得分的实例Steam异步事件中：

var type = ds_map_find_value(async_load, "event_type");
if (type == "leaderboard_upload")
    {
    var lb_ID = ds_map_find_value(async_load, "post_id");
    if lb_ID == upload_ID
        {
        var lb_name = ds_map_find_value(async_load, "lb_name");
        var lb_done = ds_map_find_value(async_load, "success");
        var lb_score = ds_map_find_value(async_load, "score");
        var lb_updated = ds_map_find_value(async_load, "updated");
        show_debug_message("leaderboard post id:" + string(lb_ID) + " to lb:" + string(lb_name) + " with score:" + string(lb_score) + " updated=" + string(lb_updated));
        if (lb_done)
            {
            show_debug_message("- Succeeded");
            }
        else
            {
            show_debug_message("- Failed");
            }
        }
    }

在示例中，我们只是将返回值作为调试消息输出到编译器窗口，但是您可以使用此事件以您选择的任何方式来处理信息。
