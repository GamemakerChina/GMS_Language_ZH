steam _ugc_create_query_all_ex

该功能可用于查询UGC数据库。该函数需要将要消耗该物品的游戏的ID和/或创建该物品的游戏的ID，并且您需要对要创建的查询类型和要查询的项目类型使用以下常量和要查询的页码。请注意，查询将返回最多50个项目。

该函数返回唯一的查询句柄值，该值应存储在变量中，以供其他查询函数使用。请注意，此函数仅准备查询，但实际上不发送查询-为此，您必须调用函数steam _ugc_send_query.htm“>steam_ugc_send_query() -您可以进一步使用 steam_ugc_query_*()函数在搜索请求实际发送之前对其进行优化。

句法：

steam_ugc_create_query_all_ex(query_type, match_type, page, creator_app_id, consumer_app_id);

争论描述
query_type要创建的数据查询的类型（请参见下面列出的常量）。
match_type要查询的UGC项目的类型（请参见下面列出的常量）。
page要查询的页码。
creator_app_id商品创建者应用ID。
consumer_app_id消费者应用程序ID。

返回值：

Query Handle

例子：

query_handle = steam_ugc_create_query_all_ex(ugc_query_RankedByVote, page, global.AccountID, 0, global.GameID);

上面的代码创建一个查询请求，并将其句柄存储在变量中以备将来使用。
