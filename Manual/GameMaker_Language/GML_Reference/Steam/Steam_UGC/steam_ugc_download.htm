steam _ugc_download

使用此功能，您可以为任何给定的UGC项目下载预览图像。这 ugc_handle是图像的唯一标识值（您可以使用函数获得 steam_ugc_send_query()），目标文件名是下载完成后希望提供图像文件Steam

使用此功能时，它将触发Steam异步事件，以报告所请求图像文件的详细信息，其中包含以下键/值对。async_load ds_map：

  • “ id ”-调用函数返回的异步ID
  • “ result ”-运算结果（实数值）。这将是GML常数 ugc_result_success或其他一些实数。因此，您应该检查该常量，以确保调用成功，否则无法正常进行。返回的其余可能值显示为Steam “ EResult”值的结果，您应该看到 steamclientpublic.h在所有89个可能值的SDK标头中。
  • “ event_type ”-此键将保存值“ ugc_download ”
  • “ original_filename ”-此键保存服务器上图像文件的原始名称（字符串）
  • “ dest_filename ”-此键保存您传入的图像文件名（字符串）
  • “ ugc_handle ”-此键保存 ugc_handle您传递给调用函数的值

句法：

steam_ugc_download(ugc_handle, dest_filename);

争论描述
ugc_handle要下载的预览的唯一句柄。
dest_filename用于保存预览的文件名。

返回值：

Async ID

扩展示例：

在此示例中，我们首先调用该函数并将异步ID值存储在变量中：

steam_get = steam_ugc_download(steam_handle, "\UGC\Preview_file.png");

然后，它将向Steam API发送文件请求，生成一个异步事件，我们将按以下方式处理：

var event_id = async_load[? "id"];
if event_id == steam_get
    {
    var type = async_load[? "event_type"];
    if type == "ugc_download"
        {
        sprite_delete(preview_sprite);         preview_sprite = sprite_add(async_load[? "dest_filename"], 0, false, false, 0, 0);
        }
    }

上面的代码检查事件类型，然后根据下载的图像创建一个精灵。
