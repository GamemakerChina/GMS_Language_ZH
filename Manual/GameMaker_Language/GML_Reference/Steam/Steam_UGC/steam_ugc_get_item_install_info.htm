steam_ugc_get_item_install_info

这个函数可以用来检索任何给定的已发布的文件项目的信息，这些项目已被订阅并下载到你游戏的Steam本地存储区。你给出项目的ID，并为一个空的DS地图提供索引，如果项目存在，该地图将被填充以下键/值对。

  • " size_on_disk" - 磁盘上的文件大小（字节）。
  • "legacy_item" - 将为真或假 ，取决于它是否是一个遗留文件。
  • "文件夹" - 这是所安装内容的完整路径（请参考Steam SDK文档中的 "物品安装"，因为用旧方法上传的"遗留"物品会被区别对待）。

如果该项目存在，那么该函数将返回true并填充地图，否则将返回false，地图将保持为空。

语法。

steam_ugc_get_item_install_info(published_file_id, info_map)。

争论 描述
published_file_id 要更新的UGC的唯一手柄。
信息地图 一个（先前创建的）DS地图索引。

返回。

布尔型

例子。

var item_map = ds_map_create();
steam_ugc_get_item_install_info( global.fileID, item_map);

上面的代码将查询全局变量 "fileID "中索引的项目的安装状态，使用一个DS Map来存储信息。

返回。UGC
