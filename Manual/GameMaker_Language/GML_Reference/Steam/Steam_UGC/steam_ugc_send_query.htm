steam _ugc_send_query

此功能可用于发送查询请求。您首先使用适当的定义查询 steam_ugc_create_query_*()函数将返回“查询句柄”。然后，此句柄用于设置过滤器等...，然后在此功能中使用该句柄发送查询请求。该函数返回唯一的异步ID值，该值可用于在Steam Async事件中正确检查详细信息。该事件将具有 async_loadDS映射，其中将填充以下键/值对：

  • “ id ”-触发事件的函数返回的异步ID
  • “ result ”-运算结果（实数值）。这将是GML常数 ugc_result_success或其他一些实数。因此，您应该检查该常量，以确保调用成功，否则无法正常进行。返回的其余可能值显示为Steam “ EResult”值的结果，您应该看到 steamclientpublic.h在所有89个可能值的SDK标头中。
  • “ event_type ”-字符串“ ugc_query ”
  • “ num_results ”-返回的结果数（最大50）
  • “ total_matching ”-匹配结果总数
  • “ cached_data ”-指示是否从本地磁盘缓存中检索此数据（将是 true或者 false）
  • “ results_list ”-DS列表索引索引，其中每个列表条目都是一个DS Map索引，其中包含特定项目的详细信息

当您获得异步事件类型“ ugc_query”时，您可以解析结果列表，并从每个DS映射（列表中每个项目一个映射）提取以下信息：

  • “ published_file_id” “-为项目保留唯一的已发布文件ID
  • “ file_type ”-使用的文件类型
  • “ creator_app_id ”-项目创建者Steam
  • “ consumer_app_id ”-商品消费者Steam
  • “ title ”-项目标题
  • “ description ”-商品说明
  • “ steam _id_owner ”-物品所有者Steam
  • “ time_created ”-首次创建项目的时间
  • “ time_updated ”-上次更新项目
  • “ time_added_to_user_list ”-订阅该项目的时间
  • “可见性”-项目的可见性（请参见 steam_ugc_set_item_visibility()对于返回的常量）
  • “被禁止”-该项目是否已被禁止（true ） 或不 （false ）
  • “ accepted_for_use ”-该项目是否已被接受使用（true ） 或不 （false ）
  • “ tags_truncated ”-标记的简短版本作为数组
  • “ tags ”-包含项目标签的字符串，每个标签之间用逗号分隔
  • “ handle_file ”-项目的唯一文件句柄
  • “ handle_preview_file ”-该项目的图像预览的唯一句柄（可以用作 steam_ugc_download()下载预览图像）
  • “ filename ”-项目文件的名称
  • “ file_size ”-项目文件的大小
  • “ Preview_file_size ”-预览图像的大小
  • “ url ”-项目的完整URL
  • “投票数”-收到的投票数
  • “ voices_down ”-收到的否决票数
  • “得分”-项目的总得分
  • “account_id_owner” -从该帐户ID Steam ID所有者（这可以在函数中使用 steam_ugc_create_query_user_ex()）

句法：

steam_ugc_send_query(ugc_query_handle);

争论描述
ugc_query_handle要发送的查询句柄。

返回值：

Boolean

例子：

var query_handle = steam_ugc_create_query_all(ugc_query_RankedByTrend, ugc_match_Items, 1);
steam_ugc_query_add_required_tag(query_handle, "RPG");
steam_ugc_query_set_return_long_description(query_handle, true);
steam_ugc_query_set_allow_cached_response(query_handle, true);
query_ID = steam_ugc_send_query(query_handle);

上面的代码创建一个查询请求，并将其句柄存储在一个局部变量中，以备将来在其余函数中使用，以便在发送查询之前进一步定义查询请求。
