steam _ugc_submit_item_update

此功能会将由给定的句柄加油的UGC项提交到Steam Workshop服务器，并添加要用于给定项的更改说明。更改说明应为字符串，更新句柄为调用函数时返回的值 steam_ugc_start_item_update()。使用此功能时，它将返回一个异步ID值，然后在Steam异步事件以报告该项目的创建时可以对其进行解析。该事件将在以下位置包含以下键/映射值：async_load DS地图：

  • “ id ”-调用函数返回的异步ID
  • “ result ”-运算结果（实数值）。这将是GML常数 ugc_result_success或其他一些实数。因此，您应该检查该常量，以确保调用成功，否则无法正常进行。返回的其余可能值显示为Steam “ EResult”值的结果，您应该看到 steamclientpublic.h在所有89个可能值的SDK标头中。
  • “ event_type ”-此键将保存值“ ugc_update_item ”
  • “ legal_agreement_required ”-将是 true或者 false（有关更多详细信息，请参见Steam文档）

句法：

steam_ugc_submit_item_update(ugc_update_handle, change_note);

争论描述
ugc_update_handle待更新的UGC的唯一句柄。
change_note用于项目的变更说明（字符串）。

返回值：

Async ID

例子：

var app_id = steam_get_app_id();
var updateHandle = steam_ugc_start_item_update(app_id, global.Publish_ID);
steam_ugc_set_item_title(updateHandle, "My workshop item(3)!");
steam_ugc_set_item_description( updateHandle, "testing workshop...");
steam_ugc_set_item_visibility(updateHandle, ugc_visibility_public);
var tagArray;
tagArray[0] = "Test";
tagArray[1] = "New";
steam_ugc_set_item_tags(updateHandle, tagArray);
steam_ugc_set_item_preview(updateHandle, "promo.jpg");
steam_ugc_set_item_content(updateHandle, "WorkshopContent1");
requestId = steam_ugc_submit_item_update(updateHandle, "Version 1.2");

上面的代码获取游戏ID，然后将其与先前存储的发布文件ID一起使用，以生成商品的更新句柄。然后，此句柄用于在将更新推送到Workshop服务器之前更新各种信息。
