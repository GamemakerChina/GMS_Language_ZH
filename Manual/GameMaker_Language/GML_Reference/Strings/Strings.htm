弦乐

在制作游戏时，有时需要使用文本。游戏中的文本通过使用字符串函数来处理（字符串只是表示一行文本的另一种方式），并且GameMaker Studio 2具有一套完整的函数，允许您以多种方式操作字符串，包括插入一个字符串，字符串的复制以及解析字符串以获取数字或包含的字母的能力。通常，只能通过在双引号“”内添加文本来创建字符串，并且不接受单引号字符串，也不能将字符串拆分为多行，并期望GameMaker Studio 2像换行符是换行符一样呈现它（除非字符串文字 @使用标识符，如下所述）。

值得注意的是，在创建字符串时可以使用某些约定，主要涉及转义字符。这些字符前面带有“\因此，例如，如果您想在字符串中加上引号，则将具有以下内容：

str = "Hello\"World\"";

GameMaker Studio 2还具有完整的四个字节宽的Unicode字符支持，使您可以在标准的上限（包括-但不仅限于-表情符号）中对Unicode字符进行解码和编码。要处理Unicode字符，您可以使用“\ “在任何Unicode文字之前-十六进制数字在”“之前u“， 例如 ”\u00e2 “代表”á“-，其中的数字是Unicode字符的编号。以这种方式使用Unicode时，您需要意识到GameMaker Studio 2会解释”“之后的所有数字。u ”，因此，例如，如果您想编写“áa”，则应使用：

"\u00e2\a"

或者

"\u00e2\u61"

或者

"\u00e2" + "a"

就像使用“\u00e2a “实际上会导致Unicode字符”ส ”（本质上成为“\ue2a ”）。

GameMaker Studio 2也可以处理任何十六进制文字-通常在“0x “， 例如 ”0xff “，其中数字是要使用的角色的编号。在GameMaker Studio 2这些字符是使用“\x ”，然后是十六进制值。这些和其他预定义的转义字符在下表中列出：

持续的描述
\n新队
\r回车
\b退格键（0x08）
\f换页（0x0c）
\t水平制表符（0x09）
\v垂直制表符（0x0b）
\\反斜杠本身（0x5c）
\a警报（0x07）
\u[Hex Digits]插入Unicode字符
\x[Hex Digits]插入十六进制文字字符
\[Octal Digits]插入八进制Unicode字符

注意：字符串支持换页，垂直制表符等，但这并不意味着渲染就可以，绘制字符串时这些字符可能会被忽略。

您还可以通过在整个字符串之前添加逗号来创建逐字字符串文字 @特点：

var test = @"
Line breaks
over multiple
lines
";

上面的代码将在多行中呈现字符串，就好像其中包含一个换行符转义符一样。逐字字符串文字与以前的GameMaker版本字符串文字相似，但它们也使用双引号或单引号，并且必须以 @符号，它们可以在代码文件的多行中断开，并且不支持转义字符，即 @"Hello\World"不会试图逃脱世界上的W，并且会逐字存储。请注意，尽管在使用这样的字符串文字时，如果希望在字符串中包含引号，则需要将字符串断开，即：

var test = @"Hello " + "\"" + @"World" + "\""

还要注意的另一件事是，在绘制事件中渲染时，Unicode字符9647（▯）用于替换您指定字体中可能缺少的所有字形。因此，例如，如果您的字体没有°符号，那么写90°实际上会产生90▯。
