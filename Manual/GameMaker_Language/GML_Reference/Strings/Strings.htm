在制作游戏的某个时候，你会需要使用文本。游戏中的文本是通过使用字符串函数来处理的（字符串只是表示一行文本的另一种方式），GameMaker Studio 2有一套完整的函数，允许你以多种方式处理字符串。 允许你以多种方式处理字符串，包括在另一个字符串中插入一个字符串，复制字符串，以及分析字符串中的数字或字母的能力。一般来说，一个字符串只能通过以下方式创建 在双引号""内添加文本，不接受单引号字符串，也不能在多行上分割字符串，并期望GameMakerStudio 2将其作为换行符来呈现（除非是一个 除非使用一个字符串字面 @ ，如下文所解释）。)

值得注意的是，在创建字符串时，有一些惯例可以使用，主要涉及到转义字符的使用。这些字符的前面有一个 "\"符号。因此，举例来说 例如，如果你想在一个字符串中加入引号，你会有这样的结果。

str = "Hello\"World\""。

GameMaker Studio 2还具有完整的四字节宽的Unicode字符支持，允许你在标准的上限范围内对Unicode字符进行解码和编码（包括--但不限于--表情符号）。为了处理Unicode字符，你可以 可以使用"\"在任何Unicode字面前--在 "u"前的十六进制数字，例如 "\u00e2"代表 "á"--其中 数字是Unicode字符的编号。当以这种方式处理Unicode时，你需要注意GameMaker Studio 2将解释"u"后面的所有数字，所以 例如，如果你想写 "áa"，你应该使用。

"u00e2\a"

或

"u00e2\u61"

或

"\u00e2" + "a"

因为仅仅使用"\u00e2a "实际上会导致Unicode字符 "ส"（本质上成为"\ue2a"）。

GameMaker Studio 2还可以处理任何十六进制的文字--通常写成 "0x"后面的十六进制数字，例如 "0xff"，其中的数字是要使用的字符的编号。 要使用的字符。在GameMaker Studio 2中，这些都是用"\x"写的，然后是十六进制值。这些和其他预定义的转义字符在下表中列出。

注意：字符串支持换页、垂直制表符等，但这并不意味着渲染也支持，在绘制字符串时，这些字符可能被忽略。

你也可以通过在整个字符串前面加上 @ 字符来创建逐字字符串。

var test = @"

行上断行

";

上面的代码将在多行上渲染字符串，就像包含有一个换行符一样。逐字字符串与以前GameMaker版本的字符串字面意义相似，但它们也使用双引号或单引号，并且必须以 符号为前缀。 前缀为 @ 符号，它们可以在代码文件中被分成多行，并且不支持转义字符，例如 @"Hello\World" 不会尝试转义 W 并将逐字存储。但是请注意，当使用这样的字符串字面时，如果你想把引号作为字符串的一部分，你将需要断开字符串，即。

var test = @"Hello " + "\"" + @"World" + "\""

另一件要注意的事情是，Unicode字符9647（▯）是用来替代你在绘制事件中渲染指定字体时可能缺少的任何字形的。因此，如果你的字体没有，比如说，°符号，那么写下 90°实际上会产生90▯。
