xboxlive_matchmaking_create

此功能将使用会话模板，对接漏斗和安全设备关联模板创建多人会话，您应该已经在XDP仪表板中预先创建了该模板。您可以通过调用特定游戏手柄获取的用户ID xboxlive_user_for_pad()功能和可见性将是以下常量之一：

持续的描述
xboxlive_match_visibility_open指定会话可以被其他会话加入。
xboxlive_match_visibility_private指定该会话对于不是会话成员的用户不可见。
xboxlive_match_visibility_usetemplate指定应使用会话模板中使用的值（通常应使用此值，因为如果在XDP中设置了模板的值，则您将无法覆盖它的值）。

该函数返回一个唯一的请求ID值，然后可以使用该ID来为此函数标识正确的社交异步事件。创建会话后将触发此事件，并将在变量中包含DS Map async_load具有以下键/值对：

  • “ requestid” –包含调用函数返回的请求ID
  • “状态” –包含字符串“ session_created”，用于通知您已触发的事件类型。
  • “ sessionid” –包含会话ID，如果失败则为-1
  • “错误” –成功时包含0，失败则包含-1
  • “ correlationid”-与函数相关的会话句柄 xboxlive_matchmaking_join_session()

句法：

xboxlive_matchmaking_create(user_id, visibility, template, hopper, sdatemplate, [matchattributes]);

争论描述
user_id要使用的用户ID指针
visibility上面列出的常数之一
template会话模板的名称
hopper牵线搭桥的名字
sdatemplate安全设备关联模板的名称
matchattributes [选修的]

返回值：

Request ID (integer)

例子：

var userID = xboxlive_user_for_pad(global.PadIndex);
result = xboxlive_matchmaking_create(userID, xboxlive_match_visibility_usetemplate, "MatchTicketSession", "MatchTicketHopper", "PeerServerTraffic");

上面的代码将检索给定游戏手柄上用户的用户ID，然后为他们初始化配对比赛。
