xboxlive_matchmaking_create

这个函数将使用会话模板、匹配跳板和安全设备协会模板创建一个多人游戏会话，你应该事先在XDP仪表板上创建这些模板。你可以通过调用 "用户ID "为某个特定的游戏垫检索用户ID。 垫，通过调用 xboxlive_user_for_pad()函数，可见性将是以下常数之一。

恒定 描述
xboxlive_match_visibility_open 指定该会话可以由其他人加入。
xboxlive_match_visibility_private 指定会话对不是会话成员的用户不可见。
xboxlive_match_visibility_usetemplate 指定应该使用会话模板中使用的值（通常应该使用这个，因为如果在XDP中已经设置了模板的值，你就不能再覆盖模板的值）。

该函数返回一个唯一的请求ID值，然后可以用来确定该函数的正确的社会异步事件。该事件将在会话创建后被触发，并在变量async_load中包含一个DS Map，其中包含以下键/值对。

  • "requestid" - 包含由调用函数返回的请求ID
  • "status" - 包含字符串 "session_created"，告知你已经触发了什么类型的事件。
  • "sessionid" - 包含会话ID，或者在失败的情况下为-1
  • "error" - 成功时包含0，失败时包含-1
  • "correlationid" - 用于与函数相关的会话句柄 xboxlive_matchmaking_join_session()

语法。

xboxlive_matchmaking_create(user_id, visibility, template, hopper, sdatemplate, [matchattributes]) 。

争论 描述
user_id 要使用的用户ID指针
知名度 上面列出的常数之一
模板 会话模板的名称
漏斗 匹配跳板的名称
ǞǞǞ 安全设备关联模板的名称
匹配属性 [可选]

返回。

请求ID（整数

例子。

var userID = xboxlive_user_for_pad(global.PadIndex);
result = xboxlive_matchmaking_create(userID, xboxlive_match_visibility_usetemplate, "MatchTicketSession", "MatchTicketHopper", "PeerServerTraffic") 。

以上将检索给定游戏板上用户的用户ID，然后为他们初始化一个匹配会话。
