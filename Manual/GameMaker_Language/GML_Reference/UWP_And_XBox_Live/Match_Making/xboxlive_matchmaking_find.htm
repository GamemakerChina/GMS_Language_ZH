xboxlive_matchmaking_find

此功能将使用您应事先在XDP仪表板中创建的会话模板，匹配配对料斗和安全设备关联模板搜索多人会话。您可以通过调用特定游戏手柄获取的用户ID xboxlive_user_for_pad()功能。该函数将返回唯一的请求ID值，然后可以使用该ID来为此函数标识正确的社交异步事件。创建会话后将触发此事件，并将在变量中包含DS Map async_load具有以下键/值对：

  • “ requestid” –包含调用函数返回的请求ID
  • “状态” –包含字符串“ session_created”，用于通知您已触发的事件类型。
  • “ num_results” –如果已找到会话，则包含1或多个，否则包含0。
  • “ sessionid <索引> ” –包含会话ID，或者在失败的情况下为-1（ <索引/ gt是从0到“ num_results”的值-1）。
  • “ sessionOwner <index> ” –如果已找到会话，则包含会话主机的ID（ <index>是从0到“ num_results”的值-1）。
  • “错误” –成功时包含0，失败则包含-1

句法：

xboxlive_matchmaking_find(user_id, template, hopper, sdatemplate, [matchattributes]);

争论描述
user_id要使用的用户ID指针
template会话模板的名称
hopper牵线搭桥的名字
sdatemplate安全设备关联模板的名称
matchattributes [选修的]

返回值：

Request ID (integer)

例子：

var userID = xboxone_user_for_pad(global.PadIndex);
result = xboxone_matchmaking_find(userID, "MatchTicketSession", "MatchTicketHopper", "PeerServerTraffic");

上面的代码将在给定的游戏手柄上检索用户的用户ID，然后尝试为他们找到配对活动。
