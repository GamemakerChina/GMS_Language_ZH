xboxlive_matchmaking_join_session

此功能允许您通过会话句柄加入特定的会话。这类似于加入会话邀请，但是在您拥有自己的服务器执行配对而不是依赖于xbox实时配对时很有用。

该功能要求您提供用户ID-您可以通过调用以下代码来获取特定游戏手柄的用户ID：xboxlive_user_for_pad()功能-以及您应该事先在XDP信息中心中创建的安全设备关联模板。

对于会话句柄，您需要首先从要加入的会话中检索“相关ID”。创建原始会话后，“异步社交”事件将为“session_created ”，它将在返回的地图中包含一个“ correlationid”条目（请参见 xboxlive_matchmaking_create()）。您需要将此ID传递回服务器，以便在收到配对请求时将其传递出去。一旦加入的游戏从服务器收到了“相关ID”，他们便可以调用此函数，并传递相关ID作为会话句柄参数。

如果出现错误（在这种情况下，应该有详细说明问题的控制台输出），该函数将返回-1，否则将返回请求ID值。然后，可以使用“请求ID”值来为此功能标识正确的社交异步事件。创建会话后将触发此事件，并将在变量中包含DS Map async_load具有以下键/值对：

  • “ requestid ”-包含调用函数返回的请求ID
  • “ status ”-包含字符串“ session_player_joined”，用于通知您已触发的事件类型。
  • “ sessionid ”-包含会话ID，如果失败，则为-1。
  • “ sessionOwner < index > ”-如果已找到会话，则包含会话主机的ID（< index >是从0到“ num_results”的值-1）
  • “ id ”-包含常量 MATCHMAKING_SESSION
  • “ error ”-如果一切正常，将为0；如果出现故障，则为-1

当新玩家加入会话时，DS映射将具有以下键/值对：

  • “ id ”-包含常量 MATCHMAKING_SESSION
  • “ sessionid ”-包含会话ID
  • “ status ”-包含字符串“ session_player_joined”
  • “ requestid ”-创建请求的函数返回的ID值

句法：

xboxlive_matchmaking_join_session(user_id, session_handle, sdatemplate);

争论描述
user_id要使用的用户ID指针
session_handle用于会话ID的“相关ID”（请参见上面的说明）
sdatemplate要使用的安全设备关联模板的名称

返回值：

Request ID (integer)

例子：

Request_ID = xboxlive_matchmaking_join_session(global.UserID, global.SessionID, "PeerServerTraffic");

上面的代码将发送一个请求以加入给定的多人游戏会话，并将请求ID值存储在变量中，以便可以在异步事件中对其进行检查。
