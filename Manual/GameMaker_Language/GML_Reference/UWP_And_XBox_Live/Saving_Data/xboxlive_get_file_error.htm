xboxlive_get_savedata_user

此函数将返回一个常量（如下所示）以帮助调试文件函数失败的原因，以便您可以在游戏代码中采取适当的措施。当其中一个文件函数失败时，您可以调用该函数以准确找出错误所在。

注意：使用异步保存/加载功能时，async_load映射将具有一个额外的键“ error ”，其中还将包含一个常量（如下所示）。

句法：

xboxlive_get_file_error();

返回值：

Constant

持续的描述
xboxlive_fileerror_outoflocalstorage如果在保存时发生这种情况，则意味着所有可用的保存游戏空间都已用完，并且用户拒绝（由OS提出）释放空间的提议。游戏应与用户确认他们打算继续游戏而不保存。如果他们仍然希望能够保存，则游戏应重试保存操作。
xboxlive_fileerror_quotaexceeded如果发生这种情况，游戏应通知用户在释放一些空间之前无法保存更多数据，然后向用户提供一种执行此操作的方法。
xboxlive_fileerror_noerror文件保存或加载功能没有问题。

例子：

buffer_save(buff, "SaveGame.bin");
if (xboxlive_get_file_error() != xboxlive_fileerror_noerror)
    {
    show_debug_message("Save failed. Error code = " + string(xboxlive_get_file_error()));
    }

上面的代码检查保存是否有错误，如果是，则将错误代码打印到控制台。

返回：保存数据
