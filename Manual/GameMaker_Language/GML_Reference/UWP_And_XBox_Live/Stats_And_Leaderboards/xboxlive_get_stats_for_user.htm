xboxlive_get_stats_for_user

这个函数可以用来从Xbox Live服务器中检索关于特定统计的数据。你提供由函数返回的用户ID xboxlive_get_user()，然后是你的游戏服务 配置ID（如XDP控制台上显示的那样），最后是所需的统计信息。你最多可以请求14个统计信息，但这并不保证你会得到14个统计信息的结果，因为请求的总长度是有限制的，所以最大的统计信息数取决于你的游戏。 因此，最大的统计信息数量取决于统计信息本身的名称的长度。

如果统计请求被发送，该函数将返回0，如果有错误则返回-1，回调将触发一个系统异步事件。这个事件将有 特殊的DS地图 async_load然后应该对其进行解析，以获得以下关键。

  • "event_type" - 如果事件是由这个函数触发的，应该持有字符串 "stat_result"。

如果事件类型与此功能有关，那么也会有以下的附加键。

  • "user"- 持有所请求的统计资料的用户ID。

也可能有一组额外的键值对，其中键是请求的状态名称，而值是服务器上该状态的值。请注意，如果一个状态从来没有为这个用户创建过（因为他们还没有触发事件来 设置的事件），则可能不会返回该状态的值。ds_map_exists()可以用来检查async_loadmap中的统计信息）。)

如果由于请求长度过大而导致请求失败，在控制台输出中应该有一条信息，说明错误代码。

xboxlive_get_stats_for_user - 获取结果时发生异常 - 0x80190190

当这种情况发生时，异步事件DS地图应该有一个值为 "0 "的 "成功 "键，在这种情况下，你应该尝试在一次调用中检索更少的统计信息。

语法。

xboxlive_get_stats_for_user(user_id, serviceconfig_id, statname1, ...)。

争论 描述
user_id 用户ID的指针。
服务配置_ID 该服务的配置文件ID
statname1 (2, 3, etc.) 检索信息的状态名称。

返回。

真实的

例子。

var _uid = xboxlive_get_user(0);
var _configid = "000000-0000-0000-0000-0000";
xboxlive_get_stats_for_user(xbuser, _configid, "GameProgress", "CurrentMode") 。

上面的代码得到了用户的ID，然后用它来请求该用户的特定统计信息。
