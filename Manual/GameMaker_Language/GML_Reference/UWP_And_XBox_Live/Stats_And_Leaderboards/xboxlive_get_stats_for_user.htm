xboxlive_get_stats_for_user

此功能可用于从Xbox Live服务器中检索有关特定统计信息的数据。您提供函数返回的用户ID xboxlive_get_user() ，然后是游戏的服务配置ID（如XDP控制台上所示），最后是所需的统计信息。您最多可以请求14个统计信息，但这不能保证您会获得14个统计信息结果，因为请求的总长度是有限的，因此最大统计信息数取决于名称的长度统计数据本身。

如果发送了stat请求，则该函数将返回0；如果发生错误，则该函数将返回-1，并且回调将触发System Asynchronous Event 。此活动将具有特殊的DS地图 async_load然后应解析以下密钥：

  • “EVENT_TYPE” -应持有字符串“stat_result”如果事件是由该功能触发

如果事件类型与此功能相关，那么还将有以下附加键：

  • “ user ”-保留请求的统计信息的用户ID。

可能还会有一组额外的键值对，其中键是请求的统计名称，值是服务器上该统计的值。请注意，如果从未为该用户创建统计信息（因为他们尚未触发设置该用户的事件），则该统计信息不会返回任何值（ds_map_exists()可用于检查 async_load地图）。

如果由于请求长度太大而导致请求失败，则控制台输出中应显示一条消息，指出错误代码：

xboxlive_get_stats_for_user - exception occurred getting results - 0x80190190

发生这种情况时，异步事件DS映射应具有值为“ 0”的“成功”键，在这种情况下，您应尝试在单个调用中检索较少的统计信息。

句法：

xboxlive_get_stats_for_user(user_id, serviceconfig_id, statname1, …);

争论描述
user_id用户ID指针。
serviceconfig_id服务配置文件ID
statname1 (2, 3, etc...)用于检索信息的统计名称。

返回值：

Real

例子：

var _uid = xboxlive_get_user(0);
var _configid = "00000000-0000-0000-0000-000000000000";
xboxlive_get_stats_for_user(xbuser, _configid, "GameProgress", "CurrentMode");

上面的代码获取用户ID，然后使用该ID请求有关该用户的特定统计信息。
