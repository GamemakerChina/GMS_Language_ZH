xboxlive_read_player_leaderboard

该函数将返回给定用户名的排行榜信息。排行榜必须事先在您的游戏的XDP仪表板上创建，并且调用函数将触发一个回调Social Asynchronous Event ，其中包含 async_load映射填充了相关的键/值对。 DS映射的“ id”键用于标识正确的回调（对于任何给定的异步事件，可以有多个触发函数），并且IT将与常量配对 achievement_leaderboard_info以及许多其他键/值对。该地图的确切内容如下所示：

  • “ id ”-对于此功能，应为 achievement_leaderboard_info
  • “ Leaderboardid ”-提供商面板上定义的排行榜的唯一ID。
  • “ numentries ”-排行榜中已收到的条目数。
  • “ PlayerN ”-玩家的名称，其中“ N”是一个整数值，对应于他们在排行榜条目列表中的位置。
  • “ PlayeridN ”-玩家的唯一用户ID，“ N”。
  • “ RankN ”-排行榜中玩家“ N”的排名。
  • “ ScoreN ”-玩家“ N”的分数。


该函数要求您提供以下常量之一来设置过滤器属性：

持续的描述
achievement_filter_all_players获得所有分数。
achievement_filter_friends_only仅按升序获得朋友分数。
achievement_filter_favorites_only仅按升序获得收藏夹分数
achievement_filter_friends_alt仅按降序获取朋友分数。
achievement_filter_favorites_alt仅按降序获取收藏夹分数。

句法：

xboxlive_read_player_leaderboard(leaderboard_name, user_name, num_items, friend_filter);

争论描述
leaderboard_name要读取的页首横幅的名称，与XDP上的设置相同。
user_name要读取的用户名。
num_items要检索的项目数。
friend_filter过滤器常数之一（请参见上面的说明）。

返回值：

Real

例子：

var _user = xboxlive_user_for_pad(0);
var _uid = xboxlive_user_id_for_user(_user);
xboxlive_read_player_leaderboard("MyLeaderboard", _uid, 10, achievement_filter_all_players);

上面的代码获取用户ID，然后使用该ID请求给定排行榜前10个位置的所有玩家数据。
