xboxlive_pad_for_user

Xbox用户可以为他们的用户ID分配各种键盘，每个键盘都填充一个“插槽”。这些插槽从0开始编号，并且可以保存任何填充索引，因此可以将插槽0分配给填充3，将插槽1分配给填充2。此功能可用于获取分配给任何给定插槽的填充，如果用户离开游戏板一段时间后仍未输入，这将特别有用，因为它会自行关闭，因此您可以使用此功能检查任何给定的用户ID，以确保他们拥有活动的游戏板，如果有，然后不要通过（例如）暂停游戏做出相应的反应。如果该垫处于活动状态，则将返回其索引值，但如果未激活，则该函数将返回-1。

请注意，如果任何插槽中的打击垫都处于非活动状态，则以后的插槽将“滑入”到空闲位置，因此，如果您将三个活动打击垫分配给了插槽0、1和2，并且插槽0中的打击垫处于非活动状态，则插槽1和到现在将变为插槽0和1，因此插槽0应该始终包含一个活动打击垫，除非所有打击垫都已关闭（在这种情况下它将返回-1）。

句法：

xboxlive_pad_for_user(user_id, slot);

争论描述
user_id要检查的用户ID（指针）
user_id要检查的游戏手柄插槽的索引（整数）

返回值：

Pointer

例子：

if (xboxlive_pad_for_user(user_id, 0) == -1)
    {
    for(var i = 0; i < gamepad_get_device_count(); i++;)
        {
        var u_id = xboxlive_user_for_pad(i);
        if (u_id == -1)
            {
            async_ID = xboxlive_pad_for_user(i, u_id);
            }
        }
    }

上面的代码检查给定用户ID指针的插槽0，以查看是否已分配了任何游戏手柄，如果没有，则分配一个。

返回：用户和帐户
