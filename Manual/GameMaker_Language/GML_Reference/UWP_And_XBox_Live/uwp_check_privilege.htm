uwp_check_privilege

使用此功能，您可以检查给定的用户是否具有选定的特权。该函数将在系统异步事件中返回true或false，具体取决于是否启用了特权，如果将try_resolution参数设置为true且未启用特权，则该函数还将打开系统对话框（暂停游戏） ），提示用户升级帐户或获取特权所需的任何操作。如果用户随后启用了必需的选项，则该函数将返回true。

该函数要求您对特权请求使用以下常量之一：

持续的描述
uwp_privilege_internet_browsing检查是否允许互联网浏览
uwp_privilege_social_network_sharing检查是否允许与社交网络共享
uwp_privilege_share_kinect_content检查是否允许共享Kinect控制器
uwp_privilege_video_communications检查是否允许视频通信
uwp_privilege_communications检查是否允许互联网通讯
uwp_privilege_user_created_content检查是否允许访问用户创建的内容
uwp_privilege_multiplayer_sessions检查是否允许在线多人游戏会话
uwp_privilege_sessions检查是否允许在线会话
uwp_privilege_fitness_upload检查是否允许上传健身数据

调用函数后，在XBox One ，当收到回调结果时，它将触发一个异步系统事件，该事件将具有特殊的DS映射 async_load。然后，应针对以下关键字解析此映射：

  • “ event_type ”-如果事件是由此函数触发的，则应保留值“ check_privilege_result”

如果事件类型与此函数有关，那么在XBox上调用该函数时，还将有以下附加键：

  • “结果”-将是以下一个或多个常量（组合值）：
    持续的描述价值
    uwp_privilege_result_no_issue用户没有特权问题0
    uwp_privilege_result_purchase_required用户必须购买其他东西（通常是订阅）才能访问1
    uwp_privilege_result_aborted支票被中止2
    uwp_privilege_result_banned用户被禁止4
    uwp_privilege_result_restricted通常通过家长控制来限制用户访问8

  • “特权”-将是上面列出的要在函数中使用的常数之一，因此，如果进行多次检查，则可以检查所请求的特权。

XBox One的UWP平台上运行，则不会触发Async事件，而该函数将立即返回-1、0或1，其中：

  • -1表示-1用户没有特权
  • 0表示该功能无法查询用户
  • 1表示用户具有特权。

句法：

uwp_check_privilege(user_id, privilege, attempt_resolution);

争论描述
user_id要检查的用户ID指针。
privilege要检查的特权常数（请参见上面的描述）。
attempt_resolution如果设置为，则启用尝试解决方法 true，如果设置为，则忽略 false。

返回值：

Integer

例子：

var user_one = xboxlive_get_user(0);
if (xboxlive_user_is_signed_in(user_one))
    {
    uwp_check_privilege(user_one, xboxlive_privilege_multiplayer_sessions, true);
    }

上面的代码检查用户是否登录以及是否登录，并检查多人游戏会话权限。
