网络和HTML5

GameMaker Studio 2提供了各种与互联网或服务器互动的功能，它们可以在所有可用的目标平台上使用。这些功能是：。

  • url_open
  • url_open_ext
  • url_open_full
  • url_get_domain
  • http_request
  • http_get
  • http_get_file
  • http_post_string

除了上面列出的一般功能外，HTML5目标还有一些额外的网络功能，是专门为针对它的游戏而设计的。

  • 浏览器高度
  • 浏览器宽度
  • 浏览器_输入_捕捉
  • 启用webgl_enabled
  • 可点击_存在
  • 可点击_添加
  • 可点击_添加_文本
  • 可点击的变化
  • 可点击的变化_文本
  • 可点击_设置风格
  • 可点击_删除
  • 分析_事件
  • 分析学_事件_文本
  • os_browser
  • http_get_request_crossorigin
  • http_set_request_crossorigin

除了 HTML5 目标模块的这些功能外，还有一个特殊的JavaScript 变量，可以在 html5game 文件夹的内容存储在完全不同的地方（URL）的罕见情况下使用。 通常情况下，该路径是在JavaScript文件中硬编码的，是相对于调用JavaScript的位置而言的，但您可以在调用游戏脚本之前设置该变量 g_GameMakerHTML5Dir但你可以在调用游戏脚本之前设置变量，游戏将从新定义的位置运行 从新定义的位置运行。比如说

<script>var g_GameMakerHTML5Dir= "http://mygamealternatedirectory.com/";</script>

还值得注意的是，你可以使用前缀gmcallback_来标记脚本函数，这意味着当你为HTML5编译项目时，它们不会被混淆。这在使用上面列出的 clickable_* 函数时非常有用（更多信息，请看这里）。
