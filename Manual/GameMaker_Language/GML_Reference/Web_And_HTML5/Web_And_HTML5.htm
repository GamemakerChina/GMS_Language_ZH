网络和HTML5

GameMaker Studio 2提供了用于与Internet或服务器交互的各种功能，并且可以在所有可用的目标平台上使用。这些功能是：

  • url_open
  • url_open_ext
  • url_open_full
  • url_get_domain
  • http_request
  • http_get
  • http_get_file
  • http_post_string

除了上面列出的常规功能之外， HTML5目标还有一些其他的网络功能，这些功能专门设计用于目标用户的游戏：

  • browser_height
  • browser_width
  • browser_input_capture
  • webgl_enabled
  • clickable_exists
  • clickable_add
  • clickable_add_ext
  • clickable_change
  • clickable_change_ext
  • clickable_set_style
  • clickable_delete
  • analytics_event
  • analytics_event_ext
  • os_browser
  • http_get_request_crossorigin
  • http_set_request_crossorigin

除了HTML5目标模块的这些功能外，还有一个特殊的JavaScript变量，可以在极少数情况下将html5game文件夹的内容存储在完全不同的位置（URL）时使用。通常，此路径是硬编码在JavaScript文件中的，并且相对于调用JavaScript的位置，但是您可以设置变量 g_GameMakerHTML5Dir在调用游戏脚本之前，游戏将从新定义的位置运行。例如：

<script>var g_GameMakerHTML5Dir = "http://mygamealternatedirectory.com/";</script>

还值得注意的是，您可以使用前缀标记脚本函数 gmcallback_这意味着当您为HTML5编译项目时，它们不会被混淆。使用上面列出的方法时，这非常有用 clickable_*功能（有关更多信息，请参见此处）。
