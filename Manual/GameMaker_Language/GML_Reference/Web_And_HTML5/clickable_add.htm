clickable_add

使用此功能，您可以创建一个自定义的，可单击的图标，该图标充当游戏画布外部但位于浏览器窗口本身内的“按钮”。该函数将返回一个id值，该id值必须存储在变量中，以便在引用该新图标的所有其他按钮函数中使用。此功能对于防止浏览器单击按钮时创建弹出窗口非常有用，因为这都是通过GameMaker Studio 2本身完成的。

您必须首先设置相对于浏览器窗口（左上角）的（0,0）的位置，然后为希望按钮图标具有的精灵提供纹理页面条目（请参阅：sprite_get_tpe() ）。然后，您可以指定要打开的URL或要打开或运行的脚本（来自GameMaker Studio 2的资源浏览器）。当您使用以下命令时，您指定的“目标”参数与标准JavaScript“名称”值相同 open()方法（请注意，除了“ _self”以外的所有内容都可能导致浏览器阻塞，或者询问用户是否希望允许它），并且“ params”参数与用于控制新属性的JavaScript“ specs”参数相同窗口/标签应显示（并非所有浏览器都支持所有功能）。如果使用图标运行内部脚本，则可以将后两个参数保留为空字符串“”。

有效的目标是：

目标描述
_空白的在新窗口或选项卡中打开链接的文档。
_自己在与单击相同的框架中打开链接的文档（这是默认设置）。
_父母在父框架中打开链接的文档。
_最佳在窗口的整个正文中打开链接的文档。

有效参数为：

参数描述
“高度= [ px ]”窗口的高度，最小值为100。
'宽度= [ px ]'窗口的宽度，最小值为100。
“左= [ px ]”窗口的左侧位置。
'top = [ px ]'窗口的顶部位置（仅适用于IE）。
“位置= [布尔值]”是否显示地址字段（默认为1）。
'menubar = [布尔值]'是否显示菜单栏（默认为1）。
'resizable = [ boolean ]'窗口是否可调整大小（默认为1）。
'scrollbars = [布尔值]'是否显示滚动条（默认为1）。
'状态= [布尔]'是否添加状态栏（默认为1）。
'titlebar = [ boolean ]'是否显示标题栏。除非调用应用程序是HTML Application或受信任的对话框（默认值为1），否则将忽略此设置。
'工具栏= [布尔]'是否显示浏览器工具栏（默认为是）。


还值得注意的是，如果您传入的URL参数是脚本函数的名称（即：它不是以“ http：//”开头），并且脚本被调用 gmcallback_*那么您可以将直接供稿添加到html5事件中。 GameMaker Studio 2脚本功能名称以开头，则不会混淆 gmcallback_。

句法：

clickable_add(x, y, tpe, url, target, params);

争论描述
x窗口内的x位置。
y窗口内的y位置。
tpe要使用的精灵的纹理页面条目。
url链接到的URL（网站地址）或要运行的脚本。
target这是在其中打开URL的目标区域（请参阅描述）。
params图标的各种参数（请参阅说明）。

返回值：

Real

例子：

home_but = clickable_add(32, 32, sprite_get_tpe(spr_MS_Home, 0), "http://macsweeney_games.com", "_blank", "width=700, height=500, menubar=0, toolbar=0, scrollbars=0");

上面的代码在运行游戏画布的页面的位置（32、32）处创建了一个可单击的DOM图标。该图标使用从纹理页面引用的精灵作为“ spr_MS_Home”，单击该图标将为指定的URL打开一个新窗口，并为该窗口定义属性。
