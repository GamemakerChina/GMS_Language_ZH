远程工作者

编译游戏时，通常使用“本地”工作程序，即安装GameMaker Studio 2但是，在某些情况下，您可能需要使用另一台计算机来构建项目。例如，如果您在Mac上工作，并且想要构建Windows项目，则需要配置Remote Worker并让Windows PC进行构建。本质上，当您使用远程工作程序时， GameMaker Studio 2会将运行它的（本地）计算机上的所有必需文件发送到远程工作程序计算机，然后使用该计算机为项目生成最终的可执行文件（保存（在远程计算机上），或之后将其返回到本地计算机）。但是，在使用此功能之前，您必须已在构建计算机上安装了某些工具并执行了一些设置，我们将在此处进行解释。

重要的！您可以使用本地Windows PC和远程Windows PC，或在本地Mac PC和远程Windows PC之间设置远程工作程序，但是当前您无法设置本地Windows （或Mac）PC和远程Mac PC，即：您不能有Mac远程工作器，只有Windows远程工作器。

或者，您可以单击“目标管理器”中的“眼睛”图标，然后通过以下方式获取Remote Worker安装程序：

安装程序将被称为“GameMaker Studio-Remote-Installer-XXXXXX.exe”，则需要将其复制到要用作远程工作器的PC上（您不需要GameMaker Studio 2 ，只需使用此程序即可）。

将其复制到构建PC后，请按照屏幕上的提示进行安装，并且安装程序完成后，操作系统可能会要求您通过防火墙访问网络。授予访问权限后，安装程序将在Web浏览器中打开一个页面，您必须记下此页面上显示的IP地址和端口号，因为下一步将需要它。

编译中

设置好Remote Worker并完成“本地构建设置”（请参阅上面的部分）后，就可以开始使用它来编译游戏了。为此，只需从“目标管理器”列表中选择“远程工作器”，选择要测试/编译的平台以及任何其他选项（如编译类型或配置），然后按“播放”即可。或创建可执行文件。

注意：目标管理器中的可用平台将根据Remote Worker计算机的功能而变化，并且并非所有目标都可用。另请注意，远程工作人员没有“测试”目标。

如果您只是在播放项目，则Remote Worker机器将编译并运行它，但是如果您正在创建可执行文件，则Remote Worker将编译该项目，然后返回 *.zip文件到本地计算机进行保存。

应该注意的是，“调试”按钮都没有也不是清洁按钮使用“远程工作者”时将可用。另请注意，当前一次只能执行一个远程构建。


在远程计算机上安装并运行了Remote Worker之后，它将在任务栏中显示为一个图标，您可以右键单击它以显示以下选项：

  • 打开：在默认浏览器中打开Remote Worker应用程序。
  • 显示日志：在浏览器中打开Remote Worker的日志文件。
  • 网络设置：您可以在此处过滤希望将构建发送到Remote Worker机器的机器。每个过滤器可以是CIDR样式的过滤器，也可以是允许使用的计算机的主机名。您应该注意，如果要多台计算机连接到一个远程工作者，或者要混合使用连接类型（例如：有线和wifi），则可能需要在此处编辑IP允许范围。
  • 编辑设置：打开“远程工作器”设置菜单（在上面的“设置”部分中有解释）。
  • 退出：退出远程工作者应用程序。这将意味着您将无法再从本地计算机与该应用程序连接（或编译），直到重新启动Remote Worker应用程序为止。
