源代码控制

源代码管理管理（SCM）是通过存储库和本地源使用项目的子版本或备份的方法的名称。基本上，SCM解决方案是一个独立的软件包，可以控制在您进行项目时维护，更改和比较项目版本的所有方面。这对于在团队中工作并且需要能够控制谁做什么并且不必担心丢失数据或进行以后可能需要撤消的更改的人尤其有用，但是个人可以从这种强大而灵活的中受益系统也是。

您可以使用各种SCM工具，并且GameMaker Studio 2附带了一个Git插件，可以立即使用，并且您无需安装任何额外的软件包，因为它们都与插件本身捆绑在一起。您可以在下面找到有关如何设置此插件以及如何在项目中使用SCM工具的小型教程的链接。

在这里，我们概述了每个可用选项（在上面的教程部分中对其中的大多数选项进行了更深入的说明）：

  • 创建项目存储库：此选项允许您在与项目文件相同的目录中创建本地存储库。
  • 将项目导入到存储库：此选项允许您将项目导入到外部存储库。
  • 克隆存储库：此选项将允许您将存储库从源克隆到新的目标。
  • 提交更改：使用此选项，您可以在项目中暂存已更改的文件，然后将其提交到存储库。
  • 推送更改：在提交或合并之后，此选项用于将更改推送到主存储库。
  • 提取更改：使用此选项，您可以通过从主存储库中提取更改的文件来更新本地存储库。
  • 查看历史记录：这将打开历史记录窗口，您可以在其中查看项目的所有版本历史记录，并选择回滚特定文件路径或整个修订版。
  • 显示冲突：这将打开“冲突”窗口并列出项目中可能存在的任何冲突文件，使您可以通过GameMaker Studio 2 IDE或使用特定的合并工具来处理它们。
