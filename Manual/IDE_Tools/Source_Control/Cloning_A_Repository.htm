克隆存储库

当您的项目存储在外部存储库中时，您需要记住既要推送更改又要抽取任何新的更改。这样做的原因是，在提交时，您将更改存储在本地，这使您可以脱离Internet连接工作并在准备就绪时进行同步，并通过同步将更改推送到远程存储库。进行更改意味着您希望将其他人从远程存储库中进行的所有更改都拖入本地存储库中。但是，这意味着有时人们可以编辑相同的文件，从而造成合并冲突。您需要知道如何处理这些冲突，因此我们将刻意创建一个冲突，然后向您展示如何处理它。

注意：不建议共享本地项目存储库（即使通过诸如Dropbox之类的文件同步服务），因为通常会将提交直接写入该存储库，因此几乎没有冲突保护。

为了产生冲突，我们首先需要有两个本地存储库，一个用于主存储库，一个用于我们的副本。为此，我们需要克隆存储库，可通过以下步骤完成：

  • 转到“源代码管理” >“克隆存储库” 。
  • 在“ URL”字段中，键入我们在上面的“还原文件”指南中设置的项目的路径（默认情况下，该路径为 <DRIVE>:\Users\<Username>\Documents\GameMakerStudio2 \ <ProjectName>）。
  • 然后，我们需要设置一个克隆路径：
    当我们单击“确定”按钮时，我们将收到有关尝试克隆“非裸机”存储库的以下警告：
    我们可以再次单击“确定”。
  • 现在，我们将看到一个目录浏览器，用于创建“裸”代理存储库（例如，可以安全地存储在Dropbox上的存储库）。浏览到我们要在其中创建此代理存储库的位置，然后选择（或创建）要使用的文件夹。

完成此操作后， GameMaker Studio 2会将原始项目推送到代理，然后将新项目拉到我们在“克隆存储库”窗口中设置的路径，然后自动打开文件浏览器，以便我们打开存储的项目。如果现在转到“源代码管理” >“查看历史记录”，我们将看到原始提交以及新的“推送克隆”

现在，您已经克隆了存储库，并且可以继续处理项目的本地版本。
