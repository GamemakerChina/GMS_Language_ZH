矛盾冲突

在本节中，我们将研究解决克隆的本地存储库和主存储库之间的冲突。假设您现在使用的是本地存储库，我们需要编辑一个文件，然后引起某种冲突，因此打开我们一直在使用的测试脚本，并将“ Hello World”消息更改为如下所示：

show_debug_message("Hello World, How Are You?");

现在，您应该保存项目并将其提交到本地存储库。现在，由于我们已链接到远程主存储库（即使它是我们已经克隆到同一磁盘的存储库），因此我们需要执行“推送”将更改保存到主存储库中。为此，转到Source Control >推送更改，这将在Source Control Output选项卡中为我们提供状态更新：

现在，我们需要打开我们的原始项目（而不是克隆的项目），因此浏览到您将其保存在磁盘上的位置-或只需检查“文件”菜单中的“最近的项目”，因为它应该位于当前项目的正下方-并加载它进入GameMaker Studio 2 。检查您的项目正确的一种简单方法是简单地检查您拥有的脚本，如果消息是“ Hello World”，那么您拥有正确的脚本，或者可以打开“查看历史记录”窗口并进行检查。

现在，您需要再次编辑脚本，因此使其读取如下内容：

show_debug_message("This will cause a conflict.");

可以将其保存到磁盘并提交更改，但是，如果尝试推送这些更改，则会收到以下消息：

有人已经推送了您还没有的更改！这意味着您需要对存储库进行拉动，然后查看问题的确切原因，因此请转至Source Control > Pull Changes 。这将显示两个窗口，第一个是有关文件更改的警告（与以前的操作一样，您想要在此处选择Reload而不是Save ），另一个是“冲突”窗口：

此窗口将在左侧列表中显示冲突的文件，您可以使用鼠标左键从该列表中选择文件。 （或使用/或者随着）。您可以通过右侧的按钮使用四个选项来处理冲突的文件：

  • 使用他们的-这将覆盖您对远程存储库所做的更改。
  • 使用我的-这将取代远程存储库中所做的更改。
  • 合并-这将尝试打开一个外部合并工具来处理冲突。
  • 选择合并工具-这将使您可以配置合并工具（有关更多详细信息，请参见“外部合并/差异工具”部分）。

现在，您可以使用按钮来解决冲突，但是在您进行操作之前，我们只需要打开冲突的文件以确切了解问题所在。如果您打开有冲突的脚本，它现在将如下所示：

这表明HEAD上有“这将导致冲突”（这是我们推送到本地存储库的内容），并且“ Hello World，您好吗？”是从远程存储库master中提取的。您现在可以返回到“冲突”窗口，然后单击“使用其”以拉出更改的文件并覆盖当前的HEAD文件。现在，脚本将被标记为已更改，您应该执行新的提交并推送，然后再继续。

请注意，我们实际上可以在GameMaker Studio 2 IDE中修复此问题。如果我们删除第1、2、3和5行，那么我们只剩下 show_debug_message("Hello World, How Are You?")并保存它，它将清除冲突，因为Git会假设您知道自己使用自己的文件正在做什么。但是，这容易出错，因此建议先安装合并工具，然后配置并使用该工具，而不要手动尝试合并。
