外部合并/差异工具

当您与其他人一起进行项目时，总是有可能发生冲突，因为多个人编辑同一个文件，并且大多数源代码控制系统都可以很好地尝试合并任何冲突的文件，但是当在同一地方，需要一些帮助才能知道该怎么做–这是合并工具用来向您显示更改并让您决定的地方。

由于每个合并工具都不同，因此可以使用四个宏来帮助为其指定文件，这些宏可以通过单击“选择合并工具”按钮从“冲突”窗口中设置。 ：

当您单击该按钮时， “偏好设置”窗口将在Git插件上打开：

在这里，您提供了选择的合并工具的路径，然后提供了“合并工具选项”宏（一个或多个宏），以便在决定要执行的操作时使用，并提供以下选项：

  • ${scm_mine} ：版本的路径
  • ${scm_thers}：版本的路径
  • ${scm_base}：发生冲突更改之前版本的路径
  • ${scm_merged}：合并工具应保存到的路径

作为设置的帮助，当您提供某些合并工具的路径时， GameMaker Studio 2会填充一些默认值以供使用，具体取决于所选工具：

  • KDiff3 ： ${scm_theirs} ${scm_mine} -o ${scm_merged}
  • 融化： ${scm_mine} ${scm_merged} ${scm_theirs}
  • Notepadd ++ ： ${scm_merged}
  • TortoiseMerge ： /base:${scm_base} /theirs:${scm_theirs} /mine:${scm_mine} /merged:${scm_merged}

请注意，如果您没有（或不想使用）合并工具，则可以在IDE中打开脚本并尝试从那里修复它。

所以，回到在上一节所述我们的例子中冲突的冲突......我们已经从师傅做了牵引和文件的一个冲突。这将打开“源代码管理冲突”窗口，您可以在其中单击冲突的脚本，然后单击“合并” ，因为已配置了合并工具。单击合并现在将打开该工具，您可以在其中处理冲突，然后使用提供的宏进行所需的更改。完成并关闭合并工具后， GameMaker Studio 2 IDE将显示以下确认窗口：

使用合并工具完成此提示后，只需对此提示回答“是”，然后需要提交此更改以表示合并操作结束（“源代码控制” >“提交更改” ）。现在，“提交”窗口将显示正在进行合并，并显示一些未暂存的文件：

这些未暂存的文件是我们的脚本文件，另外还有 .base， .ours和 .theirs。我们可以安全地删除所有这些，因此请点击在最上面的一个，然后+在最下面的一个上将其全部选中。接下来，我们右键单击所选文件，然后选择“删除路径”将其删除。也为我们预先填充了一条提交消息，指出这是一个合并，并且存在冲突（我们可以根据需要添加或编辑它）。

我们可以单击立即提交。

由于您可能会遇到其他文件的问题，这意味着使用远程或共享存储库时的工作流程应如下：

Make Changes > Save > Commit > Pull > Merge > Push

您还应该设置一个Diff工具，以便在对存储库进行提交时使用。这是通过Preferences设置的，与Merge工具相同：

差异工具的配置方式与合并工具大致相同，但仅使用 ${scm_base}和 ${scm_theirs}选项-在这种情况下， ${scm_base}代表未修改的文件，而 ${scm_theirs}表示文件的当前状态。工具选项应预先填充Meld ， KDiff3和TortoiseMerge的默认选项，就像它们用于合并工具一样。需要注意的是GameMaker Studio 2不支持单个文件的diff补丁的输出，因此没有默认的记事本++。

为了使用Diff工具查看当前版本与先前版本之间的更改，只需打开“提交”窗口，然后双击一个文件即可。如果该工具已正确设置，它将启动，您将能够看到文件之间的更改。
