外部合并/差异工具

当你和其他人一起做一个项目时，总是有可能出现冲突，因为多人编辑同一个文件，大多数源码控制系统可以很好地尝试合并任何冲突的文件，但是当有变化时 的时候，需要一些帮助才能知道该怎么做--这就是合并工具 的作用，它可以向你展示这些变化，并让你决定。

由于每个合并工具都是不同的，有四个宏可以用来帮助指定文件给它，可以从冲突 窗口点击标有选择合并工具的按钮来设置， 。

当你点击该按钮时，Git插件的首选项窗口将打开。

这里你提供一个所选合并工具的路径，然后提供一个合并工具选项宏（或多个宏），以便在决定 做什么的时候使用，有以下选项。

  • ${scm_mine} ：你的版本的路径
  • ${scm_thers}: 其版本的路径
  • ${scm_base} ：发生冲突的修改之前的版本路径
  • ${scm_merged}: 合并工具应保存到的路径

作为设置的帮助，当你给出某些合并工具的路径时，GameMaker Studio 2会根据选择的工具，填写一些默认值来开始使用。

  • KDiff3。 ${scm_theirs} ${scm_mine} -o ${scm_merged}
  • 融合。 ${scm_mine} ${scm_merged} ${scm_theirs}
  • Notepadd++。 ${scm_merged}
  • 龟甲合并。 /base:${scm_base} /theirs:${scm_theirs} /mine:${scm_mine} /merged:${scm_merged}

注意，如果你没有（或不想使用）合并工具，那么你可以在IDE中打开脚本，并试图从那里修复它。

进行修改 > 保存 > 提交 > 拉动 > 合并 > 推进

你还应该设置一个差异工具，以便在对版本库进行提交时使用。这是在首选项中设置的，与合并工具相同。

差异工具的配置方式与合并工具基本相同，但只使用 ${scm_base} 和 ${scm_theirs} 选项 - 在这种情况下， ${scm_base} 代表未修改的文件，而 ${scm_theirs} 代表文件的当前状态。工具选项应该 应该预先填充Meld、KDiff3 和TortoiseMerge的默认选项，就像它们在合并工具中那样。请注意，GameMaker Studio 2 不 支持输出单个文件的 文件的差异补丁，因此没有Notepad++的默认选项。
