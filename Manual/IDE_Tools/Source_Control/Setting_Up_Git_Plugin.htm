设置GIT插件

当开始使用GIT插件时，我们需要为GIT设置一个提交身份，这意味着我们需要去插件 - 源控制（Git）部分 并添加你的认证细节。

这个认证身份将用于所有未来的项目，GameMaker Studio 2为你提供了两种不同的方式来设置它。 了。

  • 用户名和密码：点击 "添加新的用户/密码认证 "按钮，你将看到以下窗口，你可以输入版本库的URL、用户名和 密码:
    如果你想让它只影响一个特定的版本库，那么就把版本库的 URL 放在最上面的区域。然而，如果你想要的是 域范围内的认证（即： "bitbucket.org" 或 "github.com" ），则只需放置域而无需其他。当有东西需要检查认证时。 它将首先寻找一个特定的资源库匹配，然后再寻找一个域匹配。这将允许你有一个默认的域名认证，并在以后用特定的细节来覆盖某些资源库。

注意：如果你使用的是外部版本库，那么你应该使用与为该版本库设置的账户相关的用户名和密码，但如果你想使用本地版本库，那么你可以使用任何用户名和密码。

  • SSH密钥对。点击 "添加新的SSH密钥对认证"，你将看到以下窗口，你可以输入存储库的URL和密码，然后给出所需的公钥和私钥文件的路径：。 公钥和私钥文件所需的路径：
    像用户名/密码认证一样，在使用 SSH 对钥时，你可以有域和存储库的特定认证。如果你已经有一个 SSH密钥对，你可以在这里设置路径，以及访问它们所需的任何密码。注意，如果你需要快速访问公钥，复制按钮 ，将自动为你复制内容到剪贴板。如果你还没有一个SSH密钥对，点击创建密钥对按钮将弹出一个文件对话框，说明将私钥放在哪里，而 公钥将在其旁边被创建 - 这将为你自动填写公钥和私钥的路径。

在处理克隆、推送和拉动请求时，这些设置将被用来与仓库（IE:GitHub, Bit Bucket, etc. ）进行通信。但是你仍然需要设置每个项目的选项，可以通过打开主选项并点击启用源代码控制选项来完成。这将激活当前项目的SCM。

注意：如果你需要再次编辑认证细节，只需双击该条目，细节窗口将打开供你编辑，但你将无法重命名URL。如果你需要改变URL，你将需要 删除认证细节并重新添加。要删除认证细节，点击主偏好窗口中该行左侧的关闭按钮。
