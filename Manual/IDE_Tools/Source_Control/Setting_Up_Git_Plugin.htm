设置GIT插件

开始使用GIT插件时，我们需要为GIT设置身份，以便提交，这意味着我们需要转到“首选项”的“插件-源代码控制（ Git ）”部分，并添加您的身份验证详细信息。

此身份验证标识将用于以后的所有项目， GameMaker Studio 2为您提供了两种不同的设置方式：

  • 用户名和密码：单击“添加新用户/通过身份验证”按钮，将显示以下窗口，您可以在其中输入存储库URL ，用户名和密码：
    如果您希望这仅影响特定的存储库，则将存储库的URL放在顶部字段中。但是，如果您要进行域范围的身份验证（即：“bitbucket.org “ 或者 ”github.com “”，然后仅放置域，别无其他。当需要检查身份验证时，它将先查找特定的存储库匹配项，然后再寻找域匹配项。这将使您对域具有默认身份验证，并覆盖稍后提供某些存储库的详细信息。

注意：如果使用的是外部存储库，则应使用与为该存储库设置的帐户关联的用户名和密码，但是，如果要使用本地存储库，则可以使用任何用户名和密码。

  • SSH密钥对：单击“添加新的SSH密钥对身份验证”将为您显示以下窗口，您可以在其中输入存储库URL和密码，然后提供所需的公用密钥和专用密钥文件的路径：
    像用户名/密码身份验证一样，使用SSH密钥对时，您可以具有域和存储库特定的身份验证。如果您已经生成了SSH密钥对，则可以在此处设置路径以及访问它们所需的任何密码。请注意，如果您需要快速访问公用密钥，复制按钮，会自动将内容复制到剪贴板。如果您还没有SSH密钥对，请单击“创建密钥对”按钮，将弹出一个文件对话框，说明放置私钥的位置，并在其旁边创建公钥-这将填写指向私钥的路径自动为您提供公钥和私钥。
