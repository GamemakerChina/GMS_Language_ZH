编译中

编译游戏可能意味着两件事之一：对其进行测试以进行编译，或者对其进行编译以创建可执行程序包或特定的目标平台。只需按“播放”按钮，即可编译游戏进行测试在IDE的顶部，它将使用指定的目标启动游戏进行测试。您还可以通过使用“调试”按钮进行测试，以“调试模式”运行游戏。这将运行游戏，但还会打开“调试”窗口，您可以在其中监视游戏的性能并查看任何问题（有关更多信息，请参见“调试”部分）。

默认情况下， GameMaker Studio 2将使用内置的虚拟机（VM）运行和调试，该虚拟机与所使用的桌面OS大致相同。但是， GameMaker Studio 2是跨平台引擎，您可以在许多不同的目标平台上测试，调试和编译项目的可执行程序包（可用的确切平台取决于您的许可证的详细信息）。要更改当前目标平台，可以单击在“目标”按钮上打开“目标窗口” ，其外观类似于以下内容（具体细节将根据您的许可证类型而有所不同）：

输出

每个目标平台可以具有一种或多种输出格式，主要的格式有：

  • VM：虚拟机（虚拟机）的目标使用每个平台的通用亚军，然后解释你的游戏的代码。通常，由于该选项构建时间较快，因此可用于测试，但与YYC选项（如果可用）相比，它不能提供相同的性能提升。您可以使用它来编译适用于小型游戏或性能永远不会成为问题的游戏的可执行程序包。
  • YYC ：YYC（ YoYo编译器）采用普通的GameMaker Studio 2输出并将其编译为目标平台的本机代码，“剔除”不需要的功能，并执行许多其他优化技术以创建较小的性能增强的可执行程序包。这可以将您的游戏性能提高至少两倍或三倍，尤其是在逻辑比较繁重的游戏上，并且对于那些大型或CPU密集型项目是理想的。编译时间可能会花费更长的时间，并且在为目标平台构建任何最终的完整资产包之前，应始终清除编译器缓存。请注意，YYC目标可能需要为所选平台安装额外的工具，否则它将不起作用-您可以从YoYo Games帮助中心以及本手册中详细介绍的各个目标“首选项”页面上找到有关此目的的更多信息。
  • JavaScript ：JavaScript目标仅可用于少数目标，例如HTML5目标，并将游戏设置为纯JavaScript。

    创建最终的可执行程序包

    将您的游戏项目编译到任何目标平台都将需要您设置正确的构建工具（请参见此处），并填写适当的Platform Preferences 。完成后，您只需单击“创建可执行文件”按钮在IDE中启动编译器生成，或从“生成”菜单中选择“创建可执行文件”。任一选项都将打开文件资源管理器窗口，您可以在其中提供希望用于游戏可执行文件的最终名称，然后单击“保存”以开始编译和构建过程。完成此操作后，将生成必要的文件，以便您可以根据需要分发它。

    重要的！如果您具有免费的30天试用许可证，那么将无法编译最终的可执行文件，并且您将需要升级到其中一个付费许可证才能编译游戏以进行分发。

    注意：在完成项目最终发行之前，您应该始终清除Asset Compiler Cache （使用“扫帚”图标在IDE的顶部）。 GameMaker Studio 2将缓存您的许多游戏文件，以使编译速度降至最低。

    每个目标选项均保存为特定于平台的格式，如下所示：

    • Ubuntu （ Linux ） Ubuntu （ Linux ）应用程序可以编译为 *.zip文件-包含x64或ARM可执行文件-取决于目标设备。您可以从YoYo Games帮助中心中找到更多信息。
