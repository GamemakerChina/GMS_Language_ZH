创建Marketplace资产

当您Marketplace菜单中选择Create Package选项时，将带您到以下工作空间： 在这里，您必须首先选择要将资产包发布为的发布者身份（应该从发布者仪表板中设置），然后提供显示名称和版本号。然后，将根据使用的显示名称和发布者帐户为您创建一个包ID，但是，只要您希望保留反向URL格式，就可以将其编辑为其他名称。然后，您需要选择要制作的资产包装的类型：

  • 资产Marketplace上出售的资产的集合（精灵，脚本，对象等）。
  • 演示：这是演示一些概念或游戏想法的演示
  • 教程：要上传的资产是一个教程，旨在在GameMaker Studio 2 IDE中运行（有关更多详细信息，请参阅此页面）

完成后，您可以使用右侧的部分从项目中选择要添加到资产包中的资产。您可以添加所有资产，也可以仅添加其中一些资产，也可以删除可能误添加的任何资产。假定您已经创建了先前要在GameMaker Studio 2上载的资产，并且当前项目包含了您需要上载的所有内容。

记住！为了能够创建或使用资产包，您必须事先创建了用户配置文件证书。您可以从“ Marketplace偏好设置”中执行此操作，这在此处进行了说明。

当您对资产的选择感到满意并且想要将包上传到Marketplace ，只需单击底部的“确定”按钮，资产包就会被创建并上传到Marketplace 。上传完成后，将打开一个新的工作区，其中包含资产页面的浏览器视图供您编辑。有关如何进行设置的更多信息，请参见以下页面：

  • 列出资产

如果您需要更新Marketplace资产，那么您所做的操作与创建原始程序包时大致相同，只是现在您需要从可用程序包的下拉列表中指定要更新的程序包：

更新时，您将无法更改程序包ID，并且应注意，显示的版本号将是当前版本号，应适当增加此版本号，否则更新将失败（因为您不能拥有两个资产）具有相同版本号的软件包）。还有一个字段，您可以在其中为资产的更改日志添加一些文本，这些文本将与资产一起上载并显示在“ Marketplace页面上。

重要的！ GameMaker Studio版制作的资产仅对使用2.3版及更高版本的用户可用。如果您要从先前版本更新资产，则该资产将仍可供未更新的用户使用，并将与更新的2.3+版本共存。

通过网页上传

您可以通过实际的Marketplace网页而非IDE上载资产，如果通过IDE上载由于任何原因而失败，则可能需要这样做。首先，您将创建程序包并尝试通过IDE进行上传，如上所示，如果失败，则请转到输出窗口并检查日志中是否有一行显示您正在创建的程序包的保存位置尝试上传之前例如：

在Windows ：

Created package file:
C:\Users\WINDOWSUSERNAME\AppData\Roaming\GameMakerStudio2\GMS2USERNAME\Marketplace\Packages\com.publishername.assetname.yymp

在macOS ：

Created package file:
/Users/MACUSERNAME/.config/GameMakerStudio2/GMS2USERNAME/Marketplace/Packages/com.publishername.assetname.yymp

您需要导航到该文件位置并复制 *.YYMP文件保存到安全位置，因为这将是您要上传到Marketplace的文件。

此时，您需要Marketplace ，导航到资产的资产列表，然后转到“版本”部分，然后单击“上载版本”按钮，如下图所示：

这将带您到新版本的上传页面，您可以在其中添加有关正在上传的版本的信息（如果需要），还可以通过单击“选择文件”按钮并选择先前创建的文件来提供文件。 *.YYMP文件：

添加文件后，将上传版本并显示“进度”栏，当上传完成并且对详细信息满意后，可以单击“保存”按钮将资产保存到列表中。请注意，资产在发布之前将处于处理阶段几分钟。
