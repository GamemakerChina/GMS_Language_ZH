创建一个市场教程

创建一个教程与创建一个普通的市场资产包类似，只是现在你必须加入特定的包含文件，以及包括一个初始的教程页面（和一个可选的视频页面）。 教程工作区看起来会是这样的：正如上面对一般市场资产所做的解释，你需要 选择一个出版商，然后给你的教程一个名称和一个唯一的ID（反向URL格式），然后在 "类型 "下拉菜单中选择教程。这将为你提供额外的选项，以添加一个基本的HTML文件来启动教程，以及一个（可选）视频文件。 以及一个（可选）视频文件和URL。视频文件应使用市场提供的模板--教程的其他部分也应如此--你可以在这里获得。 当用户加载教程时，视频将被加载到IDE的一个单独框架中。请注意，你给的开始页和视频页是包含文件中的相对路径，你可以打开资源管理器来选择 你可以打开资源管理器，选择GameMaker Studio 2填充字段的默认路径以外的其他路径。

实际上，创建你的教程需要你在你的教程项目中添加包含文件，而这些文件必须以一种非常具体的方式添加，以便IDE正确地检测哪些文件将被用于教程。 哪些文件将用于教程中。基本的文件夹结构如下图所示。

因此，我们有两个主要的文件夹。

  • 教程内容。你所有的教程 HTML ，用于设计风格的 CSS ，以及所需的图片（不是资源所需的图片，而是要在教程中显示的图片）。 以及视频模板（如果需要的话），都放在这里。如果你想把教程分成几个章节，你也可以有子文件夹，但你至少应该有一个基本的 HTML 文件 以作为教程的起始页。
  • 教程资源。这个文件夹中还有两个子文件夹，分别是图像 和声音 资产。如果你的教程需要用户添加一个精灵（例如），并且你想把它包含在教程中 那么你将把图像资源放在图像文件夹中。这些文件夹被导入到教程项目中，当用户打开精灵编辑器并添加图像时，资源管理器窗口的默认路径将是这个位置。 使得用户更容易找到必要的资产。

为了使你的教程能够在IDE中运行，你必须有这样的文件夹结构，以及有一个基本的 HTML 开始页，并有一个 CSS 文件用于样式设计。有一些特殊的链接 可以在你的教程项目中使用，这些链接可以跳到视频的某些部分，或者打开特定章节或功能的手册，这些都在市场上的官方教程模板中给出。

重要的是!我们强烈建议你在尝试创建你自己的教程之前，从市场上下载教程模板。它们包含 所有重要的HTML部分的例子，以及CSS和图像，以重新创建教程中的按钮和菜单，让你自由地担心内容，而不是如何显示它。

一旦你设置了包含的文件并提供了所需的信息，你可以点击"好"，教程就会被上传到市场，完成后在资产列表上打开一个工作区，这样你就可以编辑它并添加图标。 文本、屏幕截图等...一旦教程资产被发布，用户就可以在开始页的教程 部分找到它，在那里可以下载和安装。
