列出资产

在发布资产之前，你要做的第一件事是在浏览器中打开市场，创建一个资产清单。你不需要在这里填写所有的细节，但我们确实需要提供一些初步信息 所以你至少可以上传一个包。因此，登录您的悠悠账户，然后进入出版商 部分，选择您创建的出版身份。

在打开的市场窗口中，你现在将看到一个已经上传的资产列表（如果有的话），以及一个标有 "+添加新资产 "的按钮。 这将使你看到以下窗口。

显示名称 将作为软件包在Marketplace上的名称显示，而软件包ID将用于从GameMaker Studio 2和Marketplace发布者仪表板上识别软件包。 用来识别GameMaker Studio 2和Marketplace发布者仪表板中的软件包。软件包ID必须是被上传的资产所特有的 ，并且应该遵循反向的URL格式，例如： com.<Company Name>.<PackageName> 。

注意：带有出版商ID的资产，如果不遵循上述规则，可能无法正确上传!

这就为你的资产包创建了初始清单，现在你可以回到资产页面，添加更多的清单，或者继续为新添加的资产填写市场清单细节。列表页是 你定义所添加的资产包的类型，以及提供它所包含的细节和其他相关信息。

在这里你应该填写以下每个字段。

  • 主类别：这是你正在创建的资产的类型。一旦你设定了你的主要类别，你就可以继续为精灵选择更多的子类别，这将有助于人们在搜索市场时找到适当的资产 在市场上搜索时。
  • 套餐ID：这显示正在添加的资产的套餐ID。它不能被改变，每个ID对正在创建的市场清单是唯一的。
  • 显示名称。资产的名称，它将在市场列表页上显示（最大255字节）。
  • 描述。一个简短的描述（少于2000字），描述该资产、它的功能和可能的用途。
  • 关键字。一个或多个可以用来缩小搜索范围的词，以便在市场上找到你的资产。这些关键词应该与资产相关!
  • 年龄等级。应适用于该资产的年龄等级，标准如下。
  • 4+:这个类别的资产不包含任何令人反感的材料。
  • 9+:这个类别的资产可能包含轻度或不经常出现的卡通、幻想或现实的暴力，以及不经常或轻度的成熟、暗示或恐怖主题的内容，可能不适合9岁以下的儿童。 9岁以下的儿童。
  • 12+:这个类别的资产也可能包含不经常的轻度语言，频繁或强烈的卡通、幻想或现实暴力，以及轻度或不经常的成熟或暗示性主题，这些可能不适合12岁以下的儿童。 12岁以下的儿童。
  • 17+:这个类别的资产也可能包含：频繁和强烈的攻击性语言，频繁和强烈的卡通、幻想或现实的暴力，频繁和强烈的成熟、恐怖和暗示主题，以及性内容。 裸体、酒精、烟草和毒品，可能不适合17岁以下的儿童。
  • 价格。你的资产应该设置的定价层级。仔细考虑你在这里设置的价格，并确保其与你所提供的产品的质量和数量相适应。
  • 所需模块。在这里，你可以选择你的资产要正确使用的最低或绝对要求。
  • 支持电子邮件。一个电子邮件账户，如果人们对资产有任何问题，可以联系你寻求支持。
  • 支持URL。用户可以获得资产支持的URL（即：常见问题、论坛、联系人等）。
  • 营销URL。用户可以获得有关资产和出版商的进一步信息，以及看到视频、截图和任何其他相关营销材料的URL。
  • 隐私政策URL。如果你的资产需要设备或用户数据，或者如果资产是为儿童设计的，那么你必须提供一个隐私政策的链接。

最后，你将被要求提供一些图片，以便在市场上正确地推销你的资产。一个图标（200x200 PNG ）和一个功能图（1140x360 PNG ），然后至少一个屏幕截图 (应该是 PNG ，但可以用以下尺寸制作。1280×720、1366×768、1600×900或1920×1080），最后你可以选择加入YouTube或Vimeo的视频。
