资产浏览器

游戏的核心将由添加到资产浏览器的资产创建，资产浏览器默认位于IDE右侧。在这里，您可以添加游戏运行所需的所有内容，包括游戏室，子画面，对象，路径和许多其他内容。 GameMaker Studio 2基本游戏将需要一个运行空间（新项目将始终使用已创建的空间资产来创建），并且通常至少要包含一个对象和一个精灵，尽管您可能会使用更多！

在资产浏览器中，您将在其中创建和组织游戏资产，并修改某些其他方面，例如平台特定的设置和配置。您可以在下面找到每个不同部分的说明。

搜索栏

您可以在“搜索栏”中输入一些文本，并使用其过滤以下主要资产列表中显示的内容。例如，键入“房间”将显示名称中包含“房间”的所有资产。要将资产列表重设为默认视图，只需从搜索栏中删除任何文本即可。

  • 最近的：在这里，您可以找到最近使用的资产的列表。默认情况下，此处最多列出10个资产，但是您可以从“资源浏览器首选项”中进行更改。
  • 收藏夹：将一些资产添加到资产列表后，您随时可以访问一些资产，为了使事情变得更容易，您可以将这些资产标记为“收藏夹”，这会将它们添加到此列表中并允许您很快就会找到它们。要将资产标记为收藏，请使用鼠标右键在它上面，然后从弹出菜单中选择“收藏夹”。请注意，您可以使用鼠标右键再次unfavourite任何资产，如果你使用的主要收藏夹文件夹按钮，您可以选择清除所有以删除所有内容。

  • 保存的过滤器：使用过滤选项时，您可能需要保存所做的选择以供将来参考。为此，请按照本页上的说明设置过滤器，然后单击鼠标右键。在此选项上，然后选择保存过滤器。过滤器将保存到此列表，您可以根据需要重命名。要使用过滤器，只需双击在它上面或使用鼠标右键菜单选项“应用过滤器” 。您也可以从人民币中删除和重命名保存的过滤器菜单。

  • 游戏选项：您可以在此处快速访问项目的各种不同游戏选项。这些控件控制着项目的各个不同方面，这些方面与针对不同目标平台的设置和编译有关（可用的游戏选项将取决于您拥有的许可证类型）。您可以从“游戏选项”部分的列表中找到有关每个选项的更多信息。

对象

对象是一种特殊资源，可用于控制游戏各方面并做特定的事情。大多数情况下，它们具有与之关联的精灵，以便您在游戏室中看到它们，但有时它们被用作“幕后”控制器，以执行与用户有关的事情或用于计时等。被赋予行为，它们可以对某些事件以及彼此发生反应，并且您在游戏中看到的大多数事物都是基于对象及其相互作用的。请注意，我们之所以说“基于”，是因为您实际上并未将对象直接放置到游戏室中，而是放置了这些对象的实例，这些实例基本上是对象资源的副本（或克隆，如果您愿意）。要记住这是非常重要的事情，因为实例和对象不是同一件事，并且每个实例都有可能影响它们的自己的一组功能。

所有对象都有一组可以在对象编辑器中指定的属性，例如精灵，它是否使用物理，还是它是另一个对象的“子”对象。它们还具有一系列内置变量，这些变量随后可用于动作，脚本和代码。这些内置变量用于定义实例放置在游戏室中时的位置，动画速度，移动方向以及其他所有内容。为了控制对象随时间的行为，每个游戏帧都分为一系列事件，因此您可以将代码或动作添加到特定事件中，并且只有在触发事件时它才会运行，并且事件可以由诸如鼠标按钮按下或与其他实例发生碰撞。

对象是游戏的基本组成部分，与房间一起将成为您创建的任何项目的核心。

人民币对象菜单包含以下选项（以及该页面上方列出的选项）：

  • 打开所有事件脚本-选择此选项将在新工作区中以脚本选项卡打开对象的所有分配事件。

您可以从“对象编辑器”部分中找到有关添加对象的更多信息。

路径

在创建游戏的某个时刻，您可能需要一个对象的实例才能沿着关卡路径前进。现在，只需在房间内创建一个位置数组，然后让实例在它们之间移动就可以完成此操作，但这可能很麻烦，并且很难更改，测试和适应不同的事物。那就是您要创建路径资源的时候。

路径资源的基本思想很简单-您可以通过在路径编辑器中绘制路径来定义路径，然后可以在对象的事件中放置一个动作（或代码），以告知对象遵循该特定路径。游戏厅。您还可以设置跟随路径的速度，以及与房间内路径的位置和方向有关的许多其他动作。

您可以从“路径编辑器”部分中找到有关添加路径的更多信息。

房间数

GameMaker Studio 2制作的所有游戏都需要至少一个房间来运行（但可以有更多房间），而一个房间基本上是一个空间，您可以在其中放置组成游戏对象的实例以及游戏的动作。游戏将进行。但是，“房间编辑器”是您可以使用的最强大的资源之一，因为它不仅使您可以添加对象的实例，还可以在游戏室中设置“视图”，并向其中添加背景和图块创建游戏世界，创建独立于对象/实例系统的额外特殊效果，以及独立于对象运行代码。还可以将房间设置为继承其他房间的属性，这意味着您可以创建一个带有多个瓷砖的房间，例如，然后再创建一个继承这些瓷砖的房间，这样就不必重新创建他们再次。

人民币房间菜单包含以下选项（以及此页面上方列出的菜单）：

  • 创建子房间-这将创建一个新房间，作为所选房间的子房间。子房间是从父房间继承属性，图层和图层内容的新房间资产，并将在“房间管理器”中显示为链接到父房间。

您可以从“房间编辑器”部分中找到有关添加房间的更多信息。

剧本

脚本资源只是您已编写的一个或多个函数的容器。例如，假设您想让所有敌人对象死后都创建多个不同的实例，例如一些金，一个爆炸，一些鲜血以及播放声音。现在，您可以将所有必需的代码添加到需要它的每个对象中，但是通常最好在脚本资源中创建一个函数，然后调用它。这意味着您只需在对象中添加一行代码即可调用此新的脚本函数，并且如果您希望更改某些内容，则只需在脚本中对其进行一次更改，所有对象将自动运行新代码，而不是必须遍历每个对象并在多个地方更改同一件事。

人民币脚本菜单包含以下选项（以及该页面上方列出的选项）：

  • 转换为DnD™/ GML这会将选定的脚本转换为DnD™或GML 。如果脚本是GML ，那么它将被放置在“执行代码”动作中，而DnD™将直接转换为GML代码（如将DnD™转换为代码部分所示）。

您可以从“脚本编辑器”部分中找到有关添加脚本的更多信息。

顺序

序列资产本质上是随时间执行动态动画的其他资源的集合。它们可以包含精灵，实例，声音和其他东西，并且可以通过将资产分配给序列中的特定音轨来将它们中的每一个设置为移动或更改颜色，或在序列本身内随时间开始/停止。然后，这些轨道可以具有不同的属性，这些属性可以是静态的，也可以随时间变化。序列的所有编辑都在“序列编辑器”中进行，该编辑器包括：

  • 画布-放置组成序列的资产的位置
  • 跟踪面板-您可以在其中放置资产以创建资产轨道，以及创建参数轨道以随时间推移操纵资产
  • Dope Sheet -您可以在其中添加/删除/编辑keyframes ，并控制它们随着时间的变化。

请注意，您可以在序列中包含序列以创建复杂的动画和效果，还可以通过代码来访问和更改序列，以尽可能多地控制它们在游戏中的显示和使用方式。

您可以从以下章节中找到有关添加序列的更多信息： 序列编辑器。

Shaders

Shaders是一种非常强大的工具，可用于处理游戏渲染到屏幕上的图形，从而实现令人难以置信的快速效果，例如，可以向精灵添加微妙的色相，直至全屏失真效果。基本上，它是一个由两部分组成的程序，直接在图形卡本身上运行，由于GPU完成了所有工作并为您的游戏代码释放了CPU周期，因此运行速度非常快。完整的shader由一个顶点shader程序和一个片段shader程序（也称为像素shader ）组成。这两个微型程序都可以一起工作，以操纵图形卡在屏幕上呈现的内容。然后，这允许您实时操纵实际渲染到显示缓冲区中的位置，颜色和alpha值。

声音

声音是任何游戏的重要组成部分，既可以增加游戏的深度并以声音效果形式提供反馈，也可以以音乐形式增加气氛。 GameMaker Studio 2接受 WAV， MP3 和 OGG格式化文件。

一般来说， WAV尽管这些文件通常是较大的文件，但由于它们不需要任何类型的解码来播放，因此即使它们通常是较大的文件，它们也可以立即播放，因此它们可用于任何短的声音效果。 MP3和 OGG文件应用于背景音乐或较长时间播放或具有较大文件大小的任何效果。这些文件比WAV文件小得多，但确实具有与之相关的CPU开销，因为在播放之前必须对其进行解码。

人民币声音菜单包含以下选项（以及本页上方列出的选项）：

  • 在混音器中打开-这将在混音器中打开所有选定的声音。
  • 分配音频组-选择此选项时，将显示可用音频组的列表，然后可以根据需要选择一个以添加声音或创建一个新的音频组。如果在资源浏览器中选择了多种声音-或您选择了一个声音组文件夹-那么所有声音将被添加到所选的音频组中。有关音频组的更多信息，请参见此处。

您可以从“声音编辑器”部分中找到有关添加声音的更多信息。

平铺

拼贴集是从Sprite资源中获取的，但由于GameMaker Studio 2在创建游戏时会以不同的方式处理它们，因此它们被分类为单独的资源。基本上，图块集是单个图像， GameMaker Studio 2会根据您为各种设置提供的值将其分成单独的部分。然后，您可以在房间编辑器中使用它们（或通过代码通过程序）在您的房间中生成平铺地图。它们非常适合设计房间中的任何静态物品，例如地形，墙壁，背景墙等，因为它们没有对象所具有的开销。

时间线

在大多数游戏中，您会希望某些事情在特定的时间发生。现在，您可以尝试通过在实例中使用“警报事件”来实现此目的，但是当事情变得太复杂时，它将不再起作用，尤其是由于您仅限于十二个警报。这就是为什么我们拥有时间轴资源。在时间轴中，您可以指定必须在游戏时间的任何特定时刻执行哪些动作，并且可以在对象的不同事件和代码中使用可用于该对象的所有动作。一旦创建了时间线，便可以将其分配给一个对象，然后将该对象的实例放置或创建在房间中的指定时间执行操作和代码。

您可以从“时间轴编辑器”部分中找到有关添加时间轴的更多信息。
