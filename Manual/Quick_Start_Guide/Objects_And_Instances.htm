对象和实例

使用到目前为止所看到的资产，您可以在游戏资产浏览器中添加一些精美的图像和声音，但是它们在游戏本身中还没有做任何事情！为此，我们需要GameMaker Studio 2最重要的资产，即对象资产。对象是一种特殊的资产，我们可以用来控制游戏的各个方面并做特定的事情。大多数情况下，它们具有与之关联的精灵，以便您可以在游戏室中看到它们，但有时它们被用作“幕后”控制器对象，以执行诸如跟踪统计或设置场景切换定时之类的事情。等等。……可以赋予对象行为，它们可以对某些事件以及彼此之间做出反应，并且您在游戏中看到的大多数事物都是基于对象及其与用户和彼此之间的交互作用的。

现在，我们不能不谈论实例而谈论对象。在使用GameMaker Studio 2制作的游戏中，您在游戏中看到的角色，怪物，球，墙壁等实际上都被视为实例。实例实质上是在资产浏览器中找到然后放置在房间中的对象的“副本”（将对象视为实例的模板）。因此，当我们谈论影响或更改实例的事物时，是指房间中某个对象的一个特定副本受到影响，而其余所有事物都没有受到影响，但是当我们谈论影响或更改对象时，我们的意思是我们所做的一切从那时起创建的所有实例中也将反映到它。因此，对象是实例的模板，实例是我们放置在房间中进行游戏的对象。

要创建一个新的对象资产，就和其他所有资产一样，您只需右键单击在“对象”资产文件夹中，选择“创建” （或右键单击“资源浏览器”中的任意位置，然后选择“创建对象” ）。

这是“对象编辑器”窗口，其中显示了当前的对象属性，您要做的第一件事是为对象命名，再次使用诸如“ obj_ ”或其他一些标识符。通常，对象要绘制任何东西并注册与其他对象实例的碰撞，则需要为其分配一个Sprite，并使用按钮。这将打开资产浏览器，您可以在其中选择要使用的精灵（也可以从资产浏览器中拖动精灵并将其放到“对象编辑器”中进行分配）：

如果您还没有合适的精灵，可以单击在“新精灵”按钮上要创建新的Sprite资产并根据需要进行编辑，就像通过标准Sprite创建选项创建资产一样，只是现在，它将自动分配给开始创建资产时已打开的对象。选择精灵后，您还可以点击在编辑精灵上或编辑图像按钮可编辑精灵本身或精灵资产的图像。这可能比先在“资源浏览器”中找到精灵然后指示要编辑它的速度要快。

一旦添加或创建了精灵，您就可以更改其他一些对象属性，例如是否应使其可见（例如，它是否应具有可见的碰撞蒙版） 。对于简单的游戏，您可能无需在此处进行任何更改，现在可以继续向对象添加事件。

事件是游戏循环中谨慎的时刻，根据您为它们编写的内容使事件发生。 GameMaker Studio 2处理这些事件的周期-从房间开始到完成为止，都有一个游戏循环正在运行，其中运行一系列事件的每一步，您可以选择放置GML代码或DnD™您实例中对这些事件做出响应的操作（一步是游戏时间，由游戏速度设置控制-也称为“帧”）。

建立活动此事件在首次创建对象的实例时发生，并且是进入房间时通过房间编辑器放置在房间中的实例中发生的第一件事。这意味着此事件是初始化变量，启动时间轴，设置路径等...以及执行通常仅需要执行一次或仅在实例首次出现在房间中时执行的其他所有操作的理想场所。
步骤事件GameMaker Studio 2将游戏时间分为几步，游戏速度定义了每秒应该执行的步骤数。基本上，一个步骤是不断运行的循环，并在游戏运行时根据需要检查并触发所有事件，因此，可以想象，步骤事件是在实例存在时游戏的每个步骤都被检查的事件。请注意，“步骤事件”类别具有三个子类别，以“微调”实例中的计时，但通常，您将最常使用主“步骤事件”。
碰撞事件显然，制作游戏时，知道两个（或更多）对象实例何时发生碰撞以及发生碰撞事件非常重要。这是一个事件，您将其放置在一个对象中，然后指定应针对哪个其他对象进行碰撞检查。添加事件并选择要检查的对象后，就可以添加GML代码或DnD™来解决冲突。
抽奖活动在绘画事件类别中有多个不同的绘画事件，但是您最常使用的一个是最上面的一个，简称为“绘画”。每个游戏框架都将调用此通用抽奖事件，就像step事件一样，但是它以两种不同的方式起作用。第一种是您不将此事件添加到对象时...当没有指定的Draw事件时， GameMaker Studio 2将默认使用已在对象中其他位置应用的任何变换绘制分配给对象的精灵。对象（例如更改其大小或Alpha混合），因此，即使“事件”列表中未列出“绘制事件”，该对象仍在运行，并且当实例放置在房间中时仍会绘制到屏幕上。工作的第二种方式是添加事件本身并在其中包含任何GML或DnD™。向事件中添加一些内容会告诉GameMaker Studio 2 “我想控制您为该对象实例绘制的内容”，它完全覆盖了默认绘制。这意味着您可以为对象分配一个Sprite，然后将draw事件设置为绘制文本GameMaker Studio 2绘制文本，但不会绘制Sprite，尽管文本是可以绘制的。
破坏事件此事件是销毁实例时要执行的最后一个事件。在向对象添加行为时，通常会忽略它，但是它可能非常有用-例如，用于在杀死敌人时创建爆炸或粒子效果，或者用于在房间的另一部分重新生成对象的新实例，甚至用于将分数添加到乐谱中。
