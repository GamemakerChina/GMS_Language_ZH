快速入门指南

欢迎使用快速入门指南！本手册的这一部分旨在使您熟悉GameMaker Studio 2的最重要方面，并向您展示使您尽快制作游戏的基础知识。该指南涵盖了基本的IDE功能，还简要介绍了编程，因此无论您使用GameMaker Language （ GML ）还是可视脚本语言Drag and Drop™，您都可以尽可能快地充分利用产品。

为了说明GameMaker Studio 2工作流程并让您直接制作东西，我们准备了本指南，以带您了解GameMaker Studio 2和程序的基本知识，包括创建一般资产以及如何进行开发。编码运动和得分等基本事物。

从概念上讲，使用GameMaker Studio 2游戏是一件相当简单明了的事情，遵循以下一般步骤：

  • 创建图像资源（精灵，瓦套，字体S），并将它们添加到资源浏览器。
  • 创建对象资产以表示您要在游戏中使用的事物，然后为它们分配所需的图像资产。
  • 对这些对象进行编程以响应特定事件。
  • 将这些对象的实例添加到将要执行程序的房间中。

我们将在以下页面中详细介绍上述各项的详细信息，但这是制作游戏所需的一般操作流程。这里要注意的重要一点是，尽管有许多辅助资产也可以用来为这些项目添加额外的功能或行为，但游戏的主要“构造块”将始终是精灵，物体和房间。 shaders或时间轴）。

现在，在我们继续详细解释GameMaker Studio 2如何将上述构建基块转变成您的游戏。基本上，所有事情都发生在所谓的“游戏循环”中，该循环的速度以每秒帧数为单位，而一个“帧”在GameMaker称为Step 。默认的每秒帧数（或FPS ）为60，这意味着每隔一秒钟， GameMaker Studio 2都会运行游戏循环60次，并且每一步（循环，帧）都会发生许多事件，并且游戏会继续进行。

为了帮助理解这一点，让我们假设您有一个想要在游戏室中移动的角色。您可以创建一个精灵该字符，然后创建一个对象，并为其分配的精灵，和对象，你要补充称步骤事件的事件。在这种情况下，您会告诉它向右移动2个像素。然后，您可以将该对象的一个实例放置到游戏室中，然后在运行游戏时，该实例将开始每步向左移动2个像素（“步骤事件”每个游戏步运行一次），因此该实例在1秒钟内向右移动了120像素。
