从概念上讲，用GameMaker Studio 2制作游戏是一件相当简单和直接的事情，遵循以下一般步骤。

  • 创建图像 资产（精灵、瓦片组、字体 并将它们添加到资产浏览器中。
  • 创建对象 资产来代表你想在游戏中使用的东西，并为它们分配所需的图像资产。
  • 对这些对象进行编程，使其对特定事件做出反应。
  • 将这些对象的实例 添加到一个房间 里，在那里它们将执行它们的程序。

我们将在下面几页中详细介绍上述每个项目的具体内容，但这就是制作游戏所需的一般行动流程。这里需要注意的是，游戏的主要 "构件 "总是精灵、物体和房间。 始终是精灵、物体和房间，尽管有一些附属资产也可以用来为这些项目添加额外的功能或行为（如 着色器, 或 时间表).

现在，在我们继续更详细地解释这些事情之前，我们将快速概述GameMaker Studio 2将如何把上述构件变成你的游戏。基本上，一切都发生在所谓的游戏循环中。 它的速度以每秒 帧数计算，一个 "帧 "在GameMaker 中被称为一个步骤。默认的每秒帧数（或FPS）是60，这意味着每过一秒，GameMaker Studio 2将运行游戏循环60次，每一步（循环，帧）都会发生一些事件，游戏也会有进展。

为了帮助理解这一点，让我们想象一下，你有一个你想在游戏室中移动的角色。你将为该角色创建一个精灵 ，然后创建一个对象 并将该精灵分配给它，在该对象中你将添加 一个叫做 "步骤事件"的事件 。在这个事件中，你会告诉它向右移动2个像素。然后你将该对象的一个实例 放到游戏室中，当你运行游戏时，该实例将 开始每一步向左移动2个像素（Step事件每游戏一步运行一次），因此在1秒钟内，该实例向右移动了120个像素。
