路径首选项

路径GameMaker Studio 2 IDE中的元素以及所需的任何外部编辑器所需的不同文件路径。 IDE存在以下选项：

  • 临时目录：保存所有临时文件夹的位置。在Windows默认为

    \Users\<Username>\AppData\Local\Temp\GameMakerStudio2

    在macOS上是：

    /var/folders/<Path_Hash>/GameMakerStudio2

  • IDE缓存目录：用于保存IDE缓存的位置。 Windows默认情况下是

    \Users\<Username>\AppData\Roaming\GameMakerStudio2\Cache

    在macOS上是：

    /Users/<Username>/.config/GameMakerStudio2/Cache

  • 资产缓存目录：用于保存每个项目的资产缓存的位置。在Windows默认为

    \Users\<Username>\AppData\Roaming\GameMakerStudio2\Cache

    在macOS上是：

    /Users/<Username>/.config/GameMakerStudio2/Cache

  • “我的项目”位置： GameMaker Studio 2最初用于创建新项目的位置。在Windows默认为

    \Users\<Username>\Documents\GameMakerStudio2

    在macOS上

    /Users/<Username>/GameMakerStudio2

  • 关闭时自动删除临时目录：选中此选项将迫使GameMaker Studio 2自动删除它为每个项目创建的用于编译等的Temp文件夹。默认情况下，此设置为启用状态；取消选中该选项将关闭该设置（这意味着您将不得不在以后手动删除所有临时文件）。
  • 关闭时自动删除资产缓存GameMaker Studio 2时将删除资产编译器缓存文件夹。默认情况下，此功能是关闭的，启用它意味着每次加载和运行任何项目时，都需要重新构建缓存（这可能需要一些时间，具体取决于游戏的大小）。
  • 关闭时自动删除IDE缓存GameMaker Studio 2时将删除IDE编译器缓存文件夹。 GameMaker关闭状态，启用它意味着每次启动GameMaker Studio 2时，都需要重建缓存。
  • 删除临时文件夹：单击此按钮将删除项目的临时文件夹。
  • 删除资产缓存：单击此按钮将删除项目的编译器资产缓存。
  • 删除IDE缓存：单击此按钮将删除IDE缓存

存在以下用于设置外部编辑器路径的选项：

  • SWF文件的外部编辑器/查看器的路径：如果您正在使用SWF格式的精灵，则可以将其设置为首选查看器/编辑器的路径，然后在Sprite Editor中单击“编辑图像”按钮时，它将打开给定的程序。此处的默认值是没有路径。
  • Spine文件的外部编辑器/查看器的路径：如果您正在使用Spine格式的精灵，则可以将其设置为首选查看器/编辑器的路径，并在Sprite Editor中单击“编辑图像”按钮时，它将打开给定的程序。此处的默认值是没有路径。
