运行时喂食偏好

运行时馈送首选项用于显示和改变当前安装的 运行时间以及当前使用的运行时。GameMaker Studio 2在任何时候都可以安装几个不同的运行时工具，你可以通过这个偏好设置使用哪一个。理想情况下，你希望 使用最新的运行时，但有时它可能有问题，或者你可能需要使用较旧的运行时来进行比较，等等......因此你可以在这里改变GameMaker Studio 2用来构建游戏的运行时工具。

这个偏好的最初部分与你从哪里获取运行时工具有关，需要你输入一个RSSfeed的URL。 然后，这个feed将被扫描以获得运行时工具的更新，然后可以下载和安装。本节将默认显示主运行时馈送的URL，但你可以点击添加 按钮来创建一个新的运行时馈送，在这里你可以给运行时工具的名称和URL。 你可以给RSS feed的名称和URL，然后点击应用，告诉GameMaker Studio 2访问它。你需要关闭并重新打开首选项窗口，以显示新的饲料。

一旦创建了一个运行时馈送，它将作为一个子类别列在本节中，你应该总是有至少一个馈送被列出（默认情况下，这将是主 馈送）。无论使用哪种运行时，它们都将有 相同的功能，下面以主馈送为例概述。

RuntimeFeed首选项用于选择、安装和取消安装不同的运行时工具。在 顶端有当前正在使用的运行时工具，它的创建日期，以及它在硬盘上的存储位置。在这些细节下面，你可以找到一个所有可用运行时的列表，你可以从这个列表中选择任何一个 列表中的任何一个，以改变为用于编译项目的 "运行 "的运行时。

这个列表详细列出了所有可用的运行时，它们的版本号，发布日期，以及--如果它们被安装了--它们被安装到哪里的路径。右手边还显示了不同的状态图标，如下所示。
