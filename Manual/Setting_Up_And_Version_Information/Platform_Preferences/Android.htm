Android偏好设定

Android首选项分为以下几个部分：

在首选项的顶部，您可以设置JDK内存堆的最大大小（默认为2GB）。接下来，您必须设置各种不同的路径，以便GameMaker Studio 2知道在哪里寻找构建最终可执行程序包和测试游戏所需的各种工具。 Android SDK ， Android NDK和Java JDK ，这分为三个部分。这些路径应该已经为您自动填写，但是如果您有任何问题，可以单击右侧的按钮打开文件浏览器，并手动选择每个元素的位置。您可以在此处找到有关每个SDK元素所需的版本的详细信息。应用路径时，应先对其进行验证，然后消息将显示“ Found ”（发现）或告诉您任何错误（在这种情况下，您应修改所有信息以确保其正确无误）。

在其下方，您可以使用以下不同的打包选项：

  • 始终完整安装APK ：标记此选项将使GameMaker Studio 2从设备中删除游戏的所有先前运行程序以及所有相关文件（例如， *.ini文件），然后再安装新版本的游戏。
  • 安装在软件包上：标记此选项时，编译最终可执行文件 *.apk软件包还将自动将其安装在当前选定的目标设备上。

最后，我们来到KeyStore 。此文件将用于“签名”您所有的Android应用程序，因此请正确填写所有详细信息，并请记住，将来在创建和更新所有Android应用程序时都需要此文件。建议将所用设置的屏幕截图并备份最终文件。如果您丢失了此文件，则将无法更新已上传到商店的任何现有游戏。

要创建新的密钥库文件，需要以下详细信息：

  • Filename ：这是密钥库文件的名称。

注意：这不是您的名字或公司名称，而是要生成的文件的名称！如果您已经拥有以前项目中的密钥库文件，则可以单击此处的文件浏览器按钮，然后将GameMaker Studio 2到先前创建的密钥库文件（或单击“导入”按钮）。在这种情况下，您仍然必须填写其余的必需信息（与第一次生成文件时的信息完全一样），但是不能单击“ Generate Keystore”按钮！这将生成一个新的密钥库文件，覆盖先前的文件。

  • 通用名称：通常是您的名字。
  • 密码：密钥库文件的安全密码，必须至少六个字符。
  • 别名：这是可以用于密钥库文件的另一个名称，应该与上面输入的名称不同。
  • 别名密码：密钥库文件的别名安全密码，必须至少六个字符。
  • 组织单位：您所属公司的部门。
  • 机构：您公司的名称。
  • 位置：您所在的城镇名称。
  • 国家（地区） ：您所在国家/地区的标准两个字母代码。

填写完此信息后，您会感到满意，请按标记为GameMaker Studio 2生成密钥库的按钮以创建必要的密钥库文件。这可能需要一些时间，但是一旦完成，通常无需再次更改。请注意，您可以单击“导入”按钮导入以前创建的密钥库文件（请参见上面的“文件名”部分），还可以通过单击“生成密钥哈希”按钮来检索密钥库SHA1哈希。按下此键将生成两个特殊的安全哈希密钥，这些哈希密钥直接链接到某些网站需要正确功能的密钥库文件（例如，Facebook或Google Play的Sha1哈希）。如果需要此功能，只需按一下按钮，然后复制并粘贴生成的哈希键。
