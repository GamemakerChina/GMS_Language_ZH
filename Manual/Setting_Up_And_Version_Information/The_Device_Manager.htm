设备管理器

通过单击访问设备管理器在铅笔按钮上在目标列表中：

单击此按钮将打开“设备管理器” ，您可以在其中为要编译和测试的目标平台配置不同的设备（请注意，可用的平台将取决于您拥有的许可证，并且并非所有的平台都可见）。您可以在下面找到有关为每个目标平台配置设备的详细信息：

Android

在这里，您可以单击“添加新设备”按钮以手动将新的Android设备添加到列表中，也可以单击“检测设备”按钮以使GameMaker Studio 2自动检测通过有线连接连接的任何设备。您还可以通过单击标记为Run AVD Android虚拟设备。这将打开Android虚拟设备管理器，您可以在其中创建新的AVD（有关更多信息，请参见此处）。创建完成后，您需要从AVD管理器中启动它，然后在GameMaker Studio 2单击“检测设备”按钮，它应自动添加到设备列表中：

添加设备（物理或虚拟）后，可以使用“测试连接”按钮检查连接，或通过单击“删除设备”按钮再次将其从列表中删除。


HTML5

HTML5设备管理器用于将不同的浏览器添加到设备列表中，以在运行时定位。首先，应单击“检测浏览器”按钮，该按钮应为计算机上每个已安装的浏览器创建设备。如果未列出您的浏览器，也可以单击“添加新浏览器”按钮，然后为设备命名，并将GameMaker Studio 2指向浏览器可执行文件的路径。

可以设置其他可用字段，以帮助控制浏览器如何运行项目，并提供以下选项：

  • 浏览器命令行参数：在这里，您可以添加任何命令行参数以附加到浏览器路径。您可以从以下链接找到主要的流行浏览器的列表：
    • 铬（铬）
    • 火狐浏览器
    • IE浏览器
  • Web服务器地址：要定位的Web服务器的地址（默认值为127.0.0.1）。您可能需要根据不同API的规范进行更改，例如，Facebook要求将其设置为 http://localhost（然后在其应用程序控制台上将其列入白名单）。
  • Web服务器端口：设置Web服务器的端口号（默认为51264）。
  • 允许的Web服务器IP / CIDR IP范围：根据需要设置允许的Web服务器，主机名和CIDR IP范围。

iOS

必须先设置Mac设备并测试其连接，然后才能设置iOS设备（有关详细信息，请参见下文）。完成此操作后，您可以从窗口顶部的下拉菜单中选择要定位的Mac，然后单击“检测设备”按钮以使GameMaker Studio 2自动查找任何已连接的iOS设备，包括模拟器。连接后，设备详细信息将显示在此窗口的列表中，您可以编辑设备名称以使其更易于使用（此更改仅影响设备在目标管理器中的显示方式，而不会影响其他任何方式）。


tvOS

必须先设置Mac设备并测试其连接，然后才能设置tvOS设备（有关详细信息，请参见下文）。完成此操作后，您可以从窗口顶部的下拉菜单中选择要定位的Mac，然后单击“检测设备”按钮以使GameMaker Studio 2自动查找任何已连接的tvOS设备，包括模拟器。连接后，设备详细信息将显示在此窗口的列表中，您可以编辑设备名称以使其更易于使用（此更改仅影响设备在目标管理器中的显示方式，而不会影响其他任何方式）。


macOS

要将Mac设备添加到管理器，必须首先单击标有“添加新设备”的按钮，该按钮将添加设备并创建一些空白字段，您需要填写这些字段以启用连接。

首先，您需要命名设备，以便可以在目标列表中对其进行识别，然后提供Mac的主机名（或IP）。然后，您必须提供要链接的用户的用户名，以及他们使用的密码。如果愿意，还可以将Mac上的“安装文件夹” GameMaker Studio 2存储项目的生成文件的任何位置。

完成所有操作后，您可以单击“测试连接”按钮以确保GameMaker Studio 2可以与Mac通信。

注意：如果要为iOS开发，则必须先完成此步骤，并已建立与要用于开发的Mac的有效连接。

注意：您必须在目标Mac上启用了远程登录。这可以通过以下方式完成：转到您要登录的用户的“首选项” ->“共享并启用远程登录”。

Ubuntu （ Linux ）

要将Ubuntu （ Linux ）设备添加到管理器，必须首先单击标有“添加新设备”的按钮，该按钮将添加设备并创建一些空白字段，您需要填写这些字段以启用连接。

首先，需要命名设备，以便可以在目标列表中对其进行标识，然后提供Ubuntu计算机（或VM）的主机名（或IP）。然后，您必须提供要链接的用户的用户名，以及他们使用的密码。您还需要在Ubuntu计算机上提供要用于安装项目的安装文件夹。完成后，您可以单击“测试连接”按钮以确保GameMaker Studio 2可以与Ubuntu计算机通信。

GameMaker创建的最终可执行文件将取决于目标设备的体系结构，可以是x64或ARM 。例如，如果连接Raspberry Pi 3或4，则创建的可执行文件将是32位ARM可执行文件，而如果连接Linux PC，则将是64位x64可执行文件。

Windows UWP

必须先启用开发人员模式并在设备上设置设备门户，然后才能为Windows 10如果您不这样做， GameMaker Studio 2将无法与设备通信，也无法正确创建您的UWP应用。

启用开发人员模式的最简单方法是在Windows搜索中键入“开发人员模式”，然后单击将显示的“设置”条目。这将打开以下窗口：

您需要选择“开发人员模式”选项（可能要求Windows下载并安装一些其他软件包），然后选择其下方的“启用设备门户”选项。设备门户是允许GameMaker Studio 2 Windows 10 PC进行通信的门户，启用后，将要求您提供用户名和密码：

请注意，用户名和密码对于门户来说应该是唯一的，并且不能是您的Microsoft帐户或其他Windows凭据所使用的用户名和密码。提供此信息后，单击标为Apply的按钮，然后关闭设置管理器并返回GameMaker Studio 2 。

有关设备门户的更多信息，请参见此Microsoft文章：桌面设备门户。

现在，您可以在GameMaker Studio 2打开“设备管理器”并设置要使用的设备：

要将Windows 10 UWP兼容设备添加到管理器，必须首先单击标有“添加新设备”的按钮，该按钮将添加设备并创建一些空白字段，您需要填写这些字段以启用连接。

首先，您需要命名设备，以便可以在目标列表中对其进行标识，然后为主机提供名称-这是用于承载游戏的网络上Windows 10设备的IP地址，并且必须后缀“ :50443设置端口号。之后，添加您在设备门户中设置的用户名和密码（如前所述）。您可以将PIN留空，因为UWP Windows 10 Desktop不需要。

有关涵盖不同UWP输出（台式机，XboxOne）的更具体的设置详细信息，请参见YYG帮助台：设置UWP 。

所有不同的设备窗口在底部都有一个“清除设备”选项，当单击该选项时，将从列表中删除所有已连接的设备。
