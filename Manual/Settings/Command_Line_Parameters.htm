命令行参数

在创建游戏时，重要的是你要对可执行文件进行播放测试，以确保最终的编译是正确的，没有不可预见的错误或差异。然而，创建一个可执行文件意味着，由于你不能进入调试控制台，也不能进入编译器窗口。 编译器窗口，因此很难确定和调试你可能发现的任何问题。然而，有一个解决办法，那就是在运行可执行版本的游戏时，使用命令行参数来设置某些属性。 游戏运行时，使用命令行参数来设置某些属性，从而允许你创建输出日志文件并测试其他特殊情况。

注意：这只适用于标准 窗口 目标模块上可用。

为了让它工作，你必须首先创建一个你的游戏的Windows可执行文件，并将其安装在你的测试机上。一旦你完成了这一工作，你需要在桌面上创建一个游戏的快捷方式 。在这里我们将添加参数 传递给 *.exe （要创建一个快捷方式，右击 游戏可执行文件，然后选择发送至> 桌面（创建快捷方式））。

一旦你做了你的快捷方式，你就可以在目标路径中使用以下任何参数。

-noaudio ：这将关闭你游戏中的所有音频。

-inawindow: 强制在窗口中启动游戏，即使设置为全屏模式运行。

-output <filename>: 发送控制台输出到文件名，包括你的代码库中的任何调试信息。

-debugoutput <filename>: 发送控制台输出到文件名，不包括任何自定义调试信息，但包括来自运行器的额外信息，用于报告错误。

-software ：将强制游戏使用软件顶点处理而不是硬件来渲染游戏图形。这对那些使用板载英特尔GFX芯片组的电脑特别有用，或者在旧机器上显示出的问题 在旧机器上显示问题。

-intel ：这将开启对英特尔坏驱动的修复，无论使用的是什么GPU。对于那些坏的驱动程序，这种工作方法会减慢速度，给你的游戏带来明显的性能冲击，所以如果你不需 需要这个修复，请不要使用它。

-vanillaGFX ：使用这个功能可以关闭对驱动程序制造商的任何检查，对于测试你的游戏 "原样 "非常有用。

值得注意的是，如果需要， -output 和 -debugoutput 中的 <filename> 可以是同一个，因为文件在每次写入时都会被打开和关闭。

你现在应该去你创建游戏可执行文件快捷方式的地方，然后右击它，选择属性。

这将打开一个与上面类似的窗口。在这里，你可以在目标路径中添加你的命令行。只要沿着 到给定路径的末尾，添加一个空格，然后是参数字符串。因此，举例来说，要创建一个调试文件的输出，你需要这样做。

D:\Users\MeGameMaker\CustomLoadBar-Default-1.0.0.0\CustomLoadBar.exe -debugoutput debug.txt

现在，这将在与可执行文件相同的位置（不是快捷方式的位置，而是游戏安装的位置）创建一个名为"debug.txt"的文件，其中包含你完成的游戏的调试和编译信息。点击 "确定"，然后用这个快捷方式运行你的游戏，让它使用这些参数并帮助你进行调试。

关于如何调试你的游戏的进一步信息，请看手册中关于调试的部分。

使用命令行进行A/B测试

YoYo运行器还得益于一个特殊的命令行参数。

-game <filename> ：加载给定的游戏文件

你为什么要使用这个？好吧，通常你不会，但如果你希望你的游戏同时有一个以上的版本在运行，就可以用这个方法来完成，使A/B比较测试更容易。用这种方法，你可以和别人一起集中测试 或者，作为设计者，通过做并排测试来获得更好的感觉。

要设置这个，你需要事先知道几件事。首先，你将需要Runner本身的路径。这可以在你的Windows操作系统的程序数据文件夹中找到（这可能在默认情况下是隐藏的，所以你需要改变查看属性才能看到它），典型的格式如下 这可以在你的Windows操作系统的程序数据文件夹中找到（默认情况下可能是隐藏的，所以你需要改变查看属性才能看到它），典型的格式是这样的。

C:\ProgramData\GameMakerStudio2\Cache\runtimes\<runtime_version>\windows\Runner.exe

注意，runtime_version文件夹将取决于你目前所安装的运行时。你还需要 <yourgame>.win 文件的路径，因为这就是我们要指向运行器的地方。 找到这个的最简单方法是进入Temp 文件夹（通常位于 %localappdata%\Temp\GameMakerStudio\<game_name>\），然后搜索 *.win ，因为这将会显示所有可用的文件，你可以在这个文件夹中找到。 显示所有可用的文件夹，你可以简单地选择你需要获得完整路径的那个。

现在，我们可以像对可执行文件那样创建一个运行程序的快捷方式，然后根据需要改变参数，但这意味着必须在测试之间打开并编辑它，这违背了我们使用这一功能进行并排测试的目的。 测试的目的。相反，我们将创建几个小的 "批处理 "文件来处理这个问题。

要做到这一点，首先在你的桌面上创建一个新的文本文件，并在记事本（或任何其他文本编辑器）中打开它。添加以下内容。

开始 C:\ProgramData\GameMakerStudio2\Cache\runtimes\<runtime_version>\windows\Runner.exe -game <LOCAL TEMP PATH>\<GAMENAME>.win

你需要设置临时路径指向你的游戏 *.win 文件，然后将其保存为 "GM_Test_1.bat" 到你的桌面上（注意记事本会自动保存为 *.txt 文件，除非你从保存对话选项中选择 "所有文件"）。你的最终命令应该是这样的一个例子。

START C:\ProgramData\GameMakerStudio2\Cache\runtimes\runtime-9.9.1.199\windows\Runner.exe -game D:\Users\MarkAppData\Local\Temp\GameMakerStudio2\AlphaDog_Reboot\AlphaDog_Reboot.win

现在，创建另一个文本文件并添加同样的行，只是这次你把它指向你想做并排测试的第二个游戏版本的路径，你会把它保存为"GM_Test_2.bat"。 你可以双击 ，以显示你的游戏的不同版本，并且 把它们作为批处理文件意味着你可以很容易地再次打开它们来编辑它们并改变正在测试的游戏。
