命令行参数

在创建游戏时，请务必对可执行文件进行测试，以确保最终编译正确且没有不可预见的错误或差异，这一点很重要。但是，创建可执行文件意味着，由于您无权访问调试控制台或编译器窗口，因此很难确定并调试可能发现的任何问题。但是，有一个解决办法，那就是在运行游戏的可执行版本时，使用命令行参数设置某些属性，从而允许您创建输出日志文件并测试其他特殊情况。

注：这是可用的标准上Windows目标只有模块。

要使其正常工作，您必须首先创建Windows可执行文件并将其安装在测试计算机上。完成此操作后，您需要在桌面上为游戏创建快捷方式。在这里我们将添加要传递给 *.exe（要创建快捷方式，请右键单击游戏可执行文件，然后选择发送到>桌面（创建快捷方式） ）。

设置快捷方式后，即可在目标路径中使用以下任何参数：

-noaudio ：这将关闭游戏中的所有音频。

-inawindow ：即使将游戏设置为以全屏模式运行，也可以强制游戏在窗口中启动。

-output <filename> ：将控制台输出发送到文件名，包括代码库中包含的所有调试消息。

-debugoutput <filename> ：将控制台输出发送到文件名，不包括任何自定义调试消息，但包括来自运行程序的其他信息，用于错误报告。

-software ：将强制游戏使用软件顶点处理而不是硬件来渲染游戏图形。这对于使用板载Intel GFX芯片组或在较旧计算机上显示问题的PC尤其有用。

-intel ：无论使用什么GPU，这都将打开针对错误的Intel驱动程序的修复程序。解决这些不良驱动程序的问题会使速度变慢，从而给游戏带来明显的性能下降，因此，如果不需要此修复程序，请不要使用它。

-vanillaGFX ：使用此选项将关闭对驱动程序制造商的所有检查，对于“按原样”测试游戏非常有用。

值得注意的是 <filename>在 -output和 -debugoutput如果需要的话，它可以是相同的，因为每次写入时都会打开和关闭文件。

现在，您应该转到创建游戏可执行文件快捷方式的位置，然后右键单击它并选择“属性”。

这将打开与上面类似的窗口。在这里，您可以将命令行添加到目标路径中。只需滚动到给定路径的末尾，添加一个空格，然后添加参数字符串。因此，例如，要创建调试文件输出，您将具有以下内容：

D:\Users\Me\GameMaker\CustomLoadBar-Default-1.0.0.0\CustomLoadBar.exe -debugoutput debug.txt

现在，这将创建一个名为“debug.txt “与可执行文件位于同一位置（不是快捷方式位置，而是游戏的安装位置），其中包含来自完成游戏的调试和编译器信息。单击“确定”，然后使用快捷方式运行游戏以使其使用这些参数并帮助您进行调试。

有关如何调试游戏的更多信息，请参见“调试”手册中的章节。

使用命令行进行A / B测试

YoYo跑者还受益于特殊的命令行参数：

-game <filename> ：加载给定的游戏文件

你为什么要用这个？好吧，通常您不会这样做，但是如果您希望一次运行一个以上的游戏版本，则可以使用此版本来完成，这使得A / B比较测试变得更加容易。使用这种方法，您可以将测试重点放在其他人身上，或者作为设计师，通过并行测试来更好地了解事物。

要进行此设置，您需要事先了解几件事。首先，您将需要转轮本身的路径。可以在Windows操作系统的Program Data文件夹中找到它（默认情况下可能是隐藏的，因此您需要更改查看属性才能看到它），其格式如下：

C:\ProgramData\GameMakerStudio2\Cache\runtimes\<runtime_version>\windows\Runner.exe

请注意， runtime _version文件夹将取决于您当前安装runtimes您还需要通向 <yourgame>.win文件，因为这就是我们要向跑步者指出的内容。最简单的方法是转到Temp文件夹（通常位于 %localappdata%\Temp\GameMakerStudio \ <game_name> \），然后搜索 *.win，这将向您显示所有可用的内容，您只需选择所需的内容即可获得完整的路径。

现在，我们可以像创建可执行文件一样为跑步者创建快捷方式，然后根据需要更改参数，但这意味着必须在测试之间打开和编辑它，这使我们无法同时使用此功能方面的测试。相反，我们将创建几个微小的“批处理”文件来处理此问题。

为此，首先在桌面上创建一个新的文本文档，然后在记事本（或任何其他文本编辑器）中打开它。添加以下内容：

START C:\ProgramData\GameMakerStudio2\Cache\runtimes\<runtime_version>\windows\Runner.exe -game <LOCAL TEMP PATH>\<GAMENAME>.win

您需要设置临时路径以指向您的游戏 *.win文件，然后将其另存为“GM_Test_1.bat ”添加到您的桌面（请注意，记事本会自动另存为 *.txt除非您从“保存”对话选项中选择“所有文件”）。最终命令的外观示例如下：

START C:\ProgramData\GameMakerStudio2\Cache\runtimes\runtime-9.9.1.199\windows\Runner.exe -game D:\Users\Mark\AppData\Local\Temp\GameMakerStudio2\AlphaDog_Reboot\AlphaDog_Reboot.win

现在，创建另一个文本文件并添加同一行，只是这次将其指向要进行并行测试的游戏第二版的路径，然后将其保存为“GM_Test_2.bat “。您可以双击它们中的任何一个都可以显示您游戏的不同版本，并将它们作为批处理文件表示您可以轻松地再次打开它们以进行编辑并更改要测试的游戏。
