游戏选项

游戏选项分为两个部分，主要选项和平台设置。主要选项从常规选项开始，在常规选项中，您可以设置某些会影响游戏的内容，而与设计运行平台的平台无关。

该窗口包括以下部分：

  • 使用DnD™作为默认脚本类型：选中此选项时，将使用DnD™编辑器而不是代码编辑器打开所有新脚本和对象事件。
  • 每秒游戏帧数：此值设置了GameMaker Studio 2在运行时应尝试维护的每秒帧数的限制。该值表示游戏循环在一秒内运行的次数（单个游戏循环被视为单个“帧”），并且设置为默认值60，尽管也经常使用30。关于哪种房间速度最理想，有很多争论，但是实际答案取决于您制作的游戏类型，房间大小，实例数以及代码强度。例如，如果您有一个大房间且房速为30，则游戏可能会显得“不稳定”，因为您正在看的区域越大，眼睛越容易察觉到低房速，但是如果游戏如果速度过快，则将房间速度设置为120可能会导致游戏“滞后”，因为fps会由于完成的处理量而下降。因此，请谨慎使用此值，并尝试始终根据您的游戏需要对其进行调整。
  • 默认绘图颜色：当未使用代码设置颜色时，绘图文本或论坛（和其他内容）的默认颜色。
  • 房间区域外部的默认绘制颜色：将用于清除游戏室以外任何区域的背景的默认颜色。
  • 生成项目图像：单击此按钮将打开项目图像生成器工具。
  • 项目开始日期：项目开始的日期。
  • 项目使用时间：自启动项目以来经过的时间长度。
  • 项目GUID GameMaker Studio 2为每个项目创建的唯一标识符。如果要更改此设置，可以单击“创建新的GUID”按钮以生成一个新的GUID。
  • 为单独的纹理页面生成mipmap：此选项将使GameMaker Studio 2为在Sprite Editor的Texture（纹理）设置中标记了Seperate Texture Page（分离纹理页面）选项的sprite创建mipmap。启用mipmapping意味着将缩放该精灵的原始纹理页面并将其过滤到纹理文件内的多种分辨率中。这通常是关闭的，但是如果您使用视图在游戏室的大区域上进行放大或缩小，或者您正在使用3D游戏，则可能需要启用它，然后使用Mipmapping功能来调整游戏处理不同的mipmap级别。请注意，如果不将子画面标记为使用单个纹理页面，则仍可以从“纹理组管理器”基于每个纹理组启用mipmapping。默认情况下，此选项是关闭的。
  • Steam ID ：如果您要创建要通过Steam发行的游戏，则需要在Steam应用程序ID。使用此选项时，请确保已在“平台首选项” Steam Steam Works仪表板上正确设置了所有内容。
  • 启用源代码管理：如果希望项目在SCM（源代码管理）下，则需要进行检查。默认情况下处于关闭状态（有关源代码管理的更多信息，请参见此处）。
  • 作者：游戏作者（或发行者或工作室）的名称。
  • 上次更改的项目：此部分仅显示上次更改项目的日期/时间戳（不可更改）。
  • 接受Spine许可证：在此处可以选中显示Spine Runtime许可证并接受的框。这将允许您在游戏中使用Spine子画面（第一次尝试将Spine子画面添加到项目中且事先未在此处检查许可证时，也会显示此许可证）。

根据您所拥有的许可证可用的目标平台，资产浏览器还将为您提供其他游戏选项，但并非所有这些选项都可用或不可见。

存在以下特定于平台的游戏选项：

  • Windows
  • macOS
  • Ubuntu
  • Windows UWP
  • HTML5
  • Android
  • iOS
  • tvOS

注意：由于不同公司维护的各种法律限制，此处未列出各种控制台游戏选项。如果您有控制台许可证，则可以从YoYo Games帮助台获得此信息，但是您必须首先YoYo Games联系，并要求获得查看不同部分的权限。
