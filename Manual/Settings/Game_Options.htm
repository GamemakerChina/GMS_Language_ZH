游戏选项

游戏选项分为两个部分，主要选项和平台设置。主要选项从一般选项开始，在这里你可以设置某些东西，这些东西将影响游戏，无论它被设计在什么平台上。 它被设计成在哪个平台上运行。

该窗口有以下部分。

  • 使用DnD™ 作为默认的脚本类型。当这个选项被勾选时，所有新的脚本和对象事件都将使用DnD™编辑器而不是代码编辑器打开。
  • Game Frames Per Second: 这个值设定了GameMaker Studio 2在运行时应尽量保持的每秒帧数的限制。这个值表示游戏循环在一秒钟内运行的次数（一个游戏循环被认为是一个 "帧"）。 一秒钟内运行的次数（一个游戏循环被认为是一个 "帧"），并被设置为默认值60，尽管30也经常被使用。关于哪种房间速度是理想的，有许多争论，但实际答案取决于你正在制作的游戏类型、大小和时间。 实际答案取决于你正在制作的游戏类型、房间的大小、实例的数量以及代码的密集程度。例如，如果你有一个大房间，房间速度为30，游戏可能会显得 "不流畅"，因为你所看的区域越大。 但如果游戏是密集型的，那么将房间速度设置为120可能会导致游戏 "滞后"，因为由于正在进行的处理量而使帧数下降。所以要注意这个值 并尽量根据你的游戏需要来调整它。
  • 默认绘制颜色：当没有使用代码设置颜色时，绘制文本或论坛（和其他东西）的默认颜色。
  • 房间区域外的默认绘制颜色。默认颜色，将用于清除游戏室以外任何区域的背景。
  • 生成项目图像。点击它将打开项目图像生成工具。
  • 项目开始日期。项目启动的日期。
  • 项目使用时间：自你启动项目以来已经过去的时间长度。
  • 项目GUID：这是GameMaker Studio 2中为每个项目创建的唯一标识符。如果你想改变它，你可以点击创建新的GUID按钮来生成一个新的。
  • 为独立的纹理页生成mipmaps。这个选项将使GameMaker Studio 2为那些在Sprite Editor的Texture Settings中标记了Separate Texture Page选项的精灵创建mipmaps。启用mipmapping 意味着精灵的原始纹理页将在纹理文件中被缩放和过滤为多个分辨率。这通常是关闭的，但如果你使用视图来放大或缩小游戏室的大面积区域，或者你正在处理一个3D游戏，那么你可能会选择关闭。 那么你可能想启用它，然后使用Mipmapping功能来调整游戏如何处理不同的mipmap级别。请注意，如果你没有把精灵标记为使用单独的纹理页，你仍然可以启用 纹理组管理器中，在每个纹理组的基础上启用mipmapping。这个选项在默认情况下是关闭的。
  • Steam ID：如果你创建的游戏要通过Steam发布，那么你需要在这里提供Steam应用ID。在使用这个选项时，请确保你已经在平台首选项中添加了SteamworksSDK的路径。 并确保你在Steamworks仪表板上正确设置了一切。
  • 启用源代码控制。如果你希望项目处于SCM（源码控制管理）之下，那么你需要勾选这个。默认情况下，它是关闭的（关于源控制的更多信息，请看这里）。
  • 作者。游戏作者（或出版商或工作室）的名字。
  • 项目最后更改：这部分只是显示项目最后一次被更改的日期/时间戳（这是不可以编辑的）。
  • 接受Spine许可证。在这里你可以勾选显示Spine运行时许可证的方框并接受它。这将允许你在你的游戏中使用Spine精灵（当你第一次尝试在你的项目中添加Spine精灵时，如果你没有事先在这里勾选许可证，也会显示这个许可证）。 的时候，也会显示这个许可）。)
