当GameMaker Studio 2为Android/Amazon Fire目标构建游戏时，会用到构建工具（如果需要的话，还有支持库），应该根据你使用AndroidStudio安装的工具设置为适当的值。

  • 目标SDK表示您已经在您指定的版本上测试了您的应用程序（直到并包括）。这只是为了让Android操作系统了解它应该如何处理你的应用程序的操作系统功能。对于所有的实际 的目的，大多数应用程序都希望将目标SDK设置为最新发布的API版本（正如编译SDK设置所使用的）。这将确保你的应用程序在最新的Android设备上看起来尽可能好。请注意，从11月 2019年，谷歌已将API级别28作为必要的目标。
  • 最低SDK版本是指能够运行你的项目的最低API水平。从2019年11月起，谷歌已将API级别16作为绝对的最低要求，但如果你添加了扩展，那么它们可能需要更高的最低API级别。
  • 编译SDK的版本是项目被编译的API的版本。这意味着你可以使用包含在该版本API中的AndroidAPI功能。如果你试图使用API 16的功能--比如说--但设置了Compile SDK 15，那么你会得到一个编译错误。然而，如果你把编译SDK设置为28，那么你仍然可以在API 26设备上运行该应用（以及所有其他以前的版本）。

你可以通过点击预填充SDK值到选定的API级别按钮，从预定义的目标设置中选择一个，这将打开一个不同的API列表供你选择。选择其中的任何一个，都会在所有的 SDK文件，然后您可以根据需要使用或编辑这些值。

注意：你必须在AndroidSDK管理器中或通过AndroidStudio安装了相应的API，你的项目才能正确编译。

许可权

从这个选项卡中，你可以改变你的游戏在亚马逊火力清单中可能要求的一些权限。GameMaker Studio 2会根据游戏使用的功能自动分配权限，但是你可能会发现你的游戏需要的权限没有被正确分配（例如，当你使用自定义扩展时）。 在这种情况下，请勾选必要的权限框。

下面的清单概述了那些可用的权限。

  • WRITE_EXTERNAL_STORAGE ：将允许你的游戏写到外部存储。
  • READ_PHONE_STATE: 允许只读访问电话状态。
  • ACCESS_NETWORK_STATE ：允许你的游戏访问有关网络的信息。
  • INTERNET ：允许你的游戏打开网络插座。
  • BLUETOOTH ：将允许你的游戏连接到已配对的蓝牙设备。
  • RECORD_AUDIO ：将允许你的游戏记录来自麦克风的音频输入。

注意：如果你不确定你是否需要这些，你也许应该默认不勾选它们，让GameMaker Studio 2来处理你游戏的权限。
