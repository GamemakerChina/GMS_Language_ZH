Windows UWP游戏选项

本节概述了可用于控制如何Windows平台）游戏项目的不同选项。不同的部分是：

一般的

UWP选项的常规部分是您可以在其中设置产品信息的地方：

  • 显示名称：指定显示给用户的应用程序的友好名称（此字符串可本地化）。
  • 软件包名称：这是您创建的软件包的名称，应该与您为游戏创建的开发人员密钥所使用的软件包名称相匹配（请参见下面“包装”部分）。
  • 包裹的显示名称：这是包裹的“简称”，理想情况下应与显示名称相同
  • 发布者显示名称：这是您的公司或您希望与游戏发布者建立联系的人员的名称。
  • 版本：游戏的版本号。

设置好之后，您可以选择可以运行游戏的不同方向以获得最佳用户体验。可用的选项有：

  • 风景
  • 肖像
  • 横向翻转
  • 人像翻转

默认情况下，所有这些都将被选中，您应该取消选择不需要的那些。

设置方向选项后，您可以从以下选项中发出您的UWP应用需要的不同权限：

  • 启用Internet权限：将游戏标记为可以访问Internet。默认情况下是关闭的。
  • 启用麦克风权限：将游戏标记为有权访问设备的麦克风。默认情况下是关闭的。
  • 启用IAP沙箱：将游戏标记为使用应用内购买中的沙箱。默认情况下是关闭的。

设备系列

在这里，您可以选择在创建可执行文件时希望Windows UWP导出目标的设备系列。默认情况下，选中“支持通用设备系列”选项，这意味着该应用程序将定位所有设备和系列。为了将应用程序限制为仅支持特定设备系列，应取消选中“通用”选项，并检查相应的设备系列。也可以支持通用设备系列，但将其他设备系列限制为仅某些平台版本-在这种情况下，您将检查通用设备系列，但还要检查您希望为其支持不同平台版本的其他设备系列，并在相应的文本框中输入该系列的版本详细信息。请参见以下Microsoft支持页面上的页面，以了解此逻辑的工作方式：

  • 目标设备系列：示例

通常，您不希望接触目标版本和最低平台版本，而应将其保留为默认值，但是如果遇到构建问题（特别是如果无法Windows SDK集），则需要将它们更改为已安装的内容。如果不确定要设置的值，请在Visual Studio创建一个新的UWP项目，并查看将其设置为什么值。

有关设备系列的更多信息，请参见以下Microsoft文档：

  • 设备系列概述

图形

图形选项是您应该配置的选项，以确定您的游戏如何使用目标UWP设备的图形卡。包含以下选项供您修改：

  • 在像素之间插值颜色：打开线性插值，基本上可以“平滑”像素。对于清晰的像素图形，应该将其关闭，但是如果您有不错的alpha混合和平滑的边缘图形，则最好保留为默认状态。
  • 显示光标：启用此选项后，默认的OS光标将显示在您的游戏中。如果禁用此功能，则当鼠标悬停在游戏窗口上方时，除非您在游戏项目中创建了一个光标（默认情况下处于启用状态），否则将不会有可见的光标。
  • 启动全屏：如果选中此选项，则游戏将以全屏模式启动，否则将启动窗口显示。默认情况下是关闭的。
  • 允许全屏Switch Ubuntu OS快捷方式从全屏切换到窗口，然后再次返回。默认情况下是关闭的。
  • 缩放：您可以在此处选择保持宽高比（这样4：3的房间将在16：9处“装箱”）或完全缩放（拉伸图像以适合全屏）。

    警告！ Switch荷兰国际集团关闭应用面将禁止在设置所有缩放选项Windows游戏选项，直到它已经切换回再次打开。有关更多详细信息，请参见应用程序表面。

最后，可以选择设置纹理页面的大小。默认（也是最兼容）大小是2048x2048，但是您可以选择256x256到8192x8192之间的任意值。还有一个标记为“预览”的按钮，它将为该平台生成纹理页面，然后打开一个窗口，以便您查看它们的外观。如果希望查看纹理页面的结构并防止纹理页面大于（或小于）所需的页面，这将非常有用。有关纹理页面的更多信息，请参见此处。

注意：请注意，纹理页面的尺寸越大，游戏在较低规格的PC上的兼容性越差。

图片

在图像的第一部分部分，你必须设置各种瓷砖，你的游戏将在不同的设备是UWP可以运行在使用该选项。您可以设置以下内容（它们都应该是 *.png格式文件）：

  • 小图块图像是徽标图像，显示在搜索栏中以及用户界面的其他区域中应用名称旁边。它应该是44 x 44像素。
  • 中图块图像应为71x71像素的图像。
  • 下一个图块图像是商店徽标图块，并且应为50 x 50像素。
  • 宽图像图块是当图块采用其宽格式时出现的，并且应为310 x 150像素。如果未提供此图像，则图块只能以正方形格式显示，并且不能接受基于广泛模板类型的通知。由于用户可以决定图块使用哪种格式，因此建议您包括一个宽图像图块。另请注意，如果提供了宽图像图块，则图块最初可能会以其宽格式显示。
  • 大图像图块是大实时图块格式，应为310 x 310像素。请注意，这将仅在台式机操作系统上使用。

下一部分用于定义游戏徽标。这实际上是您的游戏将在UWP生态系统中使用的图标，并且值得注意的是这些图像可以是透明的 .png文件，在这种情况下，它们将绘制在本节中指定的“背景颜色”上。此背景色还会应用到任何应用程序拥有的对话框中的按钮颜色以及设备上商店中的“应用程序描述”页面。

设置徽标后，您可以设置游戏的启动画面。这是您开始游戏时将显示的屏幕，并且不能大于620 x 300像素，并且像徽标一样，它可以是透明的 .png文件，在这种情况下，将使用您选择的背景色。

值得注意的是， GameMaker Studio 2具有项目图像生成器工具，该工具可用于自动创建将游戏编译到的所有不同目标平台所需的所有图像。如果使用此工具，则应修改创建的图像，以确保它们是您所需要的。

包装

GameMaker Studio 2为Windows UWP开发提供了默认的开发者密钥，但是建议您自己生成并在此处指向它。 Visual Studio创建一个空项目，然后将自动生成的副本保存在安全的地方 *.pfx文件将包含在空项目中（这是开发人员密钥）。然后，您应该将其链接 *.pfx在这里归档。

链接到文件后，应单击相关的“安装”按钮，然后按照将在打开的命令提示符中的说明进行操作。如果您不这样做，则使用“运行/调试”按钮（或按F5键）将失败，并且无法在UWP平台上测试您的游戏（但是，如果您使用Create Executable ，则如果安装了可自动启动证书安装过程， *.pfx尚未注册）。

注意：在将您的应用程序上载到商店时，您需要获取正确的标题*.pfx。这是通过将您的应用程序与Visual Studio中Windows Visual Studio专门为该应用程序生成一个新的*.pfx。还值得注意的是，您可以Visual Studio中将任何应用程序与Windows应用商店关联，这意味着您可以将空白项目与Windows应用商店应用关联，然后只需将生成的*.pfx复制到安全位置即可在UWP中使用游戏选项。 YoYo Games知识库中找到有关如何完成此操作的说明。

在此部分中，您还需要指定用于生成签名证书的发布者名称。输入应始终以“CN= ”，例如： CN=YoYo游戏。

Xbox Live

UWP游戏选项的Xbox Live部分是您可以在其中激活您的UWP项目以在Xbox One控制台上使用的地方。在Xbox上运行该项目时，可以在此处激活Xbox的项目，以便执行以下操作：

  • 从应用程序内登录Xbox Live
  • 在项目中以某种方式使用Xbox Live用户身份（例如，显示gamertag或gamer图片）

但是，在执行这些操作之前，您需要确保已单击“启用Xbox Live”复选框，并且如果您是Microsoft Creators Program的一部分，则还应该选中Creators Program App选项。如果您是创建者计划的一部分，则需要添加标题ID和服务配置ID （有关更多信息，请访问Microsoft ）。

请注意，当前要测试项目中的Xbox Live功能，您将需要使用Create Executable而不是Run，因为UWP Xbox输出需要使用Windows Store签名证书（请参见上文），而不是GameMaker Studio 2随附的临时版本，当您点击“运行”时将使用该版本。

启用XBox Live之后，就可以继续在项目中使用XBox Live功能，但是请注意，对于最基本的功能，您将需要成为Creator程序的一部分；对于高级功能，您将需要ID @ XBox帐户..
