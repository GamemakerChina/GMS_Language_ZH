macOS游戏选项

本节概述了可用于控制如何macOS游戏项目的不同选项。不同的部分是：

一般的

在“产品信息”部分，首先要做的是填写游戏的显示名称，然后填写其版本号，应用程序ID ，应用程序输出文件和版权详细信息。 “应用程序输出”是您的游戏安装文件在Mac开发计算机上的存放位置。当您创建自己的 .app.zip，该文件可以放在任何地方（在执行保存对话框时指定的位置不是该文件的唯一副本 .app.zip），但 .pkg和其他文件保留在此文件夹中，使您可以通过终端来测试本地安装，从而模拟Mac Store购买等。

在这里，您还需要提供GameMaker Studio 2为您创建的最终应用程序的团队标识符。在此处进行设置将覆盖您在macOS首选项中提供的团队ID。在“团队标识符”之后，您还可以选择提供“签名标识符” ，这是Apple要求的所有非应用商店应用程序（例如Steam游戏）。

下一个选项是在游戏中Steam如果启用此选项，请确保已Steam Works SDK（请参阅此处），并且已将Steam应用程序ID添加到游戏的“常规选项”中。

最后，您可以选择禁用文件系统沙箱。选中此选项将禁用GameMaker Studio 2沙箱来保存和加载文件，从而允许您的游戏从运行游戏的系统上的任何位置访问文件。默认情况下，此选项为未选中状态，因为我们建议您GameMaker Studio 2沙箱，并且选中该选项后，某些保存/加载区域可能仍然不可用，具体取决于操作系统级别的权限。有关沙盒的更多信息，请参阅GameMaker Studio 2文件系统上的页面。

图形

图形选项是您应该配置的选项，以确定您的游戏如何使用目标Mac的图形卡。包含以下选项供您修改：

  • 允许全屏菜单和停靠：选中该选项时，如果游戏处于全屏模式，则将显示OS菜单并停靠。默认情况下是关闭的。
  • Display Cursor ：选中此macOS光标，而取消选中将意味着不显示任何光标，除非您在游戏代码中创建了一个。默认情况下是关闭的。
  • 启动全屏：如果选中此选项，则游戏将以全屏模式启动，否则将启动窗口显示。默认情况下是关闭的。
  • 允许全屏Switch macOS快捷方式从全屏切换到窗口并再次返回。默认情况下是关闭的。
  • 在像素之间插值颜色：打开线性插值，基本上可以“平滑”像素。对于清晰的像素图形，应该将其关闭，但是如果您具有不错的alpha混合和平滑的边缘图形，则最好保留它。默认情况下是关闭的。
  • 使用同步避免撕裂：这将打开或关闭垂直同步。请注意，如果您的游戏的室内速度为120，而玩家的监视器的刷新率为60，则打开此选项也将您的游戏速度也锁定为60。默认情况下是关闭的。
  • 允许窗口大小调整：选中此选项将允许用户更改游戏窗口的大小（“无边界窗口”选项必须关闭才能起作用）。默认情况下，此选项是关闭的。
  • 启用视网膜：如果在启用了视网膜的显示器上运行游戏，则选中此选项会将后备缓冲区设置为较高（实际）分辨率，而取消选中该选项会将后备缓冲区设置为视在（较低）分辨率。这意味着初始游戏窗口的宽度和高度将是第一个房间设置的房间/视口的宽度和高度的两倍。但是，这不会缩放应用程序表面以适合其大小，因此，如果您想让游戏充分利用视网膜显示，则还应将应用程序表面设置为与窗口大小相同（请注意，这将使像素增加一倍）沿宽度和高度，因此只有启用此功能并缩放应用程序表面（如果您的游戏将受益于提高的分辨率）。默认情况下，此选项是关闭的。
  • 缩放：您可以在此处选择保持宽高比（这样4：3的房间将在16：9处“装箱”）或完全缩放（拉伸图像以适合全屏）。

    警告！ Switch荷兰国际集团关闭应用面将禁止在设置所有缩放选项Windows游戏选项，直到它已经切换回再次打开。有关更多详细信息，请参见应用程序表面。

最后，可以选择设置纹理页面的大小。默认（也是最兼容）大小是2048x2048，但是您可以选择256x256到8192x8192之间的任意值。还有一个标记为“预览”的按钮，它将为该平台生成纹理页面，然后打开一个窗口，以便您查看它们的外观。如果希望查看纹理页面的结构并防止纹理页面大于（或小于）所需的页面，这将非常有用。有关纹理页面的更多信息，请参见此处。

注意：请注意，纹理页面的尺寸越大，游戏在较低规格的PC上的兼容性越差。

图片

图像部分是您提供游戏所需图像的地方。对于Mac，您需要提供一个图标文件（该文件必须位于 .png格式和1024x1024px）和启动画面（可以是 .png， .bmp ， .jpg或者 .gif格式）。启动游戏时会显示启动画面。

对于DMG软件包，您可以提供安装程序背景图像，该图像将作为DMG安装程序的背景出现。

值得注意的是， GameMaker Studio 2具有项目图像生成器工具，该工具可用于自动创建将游戏编译到的所有不同目标平台所需的所有图像。如果使用此工具，则应修改创建的图像，以确保它们是您所需要的。

包装

如果您希望完成的游戏能够为App Store准备就绪，则需要选中Build for Mac App Store选项，但是请注意，只有当您是注册开发人员并拥有必要的证书时，此功能才起作用。在此之下是应用程序权限，您仅应检查它们是否为真，因为如果不需要时选中了它们，Apple可以拒绝您的应用程序，反之亦然。这些选项仅允许您的游戏使用 http_和 url_功能。值得注意的是，如果您希望在游戏中支持GamePads，则应关闭创建可用于App Store的程序包的选项。

最后，您需要选择其应用类别（有关更多信息，请参见此处）。

社会的

此选项用于在Mac上使用Apple Sign In Extension。您可以从YoYo Games Marketplace获得此扩展，该扩展本身包含完整的使用说明。如果您不使用此扩展名，则不应勾选此选项。
