纹理页面

当您使用GameMaker Studio 2创建游戏时，您肯定会创建图形，包括精灵，拼贴和字体。这些图形存储在GameMaker Studio 2从游戏包含的所有图像资源为您构建的纹理页面上。以下是完成的纹理页面的示例：

如您所见，所有游戏图形都以这样的方式混杂在一起：它们全部适合2个大小的页面，例如512x512、1024x512等...最大页面大小为4096x4096像素。请注意，纹理页面的最大大小将取决于所选的导出平台，因为某些目标将不允许页面大于2048x2048px，即使那样，最大可用大小可能也不是所有使用目标平台操作系统的设备的最佳选择。

还要注意，如果您的图像周围有很多空白空间（即透明像素），则默认情况下将对其进行裁剪以删除所有这些“不可见”像素，并将尽可能多的图像打包到单个纹理页面上。如果这不是您希望发生的情况，则需要在“纹理组”窗口中将其禁用。

重要的是要了解不同的可用纹理设置将如何影响图像，因此让我们看一下标题为“纹理设置”的“精灵编辑器”部分，如下所示：

在这里，您可以定义如何将图像存储在纹理页面上，以及该图像是否是专门用于3D的图像并且必须单独存储，以及还必须将其分配给哪个纹理页面（如果有）以及其他一些选项（在“精灵编辑器”页面的epth中进行了说明）。我们感兴趣的选项是前两个：水平平铺和垂直平铺。这些与放置在纹理页面上时如何创建图像的边缘有关。

要平铺的图像将具有从相反侧的边缘添加到的边缘（水平或垂直），而如果未选择平铺，则精灵将被钳位并且边缘像素将重复（请注意，图块还具有在图块集属性中找到的输出边框宽度设置（仅对集合中的每个图块执行相同的操作）。下图说明了这一点：

这些设置下面是“单独的纹理页面”设置。此选项将强制GameMaker Studio 2将图像放置在其自己的唯一纹理页面上，并且没有任何与此相关的选项，因为平铺或夹紧是通过代码直接控制的。这通常用于纹理化3D模型（在这种情况下，纹理应为2的幂，例如：128x128、512x512、256x1024等），但是您可以使用它来强制将任何图像绘制到其上自己的独特页面。

注意：使用此选项时要小心，因为如果纹理页面过多，您可能会很快用完所有可用的纹理内存，尤其是在移动设备上。

现在，当您的游戏GameMaker Studio 2自动创建这些页面的方式可能并不是最佳地利用目标设备的最佳方法，原因是如果您的游戏需要来自4个不同纹理页面的资源，设备将不得不不断地将这些页面交换到（和从）内存中，这可能会导致性能下降。为此，您可以通过将其分配给特定的纹理组GameMaker Studio 2哪些图像放置在任何特定的纹理页面上。这样做的最终目的是最大程度地减少游戏中一次替换的次数（可以从“纹理组”窗口中添加纹理页面）。

应当注意，在“游戏选项”中可以找到的各个平台选项卡中，有一个按钮允许您查看已定义的纹理页面，就像为每个目标创建的一样。这对于确定要创建多少页面以及页面上的内容非常方便。

如果您的游戏中的纹理大于选定的最大纹理页面大小，则将其缩小一半（如有必要，再缩小一半），直到适合最大页面大小为止。它不会在各种纹理页面上分成多个部分。 Windows ， Ubuntu （ Linux ）或MacOS上测试游戏时，当您使用较大的纹理页面时，游戏看起来可能不错，但是在iOS （或需要较小纹理页面大小以实现最佳性能的任何其他目标）上， ）纹理可能看起来模糊或有伪影。确保所有图像都能够适合游戏将使用的最小纹理页面大小至关重要，如果没有，则应将其拆分为较小的图像，然后在游戏中将它们绘制在一起以提供一张完整的大图片的错觉。

注意：在将图形拆分到各种纹理页面上之前，应注意这并非总是必要的，也不总是最有效的工作方式。某些游戏的CPU绑定可能比GPU绑定更多（有关更多信息，请参见调试部分），而这些游戏不会从这种优化中受益匪浅，并且您还应该考虑将所有图像放在两个纹理上是否更好页面并在它们之间进行交换，而不是分散在多个页面上，这只是浪费内存。
