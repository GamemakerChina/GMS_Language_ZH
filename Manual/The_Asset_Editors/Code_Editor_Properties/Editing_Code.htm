编辑代码

GML语法键入想要用于脚本资产或事件或任何其他内容的代码的情况。执行此操作时，您将从自动完成窗口中获得帮助：

在编写代码时（通常会稍作暂停，但是可以在Preferences中设置），该窗口将弹出，并为您提供按字母顺序排列的“最佳”答案。

使用代码编辑器时，您可以单击并拖动鼠标以选择一行代码或按住然后点击并拖动以选择代码列。然后，可以使用以下命令的标准键盘快捷键复制，剪切和粘贴以此方式选择的所有内容： / +“ C ”， / +“X “ 和/ +“V ”。

值得注意的是，您可以通过多种方式注释您的代码。对代码进行注释对于保持生产率非常重要，因为注释可以使事情变得清晰并分离代码段，从而使每个部分都可以做什么以及为什么要这样做很明显。这也意味着，如果您在团队中工作，则可以给其他成员留下笔记，他们会确切地知道您所做的工作或特定代码的用途。即使您是一个单独的开发人员，在代码中保留注释也很重要，尤其是当您必须离开项目并稍后再返回时。

注释代码很简单，因为您只需使用“//在任何文本行之前，或者您可以使用“/* text here */ “。甚至还有两个键盘快捷键可用于注释和取消注释选定的代码行： / +“K进行评论，然后/ + +“K下图说明了使用这两种方法以及一些JS Doc注释编写的代码（下文有更详细的解释）：

脚本还可以添加某些JSDoc样式注释，以定义它们所包含的函数的属性，以便在代码中使用这些函数时，它们会连同其参数和其他详细信息一起自动完成显示。您可以从JSDoc脚本注释一节中找到有关此内容的更多信息。

默认情况下，所有编写的代码在每组匹配的花括号旁边都会带有代码折叠图标 {}。这些采取一点加号的形式 +或减 -图标，然后单击它们将在匹配的花括号内“折叠”代码，从而减少可视空间。

您还可以使用特殊设置手动设置代码折叠区域 #region标签以标记一段代码。运作方式是您添加一个 #region标记（以及所有注释文本）放在一段代码的顶部，然后添加一个对应的 #endregion标记到该部分的底部，然后您可以“折叠”代码，即：将其隐藏起来，以便可以随时减少在编辑器中可见的代码行数：

实际的代码如下所示：

#region This is an example region

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion Further comment here if required

您可能已经注意到，脚本文本的某些部分是彩色的。脚本编辑器“了解”现有对象，内置变量和函数等，并相应地对其进行颜色编码。颜色编码有助于避免错误，因为如果您拼错了一些名称或将关键字用作变量，或者遇到语法错误，您会立即看到。但是，如果您不喜欢颜色编码，则可以在“首选项”中将其打开和关闭（也可以使用 F10在编辑器中打开或关闭它）以及自定义使用的颜色。

您可以点击右键也可以在编辑器中的任何位置打开以下菜单：

除了希望剪切，复制和粘贴代码行的常规功能之外，您还可以选择注释或取消注释任何选定的文本，添加或删除缩进以及将代码窗口分为1、2或3单独的窗格（请注意，与 #regions如果您尚未在要编辑的代码中定义任何代码，则将不可用）。

用于拆分为窗格的最后一个选项会将实际窗口拆分为不同的部分，然后您可以在其中拖动脚本或事件，并将其与其他脚本或事件一起显示以进行比较和编辑。

这对于比较两个脚本或同时查看连接的脚本非常有用，而无需另一个窗口或在选项卡之间不断切换。

此菜单中的其他选项是切换用于Debugger的断点，以及将GML代码转换为DnD™，反之亦然。请注意，如果将代码转换为DnD™，则它不会为每行代码创建单独的动作，而是使用单个代码动作来保存事件或脚本中以前的所有内容。
