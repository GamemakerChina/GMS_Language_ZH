iOS / tvOS扩展

iOS或tvOS创建扩展，必须分两部分进行。第一部分是添加扩展本身，以及所需的文件等...，第二部分是创建扩展所需的函数和宏/常量。使用占位符文件将函数和常量添加到一起以将它们分组在一起，因此您将添加一个占位符，然后按照此处一节中的说明定义函数和宏。要添加其余文件，尽管您需要首先在编辑器的“其他功能” iOS和/或tvOS复选框，以打开相关的“扩展属性”窗口（下图显示了iOS属性窗口，但tvOS窗口完全相同）：

在这里，您提供以下详细信息：

  • 链接器标志/编译器标志：某些框架和第三方SDK需要添加额外的链接器标志和编译器标志才能在此处指定（有关详细信息，请参阅所涉及的SDK或框架的文档）。
  • 类名：您的扩展可以具有多个类，每个类都有其自己的函数和常量，因此您应该给它一个能反映其用途的名称
  • App Delegate Class Name ：您的自定义应用程序委托类的名称。设置此选项可使扩展程序覆盖/扩展核心应用程序功能。要使用此功能，您需要执行以下操作：
    1. 确保委托源文件的名称与委托类的名称相同（例如， @interface MyCustomAppDelegate应该在“ MyCustomAppDelegate.h ”）
    2. 添加 ${YYExtAppDelegateIncludes}应用程序委托头文件顶部的环境变量。在编译时，它将被父委托类的相关包含文件替换。
    3. 使用 ${YYExtAppDelegateBaseClass}环境变量作为您的自定义应用程序委托的基类。将在编译时将其替换为正确的基本委托类。

      为确保您的扩展与使用自定义应用程序委托的任何其他扩展一起使用，应从子类中的重写方法中调用任何基类方法。在调用超类方法之前，您需要确保在类层次结构中实现了该类以避免错误，例如：

    - (BOOL)application:(UIApplication *)application willFinishLaunchingWithOptions:(NSDictionary *)launchOptions
    {
        // Check if any superclasses implement this method and call it
        if([[self superclass] instancesRespondToSelector:@selector(application:willFinishLaunchingWithOptions:)])
          return [super application:application willFinishLaunchingWithOptions:launchOptions];
        else
            return TRUE;
    }

  • 系统架构：在这里，你可以在任何添加iOS系统架构s到您的分机。通过单击添加这些按钮，它将添加一个占位符框架，然后您可以通过缓慢单击来对其进行编辑在上面。您可以在此处找到有关可用系统框架的更多信息。要删除系统框架，只需选择它，然后单击按钮。
  • 注入到Info.plist ：您可以在此处添加要注入的任何代码 info.plist文件。
  • 第三方框架和捆绑包：此部分用于添加第三方和SDK捆绑包。与系统框架一样，您单击按钮添加它们，然后缓慢单击编辑，然后选择它们，然后单击将会删除它们（有关框架名称的信息，请参见所选SDK随附的文档）。如果框架/ SDK捆绑包在您的开发Mac上，则需要将路径添加到“在Mac上输入框架路径”，然后单击按钮，或者如果您的PC上包含SDK或源代码，请使用添加源按钮。
  • CocoaPods ：在这里，您可以提供CocoaPods的名称，而不是（或以及）已添加的任何SDK和框架。请注意，有一个iOS和tvOS游戏选项可将Podfile添加到GameMaker Studio 2，如果要CocoaPods ，则需要这样做。
  • CocoaPods依赖性：在此部分中，您可以列出上CocoaPods添加的CocoaPods所需的不同依赖性。 iOS和tvOS有一个Game Option ，可将Podfile.lock添加到GameMaker Studio 2，如果要在此步骤中添加CocoaPod依赖项，则是必需的。

如果您的扩展已添加任何系统框架或第三方框架，则这些将现在在“扩展属性”窗口中列出，每个扩展框架旁边都有一个复选框。如果选中该复选框，则表示启用弱链接，如果您需要用自己的值“覆盖”包含的库中的任何符号，这将很有用，但应注意，这样做会减慢链接速度。
