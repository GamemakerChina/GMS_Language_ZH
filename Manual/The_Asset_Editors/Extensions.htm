一个扩展包可以用来为GameMaker语言（GML）增加一些额外的功能和常量，或者作为一种方式，在人与人之间轻松传递 游戏资产甚至整个项目在人与人之间轻松转移。如果扩展是为了增加额外的功能，那么所使用的函数可以将你的游戏与外部SDK或用其他语言编写的源代码（例如Android的Java），以及本地GML连接起来。

通常，第三方扩展会从 游戏制作者 市场而那些你已经订阅的 的扩展会在MarketplaceLibrary窗口中列出，可以从那里安装。如果你没有从市场上得到你的扩展包，你仍然可以通过以下方式把它添加到项目中去 右键点击资产浏览器中的任何地方 ，然后选择添加现有的，然后浏览到扩展的位置 的位置来选择它。

注意：传统格式 *.GEX ，不再支持，不能导入GameMaker Studio 2项目中。

一旦你在资产浏览器中添加了一个扩展，你应该给它一个名字和一个版本号，还应该设置你想把它导出到哪些平台。通过取消选中的平台，你可以告诉GameMaker Studio 2不要把扩展程序作为这些平台的可执行文件的一部分导出。 作为这些平台的可执行文件的一部分。你还应该确保扩展的属性与目标平台相匹配，并根据以下内容编辑扩展文件的导出目标。

  • 所有目标。GML扩展。这些是使用GameMaker语言（GML）制作的扩展，与所有目标平台兼容。
  • Windows（标准、UWP、XBoxOne）。你可以使用windows DLL 文件（动态链接库）。请注意，如果你在Windows游戏选项 中勾选了 "使用x64Windows Runtime" 。 那么项目的所有DLL必须是64位兼容的。32位的DLLs将不能与64位的可执行文件一起工作。
  • macOS:macOS目标可以使用标准的 DyLib 文件进行扩展。
  • Ubuntu（Linux）。对于Ubuntu目标，你需要确保你的扩展使用 SO 文件。
  • HTML5。这些平台可以使用 JavaScript 扩展。
  • 安卓系统:这个平台需要用 Java 或 Android Framework SDK来编写扩展。
  • 视觉系统/ tvOS:这个平台的扩展是用Objective C编写的，需要一个 *.mm 和一个 *.h 文件，如果使用任何框架，则需要一个预编译的 *.a 本地文件。
  • PlayStation。对于PlayStation 3或4，你的扩展名应该是 PRX ，每个目标需要一个不同的 PRX ，以符合目标的规格（PS3或PS4）。 如果在PSVita上工作，扩展名应该是 SUPRX 。

注意：有些扩展需要遵循特定的命名规则，以便GameMaker正确识别它们所设计的平台和CPU架构。请参阅 "创建扩展"一节，了解更多信息。

由于一个扩展可以有不同的相关文件，所以有可能为多个平台创建一个扩展，在这种情况下，你要在主扩展编辑器中指定所有的 兼容平台（即：勾选它们旁边的复选框 旁边的复选框），然后在扩展的每个部分的单个文件属性中，设置这些文件应该被导出的平台。

要改变一个特定的扩展文件或占位符的目标平台，你必须首先打开扩展属性（在资产浏览器中双击 ，或 使用 ，然后选择 "打开属性"），然后在扩展属性中选择你想编辑的右边的资源文件。

注意：如果你要导出到macOS或Ubuntu目标，那么你可以设置 "解压为zip文件 "选项，以保留扩展名的文件权限。

一旦你安装了你的扩展，你可以使用它的函数和常量，就像你使用GameMaker语言中的内置函数和常量一样。如果你的游戏是跨平台的（例如，它被设计为同时运行在 Windows和Android目标），那么你应该使用操作系统的常量和函数来检查平台，然后再调用扩展的特定函数，以防止问题。 例如。

switch(os_type)
{
case os_android:
call_android_extension_function(x, y, z);
break;
case os_windows:
call_windows_extension_function(x, y, z);
break;
}

请注意，对于不同的平台，你的扩展可以共享函数名，所以如果你有一个不同的扩展文件，（例如）为Windows和Android弹出一个消息，两个扩展文件可以共享相同的函数名，只要 只要它们都被设置为只输出到各自的平台。
