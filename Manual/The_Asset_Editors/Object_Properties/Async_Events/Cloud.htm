云

Cloud Event是一种由以下事件之一的回叫触发的事件cloud_*()功能，例如cloud_synchronise() 。实际上，它会生成此事件专有的DS映射，并存储在特殊变量中 async_load （请参阅各个函数以获取代码示例，这些代码示例将进一步详细说明此事件的用法）。该DS映射具有以下结构：

  • ”status “：这将保存状态代码，其中负值表示错误，其描述将包含在“ errorString”中。值0（或正值）表示成功（有关准确值，请参见下文），并且“ resultString”将包含返回的数据或成功消息。
  • ”id “：从被调用函数返回的ID。如果您触发了一系列cloud_请求，那么您需要知道您要回复的请求，因此您可以使用该值与在您最初发送的是查找正确请求的请求。
  • ”description “：最后上传的文件的描述。
  • ”resultString ”：这包含一个字符串，它是从云返回的数据blob。
  • ”errorString “：返回任何错误的错误字符串。

下表解释了返回的“状态”映射条目的确切含义：

状态值errorString / resultString描述
-1 errorString =“未登录<SERVICE>” You have not successfully logged in to the given Cloud Service
0 resultString =恢复的数据New game data downloaded from the cloud (following a cloud_synchronise() call)
1 resultString =“已经同步” No new data since you last called cloud_synchronise
2 resultString =“ ConflictDeferral” A conflict was encountered, but the gamer chose to ignore it
3 resultString =“ GameUploadSuccess” data from cloud_string_save() or cloud_file_save() was successfully uploaded to the cloud
-n errorString =错误说明Any other negative number means a synchronisation failure
