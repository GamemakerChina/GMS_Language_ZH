云端事件是由其中一个函数的回调触发的。 cloud_*()函数触发的。 如 cloud_synchronise().它实际上产生了一个DS地图，这个地图是这个事件独有的，并被存储在 的特殊变量 async_load (请看各个函数的代码例子，这些例子解释了该事件的使用。 事件的细节）。)这个DS图有如下结构。

  • "status"。这是状态代码，负值表示错误，对错误的描述将包含在 "errorString "中。值为0（或正值）表示成功（具体数值见下文）。 准确的值），"resultString "将包含返回的数据或成功信息。
  • "id"。从被调用的函数返回的ID。如果你发出了一系列的云端请求，那么你需要知道你得到的是哪一个回应，因此你将使用这个值与你最初发送请求时存储的值进行比较，以找到正确的请求。 与你最初发送请求时存储的值进行比较，以找到正确的那一个。
  • "description"。最后上传文件的描述。
  • "resultString"。这是一个字符串，是从云端返回的数据块。
  • "errorString": 对任何错误都返回一个错误字符串。

下表解释了返回的 "状态 "地图条目的确切含义。

状态值 errorString / resultString 描述
-1 errorString = "未登录<SERVICE>" 您没有成功登录到指定的云服务中去
0 resultString = 恢复的数据 从云端下载新的游戏数据（在 cloud_synchronise() 电话后）。
1 resultString = "AlreadySynchronized" 自你上次调用云端同步后没有新数据
2 resultString = "ConflictDeferral" 遇到了冲突，但游戏者选择忽视它
3 resultString = "GameUploadSuccess" 来自 cloud_string_save() 或 cloud_file_save() 的数据已成功上传到云端
-n errorString = 错误描述 任何其他负数都意味着同步失败
