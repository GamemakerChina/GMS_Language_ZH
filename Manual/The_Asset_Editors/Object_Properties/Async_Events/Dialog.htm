对话

当它发现时才会触发对话异步事件回调从的特殊的一个异步对话功能，像get_login_async() （请参阅此功能，获取有关如何使用此事件的扩展代码示例）。

这些事件是要求某种类型的用户输入的事件，可以是名称，登录详细信息，数字或颜色等。由于大多数设备不喜欢坐在循环中等待答复，因此必须异步， GameMaker Studio 2将继续在后台运行，同时这些功能会打开一个对话框，直到获得触发该事件的所需用户输入为止。这意味着，如果在游戏进行过程中打开了对话框，则应采取步骤暂停游戏，直到收到对话框回调为止。

再次返回DS Map ，并将ID保留在特殊变量中async_load 。该映射表中保存的值将取决于所使用的功能，您应查阅本手册中每个功能的单独条目以获取更多详细信息。
