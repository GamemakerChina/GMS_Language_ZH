HTTP

HTTP事件是由http_*() 的一个函数的回调触发的。 如 http_post_string().它实际上产生了一个DS Map 它实际上产生了一个DS Map，这个地图是这个事件独有的，并被存储在特殊的变量 async_load （请看各个函数的 请看个别函数的代码例子，这些例子进一步解释了这个事件的使用）。)这个DS地图有以下的一般结构。

  • "id"。从命令中返回的ID。如果你发出了一系列的 http_ ，那么你需要知道你得到的是哪一个回复，因此你将使用这个 值与你最初发送请求时存储的值进行比较，以找到正确的那一个。
  • "status"。错误时返回小于0的值，成功时返回0，如果正在下载内容则返回1。
  • "result"。收到的数据（只有字符串），或者如果你使用了"...... "来下载文件的路径。 http_get_file().
  • "url"。你所要求的完整URL。
  • "http_status"。原始的http状态代码（如果有的话）。这将返回大多数浏览器的标准网络状态代码，例如：304表示 "未修改 "或204表示 "无内容"，等等。

上面显示了当你使用 http_post_string()函数时得到的结果，但是 http_ ，每个函数返回的地图可能略有不同，所以请参考每个函数的手册条目，以了解其返回的精确数据。

注意：由于 async_load 创建一个DS地图，这些函数在与 json_encode()和 json_decode()函数时特别有用。

如果你要求下载文件，这个地图还可能提供额外的数据。在这种情况下，"状态 "将有一个值为1，DS地图将持有这些额外的键。

  • "contentLength"。这是网络服务器所说的你应该期望收到的文件大小（如果服务器不返回这个数据，可能是-1）。
  • "sizeDownloaded"。已经下载的数据的大小。

请注意，该事件不会为每一个收到的数据包而被触发，而是在主游戏循环内的下载过程中的任何时间更新。还请注意，目前这个功能只适用于普通的Windows 目标平台。
