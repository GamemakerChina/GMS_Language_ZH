图片已加载

只要将有效URL或包含的文件路径与适用的加载文件功能一起使用，则在将图像加载到GameMaker Studio 2例如，您要加载一个精灵图像，并且仅在实例加载后将其当前实例更改为新实例。好吧，在创建事件或警报事件中（例如），您会遇到类似这样的情况：

var _url = "http://www.angusgames.com/game/background1.png";
spr = sprite_add(_url, 0, false, false, 0, 0);

现在将开始将图像加载到设备或浏览器中，但是在等待文件加载时GameMaker Studio 2取而代之的是， GameMaker Studio 2将保持正常运行，直到加载图像并回调触发图像加载事件为止，在此处创建DS Map并将其存储在特殊变量中async_load 。该地图包含以下信息：

  • ”filename “：您请求的文件的完整路径。
  • ”id “：您已加载的资源的ID。这将与您为资源分配的变量相同。
  • ”status “：返回小于0的错误值。

然后，您可以在此事件中将新加载的图像分配给sprite。下面的代码示例演示如何使用返回的信息：

if ds_map_find_value(async_load, "id") == spr
   {
   if ds_map_find_value(async_load, "status") >= 0
      {
      sprite_index = spr
      }
   }

上面的代码将首先检查 id已创建的DS映射，然后检查 status的回调。如果该值大于或等于0（信号发送成功），则来自回调的结果将用于将子画面索引设置为新加载的图像。
