保存/加载

此事件将由某些函数的回调触发，该回调与将缓冲区加载和保存到文件以及从内存加载或卸载音频有关。事件本身将包含内置的async_load DS映射，将由特定功能所需的键填充。这些在下面的部分中列出。

缓冲液

使用功能时buffer_save_async()或者buffer_load_async()数据传输完成后，将触发一个异步事件。此事件将填充 async_load 映射以下键/值对

  • ”id “：异步函数的ID，由所使用的函数返回。
  • ”status “：如果正确保存/加载数据，则返回true，否则返回false。

这样，您就可以在完成该过程时轮询保存/加载进度并显示消息或更改房间等...。

音频组

使用音频组时，您可以使用以下功能加载它们并将其从内存中卸载audio_group_load()和audio_group_unload() 。使用加载功能时，当为该组设置的全套音频文件已加载到内存中时，它将触发此事件，并使用以下键/值对填充映射：

  • ”type ”：这告诉我们正在调用的事件的类型，将为“ audiogroup_load”以加载音频。
  • ”group_id “：将返回已加载的音频组的ID（在Audio Group Editor中定义）。

当为一个组加载了所有音频后，此事件将触发，然后可用于更衣室或播放音乐曲目等。
