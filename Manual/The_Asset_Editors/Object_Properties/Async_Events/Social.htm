社会的

此事件只能由被触发回调从各种具体社会功能和UWP Xbox Live的功能S和将返回DS地图存储在变量async_load ，其中包含不同的键/值对，具体取决于触发事件的函数的回调。

DS映射将包含许多键，其中最重要的是“id键。此键将返回一个常量，然后可以在代码中检查该常量以确定事件已接收到许多回调中的哪一个。解析完此键并将返回的值与可用常量进行比较后，您可以继续提取该常量地图上的其余信息。

“参考”部分中“成就和排行榜”上的功能页详细介绍了函数和回调以及用于触发此事件的常量，但是在某些情况下，如果没有运行游戏的函数调用，也会触发该事件：

  • 从OS仪表板启动游戏时
  • 完成挑战后
  • 当另一个玩家完成挑战时
  • 通过选择挑战从OS仪表板启动游戏时

以上任何回调都会触发社交事件，并且 async_load将会生成具有以下详细信息的地图（请注意，有一个不同的“id ”键值，以定义调用事件的各种不同原因，但地图的所有其他内容都相同）：

  • ”id “-此键的值取决于触发事件的回调的类型。它可以是以下常量之一：
持续的描述
achievement_challenge_received收到挑战
achievement_challenge_completed挑战已经完成。
achievement_challenge_completed_by_remote另一位玩家已经完成了挑战。
achievement_challenge_launched游戏是在给定挑战的情况下从OS仪表板启动的。

  • ”playerid “-挑战的玩家ID。
  • ”issuerid “-发出挑战的人的ID。
  • ”state “-质询的状态，对于无效，待处理，已完成或已拒绝，其值为0-3（作为字符串）。
  • ”message “-挑战文字消息。
  • ”issueddate “-挑战的发布日期
  • ”completeddate “-挑战的完成日期。
  • ”type “-给定的挑战类型。可以是两个常数之一：
持续的描述
achievement_type_score_challenge基于得分值的挑战。
achievement_type_achievement_challenge基于成就的挑战。

  • ”identifier “-挑战的标识字符串。
  • ”score “-得分与挑战息息相关。

您可以在下面找到一小段代码作为使用示例：

var ident = ds_map_find_value(async_load, "id" );
if ident == achievement_challenge_completed;
   {
   player_id = ds_map_find_value(async_load, "playerid");
   issuer_id = ds_map_find_value(async_load, "issuerid");
 state = ds_map_find_value(async_load, "state");
   date_completed = ds_map_find_value(async_load, "completeddate");
   date_issued = ds_map_find_value(async_load, "issueddate");
   ach_type = ds_map_find_value(async_load, "type");
   ach_ident = ds_map_find_value(async_load, "identifier");
   ach_score = ds_map_find_value(async_load, "score");
   }
