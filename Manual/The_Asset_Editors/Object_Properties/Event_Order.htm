活动顺序

在考虑GameMaker Studio 2事件时，应注意，不能明确说明每个事件将要发生的确切顺序，仅因为它取决于GameMaker Studio 2的内部工作原理，并且可能会发生变化。随着软件的发展。但是，某些事件将始终以相同的顺序运行。

总是以相同方式发生的第一组事件是首次进入房间时发生的事件。不同事件将触发的顺序是：

  • 对象变量/实例变量被初始化（这些是可以在对象编辑器或Room Editor的IDE中定义的变量）。如果您有一个带有变量的对象，则将在其他所有对象之前创建它们，然后，将在之后定义任何带变量的实例，因此您可以在房间编辑器中用实例特定的对象覆盖对象变量。
  • 铬eate事件后，通过每个实例的实例创建代码-以便在创建每个实例，它会先运行其上移动到下一个实例被创建之前创建的事件，然后将其实例创建代码
  • 游戏开始事件-从房间编辑器放置在房间的所有实例中，在游戏的第一个房间中触发一次该事件（请注意，调用game_restart（）将再次触发此事件。
  • 房间创建代码-这是在“房间编辑器”中编写的一次性代码，用于首次输入房间时
  • 所有实例的房间开始事件-事件的“其他”类别之一，将针对所有实例（持久性或其他）触发

还值得注意的是，您还可以通过在“实例层属性”窗口的列表中上移或下移特定实例来设置在房间编辑器本身中创建实例的顺序。

除了那些特定的事件，无论属于三步事件和不同的绘制事件，唯一已知的顺序将始终以相同的方式发生。这些将始终保持一致，因此，如果您的代码在游戏的每个步骤中都依赖于特定的时间安排，则应使用：

  • 开始步骤事件
  • 步骤事件（请注意，步骤事件是在将实例放置到新位置之前执行的）
  • 结束步骤事件

所有用于绘制的事件也总是按以下相同顺序处理（“窗口调整大小”事件除外，该事件的触发方式不同）：

  • 开奖前活动
  • 抽奖开始事件
  • 抽奖活动
  • 抽签结束
  • 抽奖活动
  • Draw GUI Begin事件
  • 绘图GUI事件
  • 绘图GUI结束事件
