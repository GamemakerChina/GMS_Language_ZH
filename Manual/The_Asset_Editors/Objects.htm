对象编辑器

对象是我们用来控制游戏各方面并做特定事情的资源。大多数情况下，它们具有与之关联的精灵，因此您可以在游戏室中看到它们，但有时它们被用作“幕后”控制器，以执行与用户有关的事情或用于计时等。可以赋予行为，它们可以对某些事件以及彼此之间做出反应，并且您在游戏中看到的大多数事物都是基于对象及其交互作用的。现在，我们说“基于”是因为您实际上并没有将对象直接放置到游戏室中，而是放置了这些对象的实例，这些实例基本上是对象资源的副本（或克隆，如果您愿意）。要记住这是非常重要的事情，因为实例和对象不是同一件事，并且每个实例都有可能影响它们的自己的一组功能。

为了澄清这一点，请考虑使用GameMaker Studio 2制作的游戏。所有角色，怪物，球，墙等...都是您在“资源浏览器”中创建的所有对象。然后，将对象拖到“房间编辑器”中的房间中，以创建从基本对象模板复制的实例。然后可以在编辑器本身中或通过代码来更改会议室中的实例，以便可以对其进行缩放或着色，或从“基础”对象修改其他细节。因此，当我们谈论影响或更改实例的事物时，是指它会影响房间中对象的特定副本。仅单个实例受到影响，而其余所有实例均不受影响。但是，当我们谈论影响或更改对象时，是指我们正在资产浏览器中修改该对象，并且对该对象所做的任何操作也将反映在此后创建的所有实例中。因此，对象是实例的模板，实例是我们放置在房间中制作游戏的对象。

要在游戏中创建对象，可以右键单击在资源浏览器上，然后单击创建对象选项。这将打开本页顶部所示的窗口，其中包含以下部分：

细节

您可以在此处为对象命名。该名称不得包含空格或未经授权的字符（仅允许使用字母数字字符和下划线“ _”），并且应简短但易于记忆，以便以后在拥有很多资源时可以轻松地标识资源。例如，许多人使用前缀或后缀来标识资源之间的差异，例如“obj_Ship “ 或者 ”Ship_obj ”。

接下来，您可以为该对象选择一个精灵。用鼠标左键单击在子画面框上，资产管理器将显示，并带有资产浏览器中所有可用子画面的列表：

然后，您可以选择要用于该对象的对象。将精灵指定给对象后，您可以单击“编辑精灵”按钮更改Sprite属性，或者-如果您没有该对象的Sprite但希望创建一个Sprite，则可以单击Add Sprite按钮。创建一个新的精灵资产并进行分配。您还可以通过单击“编辑图像”按钮来编辑已分配的Sprite资产图像打开图像编辑器。

防撞面罩

在精灵分配区域下方，有一个用于选择特定“碰撞蒙版”的按钮。默认情况下，对象将根据分配给它的精灵生成碰撞属性，但是在某些情况下，您可能希望碰撞基于与其他精灵不同的形状。例如，您可能有一个自上而下的游戏，其中主子画面具有矩形的碰撞蒙版，但是使碰撞更平滑时，圆形蒙版可能会更好。在这些情况下，您可以单击“碰撞蒙版”按钮并选择另一个Sprite资源，因此，该对象的实例现在将绘制已分配的Sprite，但基本碰撞会脱离Mask Sprite。您可以在Sprite Editor的页面上找到有关碰撞蒙版的更多信息。

特性

在这里，为您提供了对象的以下选项：

  • 可见：可见表示该对象的实例在房间启动时是否可见。通常情况下，大多数实例都是可见的，但有时具有不可见的实例很有用-例如，您可以将它们用作控制移动怪物的航路点，或者保持跟踪值并执行某些定时动作。不可见的对象仍将对事件做出反应，如果为它们分配了精灵或蒙版，则它们也会在其他实例与它们碰撞时做出反应……它们只是看不见且不执行其Draw Event 。默认情况下，选中可见标志。
  • 实体：将实例标记为实体时，是在告诉GameMaker Studio 2在触发Collision Event之前，它应尝试解决所有冲突。这是一项非常基本的功能，实际上，它所做的就是将实例移回执行碰撞事件本身中的DnD使用范围有限。
  • 持久性：持久性对象是在更改房间时不会消失的对象，而是“持久性”并被带入新房间的对象。仅当您通过销毁操作或在代码中明确销毁它时，它才会消失。这意味着，如果将持久性对象放置在一个房间中，则该对象在以后的所有房间中仍然可用，并且将继续执行在其对象属性中分配给事件的任何操作，但Create Event除外，该事件仅在创建事件时被触发一次是最初创建的，而不是在您切换到新房间时创建的。例如，当您有一个主要角色从一个房间移到另一个房间并且您想要将实例中的变量保持不变时，这非常方便，但是您应该非常小心地正确创建和销毁持久对象，这很容易。错误会误入您的项目。

持久性和房间层

当您创建在房间中启用了持久性的实例时，将为它分配一个图层或深度（取决于您用来创建对象实例的功能）。更改房间后，如果下一个房间没有名称或深度与所分配的名称或深度相同的图层，则将为持久存在于各个房间中的实例创建一个新图层。如果您在创建实例时给了命名层，则新层也将与原始层相同，而如果您为实例分配了深度，则新层将被命名为“_layer_XXX “， 在哪里 ”XXX “是用于为图层赋予唯一名称的十六进制值。这些图层将从Room End的房间中删除。

关于图层和持久实例的另一件事要注意的是，如果您在创建时为该实例分配了一个命名层，并且以下房间中还有另一个具有相同名称的层，则将持久化的实例分配给具有相同名称的层名称，与图层的深度无关。

最后，如果持久化的实例移动到一个具有与创建该实例相同深度的层的房间，则不会将其分配给该层，而是将在相同深度创建一个新层（遵循命名约定）以上说明）。

持久性与事件

如前所述，一个持久对象将只运行一次其Create Event ，但还将在适当的时间触发其Game Start ， Game End ， Room Start和Room End事件。但是，如果您重新启动游戏（例如， game_restart()功能），所有持久对象将被删除，并且只有在游戏再次创建时才存在。

还要注意，如果您取消激活持久对象，除非在触发“房间结束”事件之前将其重新激活，否则它将不再从一个房间传递到另一个房间。重新激活需要进行完整的游戏步骤，因此无法使用实际的Room End事件来完成此操作，因为它不能使游戏重新激活对象，因此您在使用时需要牢记这一点。与永久对象一起停用，并相应地计划代码。

  • 使用物理：选中使用物理框时，您告诉GameMaker Studio 2该对象应该是物理世界的一部分（如房间编辑。选中此复选框还将打开另一个链接的窗口，您可以在其中定义该对象所有实例的物理属性。关于此的更多信息可以在“物理对象”部分中找到。
