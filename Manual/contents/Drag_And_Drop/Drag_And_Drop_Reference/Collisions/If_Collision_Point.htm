这个动作用于检查并查看在房间的特定点上是否与一个或多个给定物体的实例发生碰撞，如果检测到碰撞，将评估为真，否则为假。你给对象 和要检查的位置--可以是房间里的绝对位置，也可以是相对于实例的位置--你还可以检查Not标志，以检查在给定位置是否有碰撞，这意味着 意味着该动作只有在没有发现碰撞的情况下才会评估为真，否则将评估为假。你也可以设置Exclude Self选项，将调用实例从碰撞检查中排除。

接下来你可以勾选/不勾选返回列表的选项。勾选这个选项意味着这个动作将创建并填充一个列表 数据结构，其中包含所有与点相碰撞的实例，并将目标变量设置为DS列表ID。 的所有实例，并将目标变量设置为DS列表的ID。如果不勾选，它只会在没有检测到碰撞时将目标变量设置为小于0的值，或者设置为碰撞中某个实例的唯一ID值。 注意，如果有多个实例发生碰撞，那么在不使用列表的情况下，你没有办法知道哪个实例的ID会被返回）。

如果你选择了返回一个列表--并且目标变量没有设置为Temp--那么你也可以选择Free Target选项。当设置为"true "时，目标变量中包含的列表将被销毁，并在每次调用动作时创建一个新的列表（并返回到目标变量）。 每次调用动作时，都会创建一个新的列表（并返回到目标变量）。通过这种方式，你可以重复使用同一个变量，而不用担心内存泄漏，因为如果你不选中Free Target选项，你就得负责在使用前销毁列表。 在动作中再次使用该变量之前，你要负责销毁该列表。请注意，不管这个选项是否被选中，如果你把动作设置为返回列表，那么目标变量将总是包含一个列表ID。 因此，当实例被销毁或房间结束时，需要释放这个变量（例如，你可以使用Clean Up事件，以及Free Data Structure 动作）。 值得注意的是，你可以将释放目标选项设置为使用一个预定义的变量或表达式，这样就可以通过在其他地方设置变量来控制列表数据结构被释放的时间（或者对表达式有不同的结果 只要它的评估结果为真/假。

最后，你提供你想保存返回值或列表ID的目标变量。如果你将保存返回值的目标变量标记为Temp（本地）变量，那么动作将创建这个变量来保存返回值直到事件结束。 在这种情况下，Free Target选项没有任何作用，如果该选项被选中，你将负责销毁DS列表。如果没有提供目标变量并且勾选了Return List选项，则不会创建列表。
