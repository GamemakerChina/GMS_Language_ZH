如果对象在地点

这个动作用于检查并查看是否有一个或多个给定对象的实例与当前实例放置在房间的某一点发生碰撞，基于调用实例的碰撞掩码。如果检测到碰撞，该动作将评估为真，否则为假。你给出要检查的对象和要 "放置 "实例的位置来进行检查--这可以是房间里的一个绝对位置，也可以是相对于实例的一个位置 - 你也可以检查Not标志，以检查在给定的位置是否有碰撞，在这种情况下，如果没有发现碰撞，该动作将评估为真，否则将评估为假。

在给出对象和位置后，你可以选中/不选中返回列表的选项。勾选这个选项意味着这个动作将创建并填充一个列表 数据结构，其中包括与实例相碰撞的所有实例，并将目标变量设置为DS列表ID。 所有与该实例发生碰撞的实例，并将目标变量设置为DS列表ID。如果不勾选，它只会在没有检测到碰撞时将目标变量设置为小于0的值，或者是碰撞中某个实例的唯一ID值（注意，如果有多个实例发生碰撞，那么在不使用列表时，你无法知道哪个实例的ID会被返回）。

如果你选择了返回一个列表--并且目标变量没有设置为Temp--那么你也可以选择Free Target选项。当设置为"true "时，目标变量中包含的列表将被销毁，并在每次调用动作时创建一个新的列表（并返回到目标变量）。 每次调用动作时，都会创建一个新的列表（并返回到目标变量）。通过这种方式，你可以重复使用同一个变量，而不用担心内存泄漏，因为如果你不选中Free Target选项，你就得负责在使用前销毁列表。 在动作中再次使用该变量之前，你要负责销毁该列表。请注意，不管这个选项是否被选中，如果你把动作设置为返回列表，那么目标变量将总是包含一个列表ID。 因此，当实例被销毁或房间结束时，这个变量需要被释放（你可以使用 清理事件来实现，例如，与Free Data Structure 动作一起使用）。) 值得注意的是，你可以将释放目标选项设置为使用一个预定义的变量或表达式，这样就可以通过在其他地方设置变量来控制列表数据结构被释放的时间（或者为表达式设置不同的结果）。 只要它评估为真/假即可。

最后，你提供你想保存返回值或列表ID的目标变量。如果你将保存返回值的目标变量标记为Temp（本地）变量，那么动作将创建这个变量来保存返回值直到事件结束。 在这种情况下，Free Target选项没有任何作用，如果该选项被选中，你将负责销毁DS列表。如果没有提供目标变量并且勾选了Return List选项，则不会创建列表。

重要的是!碰撞将只登记那些具有有效碰撞掩码的实例，即：它们有一个分配给sprite_index的精灵，或一个分配给mask_index的精灵。如果碰撞中的任何一个实例 碰撞中的任何一个实例没有碰撞掩码，那么这个碰撞将不会被检测到，不管这个实例当时正在画什么。
