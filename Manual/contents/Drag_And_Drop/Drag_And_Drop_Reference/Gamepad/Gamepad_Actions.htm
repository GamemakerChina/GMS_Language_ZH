游戏手柄动作库

GameMaker Studio 2有一些专门的动作，可以用来检测来自多个连接的游戏板的模拟和数字控制。 的控制。这些动作要求你指定一个游戏板索引值，这是一个数字（从0开始计算），代表被选中的游戏板。请注意，当一个游戏板被插入你的设备时（或被移除），那么一个异步的数字就会出现。 它被移除），就会触发一个异步的系统事件（但是要处理这个问题，你可能需要使用代码）。

游戏板 "插槽 "的索引从0开始，其中0-3插槽只适用于新输入游戏板 ，即：Xbox360控制器和兼容产品。然而，你也可以检查4-11槽，包括直接输入的游戏板，这意味着当通过这些槽连接时，你可以检测许多其他型号的控制器。

输入常量

当使用游戏板动作时，输入可以来自轴、按钮或帽子，GameMaker Studio 2将把它们分配给以下内置常量（注意，"帽子 "通常只在非标准控制器上检测到）。

恒定 描述
gp_face1 顶部按钮1（这与Xbox 360控制器上的 "A "和PS控制器上的十字架相对应）。
脸谱网2 顶部按钮2（这相当于Xbox 360控制器上的 "B "和PS控制器上的圆圈）。
gp_face3 顶部按钮3（这与Xbox 360控制器上的 "X "和PS控制器上的方块对应）。
gp_face4 顶部按钮4（这相当于Xbox 360控制器上的 "Y "和PS控制器上的三角形）。
gp_shoulderl 左肩按钮
gp_shoulderlb 左肩触发器
gp_shoulderr 右肩按钮
gp_shoulderb 右肩触发器
gp_select 选择按钮（这就是DS4控制器上的PS按钮）。
启用 开始按钮（这是PS4控制器上的 "选项 "按钮）。
gp_stickl 左摇杆被按下（作为一个按钮）。
gp_stickr 按下的右摇杆（作为一个按钮）。
gp_padu D-pad向上
gp_padd D-pad down
gp_padl D-pad左键
gp_padr D-pad右键
gp_axislh 左摇杆水平轴(模拟)
gp_axislv 左摇杆垂直轴（模拟）。
gp_axisrh 右杆水平轴（模拟）。
gp_axisrv 右摇杆垂直轴（模拟）。

下面的列表显示了目前各平台的兼容性（注意，这将随着未来的更新而改变）。

  • 窗口完全支持最多允许同时连接11个设备。请注意，在Windows上，前4个游戏板插槽（0-3）是使用XInput dll处理的，这意味着只有XBox控制器可以100%兼容。 而对于其他类型的控制器，你应该检查其余的游戏板插槽（4-11）。
  • MacOS 支持一次最多连接4台设备，而且这些设备只能是Playstation3或Xbox360类型。请注意，Mac游戏选项中的 "Buildfor Mac AppStore "选项需要关闭，才能支持手柄。 选项中的 "Build for Mac AppStore "选项需要关闭，以使垫子支持工作。
  • 乌班图 也支持GamePad输入，但你可能需要从Ubuntu软件库中安装额外的库。你可以通过打开一个命令行终端并输入以下内容来轻松完成，这将安装GUI支持 以及操纵杆本身。

sudo apt-get install jstest-gtk
sudo apt-get install joystick.

  • HTML5 游戏在大多数主要浏览器上都支持游戏手柄，但Safari除外。
  • 对游戏板的支持也扩展到了 iOS 与iCade机柜。左轴映射到摇杆控制器（尽管输入是数字的，而不是模拟的），四个 "脸部 "按钮映射到机柜前面的按钮，而四个肩部按钮映射到机柜后面的按钮。 四个肩部按钮映射到机柜背面的按钮。
  • 安卓系统 出口支持NYKO控制器和普通蓝牙控制器，但只有当它们被启用时，意味着你必须在Android游戏选项的常规部分勾选iCade/蓝牙选项。它们需要API级别12才能完全工作，应该注意的是，GameMaker Studio 2将把你的设备与之配对的任何蓝牙设备注册为连接，无论它是否真的 实际连接。因此，在分配和检查 "插槽 "时，应该考虑到这一点。
  • 在PS4上，如果你想使用触摸板追踪，你需要使用的功能是 设备_mouse_* 按钮 (没有DnD™的动作)。
