显示虚拟键盘

这个动作可以用来在运行游戏的设备上显示虚拟键盘。当你调用这个动作时，你会被要求为前三个参数中的每一个提供以下常量。

  • 键盘类型。这个常数用于设置虚拟键盘上的可用键组。可用的键盘类型有
    键盘类型 说明
    kbv_type_default 当前系统的默认键盘类型。
    kbv_type_ascii 纯ASCII的键盘。
    kbv_type_url 一个为URL输入而优化的普通键盘。通常具有".com "或其他域名键，以及"/"和". "键。
    kbv_type_email 为输入电子邮件而优化的普通键盘。通常具有"@"和". "字符。
    kbv_type_numbers 仅有数字的键盘，通常显示为数字键盘。
    kbv_type_phone 电话垫键盘。通常只有数字，有 "*"和 "#"键。
    kbv_type_phone_name 为输入电话号码和姓名而优化的键盘。通常类似于ASCII键盘，但有有限的特殊字符选择。
  • 返回类型。这个常数用于设置虚拟键盘的返回/动作键上显示的内容。可用的返回类型有。
    返回类型 描述
    kbv_returnkey_default 当前系统的默认返回键标题。
    kbv_returnkey_go 将返回键的标题设置为 "Go"。
    kbv_returnkey_google 设置返回键标题为 "Google"，或在某些情况下设置为通用搜索图标。
    kbv_returnkey_join 设置回车键标题为 "Go"。
    kbv_returnkey_next 将回车键的标题设置为 "下一步"。
    kbv_returnkey_route 将返回键的标题设为 "路线"。
    kbv_returnkey_search 将回车键的标题设置为 "搜索"，或者在某些情况下设置为一个通用的搜索图标。
    kbv_returnkey_send 将回车键的标题设置为 "发送"。
    kbv_returnkey_yahoo 将返回键的标题设置为 "Yahoo"，或者在某些情况下设置为一个通用的搜索图标。
    kbv_returnkey_done 将返回键的标题设为 "完成"。
    kbv_returnkey_continue 将返回键的标题设为 "继续"。
    kbv_returnkey_emergency 将返回键的标题设为 "紧急呼叫"。
  • 自动大小写类型。这个常数用于确定通过虚拟键盘输入的单词应该如何自动大写。可用的自动大写类型是：。
    自动大写类型 说明
    kbv_autocapitalize_none 关闭自动大写功能。
    kbv_autocapitalize_words 字词将被自动大写。
    kbv_autocapitalize_sentences 句子将被自动大写。
    kbv_autocapitalize_characters 所有字符都将被大写。

最后一个参数是启用/禁用预测性文本，如果允许，则设置为"true"，否则为"false"，但请注意，允许使用预测性文本并不意味着会使用它，因为这取决于设备上用户的偏好。 因为这取决于设备上用户的偏好。当处于预测文本模式时，虚拟键盘将不会产生正常的GameMaker Studio 2按键事件。相反，它将只更新最后按下的字符和键盘字符串变量。这是由于 无法检测用于检测按键的内部文本字段的变化是来自实际的虚拟键盘按键，还是文本建议。在这些情况下，你会希望读取键盘字符串的输入，而不是 而不是读取任何种类的原始按键输入。

还需要注意的是，根据平台操作系统的不同，用户会得到不同功能的键盘，每个目标的使用有以下注意事项。

  • UWP：UWP的实现不支持return_key_type、autocapitalization_type或predictive_text_enabled。在一个函数中传递这些参数不会导致任何错误，它们只是不会对键盘产生任何 对键盘的影响。
  • 安卓/亚马逊火。在安卓系统上，不支持键盘类型kbv_type_phone_name，将显示ASCII键盘，而返回键类型只能是kbv_returnkey_go、kbv_returnkey_search、kbv_returnkey_next。 kbv_returnkey_send或kbv_returnkey_done。如果在该平台上使用其他返回键类型，系统将默认使用下面列出的替代返回键。
