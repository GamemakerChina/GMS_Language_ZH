评论代码

当与其他人合作时，当处理一个大型项目时，甚至只是为了自己的参考和调试，在事件代码和脚本中留下注释是非常重要的。在GameMaker Studio 2中，你有各种机制 来帮助你留下关于代码部分甚至是整个代码块的注释和评论，你可以在当前编辑的对象的事件列表中看到。通过这种方式，你可以给自己和你的同事留下注释，或者解释 一段特别棘手的代码，或者甚至只是留下关于某件事情的提醒，或者你希望某件事情在未来做什么。

要留下一个简单的单行评论，你可以做的第一件事是在文本前使用// 。比如说。

//初始化变量
sr = 10;
hp = 100;

你也可以留下多行注释，以表示感谢，为调试省略一整段代码，甚至留下关于你将来要如何改变代码的注释。为此，你可以使用 /* ...*/像这样。

/*
设置 GIF 变量为初始值（这些值可以编辑，所以如果你总是使用相同的值，就不需要一直使用设置器函数）

重要！请看各个 "设置 "脚本中关于每个不同变量的广泛信息。请看个别的 "设置 "脚本，以了解关于每个不同变量的更多信息。
*/

frame_num = 30;
frame_steps = 5;
frame_width = 256;
frame_height = 256;
frame_quality = gifcam_quality_high;

除了注释之外，你还可以使用特殊的#region标签来标记代码折叠的部分，每个区域可以包含一个注释。它的工作原理是，你在一段代码的顶部添加一个#region标签--连同任何注释文本--到 然后在该部分的底部添加一个相应的#endregion标签，然后你就可以 "折叠 "代码，即：把它隐藏起来，这样你就可以减少编辑器中可见的代码行数了。 的数量。

实际的代码看起来是这样的。

#region 这是一个示例区域

width = sprite_get_width(sprite);
height = sprite_get_height(sprite);
xoff = sprite_get_xoffset(sprite);
yoff = sprite_get_yoffset(sprite);

#endregion 如果需要在此进一步注释

最后，值得一提的是，在对象编辑器中处理事件时，你可以在事件的开头使用一个特殊的"description"注释来说明 事件的一般内容是什么，这将显示在对象编辑器的事件窗口中。这应该放在代码编辑器的第一行，其格式如下。

/// @description <这是一个事件的描述>。
