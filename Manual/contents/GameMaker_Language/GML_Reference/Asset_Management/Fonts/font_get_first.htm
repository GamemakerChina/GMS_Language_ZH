font_get_first（已弃用）。

在GameMaker Studio 2中定义字体时，你可以定义一个要包括的字符范围。这是因为字体本身实际上并不包含在你的游戏中（出于法律原因），但字体的图像包含在纹理页上，这就是你的游戏要使用的东西（就像其他图形资产一样）。 字体的图像包含在一个纹理页面上，这就是你的游戏将使用的东西（就像任何其他图形资产一样）。这意味着你要尽量减少使用的字符数，只指定你的游戏所需的字符范围，以保证你的游戏不受影响。 你的游戏将需要的字符范围，以便尽可能保持纹理内存的优化。这个函数可以用来查找在你的字体资产被添加到游戏中时使用的起始字符（ASCII值）。

注意： 由于处理字体的方式发生了变化，这个函数将始终返回32，因为GameMaker Studio 2将空格字符（32）强制纳入字体，并且不允许代码低于32的字符。 允许代码低于32的字符。在未来的更新中，将添加新的函数来完成类似的任务。

语法。

font_get_first(ind)。

争论 描述
ind 要检查的字体的索引。

返回。

真实的

例子。

num = font_get_first(fnt_Main)。

上述代码将存储 "fnt_Main "中索引的字体范围的第一个字母的ASCII值。
