font_replace_sprite

有了这个函数，你可以用一个 "精灵条 "来替换之前创建的精灵字体资产，其中每个子图像将是一个单独的符号或字母。索引是要使用的第一个字符的UTF8映射号，使用的范围 使用的范围与普通字体相同，以"！"开始，但请记住，你应该只包括你需要的内容，而且你应该使用 ord()函数来获得正确的 UTF8值的起始字母。

你还可以将字体的间距设置为比例（true）或非比例（false），其中比例字体的间距是基于字母的实际宽度（所以字母 "i "比字母 "w "占用的空间要小。 例如，字母 "i "比字母 "w "占用的空间要小），而非比例字体将根据子图像的宽度来确定间距，就像单行线字体一样。最后，你可以定义书写时每个字母之间的空间，这个值可以是任何整数，0表示没有空间（如果是比例字体，字母会 "接触"）。 字母将按比例 "接触"）。)

如果该函数失败，那么它将默默地失败，而不会替换给定的精灵字体。

值得注意的是，字符图中不能有任何空格，因此，如果你，例如，使用'.'和'0'，但不使用'/'，你需要包括'/'，或者在两个使用的字符之间留下一个空白子图像。 或者在两个使用的字符之间留下一个空白的子图像。下面的图片显示了一个典型的精灵字体布局。

语法。

font_replace_sprite（ind, spr, first, prop, sep）。

争论 描述
ind 应被替换的字体的索引（即在此函数之后对该资源的所有引用将指的是新字体，而不是原来的字体）。
洒水 要添加字体的精灵，基于此。
首先 第一个可用字符的索引。
榰 每个字母周围的空白是否应该被删除（如果是假的，创建的字体将是单行的，否则就会像大多数普通字体一样）。
绵绵不绝 每个字母之间的空间，0意味着一个字母的右边刚好接触到下一个字母的左边。

返回。

不适用

例子。

font_replace_sprite(global.Font, working_directory + "Sprites/spr_Font.png", ord("！"), true, 2)。

上述代码将用一个来自外部图像文件的新字体资产替换全局变量 "Font "中索引的字体资产。

返回。字体
