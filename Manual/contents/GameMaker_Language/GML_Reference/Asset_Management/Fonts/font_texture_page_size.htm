font_texture_page_size

这个内置变量可以用来获取或设置纹理页面的大小，当使用函数 font_add().在使用该函数添加字体时，GameMaker Studio 2将 创建一个所需字形的纹理页缓存，其大小由该变量定义。这个过程如下。

  • 你使用的每个字符都会从字体中缓存到一个纹理页中
  • 当纹理页满了（因为你使用了大的字符或很多不同的字符），GameMaker Studio 2将从缓存中删除字符，并用你需要的新字符替换它们。
  • GameMaker Studio 2尽量不把纹理做得太大，因为不是所有的设备都能应付大的纹理，它也尽量不把纹理做得太小，因为在纹理上删除和重新添加字符需要 CPU 时间 并导致你的游戏运行缓慢

如果你认为你需要一个比GameMaker Studio 2使用的默认1024px更大或更小的纹理页面尺寸，你可以使用这个内置的变量来设置这个值。如果你没有使用font_add()函数。 那么设置这个变量将不会对你的游戏性能产生影响，因为在IDE中添加的字体会被缓存在常规的纹理页中（你可以通过字体编辑器和目标平台的游戏选项来控制）。 目标平台的游戏选项）。)

语法。

font_texture_page_size。

返回。

实数（最大宽度/高度，像素）。

例子。

font_texture_page_size = 512;
newfont = font_add("Arial", 24, true, true, 32, 128) 。

上述代码将字体缓存纹理页的最大尺寸设置为512px，然后在游戏中添加一种新字体。
