深度

当你创建一个对象时，你可以给它分配一个初始深度，它定义了当游戏进行时该对象的实例将如何在房间里绘制，这个变量可以用来获取和改变深度值。 这个变量可以用来获取和改变游戏运行时的深度值。你通常不需要使用这个变量，因为你应该设置实例被添加到独立的层中，而这些层又被设置为一个特定的深度。 的深度，在这种情况下，一个 "临时图层 "将在给定的深度为该实例专门创建。注意，当没有实例在同一深度时，这个临时层将从内存中删除 不像普通的层，即使它们上面什么都没有，也会保留下来）。

重要的是!当你修改深度变量并由GameMaker Studio 2管理图层时，内置的 层数变量将返回-1，而不是 层ID，因为管理的层不能通过代码来操作。

在GameMaker Studio 2中，一个实例的深度值越小，该实例就会被画得 "离摄像机越近"，而深度值越大，就意味着该实例会被画得 "离摄像机越远"，也就是说。 -1000被画在-100的上面，而-100被画在0的上面，0被画在100的上面，100被画在1000的上面。

注意，具有相同深度的对象实例可以被绘制在彼此的上方或下方，即使它们在房间编辑器中看起来是一个放在另一个上面。如果你想保证某个东西被画在其他东西的上方或下方，你应该总是明确地设置深度（或层）。还要注意的是，有一个最小 (-16000)和最大(+16000)深度值，超出这个值的实例将不会被绘制，尽管它们仍然存在并处理事件。

注意： 你不能在一个实例的绘制事件中设置它的深度（所有其他事件都可以）。

语法。

深度。

返回。

实数（单精度浮点值

例子。

if y != yprevious
{
depth = -y;
}

上面的代码将检查Y的位置是否有变化，如果有，那么深度也将被设置为与之对应。
