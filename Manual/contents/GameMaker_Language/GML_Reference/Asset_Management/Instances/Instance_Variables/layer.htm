层

这个内置变量是为房间里的每个实例创建的，它包含了该实例被分配到的层的ID值。这个值可以在其他函数中使用，比如 layer_get_depth()或者 它可以被改变以将实例移动到另一个层，但请注意，如果被分配的层在当前房间中不存在，那么你会得到一个错误，这将迫使你的游戏关闭。当分配一个图层时，你必须提供唯一的 的图层ID（当使用命名的房间图层时），你必须提供由函数返回的 layer_get_id()(当使用命名的房间图层时)，或由函数 layer_create()(当 当你在运行时创建你自己的层时）。)

重要的是！如果你已经用 instance_create_depth()函数创建了实例，或者手动改变了 深度 变量，分配给实例的图层就会变成一个 "被管理的 "图层，也就是GameMakerStudio 2自动控制和管理的图层。在这些情况下，图层变量将返回-1。

语法。

层。

返回。

实数（整数 - 资源索引值）。

例子。

layer = layer_create(-1000);

上面的代码将创建一个深度为-1000的新图层，然后将实例的图层变量设置为返回的图层ID，将实例从它当前所在的图层移到正在创建的新图层。
