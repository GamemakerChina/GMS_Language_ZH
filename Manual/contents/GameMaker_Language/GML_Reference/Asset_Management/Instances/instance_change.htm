实例_改变

你可以用这个函数把一个对象的一个实例变成另一个不同对象的实例，同时决定是否执行初始实例Destroy和Clean Up事件以及新实例Create事件。通过这种方式。 你可以让（例如）一个炸弹变成一个爆炸--在这种情况下，"perf "参数可能是真的，因为你希望炸弹执行它的Destroy Event和Clean Up Event，以及爆炸执行它的Create 事件--或者你可以让你的玩家角色变成另一个角色--在这种情况下，"perf "参数可能是假的，因为你不希望实例执行它们的创建和销毁/清理事件。

值得注意的是，改变实例意味着在下一步之前，你不应该对该实例进行进一步的操作，特别是试图访问变量等......因为这将导致错误。基本上，你改变了这个实例，但是 在当前步骤结束之前，它实际上是不可用的，所以直接访问它所包含的任何变量（例如，调用obj_Changed.x）将无法工作。

WARNING!当改变一个启用了物理的实例时，物理属性将不会被转移到被改变的新实例中。因此，你应该有代码将当前实例的物理状态 "转移 "到新实例上。 的物理状态转移到新的实例上，或者在其创建事件中或在对象编辑器中定义新实例的物理属性。由于这个原因，我们建议你不要在启用物理的实例中使用这个函数，而应该使用 的组合。 instance_destroy()和instance_create_layer()。

语法。

instance_change(obj, perf);

争论 描述
obj 调用对象将转变为的新对象。
香水 是否执行该新对象的创建和销毁事件（true）或不（false）。

返回。

不适用

例子。

if keyboard_check(vk_enter)
{
instance_change(obj_Player_Swimming, false);
exit;
}

上面的代码将把调用的实例变为 "obj_Player_Swimming "的实例，当用户按下 "Enter "键时，既不执行原实例的销毁事件，也不执行新实例的创建事件，完成后退出该 代码块，完成后退出。
