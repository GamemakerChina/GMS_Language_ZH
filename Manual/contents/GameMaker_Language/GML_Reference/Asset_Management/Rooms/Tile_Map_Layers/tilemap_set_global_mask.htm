tilemap_set_global_mask

这个函数可以用来为游戏中的所有瓷砖地图设置瓷砖位掩码。这个想法是，如果你有，例如，相当小的16 x 16瓷砖集，每个瓷砖集将有总共256个不同的瓷砖，这只用了瓷砖数据 "blob "中的8位索引（从可用的19位数据中）。 占用了瓷砖数据 "blob "中索引的8位（从可用的19位数据中）。这些比特可以被 "屏蔽"，然后瓦片数据的其余部分可以被用来存储额外的值。

得到初始索引掩码值的一个简单方法是，将瓷砖组中的瓷砖数量减去1，所以一个16x16的瓷砖组有256块瓷砖，掩码值为255（或十六进制的$ff），如果瓷砖组中的瓷砖数量不是2的幂。那么就四舍五入到最接近的2的幂数来得到掩码，例如一个20x20的瓷砖组有400块瓷砖，你可以四舍五入到512，因此得到的掩码索引值是511（十六进制的$ff）。除了索引掩码之外。 通常你也想保留分配给瓷砖的翻转、旋转和镜像值，所以为了帮助解决这个问题，有几个常数可以使用。这些常数可以与索引掩码值一起 "或"，以保存 那些数据位。以下常数可以使用（用法见下面的示例代码）。

恒定 描述
瓷砖_旋转 用于设置/获取瓦片数据blob的旋转位。
瓷砖_镜面 用于设置/获取瓦片数据blob的镜像位。
瓷砖_翻转 用于设置/获取瓦片数据blob的翻转位。
瓷砖_index_mask 一个特殊的常数，用于与瓦片数据blob进行 "和"，以提取瓦片索引。

瓦片图位屏蔽是一个高级功能，允许你使用瓦片数据 "blob "的位，供你自己使用。例如，用户可以在瓦片数据结构中保留一个位来表示 "实体"，然后可以检查这个位是否有 快速碰撞。基本上，当绘制瓷砖地图时，位掩码与瓷砖数据 "和 "在一起（所以它不影响任何其他功能），这意味着不在掩码中的位被忽略，但仍然可以 但仍然可以被用户为他们自己的目的读和写。请注意，有两个不同的掩码--一个是全局掩码，一个是每张瓷砖地图掩码--在用于绘制之前，GameMakerStudio 2在内部将这些掩码 "和 "到一起。

例子。

var mask = tile_mirror | tile_flip | tile_rotate | 255;
tilemap_set_global_mask（mask）。

上面的代码将全局瓦片地图的位掩码设置为8位，保留了地图中每个瓦片的翻转、镜像和旋转的值。
