添加精灵

通过这个函数，你可以添加一张图片作为精灵，从外部来源加载，要加载的图片文件应该总是 *.png、*.gif、*.jpg/jpeg或*.json格式（*json文件用于加载用Spine制作的骨架动画精灵）。该函数返回新的精灵索引，然后必须在所有与该精灵有关的后续代码中使用。如果 你在HTML5中使用这个函数，或者从URL中获取图像，这个函数也会产生一个图像加载的异步事件。更多信息请参见本页面 更多信息。

所有要变成动画精灵的图像--除了*.gif（见下文）--都应该有一个 "条 "的格式（见下图），它将被分割成指定数量的子图像，规则是精灵宽度=条宽度/子图像。

正如你在上图中看到的，精灵被放置在一个深紫色的背景上，这可以通过设置 "removeback "参数为true来移除。它的工作原理是检查精灵的左下角像素的 的颜色，然后将其作为要移除的颜色。例如，在上图中，如果我们把左下角的像素颜色定为绿色，那么精灵的所有绿色部分都会被移除，而紫色背景的其他部分则被忽略。

如果你选择了 "removeback "选项，你可能还希望GameMakerStudio 2通过将 "smooth "参数设置为true来平滑精灵的边缘。这样做的目的是在精灵的边缘创建一个半透明的边框。 的边缘创建一个半透明的边框，然后把它的背景去掉。

最后，你还可以指定精灵的X和Y原点。这是精灵 "固定 "在使用它的实例上的点，并且总是相对于一个精灵子图像的左上角0,0计算。所以。 例如，一个32×32像素的精灵，这些值设置为(16,16)，它的原点将在中心。

如果你添加的是*.gif格式的图像，那么你只需要指定文件名和图像编号（第一帧应该是1），其他参数可以都留为0，注意只有GIF的第一张图像会被加载。 对于*.json精灵，图像编号应该是1，而 "removeback "和 "smooth "的设置将被忽略。

默认情况下，所有新的精灵都会自动计算其边界框（确切的边界框将取决于精灵的大小和透明度），但是你可能希望对此进行自定义，在这种情况下，你还应该使用函数 sprite_collision_mask().

应该注意的是，以这种方式添加精灵时使用的内存会比你预期的大。这是因为GameMaker Studio 2将把精灵作为一个纹理页来存储，它也将被存储在GPU内存中，所以总内存 将比相同精灵的图像文件所预期的要大。

注意：当你将一个精灵加载到GameMaker Studio 2中时，你必须记得在不再需要它时再次将其删除（用 sprite_delete())，否则会有内存泄漏的风险，这将减慢并最终导致游戏崩溃。 最终使你的游戏崩溃。

注意：在 iOS时，如果你从一个文件夹中加载一个包含的文件，即：路径类似于"背景/background1.png"，你不需要把文件夹作为路径的一部分。 路径的一部分，只需提供文件名即可。在所有其他平台上，你必须提供文件夹作为文件路径的一部分。

注意：在HTML5上，如果你从一个安全的服务器加载一个精灵，你可能需要设置跨源类型，并使用相对路径而不是绝对路径。参见 http_set_request_crossorigin()以了解 更多细节。

语法。

sprite_add(fname, imgnumb, removeback, smooth, xorig, yorig)。

争论 描述
fname 要添加的文件的名称（一个字符串文件路径）。
印象派 用来表示子图像的数量（1用于单张图像或用于*.gif）。
撤回 表示是否使所有具有背景颜色的像素（左下角的像素）透明。
顺利 表示如果是透明的，是否要平滑边缘。
原产地 指明原点在精灵中的X位置。
原住民 指明原点在精灵中的y位置。

返回。

实数（精灵ID值

例子。

spr = sprite_add("player_5.png", 16, true, true, 0, 0)。

上述代码将一个精灵加载到游戏中，并将其索引存储在变量 "spr "中。
