推送通知

在移动设备上，通常在任何时候只有一个应用程序可以在前台活动，但许多游戏和应用程序在一个基于时间或相互连接的环境中运行，当应用程序不在前台时，用户感兴趣的事件就会发生。在这些情况下，本地和远程推送通知可以让这些游戏在事件发生时通知他们的用户。

当操作系统传递本地或远程推送通知，而目标应用程序不在前台运行时，它会以横幅的形式向用户展示通知。如果有一个通知提示，并且用户点了 就会启动游戏，并调用一个异步推送事件，你可以检查你指定的回调字符串，并根据其值解决推送的问题。 基于它的值。如果通知送达时，应用程序正在前台运行，那么应用程序仍然会收到一个异步推送事件，但用户不会得到任何显示的信息，除非你自己创建它。

支持的平台之间有一些需要注意的区别。

  • 在安卓系统中，所有在应用程序未运行/在后台时收到的通知将被存储，并在下一次应用程序运行时由异步事件传递数据，你需要在安卓游戏选项中启用推送通知，以便它们发挥作用。
  • 在iOS上，特定通知的数据只有在通知被选中时（或在收到通知时应用在前台运行）才会被传送，你需要在iOS游戏选项中启用推送通知，才能发挥作用。

本地通知

本地推送通知是在游戏安装的设备上进行的，不需要后端服务器。你只需设置通知的日期和时间，如果游戏目前没有运行，它就会显示给用户。这种类型的 这种类型的通知对于设置 "提醒 "用户再次玩你的游戏，或提供每日游戏奖励等非常有用。 当用户点击通知时，它将启动游戏并传入一个数据字符串，然后由游戏解析，以给予奖励或其他什么。

注意：该功能仅限于在 iOS和 安卓系统目标模块。对于 安卓系统你将需要安装谷歌推送通知扩展。

以下功能可用于本地通知。

  • push_local_notification
  • push_get_first_local_notification
  • push_get_next_local_notification
  • push_cancel_local_notification

远程通知信息由服务器发送至设备平台应用商店提供的服务，然后将这些信息转发到所有安装了你的应用程序的设备上。GameMaker Studio 2在iOS和Android目标模块上支持这个功能。 iOS和Android目标模块上的支持。

GameMaker Studio 2中没有处理远程通知的功能，因为它们都必须由你的服务器生成并由各自的应用商店处理。然而，一旦设置正确，GameMaker Studio 2游戏将收到这些通知。 然后可以在异步推送事件中进行处理，就像你处理本地通知一样。

请注意，不能保证远程推送通知会被传递，而且允许的数据有效载荷相当小。这在不同的平台上是不同的，但iOS的限制特别大--苹果服务只提供最新的 通知，收件人必须选择该通知，才能将有效载荷数据传递给你的异步事件，这些通知的最大有效载荷大小为256bytes。通常情况下，远程推送通知只是表明新的数据 例如，从你的服务器上可以获得新的数据。

注意：实施服务器端完全取决于最终用户，除了从他们的知识库中获得基本设置信息外，YoYo Games不提供任何这方面的支持。
