buffer_async_group_begin

当你想开始将多个缓冲区保存到多个文件时，这个函数被调用。groupname "是一个字符串，将被用作保存文件的目录名，并在加载文件到文件路径时作为文件的一部分使用。 在以后加载文件到IDE中时应作为文件路径的一部分（使用任何一个 buffer_load()函数）。)该函数仅用于与 buffer_save_async()函数 一起使用，而且你必须通过调用 buffer_async_group_end()函数，否则文件将不会被保存出来。

请注意，对于控制台平台（例如PS4），"组名 "将被用作保存槽的描述，使用这个功能可以帮助你避免每一个被保存出来的文件的UI显示。还请注意，当 使用UWP时，你可以在本地XBox和微软商店窗口之间漫游你的保存游戏，例如，通过创建一个与ERA Xbox组相同的组。

如果os_type == os_uwp
{
buffer_async_group_begin（"root/yourgroupname"）。
}
else
{
buffer_async_group_begin("yourgroupname")。
}

语法。

buffer_async_group_begin(groupname)。

争论 描述
组名 组的名称（作为一个字符串）。

返回。

不适用

例子。

buffer_async_group_begin("SaveGame");
save1 = buffer_save_async(buff1, "Player_Save1.sav", 0, 16384);
save2 = buffer_save_async(buff2, "Player_Save2.sav", 0, 16384);
save3 = buffer_save_async(buff3, "Player_Save3.sav", 0, 16384);
save4 = buffer_save_async(buff4, "Player_Save4.sav", 0, 16384);
buffer_async_group_end() 。

上面的代码启动了一个缓冲区组，然后将其设置为异步保存4个文件。然后结束该组的定义（此时开始保存）。
