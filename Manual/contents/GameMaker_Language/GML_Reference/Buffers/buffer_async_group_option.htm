buffer_async_group_option

通过这个功能，你可以为正在保存的缓冲区组设置一些特定的平台选项。可用的选项如下。

  • "subtitle" 或" slottitle" - 这个选项的值是一个字符串，当用户在操作系统中管理他们的保存数据时，它将会显示给用户。这只在保存数据时很重要，在再次加载时不重要。
  • "showdialog" - 这个选项的值是一个布尔值true或false。如果设置为true，当你保存或加载GameMaker Studio 2时，将显示系统UI，否则它将进行quicksave/quickload 不显示用户界面。通常只有当你支持多个保存槽，允许用户选择一个槽时才需要这个选项，但如果你只支持每个用户一个槽，就把它设置为false。
  • "savepadindex" - 对于这个选项，你将指定正在保存或加载的玩家的pad索引，系统将向这个玩家的保存文件夹写入数据并从其读取数据。
  • "saveslotsize" - 这个选项要求你指定你想保存的实际大小（以字节为单位）（所以你可以做一个 buffer_seek()和 buffer_tell()来获得 例如，你可以用 "和 "来获得这个值）。)请注意，提供这个值不是必须的，因为所有的保存都预先分配了一个最小空间，通常是10MB左右。

重要的是!该功能目前只对PSVita、PS4和XBox One目标模块有效。在所有其他目标上，它不会有任何作用。

语法。

buffer_async_group_option(option, value)。

争论 描述
选择 的选项来设置。
价值 要设置的值（可以是字符串或实数，取决于选项）。

返回。

不适用

例子。

buffer_async_group_begin("save_folder_name");
buffer_async_group_option("showdialog", false);
buffer_async_group_option("slottitle", "Catch The Haggis Save");
buffer_async_group_option(" subtitle", "你所有的羊杂碎都保存在这里！"）;
save = buffer_save_async(buff, "Player_Save.sav", 0, 16384);
buffer_async_group_end();

上面的代码启动了一个缓冲区组，然后在设置4个文件异步保存之前设置了组的选项。然后结束组的定义（这时将开始保存）。
