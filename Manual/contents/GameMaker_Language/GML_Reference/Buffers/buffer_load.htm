缓冲区_加载

这个函数用来加载以前用 buffer_save()函数保存的缓冲区，以及任何包含的文件或外部加载的文件。 从外部加载。它将返回一个新的缓冲区的ID，这个ID是由函数创建的 "增长 "缓冲区（见这里），字节对齐为1。 函数调用这个缓冲区时使用。如果由于某种原因加载失败，该函数将返回-1，并在编译器输出窗口中显示一条消息，说加载失败了。

请注意，在HTML5上，任何从本地存储加载的缓冲区都将使用base64解码（因为它们被保存为base64字符串），但在Included Files或任何外部文件上使用此函数将不使用base64解码，这在大多数情况下应该是好的 对大多数情况来说应该是好的。

注意：当你不再需要这样的动态创建的资源时，必须将其从内存中删除，以防止内存泄漏，所以当你用完你创建的缓冲区后，应该再次释放它，这一点很重要。 使用 buffer_delete().

语法。

buffer_load(filename)。

争论 描述
文件名 要加载的文件的名称。

返回。

真正的

例子。

player_buffer = buffer_load("Player_Save.sav")。

上述代码将把以前保存的缓冲区加载到内存中，创建一个新的缓冲区。这个新缓冲区的索引被储存在变量 "player_buffer "中。
