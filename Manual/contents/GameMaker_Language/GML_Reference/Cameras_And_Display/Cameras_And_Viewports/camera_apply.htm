摄像机_应用

这个函数将把给定的摄像机设置立即应用于正在渲染的当前摄像机视图。虽然你可以使用特定的camera_set_*函数来设置任何视角的摄像机，但新的设置将不会被用于渲染，直到下一帧。 但通过这个函数，你可以立即做出这些改变。如果你使用的是多个视图，那么这个函数应该在绘制事件中使用，你可以使用变量 view_current 来选择性地将给定的摄像机只应用于当前正在渲染的摄像机视图。请注意，这并不改变摄像机的ID值，只是改变该摄像机的设置，如果你在默认的摄像机上使用这个函数（即：GameMaker Studio 2的内部摄像机在渲染时使用的 如果你在默认摄像机（即：GameMaker Studio 2在没有视图摄像机时使用的内部摄像机）上使用这个函数，那么在下一个游戏帧中，这些变化将被绘制房间的内部设置所覆盖。

语法。

Camera_apply(camera_id)

争论 描述
摄像机_ID 创建摄像机时返回的唯一摄像机ID值。

返回。

不适用

例子。

如果view_current == 0
{
camera_apply(cutscene_cam)。
}

上面的代码（在绘图事件中）检查当前正在渲染的摄像机视图，如果是摄像机视图[0]，那么变量 "cutscene_cam "所引用的摄像机的设置就会应用到它。
