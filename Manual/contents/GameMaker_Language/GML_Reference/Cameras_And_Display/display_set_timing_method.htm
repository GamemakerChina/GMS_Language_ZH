display_set_timing_method

这个函数可以用来设置渲染游戏时使用的计时方法。该方法可以是下列常数之一。

恒定 描述
tm_sleep 使用睡眠余量值作为主要计时方法
tm_countvsyncs 使用vsync计时作为主要的计时方法（所有支持的平台都默认）。

vsync计时方法使用目标平台对垂直同步的支持来为游戏的渲染计时计算提供一个锚点，而将模式设置为睡眠保证金将只是试图确保每一帧持续正确的 而将模式设置为休眠模式，只是试图通过等待或休眠来确保每一帧持续正确的时间（比如1/30秒或1/60秒）。一般来说，默认的vsync时间将给出最平滑的结果，但请注意，当使用vsync方法时，睡眠保证金的值仍然是相关的，尽管它的影响会减少。 我们建议在使用这种方法时将其保持在默认值。

在PS4、Ubuntu和HTML5之外的所有平台上，GameMaker Studio 2默认会使用vsync计时方法，而在不支持的平台上，只能使用睡眠边缘计时。

如果你想设置睡眠保证金，你可以使用函数 display_set_sleep_margin()来实现，你可以通过函数找到当前正在使用的计时方法 display_get_timing_method().

语法。

display_set_timing_method(method)。

争论 描述
方法 要使用的计时方法（见上面的常数列表）。

返回。

不适用

例子。

如果 display_get_timing_method() != tm_sleep
{
display_set_timing_method(tm_sleep);
if display_get_sleep_margin() != 20
{
display_set_sleep_margin(20);
}
}

上面的代码将检查计时方法，然后如果它没有被设置为tm_sleep，它将被设置为tm_sleep，并且睡眠幅度设置为20。
