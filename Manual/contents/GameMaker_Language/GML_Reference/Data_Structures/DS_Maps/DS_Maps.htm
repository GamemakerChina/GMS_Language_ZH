DS地图

DS地图数据结构是一个非常有用的结构，因为它可以让你存储键和值对。例如，在你的游戏中，一个角色可以拥有一定数量的不同物品（键），对于每个单独的物品，他可以有一定数量的 值），比如在RPG游戏中，你可以有10个健康药水、5个法力药水和100个金币。地图将这些物品对集中在一起，你可以在地图上添加物品对，并使用一些简单的函数搜索与某些键对应的值 也可以使用一些简单的函数搜索对应的值。然而，在你使用地图之前，有几件事情你应该了解一下!地图不是以任何（可识别的）方式进行排序的，这意味着要找到某个键，你可能需要遍历整个 整个过程（这很慢）。也没有办法保持两个相同的键，你也不能给一个键分配两个值。

注意：与所有的动态资源一样，数据结构会占用内存，所以当不再需要时，应该总是销毁，以防止内存泄漏，这将减缓并最终使你的游戏崩溃。

注意：如果你需要检查一个数据结构是否存在，你可以使用 ds_exists()函数。

除了下面列出的特定函数外，你都可以使用一个表达式（称为访问器）来添加或修改你的DS地图的内容。这个访问器看起来类似于一个1D数组，语法如下。

map_index[? key]

你可以从GML概述的Accessors页面中找到更多的信息和例子。

  • ds_map_find_next
  • ds_map_find_previous
  • ds_map_find_value
  • ds_map_keys_to_array
  • ds_map_values_to_array
  • ds_map_set
  • ds_map_read
  • ds_map_write
  • ds_map_destroy

还有四个函数可用于保存和加载DS地图。这些功能将混淆地图，并将其存储在目标平台上的一个安全位置，使用的文件格式意味着最终文件不能在设备间传输 在设备之间传输。

  • ds_map_secure_save
  • ds_map_secure_save_buffer
  • ds_map_secure_load
  • ds_map_secure_load_buffer

DS地图的用途之一是在处理JSON时，因此有一些专门的功能。

  • ds_map_add_list
  • ds_map_add_map
  • ds_map_replace_list
  • ds_map_replace_map
  • ds_map_is_list
  • ds_map_is_map

注意：虽然这些函数允许你在地图中添加列表和地图，但它们对JSON以外的任何东西都是无用的，如果写入磁盘或以任何其他方式访问，嵌套的地图和列表将不会被正确读取。
