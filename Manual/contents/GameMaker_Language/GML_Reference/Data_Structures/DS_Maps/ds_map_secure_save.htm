ds_map_secure_save

这个函数将把给定的DS地图的内容保存到一个与它所创建的设备相链接的文件中（意味着如果转移到任何其他设备上，它就不能被读取）。文件本身可以有几乎任何扩展名（例如，*.dat, *.json，*.bin，等等），并将被混淆并存储在目标平台的本地存储中。然后，你可以用以下函数重新加载ds_map ds_map_secure_load().注意，如果 被保存的DS地图包含一个数组，这个数组在保存时将被转换为DS列表。

重要的是!安全保存的文件的特点之一是它被锁定在创建它的设备上，所以你不能把保存在一个设备上的文件加载到另一个设备上运行的项目中。

语法。

ds_map_secure_save(map, filename)。

争论 描述
地图 要使用的数据结构的ID
文件名 要保存地图的文件名

返回。

布尔型

例子。

ds_map_secure_save(purch_map, "p_data.dat")。

上述代码将把变量 "p_data "中索引的DS地图保存到给定的文件中，以便以后检索。

返回。DS地图
