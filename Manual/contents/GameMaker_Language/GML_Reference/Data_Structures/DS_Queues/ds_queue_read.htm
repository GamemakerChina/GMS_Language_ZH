ds_queue_read

通过这个函数，你可以重新创建一个已保存的DS队列（一个以前被写成字符串的队列，使用 ds_queue_write()).你必须首先创建一个新的DS队列来读取字符串，如果DS队列已经存在并且有信息存储在其中，那么在读取之前将清除它。 存在并且有信息存储在其中，那么在读取之前将被清空。在为你的游戏创建保存/加载机制时，这个函数是非常重要的。注意，如果该数据结构是用以前的 GameMaker 创建的，你应该添加可选的参数 "legacy"，将其设置为 "true"，因为这个版本的字符串格式已经改变。

语法。

ds_queue_read(id, str [, legacy])。

争论 描述
身份证 要读入的数据结构的ID。
弦外之音 要读取的字符串。
遗产（可选 可以是真或假，也可以完全省略。

返回。

不适用

例子。

queue = ds_queue_create();
ini_open(" save.ini");
var str = ini_read_string("Queues", "0", "");
if (str != " ")
{
ds_queue_read(queue, str);
}
ini_close()。

上面的代码创建了一个队列，并将索引存储在变量 "queue "中。然后，它打开一个ini文件并从中读取一个字符串，首先检查以确保该字符串没有被返回为空。然后，这个字符串被读到 新建的DS队列。
