调试

在制作游戏的过程中，通常非常重要的一点是，你要知道游戏机制在特定时间点的某些细节，或者能够为某些功能输入某些键值，以测试游戏的不同方法和想法。为此 为此，GameMaker Studio 2为你提供了一些调试函数，这些函数在调试模式下（或在Windows目标上）会显示信息并要求输入，但在为其他目标模块编译时，这些函数会被忽略，原因是 这些函数使游戏处于闭环状态，直到输入被解决。

注意：尽管它们可能在某些平台上工作，但它们不能用于除调试以外的其他用途。

  • debug_mode
  • debug_event
  • debug_get_callstack
  • 异常_unhandled_handler
  • get_integer
  • get_string
  • show_error
  • 显示信息
  • 显示_问题
  • show_debug_message
  • show_debug_overlay
  • 代码_is_compiled
  • fps
  • fps_real

除了这些专业功能，为了帮助你创建游戏，GameMaker Studio 2还有一长串错误信息，可以在编写游戏代码、编译游戏或游戏本身运行时显示。 这些信息指出了有关的具体错误，可以用来确定特定对象或脚本功能中的问题，对调试游戏的过程有很大帮助。然而，错误信息的确切含义可能并不总是那么明显，因此下面的章节将介绍这些信息。 错误信息的意思，所以下面的章节列出了所有可能的信息，并解释了它们的含义，以使你的生活更轻松。

  • 跑步者的错误
