gpu_set_blendmode

当GameMaker Studio 2要绘制一个像素时，有一个源颜色（我们要绘制的像素的颜色）和一个目标颜色（我们要绘制的像素中已经有的颜色），所以当确定像素的新颜色时 时，源色和目的色是根据所选择的混合模式来计算的。颜色的每一个分量都被存储为0到1之间的浮点值，新的颜色是通过将源颜色的每一个分量乘以某个系数来计算的。 源颜色的每个分量乘以某个系数，目标颜色的每个分量乘以另一个系数，然后将结果逐个分量相加来计算。

这个函数允许你使用以下常数以四种基本方式设置混合模式。

恒定 描述 扩展混合模式
bm_normal 正常混合（默认混合模式）。 (bm_src_alpha,bm_inv_src_alpha)
bm_add 加法混合。光线区域的发光度值被添加。 (bm_src_alpha,bm_one)
bm_subtract 减法混合。光线区域的发光度值被减去。 (bm_zero,bm_inv_src_colour)
bm_max 最大限度的混合。类似于加法混合。 (bm_src_alpha,bm_inv_src_colour)

从上表可以看出，这四种混合模式实际上是扩展混合模式的组合，可以在描述的页面上找到 gpu_set_blendmode_ext().

为了帮助你从混合模式中获得最大的收益，并帮助理解它们是如何工作的，以及它们是如何影响绘制在屏幕上的最终图像的，我们建议你阅读以下指南。

  • Blendmodes使用指南

语法。

gpu_set_blendmode(mode)。

争论 描述
模式 要使用的混合模式（见上表）。

返回。

不适用

例子。

gpu_set_blendmode(bm_add);
draw_circle_colour(100, 100, 50, c_white, c_black, 0);
gpu_set_blendmode(bm_normal) 。

这将使黑色变成透明，创造出一种 "发光 "的效果，白色在外面很强，从圆心开始逐渐减弱。混合模式在画完圆后被重置，以确保加法混合 不被应用于它之后的其他东西。
