部分_类型_斯普利特

这个函数可以用来设置一个粒子类型，以使用游戏资产中的一个自定义精灵。你可以选择精灵，然后告诉GameMaker Studio 2是否使用该精灵并为其制作动画，如果你选择动画，那么你还可以选择 让子图像以一定的间隔变化，这样动画就会在创建的每个粒子的寿命开始和结束时开始和结束（因此，如果你把寿命设置为最小15步，最大30步，无论 寿命的数值，动画都会被制作成适合的，寿命短的动画运行得快，寿命长的动画运行得慢）。)你也可以选择选择一个随机的子图像，这样，如果精灵没有动画，就会随机选择一个不同的图像来创建精灵。 随机选择一个不同的图像来创建精灵，或者如果粒子是有动画的，它将在一个随机点开始动画。

语法。

part_type_sprite(ind, sprite, animate, stretch, random)。

争论 描述
ind 要改变的粒子类型的索引。
精灵 要设置粒子类型的精灵的索引。
动画 是否正常跟随精灵的动画（true）或不跟随（false）。
伸展 是否拉伸精灵的动画以匹配粒子的寿命（true）或不（false）。
随机 是否选择一个随机的子图像（真）或不（假）。

返回。

不适用

例子。

global.p1 = part_type_create();
part_type_sprite(global.p1 , spr_Coins, 1, 0, 0);
part_type_size(global.p1, 1, 3, 0, 0);
part_type_scale(global.p1, 1, 1);
part_type_colour1(global.p1, c_white) ;
part_type_alpha2(global.p1, 1, 0);
part_type_speed(global.p1, 2, 4, 0, 0);
part_type_direction(global.p1, 0, 180, 0, 0);
part_type_gravity(global.p1, 0.20, 270);
part_type_orientation(global.p1, 0, 0, 0, 1);
part_type_blend(global.p1, 1);
part_type_life(global.p1。 15, 60);

上述代码将全局变量 "p1 "中索引的粒子类型设置为 "spr_Coins "中索引的精灵的形状，然后设置其他各种粒子属性。
