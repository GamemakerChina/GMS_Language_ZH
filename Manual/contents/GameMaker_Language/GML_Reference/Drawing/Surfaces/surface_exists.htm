表面_存在

由于表面的不稳定性，这个功能在处理表面时是必不可少的。表面总是被保存在纹理内存中，这意味着它们可以在不同的时刻被销毁（例如，当屏幕保护程序在windows上启动时，或 的时候），所以在对表面做任何事情之前，你都应该检查它是否存在（包括把它画到屏幕上）。下面的示例代码显示了该命令在实例的绘制事件中的一个典型用法 来检查表面，如果它被删除了，就重新创建它（注意，这个表面最初是在对象的创建事件中创建的）。

语法。

surface_exists(surface_id)。

争论 描述
surface_id 要检查的表面的ID。

返回。

布尔型

例子。

if !surface_exists(surface)
{
surf = surface_create(1024, 1024);
}
else
{
if view_current = 1
{
draw_surface(surface,0,0);
}
}

上面的代码将检查变量 "surf "中索引的曲面是否存在，如果不存在，它将重新创建它。如果它确实存在，它将检查当前正在绘制的视图，如果是视图[1]，它将绘制这个曲面。
