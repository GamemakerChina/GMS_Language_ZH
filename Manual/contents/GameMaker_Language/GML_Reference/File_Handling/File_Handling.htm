文件处理

在创建你的游戏时，有必要从外部存储信息并能够检索这些信息。例如，你可能希望存储登录信息，或高分，或可能是玩家通过你的 级别的进展。不管是什么原因，你都需要知道如何使用GameMaker Studio 2的不同文件功能。

重要的是要意识到GameMaker Studio 2默认是沙盒的 ，这意味着在使用可用的文件系统功能时有一些限制和小麻烦。 值得注意的是，文件系统的工作方式会因你所针对的平台而略有不同。欲了解更多信息，请参见以下章节。

  • 文件系统

你可以从所选平台的游戏选项中关闭桌面目标（Windows、macOS和Ubuntu（Linux））的沙箱，这意味着你可以在操作系统权限的范围内，从任何地方保存和加载。

如果你已经得到了Steam的许可，可以在你的游戏中使用他们专用的API，你会发现还有一套新的文件功能可以使用。这些功能将在本手册的SteamAPI部分进行解释。 手册中的Steam API部分。

最后，你还应该注意到，某些目标平台之间有一些明显的差异，在使用文件功能时将会被考虑到。这些是

  • iOS在iOS上，如果你从一个文件夹中加载一个包含的文件，即：路径类似于"Backgrounds/background1.png"，你不需要把文件夹作为路径的一部分，只需要提供文件名就足够了。 就足够了。在所有其他平台上，你必须提供文件夹作为文件路径的一部分。
  • macOS和Ubuntu（Linux）。在macOS和Ubuntu上，文件名和路径在这些平台上是区分大小写的，因此为了防止任何潜在的问题，GameMaker Studio 2将自动使所有名称/路径默认使用小写。
  • HTML5。在HTML5上，保存和加载到本地存储，以及从本地存储加载，将按照不同功能页面中的解释进行，但是如果你从服务器加载文件，你必须只使用buffer_load_async()函数，因为同步加载在大多数浏览器上已经被废弃，最终将被淘汰。还要注意的是，你不能在浏览器本地存储中创建或销毁目录。
