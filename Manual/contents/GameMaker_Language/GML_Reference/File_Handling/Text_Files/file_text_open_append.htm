文件_text_open_append

该函数打开指定文件名的文本文件供写入（如果文件不存在，则创建），并返回该文件的唯一ID，该ID应存储在一个变量中，因为它将用于所有与该文件有关的进一步操作。 都会用到这个ID。写入文件的位置被设置为该文件包含的最后一行文本。注意，如果文件不能被创建（例如，由于非法的文件名），该函数将返回-1。

注意：你在任何时候最多只能打开32个文件。你也应该在结束时总是关闭文件，因为这样可以写入信息并释放与文件相关的内存。

警告!由于GameMaker Studio 2是沙盒式的，这个功能可能不会像你预期的那样工作!请参阅文件系统部分以了解更多信息。

语法。

file_text_open_append(fname)。

争论 描述
fname 要追加的文件的名称。

返回。

真实的

例子。

file = file_text_open_append（working_directory + "save.txt"）。

这将从与游戏相同的目录中打开 "save.txt"，并在变量 "file "中存储文件ID。

返回。文本文件
