游戏手柄_测试_映射

这个函数可以用来在那些允许的目标上设置游戏手柄的映射。你为要设置的游戏板提供 "槽 "的索引，然后提供地图字符串，它应该是用SDL格式创建的，有以下几个字。

<guid>,<description>,平台:<platform-name>,<bindings>。

每个字段中应包括的细节是。

  • guid- 由数字和字母组成的字符串，用于唯一地识别设备类型（可以使用 gamepad_get_guid()函数获取)
  • description- 设备的Ascii描述（可以用 gamepad_get_description()函数获取)
  • platform-name- 将是 "platform:android"、"platform:windows"、"linux "或 "mac"（注意，当你调用这个函数时，它会自动追加到字符串中，所以你在构建自己的映射字符串时不需要提供它。 所以你在构建你自己的映射字符串时不需要提供它)
  • 绑定- 一组用','分隔的条目，将实际输入与特定的GML常量绑定在一起

逗号分隔的条目列表遵循每个条目的特定格式。

<Abstract-gp-name>:<value-binding>。

这里<value-binding>（即：直接的游戏手柄输入）可以是以下任何一种。

  • a<N> -这代表游戏板的一个轴，其中<N>是0到1的一个轴。 gamepad_axis_count()-1
  • b<N> -这代表游戏板的一个按钮，其中<N>是0到1的一个按钮。 gamepad_button_count()-1
  • h<N>.<mask> - 这代表游戏板的一个帽子，其中<N>代表一个从0到-1的帽子，<mask>是1、2或8。 gamepad_hat_count()-1，而<mask>是1、2、4或8。

每个输入值都应该绑定到一个SDL名称上，这个名称表示为<>abstract-gp-name> ，而这个名称又被绑定到一个GML常量上。下表显示了SDL名称和GameMaker Studio 2常量之间的等价关系。

当你构建了你的映射字符串，它应该看起来像这样。

"050000005e040000fd020000ffff3f00,Xbox Wireless Controller,a:b0,b:b1,start:b4,dpdown:h0.4,dpleft:h0.8,dpright:h0.2,dpup:h0.1,Guide:b6,leftshoulder:b9,leftstick:b7,lefttrigger:a4,leftx:a0,lefty:a1,righthoulder:b10,rightstick:b8,righttrigger:a5,rightx:a2,righty:a3,x:b2,y:b3,platform:android"

语法。

gamepad_test_mapping(index, mapping_string)。

争论 描述
指数 设置哪个游戏板索引 "槽"。
颜色 要使用的地图字符串（更多信息见描述）。

返回。

不适用

例子。

var mapping = gamepad_get_guid(global.padIndex) + "," + gamepad_get_description(global.padIndex);
var len = array_length(global.PadInstances);
for( i = 0; i < len; i += 2)
{
var left = global.PadInstances[i];
var right = global.PadInstances[i+1];
mapping += ", " + left.sdlLabel + " :" + right.binding;
}
gamepad_test_mapping(global.padIndex, mapping)。

上述代码将循环浏览一些实例，并使用它们所包含的不同变量的值来创建一个映射字符串，然后将其设置为在给定的插槽索引中的游戏手柄上使用。

返回。游戏手柄输入
