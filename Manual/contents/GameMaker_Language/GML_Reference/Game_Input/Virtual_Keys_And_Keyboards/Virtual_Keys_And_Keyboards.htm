虚拟按键和虚拟键盘

在为触摸屏设备开发游戏时，从用户那里获得键盘输入往往很重要。当使用GameMaker Studio 2时，这可以通过以下两种方式之一来实现。

  • 创建你自己的屏幕虚拟按键来获取输入信息
  • 弹出操作系统的虚拟键盘并从中获取输入信息

你使用什么将在很大程度上取决于游戏和你的需要，但一般来说，当你想用屏幕上的按钮来控制玩家或菜单时，你会使用虚拟按键，而当你需要玩家输入时，你会使用虚拟键盘。 比如获取名字或设置数值。

为了使你的游戏能够在有触摸屏的移动设备上正常工作，你可能需要能够指定屏幕上的区域，以便在用户触摸时对某些事情作出反应。这可以通过将一个 "虚拟键 "分配给内部映射的标准键盘键来实现。 到一个内部映射的标准键盘键，然后使用它们产生的键盘事件来控制你的应用程序。值得注意的是，这些 "虚拟键 "被映射到屏幕位置而不是房间位置 所以X/Y值是绝对的，基于显示器的尺寸。这意味着你不需要担心使用视图或相对的房间坐标，你可以简单地在一个对象的创建事件中定义按键（你只需要在每个房间定义一次 每个房间只需要定义一次虚拟按键），然后在Draw GUI事件中绘制你的按键精灵，GUI层的大小与屏幕相同。

注意： 这些功能在使用触摸屏的Windows、Ubuntu（Linux）和MacOS等目标平台上无法使用，尽管它们会将鼠标点击作为 "触摸 "来接收。

可用的虚拟按键功能有：。

  • 虚拟键_添加
  • 虚拟键_显示
  • 虚位以待
  • 虚拟键删除

如果你在触摸屏设备上的游戏中需要文字或数字输入，那么你可以使用提供的虚拟键盘功能来使用设备的操作系统键盘。这些功能将调用或隐藏操作系统键盘，并接受来自它的输入。 但它不会触发常规的键盘事件，而是用用户的输入更新 keyboard_string变量（这是由于虚拟键盘可以 输入整个单词而不仅仅是单个字符）。)

打开或关闭虚拟键盘将产生系统异步事件，其中的 async_load键盘 地图将被填充以下键/值对。

  • "event_type" - 收到的系统事件的类型，对于虚拟键盘来说，它将是 "虚拟键盘状态"。
  • "screen_height" - 虚拟键盘的高度（像素）。如果键盘是不可见的，这将是0。
  • "keyboard_status" - 键盘的当前状态，以下列字符串之一返回。
    • "隐藏"
    • "隐藏"
    • "显示"
    • "可见"

可用的虚拟键盘功能有：。

注意： 这些功能只对 安卓(包括AndroidTV)。 iOS (包括 tvOS)，以及UWP 目标平台。

  • 键盘_虚拟显示
  • 键盘_虚拟隐藏
  • 键盘_虚拟状态
  • 键盘_虚拟高度
