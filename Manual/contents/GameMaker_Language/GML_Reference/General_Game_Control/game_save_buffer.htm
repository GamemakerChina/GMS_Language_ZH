游戏保存缓冲区（game_save_buffer

这是一个变种的 game_save()函数的一个变种，所以请先阅读它的页面，因为它包含了与它的使用和这个函数相关的重要信息。

通过这个函数，你可以将游戏的当前状态保存到先前创建的 "增长 "缓冲区（见缓冲区），然后可以在任何时候用 game_load_buffer() 功能。这个函数被设计为一次只在一个房间使用（即：不建议在一个房间做缓冲区保存，然后尝试从另一个房间加载缓冲区），最好只用于检查点或关卡重启。

注意：这个功能非常有限，它是为初学者设计的，以便快速建立和运行一个检查点系统，但更高级的用户可能更喜欢使用文件功能来编码自己的系统。 由于游戏不会保存任何动态资源，如数据结构、表面、添加的精灵等。

语法。

game_save_buffer(buffer)。

争论 描述
缓冲区 要保存到的缓冲区ID。

返回。

不适用

例子。

if keyboard_check_pressed(ord("S"))
{
global.Checkpoint = true;
game_save_buffer(save_buff);
}

上面的代码将设置一个全局变量为真，然后在按下 "S "键时将当前游戏状态保存到变量 "save_buff "中索引的缓冲区。

返回。一般游戏控制
