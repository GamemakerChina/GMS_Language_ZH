点_产品_三维_正常化

点积是一个表达两个向量之间角度关系的数值，通过将两个向量相乘，然后将结果相加而得到。点积 "这一名称来自于中间的点"-" 衍生出来的（另一个名称 "标量积 "强调了结果的标量性质，而不是矢量性质），通常用来表示这种操作。

实际的数学公式可以这样写。

在2D中，向量 a[x1,y1] 和 b[x2,y2] 的点积是 x1x2 + y1y2 ，所以在3D中，向量 a[x1,y1,z1] 和 b[x1,y1,z1] 的点积是 x1x2 + y1y2 + z1z2 。因此，GameMaker Studio 2中的dot_product_3d的计算方法是：。

a-b=（x1*x2）+（y1*y2）+（z1*z2）。

归一化点积是怎么回事？归一化点积已经被修正，使其返回值在-1和1的范围内（更详细的信息见矢量图 更详细的信息见矢量），这在很多情况下是非常有用的，特别是在处理照明和其他3D功能时。

语法。

dot_product_3d_normalised(x1, y1, z1, x2, y2, z2)

争论 描述
x1 第一个矢量的X坐标。
y1 第一个矢量的Y坐标。
z1 第一个矢量的Z坐标。
x2 第二个矢量的X坐标。
y2 第二个矢量的y坐标。
z2 第二个矢量的Z坐标。

返回。

真实的

例子。

var x1, y1, x2, y2;
x1 = 0;
y1 = 1;
z1 = 0;
x2 = o_Player.x - x;
y2 = o_Player.y - y;
z2 = o_Player.z - z;
if dot_product_3d_normalised(x1, y1, z1, x2, y2, z2) > 0 above=true otherwise above=false;

上面的代码沿着实例的y（向上）轴创建了一个法线向量，然后把玩家对象 "o_Player "的向量拿到自己身上。最后，它计算这两个向量的点积，如果它大于tan 0，就把变量 "以上 "为真，如果小于或等于0，则将其设置为 "假"。

返回。角度和距离
