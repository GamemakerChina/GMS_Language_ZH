碰撞点

Collision_point 检查由参数x1,y1指定的点是否与参数 "obj "指定的任何对象实例发生碰撞。这种检查可以是精确的，也可以不是，但要启用精确碰撞，你要检查的对象或实例也必须为其精灵启用精确碰撞。 碰撞，你要检查的对象或实例也必须为其精灵启用精确碰撞。如果没有，默认的检查是基于边界盒的。下面的图片说明了这是如何工作的。

记住，要考虑精确碰撞，对象精灵和碰撞函数都必须将精确标记为开。 和碰撞函数都必须将精确标记为开启。还应该注意的是，函数的返回值可以是任何一个被认为是碰撞的实例的id，所以如果三个实例在这一点上重叠，它们的任何一个id都可以是函数的返回值。 他们中的任何一个id都可以成为函数的返回值。

注意，你可以提供一个实例来代替对象索引 id来检查一个特定的实例，或者实例关键字 all ，或者 other （取决于事件和当前的代码范围）。

语法。

collision_point(x, y, obj, prec, notme)。

争论 描述
x 要检查的点的X坐标。
y 要检查的点的y坐标。
obj 检查实例碰撞的对象。
沉淀 检查是基于精确的碰撞（真，比较慢）还是基于其一般的边界盒（假，比较快）。
不是我 如果相关的话，调用实例是否应该被排除（true）或不排除（false）。

返回。

实例ID或没有人

例子。

if collision_point(x, y, obj_Cursor, false, true)
{
Score += 10S;
}

这里我们要检查的是有 "obj_Cursor "这个对象的代码的位置上的点。如果有，那么我们就在分数变量上加10。

返回。碰撞
