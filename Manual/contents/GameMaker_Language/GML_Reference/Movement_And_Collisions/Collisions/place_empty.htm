地方_空

你可以使用这个函数来检查并查看调用的实例是否会与你游戏中的任何其他物体的实例或所有实例发生碰撞。为了正确解决这个碰撞问题，运行代码的实例必须有一个有效的碰撞 掩码（无论是对精灵本身，还是通过 mask_index)，它将只登记与那些也有有效掩码的实例的碰撞。

该函数正在测试如果调用的实例被放置在一个特定的位置上是否有碰撞，你可以提供一个可选的参数来细化检查，只检查一个位置是否与给定类型的实例没有碰撞。 类型的实例碰撞。注意，如果没有提供可选的对象ID，检查将针对房间内的所有实例进行。碰撞检查要么是精确的，要么是基于实例的边界框，这取决于选择了什么样的碰撞 但为了使精确碰撞正常工作，检查中的所有实例都应该有精确的碰撞掩码。

注意，在进行检查之前，给定的x/y坐标将被浮动到最接近的整数。

语法。

place_empty(x, y, [object_id])。

争论 描述
x 要检查的X位置。
y 要检查的y位置。
[object_id] 可选的!要检查的对象。

返回。

布尔型

例子。

if place_empty(mouse_x, mouse_y, obj_Enemy)
{
x = mouse_x;
y = mouse_y;
}

上述代码将检查是否与任何其他对象 "obj_Enemy "的实例发生碰撞，就像调用的实例被放置在与鼠标相同的位置。如果没有检测到碰撞，那么该实例的x/y坐标就会被 设置为鼠标的坐标。
