mp_grid_create

通过这个函数，你可以为运动规划函数创建一个mp_grid。它返回一个索引，该索引应存储在一个变量中，以便在所有其他的MP网格函数调用中使用。x和y坐标表示网格左上角的位置。 的左上角位置，hcells和vcells表示要在网格中创建的水平和垂直单元的数量，而单元的宽度和高度表示单元的尺寸，单位是像素。你可以在同一时间创建和维护多个网格结构 如果你需要它们做不同的事情，你可以在同一时刻创建并维护多个网格结构，但要注意每个网格都需要内存，而且单元格越多，需要的内存就越多。还要注意的是，MP函数的使用是相当繁重的，所以你越是频繁地调用它们，单元格就越高。 你调用它们的频率越高，单元格的分辨率越高，那么你就越有可能出现性能下降的情况。

语法。

mp_grid_create（left, top, hcells, vcells, cellwidth, cellheight）。

争论 描述
左边 房间内mp_grid的起始x坐标
顶部 房间中mp_grid的起始y坐标
hcells mp_grid将包含的水平单元格的数量
vcells mp_grid将包含的垂直单元的数量
细胞宽度 mp_grid的每个单元格的宽度（像素）。
细胞高度 mp_grid的每个单元格的高度（像素）。

返回。

真实的

例子。

global.grid = mp_grid_create(0, 0, room_width / 32, room_height /32, 32, 32)。

上面的代码创建了一个全局变量 "grid"，然后生成了一个mp_grid，并将其索引（id）分配给该变量，以便在所有进一步的mp_grid函数调用中使用。这个网格覆盖了整个房间，单元的分辨率为32x32像素。这意味着 例如，在一个640x480的房间里，网格将包含300个单元：20个水平单元（640/32）乘以15个垂直单元（480/32）得到300个单元
