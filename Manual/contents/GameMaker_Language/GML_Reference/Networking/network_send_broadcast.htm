网络_send_broadcast

有了这个功能，你可以将缓冲区内的数据广播到一定范围的IP地址（关于缓冲区的更多信息，见参考-缓冲区）。这个范围仅限于运行服务器的设备。 例如，如果设备的IP是92.168.11.130，那么数据将被广播到92.168.11.*的范围。该函数将返回所发送数据的字节数，如果发送失败，则返回一个小于0的数字。

注意：这个功能只有在与UDP一起使用时才能工作--你的服务器需要是TCP的，你的客户需要有一个UDP客户插口，用 network_create_socket_ext() 来接收来自服务器的任何广播。

注意：这个功能在运行在HTML5目标上的项目中使用时将不起作用。

语法。

network_send_broadcast(socket, port, buffer, size)。

争论 描述
插座 要使用的插座的ID。
港口 服务器将使用的端口。
缓冲区 要获取数据的缓冲区的ID。
尺寸 数据的大小（以字节为单位）。

返回。

真实的

例子。

buffer_seek(broadcast_buffer, buffer_seek_start, 0);
buffer_write(broadcast_buffer, buffer_string, global.ServerName);
network_send_broadcast(server, 6511, broadcast_buffer, buffer_tell(broadcast_buffer) )。

上面的代码写出了当前服务器的名称字符串（存储在 "global.ServerName "中），然后把它写到一个二进制缓冲区，其ID为 "broadcast_buffer"。然后，这个数据被广播到本地的一个IP范围（设备IP目前被暗示为广播基础范围）。 目前暗示为广播基础范围）到6511端口。
