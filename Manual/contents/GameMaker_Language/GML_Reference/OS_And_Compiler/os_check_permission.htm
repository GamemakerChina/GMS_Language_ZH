os_check_permission

通过这个函数，你可以检查用户是否已经授予了游戏一个特定的权限。你可以用"android.permission.<permission>"的格式提供要检查的权限字符串，所以要检查RECORD_AUDIO权限（例如），你可以调用

os_check_permission("android.permission.RECORD_AUDIO")。

该函数将返回将返回三个常量中的一个--如下图所示--以告诉游戏权限的状态如何。关于应用程序权限的更多信息，请参见Android文档。

重要的是!此功能只针对 安卓系统目标。

语法。

os_check_permission(permission)

争论 描述
许可 要检查的权限（一个字符串）。

返回。

恒定

恒定 描述
授予的许可 这表明许可已经被授予
许可证被拒绝 这表明许可尚未被授予
拒绝请求的权限 这表明该权限已被手机设置所阻止，或者用户之前拒绝了该请求并选择了 "不要再问"。

例子。

if (os_type == os_android)
{
if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
{
os_request_permission("android.permission.INTERNET")。
}
}

上面的代码检查操作系统的类型，如果是安卓，它就对权限进行检查，如果 "INTERNET "的权限还没有被授予，它就会请求它。
