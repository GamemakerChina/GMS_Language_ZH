os_is_network_connected

通过这个函数，你可以检查并查看你的设备目前是否有互联网连接，如果有，它将返回true，如果没有，它将返回false，根据操作系统，它可能会在返回之前尝试建立一个连接。 一个值。

该函数有一个可选参数，用于在调用时启用/禁用其尝试建立连接，该选项目前仅适用于任天堂 交换机目标。当设置为 true ，该函数将尝试 进行连接，而当设置为 false ，它将仅仅限于检查连接。请注意，尝试建立连接可能会导致Switch操作系统挂起几秒钟的时间。

注意： 该函数检查控制连接的内部设备API，因此如果有蓝牙连接、Wi-Fi连接，甚至只是一个允许上网的普通网络连接，都可能返回真。

语法。

os_is_network_connected(attempt_connection);

争论 描述
尝试连接 可选的!仅适用于Nintendo Switch目标。设置为 true ，以便在调用时尝试操作系统级别的连接。

返回。

布尔型

例子。

if os_is_network_connected()
{
global.connected = true;
}

上面的代码检查设备是否有互联网连接，如果有，就设置一个全局变量。
