物理学_夹具_删除

因为一旦灯具被绑定到一个实例上，该实例就会保持灯具的属性，我们可以用这个函数从内存中删除灯具，特别是如果我们不再打算再次使用它。请注意，如果在不再需要的情况下没有删除固定设备 可能会导致内存泄漏，这将减慢速度并最终导致游戏崩溃，所以你应该在定义和绑定灯具后注意适当地进行清理。

语法。

physics_fixture_delete(fixture)

争论 描述
固定设备 要从内存中删除的灯具

返回。

不适用

例子。

var fix, inst;
fix = physics_fixture_create();
physics_fixture_set_circle_shape(fix, 16);
physics_fixture_set_density(fix, 1.0);
inst = instance_create_layer(x, y, "Instances", genericBodyObject);
physics_fixture_bind(fix, inst);
physics_fixture_delete(fix)。

上面的代码将创建一个夹具并将其索引分配给变量 "fix"。然后，它定义了夹具的形状和密度，然后将其绑定到创建的实例上，该实例的索引存储在变量 "inst "中。 最后，该夹具被删除，以防止内存泄漏，因为它不再需要了。

返回。固定装置
