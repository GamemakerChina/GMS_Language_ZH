物理学_关节_摩擦力_创建

摩擦关节与物理模拟中的所有其他关节有点不同，因为创建的连接不会限制实例的位置或运动，而是限制其速度和旋转。它的工作原理是取最大的输入值 力和扭矩的最大输入值，并将其应用于第二个夹具，使其速度和角动量下降到与第一个实例相同的值。因此，如果你有一个静止的实例和一个移动的实例，然后用一个摩擦力连接它们 接头，运动的实例将逐渐减速，直到它也静止不动。如果两个实例都在移动，那么第二个实例的移动速度将被修改为与第一个实例的移动速度一致。

与大多数其他接头一样，你要提供连接的实例，以及在房间内创建接头的位置。 以及在房间内创建接头的位置。然后你提供最大的力和最大的扭矩（方向和旋转摩擦力），如果你把 "col "值设置为 "true"，那么这两个实例可以相互作用和碰撞。 如果你把 "col "值设置为 "true"，那么这两个实例就可以相互作用和碰撞，但前提是它们有碰撞事件，然而如果它被设置为"false"，它们无论如何都不会碰撞。

语法。

physics_joint_friction_create(inst1, inst2, anchor_x, anchor_y, max_force, max_torque, col)

争论 描述
inst1 第一个例子是与联合的连接
仪表盘2 第二种情况是与联合体连接
anchor_x 关节的X坐标，在游戏世界中。
anchor_y 在游戏世界中，关节的Y坐标。
max_force 将被施加的最大摩擦力
max_torque 将被施加的最大旋转力
殖民地 两个实例是否可以碰撞（真）或不（假）。

返回。

整数（关节的索引）。

例子。

var mainFixture, o_id;
mainFixture = physics_fixture_create();
physics_fixture_set_circle_shape(mainFixture, sprite_get_width(sprite_index) / 2);
o_id = instance_create_layer(x+300, y, "Instances", obj_Rudder);
physics_fixture_bind(mainFixture, id);
physics_fixture_bind(mainFixture, o_id);
physics_joint_friction_create(id, o_id, x, y, 10, 0.3, true);
physics_fixture_delete(mainFixture) 。

上面的代码将创建一个夹具，然后将其绑定到两个实例上，然后用摩擦力连接它们。
