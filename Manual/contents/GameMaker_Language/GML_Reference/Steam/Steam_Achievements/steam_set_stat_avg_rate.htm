扩展实例。

由于平均统计功能的理解可能比较复杂，我们将通过下面的例子来说明其使用。考虑一下这样的情况：你想跟踪一个平均统计数字，比如 "每小时获得的积分"。一种方法是 一种方法是有两个统计量：一个是整数统计量 "TotalPoints"，一个是浮点数统计量"TotalPlayTimeHours"，然后用总点数除以总时间，得到 "每小时点数 "值。

然而，一旦玩家积累了大量的游戏时间，计算出的平均数就会变化得极其缓慢，而且用户玩的游戏越多，这个平均数的反应就越小。如果用户花了100个小时玩这个游戏 如果用户花了100个小时玩游戏，计算出来的平均数会 "滞后 "50个小时，如果他们提高技能，他们就不会看到他们所期望的 "每小时积分 "的增加。为了解决这个问题，我们可以使用一个 "滑动窗口" 只计算最后10个小时的 "每小时积分"。

因此，为了使用这个函数，我们需要创建一个名为 "AvgPointsPerHour "的Steam状态（在工作室的游戏控制面板中），并将其窗口属性设置为10。现在在你的游戏中，你需要添加一些全局变量 加入到一个实例中，在开始的时候。

global.Points = 0;
global.Time = 0;

然后你会有一些控制器对象在一个警报器中每秒计算全局 "时间 "变量（例如），而你的游戏会影响全局 "点数 "变量。在游戏过程中，每隔一段时间（同样，在一个控制器对象中，也许是在一个警报器中）就会有一个全局 "时间 "变量。 在一个控制器对象中，也许在一个警报器中，或者在轮询 "时间 "值的间隔中），你将这样设置状态。

steam_set_stat_avg_rate("AvgPointsPerHour", global.Points, (global.Time / 3600));
global.Points = 0;
global.Time = 0。

请注意，我们将时间除以3600，因为我们想要的是以小时为单位的时间，而不是以秒为单位的时间，之后我们将全局 "点数 "变量和全局 "时间 "变量重置为0，这样在下次调用该函数时，我们就可以 得到一个新的统计量的平均值。现在，Steam要做的是利用你发送的这个值，在为我们的 "窗口 "设定的时间内创建一个平均值。
