蒸汽_获取配额_总数

在使用Steam云存储和同步文件时，您必须设置您的游戏所需的空间配额。这个配额在每个支持云计算的游戏中都是以每个用户、每个游戏为基础来执行的，因此，举例来说，如果游戏X的配额为 的配额是1兆字节，那么每个拥有游戏X的Steam账户最多可以在云端存储与该游戏相关的1兆字节的数据。用户拥有的任何其他支持云计算的游戏（例如，游戏Y）将不会受到游戏X所存储数据的影响。 新的Steamworks游戏的默认配额是1GB，但你可以在Steamworks的控制面板中改变你的游戏。

注意：一旦配额用完，文件写入将失败。如果你认为你的游戏用户的配额有可能被用完，你应该创建代码来处理它，因为如果你什么都不做，你就会让用户处于这样的境地 他们无法解决问题，这将导致用户对你的游戏产生不良体验。

语法。

steam_get_quota_total()。

返回。

真实的

例子。

ifsteam_is_cloud_enabled_for_app()
{
quota =steam_get_quota_total();
}

上面的代码检查游戏是否启用了steam云，如果是，它就会得到存储配额的大小，并将其存储在一个变量中。
