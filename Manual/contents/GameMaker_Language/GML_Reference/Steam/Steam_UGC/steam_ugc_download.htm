蒸汽_ugc_download

通过这个函数，你可以为任何给定的UGC项目下载一个预览图片。ugc_handle是图片的唯一识别值（你可以用函数 steam_ugc_send_query()), 目标文件名是你希望在下载完成后给图像文件的名称（和在Steam沙盒中的本地路径）。

当使用这个函数时，它将触发一个Steam异步事件，报告所请求的图像文件的细节，包含以下键/值对，在 async_loadds_map。

  • " id" - 调用函数返回的异步ID
  • "result" - 操作的结果（一个实值）。这将是GML常数ugc_result_success或其他的实数。所以你应该检查这个常数，以确保调用是成功的，并且如果 否则就会出现工作不正常的情况。其余可能返回的值显示为Steam"EResult "值的结果，你应该在SDK头文件中看到steamclientpublic.h的所有89个可能值。
  • "event_type" - 这个键将持有 "ugc_download "的值。
  • "original_filename" - 这个键保存服务器上图像文件的原始名称（一个字符串）。
  • "dest_filename" - 这个键保存你传入的图像文件名（一个字符串）。
  • " ugc_handle" - 这个键保存你传递给调用函数的ugc_handle值。

语法。

steam_ugc_download(ugc_handle, dest_filename)。

争论 描述
ugc_handle 要下载的预览的唯一手柄。
dest_filename 用来保存预览的文件名。

返回。

异步ID

扩展实例。

在这个例子中，我们首先调用函数并将异步ID值存储在一个变量中。

steam_get =steam_ugc_download(steam_handle, "\UGC\Preview_file.png")。

然后，这将向SteamAPI发送一个文件请求，产生一个异步事件，我们将按如下方式处理。

var event_id = async_load[? "id"];
if event_id == steam_get
{
var type = async_load[? "event_type"];
if type == "ugc_download"
{
sprite_delete(preview_sprite); preview_sprite = sprite_add(async_load[? "dest_filename"], 0, false, false, 0, 0) ;
}
}
