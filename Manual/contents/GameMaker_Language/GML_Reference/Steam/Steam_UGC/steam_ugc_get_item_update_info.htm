steam_ugc_get_item_update_info

这个函数可以用来检索关于给定文件ID的当前下载状态的信息。你给出项目的ID，并为一个空的DS地图提供索引，如果项目存在，DS地图将被填充以下键/值对。 如果该项目存在的话，它将被填入以下键/值对。

  • " needs_update" - 根据项目是否需要更新，将返回真或假。
  • " is_downloading" - 将返回true或false ，取决于该项目当前是否正在下载。
  • " bytes_downloaded" - 已经下载的字节数。
  • " bytes_total" - 该项目在磁盘上所需的总大小（字节数）。

如果该项目存在，那么该函数将返回true并填充地图，否则将返回false，地图将保持为空。

语法。

steam_ugc_get_item_update_info(published_file_id, info_map)。

争论 描述
published_file_id 要检查的UGC的唯一文件ID。
信息地图 一个（先前创建的）DS地图索引。

返回。

布尔型

例子。

var info_map = ds_map_create();
var info =steam_ugc_get_item_update_info( global.fileID, info_map);
if info
{
draw_text(32, 15, "needs_update: " + string(info_map[? "needs_update"]));
draw_text(32, 30, "is_downloading: " + string(info_map[? "is_downloading"));
draw_text(32, 45, "byte_downloaded: " + string(info_map[? "bytes_downloaded"]);
draw_text(32, 60, "bytes_total: " + string(info_map[? "bytes_total"]));
}

上述代码将查询全局变量 "fileID "中索引的项目的下载状态，使用DS Map来存储信息。

返回。UGC
