匹配性的查找

这个函数将使用会话模板、匹配跳板和安全设备协会模板搜索多人游戏会话，你应该事先在XDP仪表盘中创建这些模板。你可以通过调取一个特定的用户ID来检索 你可以通过调用该 xboxlive_user_for_pad()函数。该函数将返回一个唯一的请求ID值，然后可以用来确定该函数的正确的社交异步事件。这个事件将在会话被创建时被触发，并在变量中包含一个DS Map async_load中包含一个DS Map，有以下 键/值对。

  • "requestid" - 包含由调用函数返回的请求ID
  • "status" - 包含字符串 "session_created"，告知你已经触发了什么类型的事件。
  • "num_results" - 如果发现了一个会话，则包含1或更多，否则包含0。
  • "sessionid<index> " - 包含会话ID，如果失败则为-1（<index/gt;是一个从0到 "num_results"-1的值）。
  • "sessionOwner<index>" - 包含会话主机的ID，如果已经找到了一个会话（<index>是一个从0到"num_results"-1的值）。
  • "error" - 成功时包含0，失败时包含-1

语法。

xboxlive_matchmaking_find(user_id, template, hopper, sdatemplate, [matchattributes]) 。

争论 描述
user_id 要使用的用户ID指针
模板 会话模板的名称
漏斗 匹配跳板的名称
ǞǞǞ 安全设备关联模板的名称
匹配属性 [可选]

返回。

请求ID（整数

例子。

var userID = xboxone_user_for_pad(global.PadIndex);
result = xboxone_matchmaking_find(userID, "MatchTicketSession", "MatchTicketHopper", "PeerServerTraffic") 。

以上将检索给定游戏板上用户的用户ID，然后尝试为他们找到一个匹配会话。
