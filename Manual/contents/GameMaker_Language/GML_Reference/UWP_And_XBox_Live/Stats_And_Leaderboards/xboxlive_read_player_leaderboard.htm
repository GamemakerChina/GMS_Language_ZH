xboxlive_read_player_leaderboard

该函数将返回给定用户名的排行榜信息。排行榜必须是先前在你的游戏的XDP仪表板上创建的，调用该函数将触发一个回调的Social Asynchronous Event，其中包括 异步加载地图上的相关键/值对。这个DS地图的 "id "键被用来识别正确的回调 (对于任何给定的异步事件，可以有一个以上的触发函数)，并且IT将与常数achievement_leaderboard_info以及其他一些键/值对配对。该地图的具体内容如下所示。

  • " id" - 对于这个函数，它应该是achievement_leaderboard_info
  • "领先优势" - 在提供者仪表板上定义的排行榜的唯一ID。
  • "纪念品" - 你在排行榜上获得的条目数。
  • "闲人免进" - 玩家的名字，其中 "N "是一个整数值，对应于他们在排行榜条目列表中的位置。
  • "玩家IDN" - 球员的唯一用户ID，"N"。
  • "等级N" - 玩家 "N "在排行榜上的排名。
  • "分数N" - 球员 "N "的得分。


该函数要求你给出以下常数之一来设置过滤器属性。

恒定 描述
成就过滤器_所有玩家 获得所有的分数。
仅限朋友的成就_过滤器 只获得朋友的分数，按升序排列。
仅限成就_过滤器_收藏品 只获取最喜欢的分数，按升序排列
成就过滤器_朋友_alt 只获取朋友的分数，按降序排列。
成就_过滤器_收藏夹_alt 只获取按降序排列的最喜欢的分数。

语法。

xboxlive_read_player_leaderboard(leaderboard_name, user_name, num_items, friend_filter)。

争论 描述
排行榜名称 要读取的排行榜名称，如XDP上设置的那样。
用户名称 要读取的用户名称。
num_items 要检索的项目的数量。
友好过滤器 滤波器常数之一（见上面的描述）。

返回。

真实的

例子。

var _user = xboxlive_user_for_pad(0);
var _uid = xboxlive_user_id_for_user(_user);
xboxlive_read_player_leaderboard("MyLeaderboard", _uid, 10, achievement_filter_all_players) 。

上面的代码得到用户ID，然后用它来请求给定排行榜前10名的所有玩家数据。
