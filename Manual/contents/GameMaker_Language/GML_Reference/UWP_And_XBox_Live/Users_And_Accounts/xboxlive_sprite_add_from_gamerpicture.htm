淘宝网上卖的是什么？

通过这个函数，你可以得到一个给定的用户ID指针的游戏者图片。该函数以异步方式工作，并将触发一个图像加载的异步 事件来通知你该函数已经完成（就像HTML5/URL调用到 sprite_add()).异步图像加载事件将有一个DS地图存储在 的内置变量 异步加载.该地图包含以下信息。

  • "文件名"。你请求的文件的完整路径。
  • " id":你所加载的资源的ID。这将与你分配给该资源的变量相同。
  • "状态"。错误时返回小于0的值。

注意：当你在运行时将一个精灵动态加载到你的游戏中时，你必须记得在不再需要它时再次使用 sprite_delete()删除它。 否则会有内存泄漏的风险，这将使你的游戏变慢并最终崩溃。

语法。

xboxlive_sprite_add_from_gamerpicture(user_id, imagesize, xorig, yorig)。

争论 描述
user_id 获取游戏者图片的用户ID（一个指针）。
图像大小 要返回的精灵的尺寸，单位是像素
原产地 指明原点在精灵中的X位置
原住民 指明原点在精灵中的y位置

返回。

真实的

例子。

var _a = 0;
var _num = xboxlive_get_user_count();
for (var i = 0; i < _num; ++i;)
{
var _uid = xboxlive_get_user（i）;
if _uid != pointer_null
{
global.UserName[_a] = xboxlive_gamedisplayname_for_user（_uid）;
global.UserScore[_a] = xboxlive_gamerscore_for_user（_uid）;
global.UserRep[_a] = xboxlive_reputation_for_user（_uid）;
global.UserAvatar[_a] = xboxlive_sprite_add_from_gamerpicture（_uid, 256, 0, 0）;
++a;
}
}

上面的代码循环浏览登录的用户，并将他们的游戏者数据存储在各种全局数组中。
