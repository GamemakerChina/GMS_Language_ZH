远程工作者

当你编译你的游戏时，你通常使用 "本地 "工作者，也就是GameMakerStudio 2已经安装的计算机。但是在某些情况下，你可能需要使用另一台计算机来构建你的项目。比如说。 如果你在Mac上工作，而你想构建一个Windows项目，那么你就需要配置一个远程工作者，并让一台Windows电脑来进行构建。基本上，当你使用一个远程工作者时，GameMaker Studio 2将从（本地）发送所有的 所需的文件从它正在运行的（本地）机器上发送到远程工作者机器上，然后使用该机器来构建项目的最终可执行文件（要么保存在远程机器上，要么事后返回到本地机器上）。 然而，在你使用这一功能之前，你必须在构建机器上安装某些工具，并进行一些设置，我们将在这里解释。

重要的是!你可以在本地Windows电脑和远程Windows电脑之间，或在本地Mac电脑和远程Windows电脑之间设置一个远程工作者，但目前你不能设置本地Windows（或Mac）电脑和远程Mac电脑，即：你不能 即：你不能有一个Mac的远程工作者，只能有一个Windows的。

初始设置

要设置一个远程工作者，你需要首先在要使用的电脑上安装远程工作者构建工具应用程序。你可以在本地机器上的GameMaker Studio 2的Build菜单中找到这个工具，然后选择 "ShowRemote Worker Installer "选项。

或者，你可以点击目标管理器中的 "眼睛 "图标，并通过这种方式获得 远程工作站的安装程序。

安装程序将被称为 "GameMakerStudio-Remote-Installer-X.X.XXX.exe" 你需要把它复制到你想用作远程工作者的电脑上（你不需要在远程工作者上安装GameMaker Studio 2，只需要这个程序）。

一旦你把它复制到构建的PC上，按照屏幕上的提示进行安装，当安装程序完成后，操作系统可能要求你通过防火墙访问你的网络。一旦你同意访问，安装程序将打开 你必须记下这个页面上显示的IP地址和端口号，因为下一步你将需要它。

你现在需要回到你安装GameMaker Studio的机器上 2的机器，点击目标管理器按钮 ，打开目标管理器窗口。 然后进入Worker部分，点击Add Worker按钮 。这将打开 以下窗口。

在这里，您需要给您的远程工作者一个名称（这就是将在目标管理器的工作者列表中显示的名称）。 显示在目标管理器的Worker列表中），然后在Host Name（主机名）部分，添加地址以及之前显示和注意到的Worker机器的端口。

如果一切正常，你现在应该得到一个消息，说你可以成功地连接到远程工作者的机器上。一旦你安装了远程工作机并连接到你的主IDE计算机，你将需要去设置每个平台的本地 在能够编译你的项目之前，你需要为每个平台设置本地构建设置（更多细节见下面的章节）。

远程设置

设置 页面显示了Remote Worker应用程序的不同路径和构建设置。在设置的顶部有不同的路径，这些路径将被Remote Worker应用程序使用。 如果需要的话，应该修改这些路径并设置为其他路径（一般来说，默认路径应该是可以的）。你可以用以下任何一种方法访问Remote Worker设置页面。

  • 点击在你的远程机器的浏览器中打开的远程工作者页面顶部的 "设置 "链接。
  • 使用远程工作者托盘图标上的人民币，在远程机器的浏览器中打开设置。
  • 在您的本地机器上，打开目标管理器并点击 "齿轮 "图标，在您的本地机器上的一个窗口中打开工人设置。

这将显示设置页面，你可以看到（并设置）不同的选项，允许你设置构建项目的路径。


本地构建设置

在检查了要使用的远程工作程序的路径后，你可以设置（如果需要）不同的本地构建设置。不同的平台将需要在构建电脑上安装不同的构建工具，而远程 工人需要能够通过设置 页面使用它们来构建你的游戏。这里的第一个设置是常规 设置。

这里你可以设置帮助服务器的端口 和调试器使用的端口。 帮助服务器的端口和调试器 使用的端口。如果你在使用调试模块时有任何连接问题，可以改变这些端口，但一般来说，应该保持其默认值。

汇编

一旦你设置好了Remote Worker并完成了本地构建设置（见上面的章节），你就可以开始使用它来编译你的游戏了。为此，你只需从目标管理器中选择Remote Worker 列表中选择Remote Worker，选择要测试/编译的平台，以及任何其他选项（如编译类型或配置），然后按Play 或Create Executable 。

注意：目标管理器中的可用平台将根据远程工作者计算机的能力而改变，并非所有目标都可用。还要注意的是，没有针对远程工作者的 "测试 "目标。

如果你只是简单地播放项目，那么Remote Worker机器将编译并运行它，但如果你要创建一个可执行文件，那么Remote Worker将编译项目，然后返回一个 *.zip 文件到 本地机器进行保存。

应该注意的是，在使用远程工作器时，Debug按钮 ，Clean按钮 ，都将无法使用。还要注意的是，目前你一次只能执行一个远程构建。


一旦你在远程机器上安装并运行了Remote Worker，它就会以图标的形式出现在任务栏中，你可以右键点击它，调出以下选项。

  • 打开。在你的默认浏览器中打开Remote Worker应用程序。
  • 显示日志。在浏览器中打开Remote Worker的日志文件。
  • 网络设置。在这里你可以过滤你希望哪些机器能够发送构建到远程工作者机器上。每个过滤器可以是一个CIDR风格的过滤器，也可以是一个要允许的机器的主机名。你应该知道，如果你想让多台机器连接到一个远程工作者，或者你混合连接类型（即：有线和wifi），那么你可能需要在这里编辑允许的IP范围。
  • 编辑设置。打开远程工作者设置菜单（在上面的设置部分有解释）。
  • 退出。退出Remote Worker应用程序。这意味着你不能再从本地机器连接（也不能编译）该应用程序，直到你重新启动Remote Worker应用程序。
