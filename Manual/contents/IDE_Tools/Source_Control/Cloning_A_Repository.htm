克隆一个存储库

当你的项目存储在一个外部仓库时，你需要记住推送你的修改和拉取任何新的修改。原因是，当你提交时，你把你的修改存储在本地，这允许你在远离互联网的地方工作，并在你准备好的时候同步。 当你准备好的时候就可以同步，为了同步，你要把你的修改推送到远程版本库中。拉取修改意味着你要把别人从远程版本库中的任何修改拉到你的本地版本库。然而。 这意味着有时人们可能会编辑相同的文件，这又会产生合并冲突。你需要知道如何处理这些冲突，所以我们现在要故意制造一个，然后告诉你 如何处理它。

注意：不建议共享本地项目库（即使是通过Dropbox这样的文件同步服务），因为提交一般是直接写入库中的，因此你的冲突保护非常少。

为了产生我们的冲突，我们首先需要有两个本地版本库，一个是主版本库，一个是我们的副本。为了创建这个，我们需要克隆版本库，这可以通过以下步骤完成。

  • 进入Source Control> Clone Repository。
  • 在URL字段中，键入我们在上面的还原文件指南中设置的项目路径（默认情况下，这将是 <DRIVE>:\Users\<Username>\Documents\GameMakerStudio2\<ProjectName>）。
  • 然后我们需要设置一个克隆的路径。
    当我们点击 "Okay "按钮时，我们会收到以下关于试图克隆一个 "非空白 "版本库的警告：
    我们可以在这里再次点击 "Okay"。
  • 现在我们将看到一个目录资源管理器，以创建一个 "裸 "代理仓库（这是一个你可以安全存储在Dropbox上的仓库，例如）。浏览到我们要创建代理仓库的地方，选择（或创建）要使用的文件夹。 要使用的文件夹。

一旦我们这样做了，GameMaker Studio 2就会把原来的项目推送到代理，并把一个新的项目拉到我们在CloneRepository窗口中设置的路径，然后自动打开一个文件浏览器，这样我们就可以打开存储的 项目。如果我们现在进入源代码控制>查看历史，我们会看到我们的原始提交加上一个新的 "推送给克隆"。

你现在已经克隆了你的版本库，可以继续在本地版本的项目上工作。
