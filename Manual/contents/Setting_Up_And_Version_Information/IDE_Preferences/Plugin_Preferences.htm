插件首选项

插件首选项用于控制已经添加到GameMaker Studio 2IDE中的任何额外插件。默认情况下，它有一个部分专门用于源代码控制插件，用于将GameMaker Studio 2项目与 Git.

资源控制（Git

本节涉及你需要设置的首选项，以获得 Git SCM插件与GameMaker Studio 2一起工作。首先，你有一组 G 的一般 选项。

  • 自动打开源控制输出窗口。当勾选此项时，当有变化时，源控制输出窗口将自动显示。这在默认情况下是打开的。
  • 冗长的日志记录。启用Git输出日志的粗略记录。这在默认情况下是关闭的。

接下来是 用户身份部分，有以下需要填写的字段。

  • 提交用的用户名。在这里你必须输入你将用来向Git提交的用户名（这必须与你设置Git仓库时指定的用户名相同）。
  • 提交时使用的电子邮件。与此Git身份相关的电子邮件。

在设置完用户身份后，您可以通过点击标有 "添加新的用户/密码认证 "的按钮来添加一个新的用户名/密码组合进行认证。 添加新的用户/密码认证，这将打开以下窗口，有不同的字段供您填写。

你也可以 通过点击带有该标签的按钮添加一个新的SSH密钥对认证。这将打开以下窗口供你填写。

在这些按钮下面，你可以设置 合并工具和 差异工具，并使用这些选项。

  • 合并工具的路径：在这里你可以指定Git合并工具的完整文件路径。
  • 合并工具的选项。在这个领域，你可以添加任何额外的命令，以便在你使用合并工具时运行。
  • Diff工具的路径：这里你可以指定Gitdiff工具的完整文件路径。
  • Diff工具的选项。在这个字段中，你可以添加任何额外的命令，以便在你使用Diff工具时运行。

最后，在设置好这些之后，你还有最后一个选择。

  • 自动启用源码控制。勾选后，所有新项目都将自动启用源码控制。默认情况下这是关闭的。
