Windows UWP游戏选项

本节概述了可用于控制你的UWP（通用Windows平台）游戏项目如何被编译的不同选项。不同的部分是。

一般

UWP选项的一般部分是你可以设置产品信息的地方。

  • 显示名称。指定显示给用户的应用程序的友好名称（这个字符串是可本地化的）。
  • 软件包名称。这是你所创建的软件包的名称，应该与你为游戏创建的开发者密钥所用的名称一致（见下面关于包装的部分）。
  • 包的显示名称。这是包的 "短名称"，最好与显示名称相同。
  • 发行商显示名称。这是你的公司或个人的名字，你希望将其作为游戏发行商。
  • 版本。你的游戏的版本号。

设置之后，你可以选择你的游戏可以运行的不同方向 ，以获得最佳的用户体验。可用的选项是。

  • 景观
  • 形象代言人
  • 景观翻转
  • 纵向翻转的

默认情况下，所有这些都将被选中，你应该取消对那些你不需要的东西的选择。

在设置方向选项后，你可以从以下选项中发出你的UWP应用程序需要的不同权限 的信号。

  • 启用互联网权限。将游戏标记为可以访问互联网。这在默认情况下是关闭的。
  • 启用麦克风权限。标志着游戏可以访问设备的麦克风。这在默认情况下是关闭的。
  • 启用IAP沙盒。将游戏标记为使用沙盒的应用程序内购买。这在默认情况下是关闭的。

图形

图形选项是你应该配置的，以确定你的游戏将如何使用目标UWP设备的显卡。包括以下选项供你修改 包括以下选项供你修改。

  • 在像素之间进行颜色插值。打开线性插值，这基本上是 "平滑 "像素。对于清晰的像素图形，它应该是关闭的。 关闭，但如果你有漂亮的阿尔法混合和平滑边缘的图形，最好打开。
  • 显示光标。当这个选项被启用时，默认的操作系统光标将显示在你的游戏中。如果这个选项被禁用，那么当鼠标在游戏窗口上时，将没有可见的光标，除非你在游戏项目中创建了一个光标。 的一部分（默认情况下是打开的）。
  • 开始全屏。如果选中这个选项，游戏将以全屏模式启动，否则将以窗口模式启动。默认情况下这是关闭的。
  • 允许全屏切换：勾选此项后，用户可以使用标准的Ubuntu操作系统快捷键从全屏切换到窗口，然后再返回。默认情况下这是关闭的。
  • 缩放。在这里，你可以选择保持长宽比（所以4:3的房间将在16:9上被 "字母框"）或完全缩放（拉伸图像以适应整个屏幕）。

    警告! 关闭应用程序表面将禁用Windows游戏选项中设置的所有缩放选项，直到它被重新打开。更多的细节请看应用程序表面。

最后是设置纹理页的大小的选项。默认（也是最兼容的）尺寸是2048x2048，但你可以从256x256到8192x8192之间的任何地方进行选择。在这里 也有一个按钮标明 预览 的按钮，它将为这个平台生成纹理页，然后打开一个窗口，以便你能看到它们的样子。如果你想看看纹理页是如何结构的，这可能非常有用。 结构，并防止纹理页大于（或小于）所需。关于纹理页的更多信息，请看这里。

注意：请注意，纹理页的尺寸越大，你的游戏在规格较低的PC上的兼容性就越差。

图片

在图像部分的第一部分，你可以选择设置各种瓷砖 ，你的游戏将在不同的 终端上使用的瓷砖。你可以设置以下内容（它们都应该是 *.png 格式的文件）。

  • 小瓦片图像是出现在搜索栏和用户界面其他区域的应用程序名称旁边的标志图像。它应该是44 x 44像素。
  • 中号瓦片图像应该是一个71x71像素的图像。
  • 下一个瓦片图像是商店标志瓦片 ，它应该是50 x 50像素。
  • 宽图像瓦片是当瓦片处于宽格式时出现的图像，应该是310 x 150像素。如果没有提供这个图像，瓦片只能以方形格式显示，不能接受基于宽幅模板类型的通知。 基于宽幅模板类型的通知。由于用户可以决定瓦片使用哪种格式，所以建议你包括一个宽图像瓦片。还要注意的是，如果提供了宽图像瓦片，瓦片最初可能以其宽格式出现。
  • 大图像瓦片是大的实时瓦片格式，应该是310 x 310像素。请注意，这只会在桌面操作系统上使用。

淘宝网

UWP游戏选项的Xbox Live部分是你可以激活你的UWP项目在Xbox One主机上使用的地方。激活该项目 允许你在Xbox One上运行该项目时进行以下操作。

  • 从应用程序中登录Xbox Live
  • 在项目中以某种方式使用Xbox Live用户身份（例如，显示游戏名或玩家照片）。

然而，在你做这些事情之前，你需要确保你已经点击了启用Xbox Live的复选框，如果你是微软创造者计划的一部分，你也应该勾选创造者计划应用程序选项。如果你是创造者计划的一部分，你将需要添加标题ID和服务配置ID（关于这些的更多信息可以在 可以从微软那里找到更多信息）。)

注意，目前要在你的项目中测试Xbox Live功能，你需要使用Create Executable而不是Run，因为UWP的Xbox输出需要使用你作为Packaging 信息的一部分（见上文），而不是GameMaker Studio 2提供的临时证书，当你点击 "运行 "时就会使用这个证书。

一旦你启用了XBox Live，你就可以在你的项目中使用XBox Live功能，但要注意的是，对于最基本的功能，你需要成为创造者计划的一部分，对于高级功能，你需要一个ID@XBox账户。
