在产品信息 部分，首先要做的是填写你的游戏的显示名称 然后填写其版本号、应用程序ID、应用程序输出文件和版权细节。App Output是你的游戏安装文件在你的Mac开发机上的位置。 机上的位置。当你创建你的 .app.zip ，该文件可以去任何地方（你在做保存对话时指定的位置不是该 .app.zip 的唯一副本），但 .pkg 和其他文件被留在这个文件夹中，允许你通过终端测试本地安装，以模拟Mac商店购买等方式。

在这里，你还需要给出你希望用于签署GameMaker Studio 2为你创建的最终应用程序的团队标识符。在这里设置它将取代你在macOS首选项中提供的团队ID。 在团队标识符之后，你还可以选择提供签名标识符，这是苹果对所有非应用商店应用程序（例如Steam游戏）的要求。

图形

图形选项是那些你应该配置的选项，以决定你的游戏如何使用目标Mac的显卡。 包括以下选项供你修改。

  • 允许全屏下的菜单和Dock。勾选后，如果游戏处于全屏模式，该选项将显示操作系统的菜单和底座。这在默认情况下是关闭的。
  • 显示光标。当这个选项被选中时，常规的macOS光标将被显示出来，而取消这个选项将意味着没有光标被显示，除非你在游戏代码中创建了一个光标。默认情况下这是关闭的。
  • 开始全屏。如果选中这个选项，游戏将以全屏模式启动，否则将以窗口模式启动。默认情况下这是关闭的。
  • 允许全屏切换：勾选此项后，用户可以使用标准的macOS快捷键从全屏切换到窗口，然后再返回。默认情况下这是关闭的。
  • 在像素之间进行颜色插值。打开线性插值，这基本上是 "平滑 "像素。对于清晰的像素图形，它应该是关闭的，但是 如果你有很好的阿尔法混合和平滑边缘的图形，最好打开。这在默认情况下是关闭的。
  • 使用同步来避免撕裂。这将切换v-sync的开启或关闭。注意，如果你有一个房间速度为120的游戏，而玩家的显示器刷新率为60，打开这个选项也会把你的游戏速度锁定在60。 60，打开这个选项将把你的游戏速度也锁定为60。这在默认情况下是关闭的。
  • 允许窗口调整大小。勾选此选项允许用户改变游戏窗口的大小（无边界窗口选项需要关闭才能发挥作用）。该选项默认为关闭。
  • 启用视网膜：当游戏在支持视网膜的显示器上运行时，勾选此选项将把后面的缓冲区设置为较高（实际）的分辨率，而不勾选此选项将把后面的缓冲区设置为表面（较低）分辨率。 这意味着初始游戏窗口将被设置为第一个房间/视图端口的宽度和高度的两倍。然而，这并不能使应用表面适应，因此，如果你想让你的游戏在视网膜显示器上发挥最大作用，你应该 所以如果你想让你的游戏充分利用视网膜显示器，你也应该把应用程序的表面设置为与窗口大小相同（注意，这将使宽度和高度的像素增加一倍，所以只有在你的游戏将受益于增加的分辨率的情况下才会启用这个功能并缩放应用程序的表面）。

    这个选项默认是关闭的，当你使用虚拟机运行或调试游戏时，启用它不会显示任何差异，因为在这种情况下，运行器是预先建立的。只有当你通过VM构建一个可执行文件，或者使用YYC测试或 构建游戏，因为这将用你自定义的游戏选项重建运行器。
  • 缩放。在这里你可以选择保持长宽比（所以4:3的房间在16:9的屏幕上会有 "字母框"）或完全缩放（拉伸图像以适应全屏）。

    警告! 关闭应用程序表面将禁用Windows游戏选项中设置的所有缩放选项，直到它被重新打开。更多的细节请看应用程序表面。

最后是设置纹理页的大小的选项。默认（也是最兼容的）尺寸是2048x2048，但你可以从256x256到8192x8192之间的任何地方进行选择。在这里 也有一个按钮标明 预览 的按钮，它将为这个平台生成纹理页，然后打开一个窗口，以便你能看到它们的样子。如果你想看看纹理页是如何结构的，这可能非常有用。 结构，并防止纹理页大于（或小于）所需。关于纹理页的更多信息，请看这里。

注意：请注意，纹理页的尺寸越大，你的游戏在规格较低的PC上的兼容性就越差。

图片

图像部分是你提供你的游戏所需图像的地方。对于Mac，你需要提供一个图标 文件 (必须是 .png 格式和1024x1024px)和一个闪屏(可以是 .png, .bmp, .jpg 或 .gif 格式)。在游戏加载时，将显示闪屏。

对于DMG软件包，你可以提供一个安装程序背景图像，它将作为DMG安装程序的背景出现。

值得注意的是，GameMaker Studio 2有一个项目图像生成器工具，可以用来自动创建所有不同的目标平台所需的图像。 被编译到的所有不同的目标平台上。如果你使用这个工具，你应该修改创建的图像以确保它们是你所需要的。
