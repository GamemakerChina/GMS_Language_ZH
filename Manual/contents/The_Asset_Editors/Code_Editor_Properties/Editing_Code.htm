值得注意的是，你可以用各种方式注释 你的代码。注释你的代码对于保持生产力是非常重要的，因为它可以使事情变得清晰，并以这样的方式将代码的各个部分分开，使每个部分的作用和原因都很明显。 部分是做什么的，以及它为什么在那里。这也意味着，如果你在一个团队中工作，那么你可以给其他成员留下注释，他们会清楚地知道你做了什么，或者某段代码是做什么的。即使你是一个单独的开发者。 在你的代码中留下注释是很重要的，特别是当你不得不离开这个项目并在以后的日子里再回来的时候。

注释你的代码很简单，因为你只需在任何一行文本前使用"//"，或者你可以通过使用"/* text here */"注释整个段落。甚至有两个键盘 甚至有两个键盘快捷键来注释和取消注释选定的代码行。 / + "K" 来注释， / + + "K" 来取消注释。下面的图片展示了使用这两种方法编写的代码 以及一些JS文档注释（下面有更详细的解释）。

脚本也可以添加某些JSDoc风格的注释来定义它们所包含的函数的属性，这样，当你在代码中使用 函数时，它们会在自动完成中显示出来，同时还有它们的参数和其他细节。你可以从JSDoc脚本注释一节中了解到更多的信息。

默认情况下，所有编写的代码在每一组匹配的大括号旁边都有代码折叠图标 {} 。这些图标的形式是一个小的加号 + 或减号 - ，点击这些图标将把代码 "折叠 "在匹配的大括号内，这样它所占用的视觉空间就会减少。

你也可以使用特殊的 #region 标签手动设置代码折叠区域，以标记代码的一个部分。它的工作原理是，你在一段代码的顶部添加一个 #region 标签（连同任何注释文本），然后在该部分的底部添加一个相应的 标签，然后你就可以 "折叠 "代码。 然后在该部分的底部添加一个相应的 #endregion 标签，然后你就可以 "折叠 "代码，即：将其隐藏起来，这样你就可以减少编辑器中可见的代码行数了。 在任何时候都是如此。

你可能已经注意到了，脚本文本的部分内容是有颜色的。脚本编辑器 "知道 "现有的对象、内置变量和函数等等，并对它们进行相应的颜色编码。颜色编码对避免错误有很大的帮助 因为你可以立即看到你是否拼错了某个名字，是否把某个关键字当作了变量，是否有语法错误。然而，如果你不喜欢颜色编码，你可以在首选项中打开或关闭它（你也可以在编辑器中使用 F10 来打开或关闭它），也可以自定义使用的颜色。

你也可以在编辑器的任何地方右击 ，打开以下菜单。

除了你期望的剪切、复制和粘贴代码行的常规功能外，你还可以选择注释或取消注释 任何选定的文本，添加或删除缩进，以及将代码窗口分成1、2或3个独立的窗格（注意，如果你没有在正在编辑的代码中定义任何选项，那么与 #regions 有关的选项将不可用）。

最后一个分割窗格的选项将把实际的窗口分割成不同的部分，然后你可以拖动一个脚本或事件，让它与其他的脚本或事件一起显示，以便进行比较和编辑。

这对于比较两个脚本或同时查看相连的脚本特别有用，而不需要另一个窗口 或在标签页之间不断切换。
